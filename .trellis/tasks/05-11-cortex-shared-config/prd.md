# cortex 共享配置 ~/.cortex/

## Goal

在 `~/.cortex/` 落一份用户级共享配置, 整合现散落于环境变量与脚本默认值的 cortex 运行参数 (vault 路径、默认 lang、Claude settings、安装路径等), 方便 hooks/scripts/cron/skills/agents 跨入口复用同一份事实源。

## What I already know

- 现有 CORTEX_* 环境变量 (来自 `docs/编程式调用.md` + 代码):
  - `CORTEX_VAULT` (必需) vault 根路径
  - `CORTEX_LANG` (可选) 覆盖 vault.lang; 默认从 `_meta/version.json` detect
  - `CORTEX_SETTINGS` (可选) Claude settings 路径; 默认 `~/.claude/settings.glm-4.5-flash.json`
  - `CORTEX_INSTALL_PATH` (可选) marketplace 安装路径覆盖
  - `CORTEX_DRY_RUN=1` (可选) 干跑

- 现有消费点:
  - `scripts/cron/run.sh` L47-49 fallback 链 `CORTEX_VAULT → OBSIDIAN_VAULT → ~/persons/knowledge/obsidian`
  - `scripts/cron/{lint,fold,dashboard}.sh` 接 `--vault/--lang/--settings/--dry-run` CLI 参数
  - `scripts/install_cron.sh` 生成 cron 行内嵌 `CORTEX_VAULT="${VAULT}"`
  - `hooks/_lib/cortex_locale.py` `detect_vault_lang()`
  - `hooks/_lib/save_session.py` `read_vault_lang()`
  - `lint/run.py` `--lang` arg + `_load_vault_lang` fallback

- 现状: `~/.cortex/` **不存在**; 用户每次手动 export 或在 cron 内写死

## Requirements

### 配置文件
- 路径: `~/.cortex/config.json`
- Schema (扁平 JSON, 全字段可选):
  ```json
  {
    "vault": "/abs/path/to/vault",
    "lang": "zh-CN",
    "settings": "/abs/path/to/claude-settings.json",
    "install_path": "/abs/path/to/cortex-plugin"
  }
  ```
- JSON 损坏 → fail-fast (stderr 明指修复路径)
- 字段缺失 → 视为未配置, 走 fallback 链 (不报错)

### 优先级 (lookup 顺序)
- vault: `CORTEX_VAULT` env > config.vault > `OBSIDIAN_VAULT` env > **报错退出** (移除 `~/persons/knowledge/obsidian` 硬编码)
- lang: `CORTEX_LANG` env > vault `_meta/version.json` > config.lang > "zh-CN" (现有默认)
- settings: `CORTEX_SETTINGS` env > config.settings > `~/.claude/settings.glm-4.5-flash.json`
- install_path: `CORTEX_INSTALL_PATH` env > config.install_path > 内置 detect 链

### 消费点接入 (MVP 全量)
- **cron**: `scripts/cron/run.sh` (vault/lang/settings fallback) + `lint.sh/fold.sh/dashboard.sh` (透传)
- **install**: `scripts/install_cron.sh` (读 config 作 prompt 默认值)
- **hooks**: `hooks/_lib/cortex_locale.py::detect_vault_lang` + `save_session.py::read_vault_lang` (lang fallback)
- **lint**: `lint/run.py::_load_vault_lang` (lang fallback)

### CLI
- `cortex config get <key>` — 输出单字段
- `cortex config set <key> <value>` — 写单字段 (验证路径存在性)
- `cortex config init` — 交互式 (AskUserQuestion 风格交互) 或非交互 (`--vault X --lang Y ...`)
- `cortex config validate` — 检查 JSON 合法 + 路径存在 + lang code 合法 (供 doctor 调)

### doctor 集成
- `cortex-doctor` 增项: config 存在性 + `config validate` 输出 + `~/.cortex/scripts/` wrapper 完整性

### Wrapper 脚本 (`~/.cortex/scripts/`)
七件 thin proxy, 生成时嵌入绝对 marketplace 路径:
- `lint.sh` → `<marketplace>/cortex/scripts/cron/lint.sh "$@"`
- `fold.sh` → `<marketplace>/cortex/scripts/cron/fold.sh "$@"`
- `dashboard.sh` → `<marketplace>/cortex/scripts/cron/dashboard.sh "$@"`
- `doctor.sh` → `<marketplace>/cortex/scripts/cortex_doctor.py "$@"` (或对应入口)
- `install_cron.sh` → `<marketplace>/cortex/scripts/install_cron.sh "$@"`
- `config.sh` → `python3 <marketplace>/cortex/scripts/cortex_config.py "$@"`
- `update.sh` → 内嵌 `claude plugins marketplace update ccplugin-market && claude plugins update cortex@ccplugin-market`

Wrapper 模板要求:
- shebang `#!/usr/bin/env bash` + `set -euo pipefail`
- 顶部注释标记 `# generated by cortex install.sh on <ISO date>; do not edit by hand`
- 嵌入路径写死, 不依赖 `$PATH` 或动态 detect

### 一键安装 (`install.sh`)
插件根 `plugins/tools/cortex/install.sh`, 人机交互友好:
- 探测 marketplace 安装路径 (复用 `CORTEX_INSTALL_PATH` 逻辑)
- 交互式 prompt: vault 路径 / lang / settings (env 已设则预填)
- 写 `~/.cortex/config.json`
- 生成 `~/.cortex/scripts/*.sh` 七件套 (chmod +x)
- 可选: 询问是否安装 cron (调 wrapper 的 install_cron.sh)
- 末尾打印 next step (e.g. `~/.cortex/scripts/doctor.sh`)
- `--non-interactive` flag 走纯 env 模式

## Technical Notes

### 关键路径
- 配置: `~/.cortex/config.json`
- Wrapper 目录: `~/.cortex/scripts/` (用户面入口)
- Python loader 位置: `plugins/tools/cortex/hooks/_lib/cortex_config.py` (新)
- Bash loader 位置: `plugins/tools/cortex/scripts/lib/config.sh` (新, source 自 cron/install 脚本)
- CLI 入口: `plugins/tools/cortex/scripts/cortex_config.py` (新, 经 `cortex config ...` 调度)
- 一键安装: `plugins/tools/cortex/install.sh` (新, 根级入口)
- Wrapper 模板/生成器: `plugins/tools/cortex/scripts/install_wrappers.sh` (新, 被 install.sh 调) 或内嵌 install.sh

### 实现要点
- Python: stdlib `json` + `pathlib`; 暴露 `load_config() -> dict` + `resolve(key, env_var, fallback) -> value`
- Bash: `jq -r '.vault // empty'` 读字段; 文件不存在 / jq 失败 → 视为空 config (但 jq 语法错时 fail-fast)
- `cortex config` CLI 复用 Python loader
- `set` 路径验证: `vault/settings/install_path` 检查 `Path.exists()`; `lang` 验证 locale code 格式
- `init` 交互模式优先环境变量预填默认值
