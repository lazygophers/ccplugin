# PRD — ~/.cortex/scripts/ 暴露完整 cortex 能力

## 背景

现 `~/.cortex/scripts/` 含 7 文件,调 claude 的仅:
- `doctor.sh` (cortex-doctor)
- `lint.sh` (cron 模式, exec cron/lint.sh)
- `fold.sh` (cron 模式)
- `dashboard.sh` (cron 模式)

cortex 插件 13 个 skill 中,核心能力 (ingest / search / save / refactor) **无命令行 wrapper**,用户只能在 claude 会话内调 skill,不能 shell 直跑。

## 目标

`~/.cortex/scripts/` 加 4 新 wrapper + 扩 lint.sh:

1. `ingest.sh <source>` — 调 cortex-ingest skill,自动判 url/file/git/dir
2. `search.sh <query>` — 调 cortex-search skill
3. `save.sh <kind> <title>` — 调 cortex-save skill,stdin 读 body
4. `refactor.sh <subcommand> [args...]` — 调 cortex-refactor skill,透传子命令
5. `lint.sh [--fix]` — 加 `--fix` flag,触发交互修复模式 (调 cortex-lint skill 走 AskUserQuestion 流程,而非 cron 模式)

## 范围

### 修改

- `plugins/tools/cortex/scripts/install_wrappers.sh` — 加 4 个 emit 段 (ingest/search/save/refactor),lint.sh emit 改支持 --fix 分支
- `plugins/tools/cortex/tests/bash/test_install_wrappers.sh` — 新 wrapper 验证

### 不在范围

- 不动 mcp/ / hooks/ / install.sh / P0-P6 已交付
- skill SKILL.md 本身不动 (skill 已具备能力, wrapper 只是入口)

## 详细规范

### 1. wrapper 通用模板

参照现 doctor.sh:

```bash
#!/usr/bin/env bash
# generated by cortex install on <ISO>; do not edit by hand
set -euo pipefail
SKILL_PATH="<INSTALL_PATH>/skills/<skill-name>/SKILL.md"
[[ ! -f "$SKILL_PATH" ]] && { echo "<skill-name> SKILL.md missing: $SKILL_PATH" >&2; exit 1; }
LIB_PATH="<INSTALL_PATH>/scripts/lib/stream_progress.sh"
[[ ! -f "$LIB_PATH" ]] && { echo "stream_progress.sh missing: $LIB_PATH" >&2; exit 1; }
source "$LIB_PATH"
export CORTEX_JOB_LABEL="cortex-<skill>"
cortex_stream_runner claude --bare -p \
  --append-system-prompt "$(cat "$SKILL_PATH")" \
  "<prompt 含 $@>" "$@"
```

### 2. ingest.sh

```bash
emit ingest.sh "$(cat <<EOB
SKILL_PATH="$INSTALL_PATH/skills/cortex-ingest/SKILL.md"
...
cortex_stream_runner claude --bare -p \\
  --append-system-prompt "\$(cat "\$SKILL_PATH")" \\
  "摄取以下源到 cortex vault. 源: \$* (自动判断 url/file/git/dir). 按 cortex-ingest skill 流程: url_security → fetch/read → html_sanitize → masking → save."
EOB
)"
```

用法:
```bash
~/.cortex/scripts/ingest.sh https://example.com/article
~/.cortex/scripts/ingest.sh ~/Downloads/paper.pdf
~/.cortex/scripts/ingest.sh https://github.com/foo/bar.git
~/.cortex/scripts/ingest.sh ~/notes/  # 目录
```

### 3. search.sh

### 4. save.sh

```bash
emit save.sh "$(cat <<EOB
SKILL_PATH="$INSTALL_PATH/skills/cortex-save/SKILL.md"
...

  --append-system-prompt "\$(cat "\$SKILL_PATH")" \\
  "落档到 cortex vault: kind=\$KIND, title=\$TITLE. body 如下 (经 masking 后写盘):\\n\\n\$BODY"
EOB
)"
```

用法:`echo "some finding" | ~/.cortex/scripts/save.sh concept "auth pattern"`。

### 6. lint.sh 加 --fix 模式

现 lint.sh 是 `exec bash $INSTALL_PATH/scripts/cron/lint.sh "$@"` thin wrapper。

改:

```bash
emit lint.sh "$(cat <<EOB
# cron 模式 (无 --fix): exec cron/lint.sh (read-only, JSON 输出)
# 交互修复模式 (--fix): cortex_stream_runner + cortex-lint skill (AskUserQuestion)
if [[ "\${1:-}" == "--fix" ]]; then
  shift
  SKILL_PATH="$INSTALL_PATH/skills/cortex-lint/SKILL.md"
  [[ ! -f "\$SKILL_PATH" ]] && { echo "cortex-lint SKILL.md missing: \$SKILL_PATH" >&2; exit 1; }
  LIB_PATH="$INSTALL_PATH/scripts/lib/stream_progress.sh"
  [[ ! -f "\$LIB_PATH" ]] && { echo "stream_progress.sh missing: \$LIB_PATH" >&2; exit 1; }
  source "\$LIB_PATH"
  export CORTEX_JOB_LABEL="cortex-lint-fix"
  cortex_stream_runner claude --bare -p \\
    --append-system-prompt "\$(cat "\$SKILL_PATH")" \\
    "对 cortex vault 跑 lint 并 --fix 交互修复. 按 cortex-lint skill 工作流: 跑 run.py 获 JSON 报告 → 解析 errors[] → vault-structure-violation 各项用 AskUserQuestion 工具询问 (move/delete/whitelist/skip), 其它 autofix 项直接落. \$*"
else
  exec bash "$INSTALL_PATH/scripts/cron/lint.sh" "\$@"
fi
EOB
)"
```

### 7. install_wrappers.sh 改动汇总

替换 line 95 现:
```bash
emit_exec lint.sh           "bash \"$INSTALL_PATH/scripts/cron/lint.sh\""
```

改为完整 emit 模板 (含 --fix 分支)。

在 line 104 后追加 4 个 emit 段 (ingest/search/save/refactor)。

每个 wrapper 都注入 `CORTEX_JOB_LABEL` 给 stream_progress.sh 用。

## 验收

1. `bash ./plugins/tools/cortex/install.sh --use-source` 跑后 `~/.cortex/scripts/` 含:
   - doctor.sh / lint.sh / fold.sh / dashboard.sh (现有)
   - ingest.sh / search.sh / save.sh / refactor.sh (新)
2. `bash -n ~/.cortex/scripts/*.sh` 全绿
3. `~/.cortex/scripts/search.sh "test"` → rich Live UI 启动调 cortex-search skill
4. `~/.cortex/scripts/ingest.sh https://example.com` → rich Live UI 调 cortex-ingest
5. `echo "hi" | ~/.cortex/scripts/save.sh log "test"` → rich Live UI 调 cortex-save
6. `~/.cortex/scripts/refactor.sh rename a.md b.md` → rich Live UI 调 cortex-refactor
7. `~/.cortex/scripts/lint.sh --fix` → 走 cortex-lint skill 交互模式
8. `~/.cortex/scripts/lint.sh` (无 --fix) → 走 cron/lint.sh JSON 输出 (现状不变)
9. `bash plugins/tools/cortex/tests/run.sh` 不回归 (215 python + 8 bash pre-existing)
10. tests/bash/test_install_wrappers.sh 加新 wrapper 存在验证

## 不变量

- 纯 bash, bash 3.2 兼容
- 所有 wrapper `set -euo pipefail`
- 通过 cortex_stream_runner 走 rich Live (与 doctor.sh 一致)
- SKILL_PATH 存在性检测 + 错误退出 (1)
- CORTEX_JOB_LABEL 设置每 wrapper
- 不破坏 emit / emit_exec 函数本身
- cron 模式 lint.sh 不变 (向后兼容)

## 风险

- **save.sh stdin 大文本注入 prompt**: claude prompt 长度限,大 body 截断. **缓解**: 文档建议短文本;长文本用 `cortex_save` MCP tool 直调
- **refactor.sh dry-run 默认**: 用户必须明示 --apply 才落. cortex-refactor SKILL.md 已内置该 contract
- **ingest.sh 自动判断 source 类型**: 依赖 skill 内逻辑 (url 匹配 ^https?:// / file 路径存在 / git URL 匹配 .git$). 已是 cortex-ingest 现有职责
- **lint.sh --fix 非 tty 行为**: --fix 走 AskUserQuestion, 必须 tty. 检测 `[[ -t 0 ]]` 加 guard? 简化: skill 内会失败明示, 不在 wrapper 层加
