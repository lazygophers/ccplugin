# promo-posts — 多平台推广文案生成框架

同一件东西, 在不同平台要用不同的"语言"。GitHub 上读者是已在评估代码的工程师, 你给他营销腔会掉头就走; 小红书上读者在刷生活灵感, 你甩技术参数她划走。**每个平台是一个不同的房间, 进门先守那个房间的规矩。** 本 skill 把"同一素材 → 各平台地道文案"的改写落地为可执行流程。

> 本 skill **只生成文案**, 不发布、不调 API、不自动发帖。产出 = 各平台可直接复制粘贴的帖子文本(+ 配图/标签建议)。

## 核心戒律(四条)

1. **先给价值, 再谈自己。** 任何社区的反硬广本质都是"你在索取(求关注/求点击)还是在给予(干货/灵感/解决方案)"。所有平台通吃的一条: 帖子主体是读者能拿走的东西, 推广是副产品。
2. **守房间规矩。** 字数、标题、标签、调性、CTA 形态各平台不同, 照搬一稿到处发 = 哪里都不地道。
3. **不编算法民俗。** "外链降权 50%""hashtag 超 3 个触发 spam""reply 权重 27 倍"等多是第三方博客推断, 平台官方未证实(X 官方开源算法已移除手工特征)。**这类按"实操建议"写, 绝不当成平台铁律断言。** 能断言的只有官方明文(字数限制、明文 spam 政策)。
4. **去 AIGC 化, 写得像真人。** 每一篇都要过 AI 腔自检——删套路连接词(首先/其次/综上/In conclusion)、打破句长均匀、注入只有真人知道的真实细节(数字/踩坑/场景)、套平台方言。多数平台已打压 AI 空泛内容, 人味同时过 AI 检测、过平台算法、过读者。**最稳的去 AI 味是注入真实信息, 不是靠模板。** 不承诺"保证过检测"(检测是概率)。详见 `references/anti-ai-detection.md`。

## 第一动作: 提素材内核(所有平台共用)

写任何平台前, 先把素材榨成这 5 个槽位——这是后续所有改写的原料:

| 槽位 | 问 | 例 |
| --- | --- | --- |
| **一句话价值** | 它帮谁、解决什么、凭什么 | "一个让 Claude Code 多平台改写推广文案的 skill" |
| **目标受众** | 谁会关心、在什么场景 | 独立开发者 / 内容创作者 |
| **可信证据** | demo / 数据 / 截图 / 对比 / 真实体验 | GIF、star 数、before-after |
| **CTA 目标** | 你想让读者做什么(单一) | 试用 / star / 收藏 / 讨论 / 关注 |
| **禁忌** | 不能说什么(夸大 / 违禁词 / 引流红线) | 广告法极限词、未证实疗效式宣称 |

> 🔴 **CHECKPOINT(改写前)**: 5 槽位必须填满, 尤其"可信证据"——没有证据的推广在任何技术社区都会被当软广, 且**真实细节也是去 AI 腔的核心原料**(AI 编不出私有数字/踩坑)。缺证据 → 先问用户要(截图/数据/真实使用故事), 不要替用户编造。

## 工作流 A: 一键多平台全套(按序)

| 步 | 动作 |
| --- | --- |
| 1 | **提内核**(上方 5 槽位) |
| 2 | **选平台**: 用户没指定 → 据素材性质推荐(开源工具→dev 阵营; 消费品/生活向→种草阵营; 都沾→两边各出) |
| 3 | **逐平台适配**: 对每个平台查"平台调性速查"对应行 + 读 `references/` 详档, 各出一篇 |
| 4 | **逐篇自检**: 过该平台禁忌 + 通用反硬广黑名单 + **去 AI 腔自检**(`references/anti-ai-detection.md`); 标题/字数/标签是否合规 |
| 5 | **打包交付**: 每篇标注「平台 / 标题 / 正文 / 标签 / 配图建议 / CTA」, 用户复制即用 |

## 工作流 B: 单平台精修(按序)

1. 确认平台 + 读 `references/` 该平台详档。
2. 提内核(若未提)。
3. 按该平台**标题公式 + 开头 hook + 正文结构 + CTA 形态**逐段打磨(方法见 `references/playbook.md`)。
4. 过该平台头号禁忌 + 违禁词(种草/公众号尤其查广告法极限词)+ 去 AI 腔自检。
5. 给 2-3 个标题备选(标题决定打开率)。

## 平台调性速查

> 字数为"约值/社区惯例", 仅 X 字符限制是官方明文(280 / Premium 25000)。其余以平台当前实际为准。

### 开发者 / 技术社区阵营(调性: 真诚、可验证、低营销味、show-don't-tell)

| 平台 | 受众/形态 | 体量 | 标题/开头 | 标签 | CTA 得体形态 | 头号禁忌 |
| --- | --- | --- | --- | --- | --- | --- |
| **GitHub** | 评估代码的工程师(README/Release/Discussion) | README 第一屏定生死 | 一句话价值 + demo GIF | 精选 badge(version/build/license) | "Star if useful""Quick start""Contributions welcome" | 刷 star / 奖励换 star / 去别人 issue 发广告(**违反 AUP**) |
| **linux.do** | 中文开发者论坛(Discourse, 信任等级制) | 中长帖, 重正文 | 朴实直述, 不标题党 | 选对分类节点 | 真诚分享 + 邀讨论 | 纯硬广 / 伸手党式引流 → 被踩被封 |
| **Twitter/X** | 英文技术圈 | 单帖 **280 字符**(Premium 25000) | 首条 = hook(bold claim/具体承诺), 不放链接 | 0-1 个 hashtag | 外链放**回复第 2 条**; 收尾引 reply | 首条堆链接 + 多 hashtag; 营销腔 |
| **Hacker News** | 极挑剔的技术读者(Show HN) | 标题 + 简短正文 | `Show HN: 工具名 – 一句话做什么`(no clickbait) | 无标签体系 | 帖内老实说做了啥、求反馈 | 标题党 / 营销腔 / 夸大 |
| **Reddit** | 细分 subreddit | 视 sub 而定 | 具体、不党 | 守 subreddit 规则 | 先贡献再偶尔自荐(**9:1 自荐潜规则**) | 无视 sub 规则刷自我推广 → 封 |
| **V2EX** | 中文极客 | 中短帖 | 平实 | 发对节点(如「分享创造」) | 分享创造心态 | 错节点 / 硬广引流 |
| **掘金** | 中文技术写作社区 | 技术长文 | 干货式标题 | 技术标签 | 文末轻提项目 | 通篇软文无干货 |

| 平台 | 受众/形态 | 体量 | 标题/开头 | 标签 | CTA 得体形态 | 头号禁忌 |
| --- | --- | --- | --- | --- | --- | --- |
| **小红书** | 找生活/消费灵感(笔记) | 标题约 ≤20 字, 正文短、分段、emoji | 痛点/利益钩子 + emoji, 标题决定打开 | 话题 # 若干(垂直相关) | "姐妹冲""码住""评论区问我" | 广告法极限词(最/第一/国家级)、硬导流外链 → 限流 |
| **知乎** | 求专业解答(回答/文章/想法) | 长文干货, 问答体 | 先给结论/反常识钩子 | 话题标签 | 利益相关声明 + 干货中自然带出 | 软文无干货 / 抖机灵 / 不声明利益相关 |
| **微信公众号** | 订阅读者(文章) | 长文, 重排版 | 标题决定打开率(克制党) | 无公开标签 | 文末引导(在看/转发/阅读原文) | 诱导分享(平台限制)、过度标题党 |
| **即刻** | 轻社交动态(圈子) | 短动态 | 轻松真诚口语 | 发对圈子 | 朋友安利口吻 | 端着 / 硬广腔 |

## 两大阵营的根本差异(选错调性=全废)

| 维度 | 开发者/技术社区 | 大众种草/问答 |
| --- | --- | --- |
| 读者想要 | 可验证的技术价值、能跑的代码 | 情绪共鸣、生活解决方案、灵感 |
| 信任来源 | demo / 数据 / 开源 / 老实 | 真实体验 / 人设 / 细节场景 |
| 营销味容忍度 | 极低(salesy 即扣分) | 中(但要"软", 极限词违法) |
| 标题 | 具体、不党 | 钩子、痛点、利益前置 |
| 最致命错误 | 夸大 + 营销腔 | 干巴巴讲参数、像广告 |

## 反硬广通用黑名单(跨平台禁做)

| 禁 | 为什么 | 替代 |
| --- | --- | --- |
| 一稿到处复制粘贴 | 每个平台调性不同, 哪里都不地道 | 提一次内核, 各平台各改写 |
| 通篇夸自己无读者价值 | 索取型内容被反感/折叠 | 主体是读者能拿走的干货/灵感, 推广做副产品 |
| 把未证实算法数字当铁律 | "降权 X%""权重 N 倍"多是博客推断 | 标为"实操建议"; 只断言官方明文 |
| 开发者平台用营销腔 | 技术圈极敏感, 直接扣分 | show-don't-tell, 给 demo 和数据 |
| 小红书/公众号用极限词 | "最/第一/国家级"违广告法 → 限流删帖 | 换体验化表达("我用下来最顺手"亦慎用) |
| 编造数据/好评/使用故事 | 翻车毁信任, 违规 | 证据不足就问用户要, 没有就不写 |
| GitHub 求 star / 奖励换 star | **违反 GitHub AUP**, 可删号 | "Star if useful" + 真实试用/贡献引导 |
| 硬塞外链导流 | 多平台限流/折叠外链 | 链接放回复/文末, 主体先给价值 |
| 标题党货不对板 | 短期打开高, 长期毁号 | 钩子要兑现, 标题与正文一致 |
| 满篇 AI 腔(首先/其次/综上/强大高效/seamless/game-changer) | 被 AI 检测判定 + 平台打压 AI 空泛内容 + 读者反感 | 删套话、打破句长、注真实细节、套平台方言(`anti-ai-detection.md`) |
| 靠加错别字/乱符号骗 AI 检测 | 伤可读性, 平台反判低质 | 用真实信息+口语化去 AI 味, 不破坏通顺 |

## 失败模式(触发 → 一线修复 → 仍失败兜底)

| 触发 | 一线修复 | 仍失败兜底 |
| --- | --- | --- |
| 素材缺可信证据(无 demo/数据/截图) | 🔴 停, 问用户要真实证据(截图/数字/使用故事) | 用户给不出 → 只写「能验证的部分」, 缺证据处留白, 禁编造数据/好评 |
| 文案被平台判软广 / 折叠 | 砍推广主张, 主体换成读者能拿走的干货/灵感, 推广降副产品 | 仍折叠 → 该平台改纯价值贴, CTA 移到评论/文末 |
| 种草/公众号误用广告法极限词(最/第一/国家级)触限流 | 换体验化表达(「我用下来顺手」), 删绝对化词 | 拿不准是否违禁 → 标「疑似极限词, 需再核」, 不硬发 |
| 去 AI 腔后仍疑被 AI 检测判定 | 注真实私有细节(数字/踩坑/场景)+ 打破句长均匀, 不靠错别字 | 检测是概率 → 告知用户「已降特征但不保证过检」, 不承诺 |
| 平台规则/字数/审核已变(速查表过时) | 以平台当下实际为准, 约束按官方明文重新校准 | 无法核实当下规则 → 标「规则可能已变, 发前按目标平台二次校准」 |

## 适用边界

- 本 skill 管**文案生成**, 不管发布、不管投流、不保证转化或流量(平台算法不可控且多为黑箱)。
- 平台规则/字数/审核**变动快**: 仅 X 字符限制(280/25000)与 GitHub AUP 红线为本轮核实的官方明文(2026-06); 其余体量/标签为社区惯例, 以平台当前实际为准。
- "怎么发更火"的时机/运营技巧不是本 skill 重点(用户只需文案); 速查表只保留影响**文案写法**的约束。
