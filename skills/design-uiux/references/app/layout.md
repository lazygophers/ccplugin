# App 媒介 · 布局与信息架构

App 布局受平台习惯强约束。先定平台 → 再定信息架构 → 再选导航模式。场景适配见 [scenes.md](scenes.md)，组件见 [components.md](components.md)；色彩 / 风格在姊妹 skill `design-color`。

## 平台习惯差异（选型硬约束）

### iOS vs Android

| 维度 | iOS | Android |
|------|-----|---------|
| 导航 | 底部 Tab + 栈推送；返回靠系统手势 | 顶栏汉堡或底部 Tab；返回靠系统返回键 |
| 触控目标 | ≥ 44pt | ≥ 48dp |
| 字体 | San Francisco | Roboto |
| 圆角 | 大圆角（12-16） | Material 圆角阶（4/8/16） |
| 模态 | 从底部滑入 Sheet | 居中 Dialog 或 Bottom Sheet |
| 色彩 | 系统强调色跟随主题 | Dynamic Color 取壁纸 |
| 列表 | 右滑出操作 | 长按出菜单 / SwipeDismiss |

原则：**遵循平台 > 跨平台一致**。跨平台框架（Flutter / RN）也尽量适配各自平台。

### 桌面应用

- 多窗口 / 多面板（侧栏 + 主区 + 检视器），利用宽屏
- 菜单栏（macOS 顶部 / Windows 应用内）
- 键盘快捷键一等公民（Cmd/Ctrl + 字母）
- 右键菜单、拖拽（文件 / 面板重排）
- 高密度信息（小字号、紧凑间距可接受）

## 信息架构

- 主流程 ≤3 步到达核心功能
- 次级功能藏合理层级，别堆首页
- 内容优先级：首屏放最高频任务
- 层级深度：移动 ≤3 层，桌面可更深（有面包屑）

## 导航模式选择

| 模式 | 何时用 |
|------|--------|
| 底部 Tab（2-5 主区） | 主功能并列、需快速切换 |
| 侧栏抽屉 | 功能多、次级深、主区 ≤1 |
| 顶部 Segmented | 2-3 同级视图切换 |
| 栈式推送 | 线性流程（设置层级、详情展开） |
| 桌面侧栏 | 多模块后台、文档应用 |

当前位置必须明显——活跃态色 + 字重，不只靠下划线。

## 屏幕布局骨架

| 平台 | 顶栏 | 底栏 | 内容区 |
|------|------|------|--------|
| iOS | NavBar（标题居中、左右行动） | TabBar（主区切换） | 单栏滚动 |
| Android | TopAppBar（标题左、行动右） | BottomBar 或悬浮 FAB | 单栏滚动 |
| 桌面 | 标题栏 + 菜单 | 状态栏（可选） | 多面板栅格 |
