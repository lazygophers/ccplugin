# TUI 媒介 · 场景自适应

TUI 要在不同终端能力、尺寸、主题下都可用。场景轴决定何时切换。布局见 [layout.md](layout.md)，组件见 [components.md](components.md)；色彩 / 风格在姊妹 skill `design-color`。

## 场景轴速查

| 场景轴 | 切换什么 | 触发 |
|--------|---------|------|
| 色域 | 调色板（真彩→256→16） | 终端能力检测 |
| 亮 / 暗主题 | 中性阶、前景背景对调 | 终端背景 / 手动 |
| 窗口尺寸 | 布局折叠 / 滚动 | resize 事件 |
| 鼠标支持 | 启用 / 禁用鼠标交互 | 终端能力 + 用户 |
| 动效偏好 | 动画 / 转场启用 | 配置 |
| 非交互 | 退出到纯文本输出 | 非 TTY / 管道 |

## 色域检测与降级

| 色域 | 覆盖 | 选择 |
|------|------|------|
| 8/16 色 | 基本所有终端 | 最大兼容、朴素 |
| 256 色 | 现代终端 | 推荐默认 |
| 真彩色（24bit） | 多数现代终端 | 丰富，需降级链 |

- 检测 `$TERM` / `COLORTERM=truecolor` 判能力
- 降级链：真彩 → 256 → 16，禁假设用户终端支持某色
- 语义色固定：error=红、success=绿、warning=黄、info=蓝

## 亮 / 暗主题

- 读终端背景（或让用户选）
- 暗底用亮文字、亮底用暗文字
- 用主题 token（`--tui-bg` `--tui-fg`）切换，禁硬编码

## 窗口 resize

- 监听 `SIGWINCH`（或框架的 resize 事件）
- 重绘：只重绘变化区域（diff rendering），整屏重绘会闪
- 窄屏折叠：三栏 → 双栏 → 单栏
- 极窄（< 80 宽）给提示「请放大窗口」而非硬崩

## 鼠标场景

- 可选支持（部分终端 / 复制模式冲突）
- 键盘为主、鼠标为辅
- 关键操作必须有键盘等价

## 非交互降级

- 非 TTY（管道 / 重定向）退出到纯文本输出
- 禁在全屏 TUI 模式下被管道读（输出乱）
- `tool list | grep` 时 `tool` 应输出纯文本
