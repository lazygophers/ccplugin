# 图像设计模式

产出固定尺寸 HTML，导出 PNG。一张图一个 HTML。

## 物料规格

| 物料 | 尺寸 | 要点 |
|------|------|------|
| 文章封面 | 16:9 / 2.35:1 / 1:1 | 主视觉 + 标题，留 logo 位 |
| 文章配图 | 内容驱动 | 一图一概念，与段落语义对应 |
| 小红书卡片 | 1080×1440（3:4） | 系列卡片统一视觉系统；封面卡 + 内容卡 + 尾卡 |
| 海报 | A3 / A4 / 1080×1920 | 层级、留白、焦点 |
| 邀请函 / 证书 | 按媒介 | 装饰适度，信息优先 |

## 五维选型

每张图定这五个维度（三方向硬门时，三个方向就在五维上做差异化组合）：

| 维度 | 选项示例 |
|------|---------|
| **类型** | 摄影 / 插画 / 几何 / 抽象 / 字体为主 |
| **色板** | 见 [design-styles.md](design-styles.md) 配色色系 |
| **渲染风格** | 扁平 / 拟物 / 线条 / 渐变 / 噪点 / 拼贴 |
| **文字处理** | 无字 / 标题为主 / 数据标注 / 排版为主 |
| **情绪** | 冷静 / 热烈 / 神秘 / 温暖 / 科技 / 复古 |

## 系列物料（小红书卡片 / 多图文章）

系列必须统一视觉系统：

- 统一色板（同一套主色 + 辅助色贯穿全系列）
- 统一字体阶（标题 / 正文 / 标注字号全系列一致）
- 统一栅格（卡片边距、栏宽、安全区固定）
- 统一图标 / 装饰语言（线性 or 填充，不混）
- 差异化只在内容与主视觉，框架不动

结构：封面卡（抓眼球、定调）→ 内容卡（每张一要点）→ 尾卡（总结 / CTA）。

## 排版层级（图像内的视觉动线）

海报 / 封面类，层级必须清晰：

1. **主视觉 / 主标题**（最大权重，视觉入口）
2. **副信息**（次级、支撑）
3. **细节 / 落款**（最小、角落）

自检：眯眼看图，三秒内能否抓到主信息？抓不到 = 层级没拉开。

留白是主角不是配角——信息塞满 = 没有焦点。

## 文字在图上

- 标题字号阶梯：主 72+ / 副 36+ / 标注 18+
- 文字与背景对比足够（暗底白字、亮底深字，避免中灰对中灰）
- 中文字号比英文大一档才好读
- 标题用字重拉开层级（700 vs 400），而非全是 500

## 导出

- headless Chrome 按固定 viewport（= 图像尺寸）截图 → PNG
- 设备像素比 `deviceScaleFactor: 2` 出高清（ retina ）
- 批量系列卡片 → 脚本遍历每个 HTML 截图，见 [export-pipeline.md](export-pipeline.md)
- 需透明背景 → `body { background: transparent }` + PNG alpha
