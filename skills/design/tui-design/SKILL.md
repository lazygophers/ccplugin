# TUI-Design · 终端 UI 设计

TUI 是终端里的图形界面：全屏、字符格栅、键盘为主、256/真彩色。约束硬（字符格子、有限色、终端差异），设计就是在这些约束里做信息架构与交互。

## 适用 / 不适用

适用：全屏终端应用（文件管理器、git 客户端、仪表盘、邮件客户端、IDE）。

不适用：一次性命令行工具（→ cli-design）、Web（→ html-design）、App（→ app-design）。

## 核心硬门 · 三方向初稿（100% 必走）

新 TUI 设计先出**三个差异化布局方案**（如：双栏 Master-Detail vs 三栏侧栏-主-检视 vs 顶部 Tab 多视图），等用户选定。

## 决策路由

| 决策层 | 选什么 |
|--------|--------|
| 框架 | textual(Python) / bubbletea(Go) / ratatui(Rust) / curses |
| 布局 | 单栏 / 双栏 Master-Detail / 三栏 / 顶部 Tab 多视图 |
| 焦点流转 | 栏间切换、栏内上下、模态切换 |
| 色彩 | 16 色 / 256 色 / 真彩色，主题适配终端 |
| 键位 | 全键盘可达、助记符、vim 风格 vs 通用 |
| 输出 | 纯交互 vs 命令面板混合 |

## 框架选型

| 框架 | 语言 | 特点 |
|------|------|------|
| Textual | Python | 现代异步、CSS 样式、组件丰富、易上手 |
| Bubble Tea | Go | Elm 架构（Model-Update-View）、轻快 |
| Ratatui | Rust | 性能、安全、细粒度控制 |
| ncurses | C/FFI | 老牌、底层、跨终端兼容性最强 |
| Ink | JS/TS | React 模型写 CLI/TUI |

选型看：宿主语言生态、性能要求、团队熟悉度、是否需 CSS 样式层。

## 布局模式

| 布局 | 何时用 |
|------|--------|
| 单栏（列表 + 底栏） | 简单浏览、菜单 |
| 双栏 Master-Detail | 列表选条目、右侧看详情（邮件、文件） |
| 三栏（侧栏-主-检视） | 复杂应用、IDE、数据库工具 |
| 顶部 Tab 多视图 | 多独立功能、切换不丢上下文 |
| 底部命令栏 | 命令面板 + 交互混合（vim / emacs 风） |

### 栅格约束

- 最小尺寸假设：80×24（经典终端），设计在该尺寸不塌
- 响应式：宽屏展开多栏，窄屏折叠堆叠或隐藏次要栏
- 留边距（padding 1-2 字符），贴边拥挤
- 分隔线用 `─│┌┐└┘├┤` box-drawing 字符

## 组件

| 组件 | 说明 |
|------|------|
| 列表 List | 可选中、可滚动、高亮当前、标记状态 |
| 表格 Table | 列对齐、可排序、长内容截断+省略 |
| 树 Tree | 层级、可折叠 |
| 表单 Form | 输入框、选择、校验、焦点流转 |
| 底栏 StatusBar | 当前模式、光标位置、快捷键提示 |
| 顶栏 TabBar | 多视图切换标记 |
| 弹层 Modal/Overlay | 确认、搜索、命令面板 |
| 进度 Progress | 进度条、spinner |
| 滚动条 | 内容超屏时可见滚动位置 |

### 状态完整性

default / hover(终端无鼠标时可省) / focused / selected / disabled / loading / empty / error。

## 焦点流转

TUI 核心是键盘焦点导航。设计明确：

- **栏间切换**：`Tab` / `Shift+Tab` 正逆向；或 `hjkl` / 方向键
- **栏内导航**：`j/k` 或 `↑↓` 上下，`g/G` 顶底，`PgUp/PgDn` 翻页
- **模态切换**：进入编辑模态（插入）、退出（ESC）、命令模态（`:`）
- **焦点可见**：当前焦点栏 / 控件必须有明显视觉标记（反色 / 边框高亮 / 色块）

焦点不可见 = 用户不知道在哪。这是 TUI 最常见可用性问题。

## 色彩约束

### 色域选择

| 色域 | 覆盖 | 选择 |
|------|------|------|
| 8/16 色 | 基本所有终端 | 最大兼容、朴素 |
| 256 色 | 现代终端 | 推荐默认，色彩够用 |
| 真彩色（24bit） | 多数现代终端 | 丰富，但需检测降级 |

- **检测终端能力**降级：真彩→256→16，禁假设用户终端支持某色
- **亮 / 暗主题适配**：读终端背景（或让用户选），暗底用亮文字、亮底用暗文字
- **语义色固定**：error=红、success=绿、warning=黄、info=蓝，跨主题一致
- **对比足够**：终端字体小，低对比更难读，别用深灰底深灰字

### 色彩克制

终端色滥用易花。原则：

- 默认色（无背景）为主，强调色点缀
- 背景色只用于焦点 / 选中 / 状态标记
- 全屏彩色背景疲劳，小范围用

## 键位设计

| 风格 | 特点 | 受众 |
|------|------|------|
| Vim 风（hjkl/模态） | 高效、免方向键、学习陡 | 开发者、power user |
| 通用风（方向键/Tab） | 直觉、低学习成本 | 普通用户 |
| Emacs 风（C-n/C-p） | 兼容性强、组合多 | 长期用户 |

- 全键盘可达（不依赖鼠标）
- 常用键单键，次级组合键
- 危险操作（删除 / 退出）用组合键或确认，防误触（`q` 退出要小心——别和输入冲突）
- 快捷键提示在底栏常驻（发现性）
- 支持 `?` 弹快捷键帮助

### 命令面板混合

现代 TUI 趋势（如 fzf、helm）：交互式 + 命令面板（`:` 或 `Ctrl+P` 触发搜索执行）。功能多时避免键位爆炸，命令面板收纳长尾操作。

## 终端兼容性

- **resize 处理**：窗口变化重绘，禁崩
- **鼠标支持**：可选，部分终端 / 复制模式冲突，键盘为主、鼠标为辅
- **非交互降级**：管道或非 TTY 下退出到纯文本输出
- **渲染抖动**：只重绘变化区域（diff rendering），整屏重绘会闪
