# 信息架构与导航（无关平台）

信息架构（IA）决定内容怎么组织、用户怎么找到。跨平台通用。原则见 [principles.md](principles.md)，交互见 [interaction-design.md](interaction-design.md)，可用性见 [usability-a11y.md](usability-a11y.md)。

## IA 核心

IA 回答三个问题：

1. **有什么内容 / 功能**（inventory）
2. **怎么分组 / 分层**（taxonomy / hierarchy）
3. **怎么导航到**（navigation / search）

## 内容盘点与分组

- 列全部内容 / 功能，按用户心智模型分组（不是按内部部门）
- 每组有明确边界，交叉处显式归属
- 分组数：同层 5±2 组（米勒数），过多合并、过少拆分

## 层级深度 vs 广度

| 取舍 | 适用 |
|------|------|
| 浅而宽（少层多项） | 内容平权、浏览型（电商、门户） |
| 深而窄（多层少项） | 内容有强层级、检索型（设置、文件系统） |

原则：

- 重要内容 ≤3 次点击到达（规则非铁律，每次点击「该往哪」清晰更重要）
- 每层选项数 ≤7（人短期记忆上限）
- 平衡深度与广度，避免「扁平到塞爆」或「深到迷路」

## 导航模式（跨平台）

| 模式 | 特点 | 平台映射 |
|------|------|---------|
| 全局导航 | 始终可见、主区切换 | Web 顶栏 / App Tab / TUI 顶栏 Tab |
| 局部导航 | 区内子级切换 | 侧栏 / 面包屑 / Segmented |
| 上下文导航 | 相关内容跳转 | 「相关推荐」/ 链接 |
| 搜索 / 命令面板 | 直达，绕过层级 | 搜索框 / `Ctrl-P` / CLI |

### 当前位置可见

- 用户随时知道：在哪、能去哪、来过哪
- 工具：高亮当前导航项、面包屑、页面标题、URL / 路径
- 弱化的导航 = 迷路

## 用户心智模型

- 按用户语言组织，不按系统术语（「我的订单」非「订单记录列表」）
- 卡片分类法：让用户分组，验证 IA 是否符合预期
- 别假设——测试真实用户怎么找东西

## 内容优先级

- 首屏 / 首页：最高频任务 + 入口
- 次级：低频但必要
- 长尾：搜索 / 命令面板兜底
- 移动端更激进：只留核心，次要的藏深一点

## 搜索与筛选

- 内容多（>50 条目）必给搜索
- 搜索框显眼（右上 / 顶部）
- 筛选 / 排序配合列表，帮用户缩窄
- 搜索支持模糊、容错、快捷键

## URL / 路径 / 深链

- 每个可达状态有可分享的地址（Web URL / CLI 命令 / App deep link）
- 深链让用户直达、可回溯、可书签

## 自检清单

- [ ] 分组符合用户心智模型
- [ ] 每层选项 ≤7
- [ ] 重要内容 ≤3 步到达
- [ ] 当前位置始终可见
- [ ] 内容多时有搜索
- [ ] 每状态可深链
