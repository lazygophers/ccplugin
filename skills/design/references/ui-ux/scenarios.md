# 主流场景 UI/UX 设计建议

通用方法论见本目录其余文件。本文件给**具体场景**的专属设计建议——每场景列：核心目标 / 关键页面 / UX 要点 / 常见坑 / 推荐模式。

> 选场景：先在本表定位业务类型 → 拿要点与避坑 → 去对应 medium（[html](../html/INDEX.md) / [app](../app/INDEX.md) / [cli](../cli/INDEX.md) / [tui](../tui/INDEX.md)）落地组件。

## 场景速查

| 场景 | 核心矛盾 | 优先级 |
|------|---------|--------|
| 电商 / 购物 | 转化率 vs 信息密度 | 信任 + 决策效率 |
| SaaS 后台 / Dashboard | 功能密集 vs 认知负荷 | 信息架构 + 一致性 |
| 数据可视化 / BI | 数据准确 vs 可读 | 编码 + 钻取 |
| 内容站 / 博客 / 媒体 | 阅读体验 vs 商业化 | 排版 + 留白 |
| 落地页 / 营销页 | 转化 vs 简洁 | 注意力 + CTA |
| 登录注册 / 认证 | 安全 vs 摩擦 | 最短路径 + 可恢复 |
| Onboarding / 新手引导 | 留存 vs 打扰 | 渐进披露 |
| 搜索 / 筛选 | 召回 vs 精准 | 反馈 + 可修正 |
| 设置 / 偏好 | 灵活 vs 可找 | 分组 + 默认值 |
| 社交 / 信息流 | 粘性 vs 焦虑 | 节奏 + 控制 |
| 教育 / 学习 | 坚持 vs 挫败 | 进度 + 反馈 |
| 长表单 / 数据录入 | 完整 vs 疲劳 | 分段 + 即时校验 |
| 实时协作 / 编辑器 | 并发冲突 vs 流畅 | 在场感知 + 撤销 |
| 通知 / 消息中心 | 及时 vs 打扰 | 分级 + 聚合 |

## 1 · 电商 / 购物

**核心目标**：缩短「看到商品 → 下单」路径，建立信任。

**关键页面**：商品列表（PLP）/ 商品详情（PDP）/ 购物车 / 结账 / 订单确认。

**UX 要点**
- PLP：首图优先、价格显眼、库存状态（「仅剩 3 件」制造稀缺但不骗）、筛选侧栏常驻、排序默认「相关度」。
- PDP：图大可缩放、规格选择即时反馈（选错提示在控件旁非弹窗）、评价区含图、相关推荐「买了这个还买了」。
- 购物车：可改数量不跳页、小计实时、优惠码就近输入、运费预估前置（别到结账才出运费吓人）。
- 结账：**guest checkout 必须有**、步骤 ≤3、地址自动补全、支付方式图标化、进度条可见。
- 信任信号：退货政策、安全支付徽标、真实评价（含差评）、客服入口。

**常见坑**
- 强制注册才下单（流失 No.1）
- 运费 / 税最后一步才露（信任崩塌）
- 评价全是五星无图（假）
- 规格选错靠弹窗报错（打断）

**推荐模式**：Amazon 式详情骨架、Shopify 式结账、Stripe 式支付表单。

## 2 · SaaS 后台 / Dashboard

**核心目标**：功能密集但不压垮用户，让常用动作一步可达。

**关键页面**：概览 Dashboard / 列表 + 详情 / 表单编辑 / 设置。

**UX 要点**
- 导航：左侧常驻 sidebar（≤7 项一级，超出归「更多」）、面包屑、当前位置高亮。
- 概览：关键指标卡顶部（数字 + 趋势 + 对比基准）、空态有引导不是死页。
- 列表：批量操作工具栏、可配置列、记忆筛选、行内快捷操作（hover 出现）。
- 表单：侧抽屉编辑（不跳页）、自动保存草稿、离开提示未保存。
- 权限：无权功能隐藏而非灰显（减认知噪声）。
- 密度：提供「舒适 / 紧凑」密度切换。

**常见坑**
- 一级导航 >10 项（找不着）
- 所有操作都跳页（慢）
- 无空态设计（新用户进来全空不知干啥）
- 删除无二次确认或无撤销（误删灾难）

**推荐模式**：Linear 式命令面板（Cmd+K）、Notion 式侧抽屉、Vercel 式 dashboard。

## 3 · 数据可视化 / BI 看板

**核心目标**：数据一眼读出含义，支持钻取定位问题。

**UX 要点**
- 选图：趋势→折线、对比→柱、占比→环（非饼）、分布→直方、相关→散点。见 [../html/data-viz.md](../html/data-viz.md)。
- 编码：颜色只编码分类 / 顺序，不装饰；顺序数据用单色梯度。
- 标注：轴标签清晰、单位显式、关键值标数、悬停 tooltip 给精确值。
- 钻取：点击图表下钻、面包屑回溯、可联动筛选其他图。
- 空数据态：无数据时给说明 + 引导，不渲染空轴。
- 色盲安全：用 Okabe-Ito（见 [../color/palettes-catalog.md](../color/palettes-catalog.md) E 段）。

**常见坑**
- 饼图超 5 类（没法读）
- 3D 图表（扭曲感知）
- 双 Y 轴无标注（误导）
- 色阶不感知均匀（Rainbow 色阶）

**推荐模式**：AntV / ECharts 设计语言、Observable 笔记本式。

## 4 · 内容站 / 博客 / 媒体

**核心目标**：沉浸阅读，排版服务于内容。

**UX 要点**
- 正文宽 45-75 字符（中文 30-40 字）、行高 1.6-1.8、段落间距 >行高。
- 字体：中文用系统衬线（Songti / Source Han Serif）或无衬线、英文 Georgia / system-ui。
- 层级：标题阶 ≤4 级、H1 唯一、blockquote / 代码块视觉区分。
- 目录（TOC）：长文常驻侧栏、滚动跟随高亮。
- 阅读进度条顶部、返回顶部按钮。
- 暗模式：纯黑底 + 降饱和文字（`#e6edf3` 非 `#fff`）。
- 干扰剥离：阅读模式去侧栏广告。

**常见坑**
- 正文过宽（>80 字符难追踪行）
- 代码块无语言标注 / 无复制
- 段落无间距挤一团
- 移动端字号 <16px（iOS 自动放大）

**推荐模式**：Medium / Substack 排版、纽约时报交互专题。

## 5 · 落地页 / 营销页

**核心目标**：3 秒说清价值，引导转化（注册 / 下载 / 购买）。

**UX 要点**
- 首屏（above fold）：一句话价值主张 + 主 CTA + 信任徽标，3 秒可读。
- 视觉锚点：产品截图 / demo 视频 > 抽象插图。
- CTA：动词明确（「免费开始」>「提交」）、对比色、单主 CTA（别给多选）。
- 社会证明：用户 logo 墙 / 评价 / 数据（「10 万+团队在用」）。
- 节奏：特性 → 收益（不是功能罗列）、交替图文段。
- 底部：二次 CTA + FAQ。

**常见坑**
- 首屏全视频自动播放（慢 + 吓人）
- CTA 多个同权重（选择瘫痪）
- 虚假关闭按钮（骗点击，信任归零）
- 移动端首屏被图片占满看不到价值

**推荐模式**：Stripe / Linear 落地页、Apple 产品页节奏。

## 6 · 登录注册 / 认证

**核心目标**：最短路径进入，失败可恢复。

**UX 要点**
- 优先 SSO / 社交登录（降低注册摩擦）。
- 密码框：显隐切换、强度提示实时、看清要求（长度 / 字符类）在输入前。
- 邮箱：失焦即校验格式、错就地下提示非弹窗。
- 验证码：优先 magic link / OTP 邮件，图形验证码最后手段。
- 2FA：记住此设备 30 天、备用码必给。
- 错误信息：说清错在哪（「密码至少 8 位」非「无效」）。
- 注册后：直接进产品，别强制填一堆资料。

**常见坑**
- 密码要求输入后才告知
- 错误信息「用户名或密码错误」（不告诉哪个，但安全；可接受）
- 无忘记密码入口或流程绕
- 注册页字段 >5 个

**推荐模式**：Clerk / Auth0 登录组件、Google sign-in 流。

## 7 · Onboarding / 新手引导

**核心目标**：用户首次到达 Aha moment，不打扰。

**UX 要点**
- 渐进披露：首次只教最核心 1-2 个动作，其余延后。
- 聚焦遮罩（spotlight）：灰底高亮目标控件，配一句话。
- 空态即教学：列表空时教如何创建（「点击这里添加第一个」）。
- 可跳过：引导始终有「跳过 / 稍后」，别绑架。
- 进度感：多步引导有进度（2/5）。
- 模板 / 示例数据：新用户进来有预填内容可改，非空白。
- 迷你 checklist：首次侧栏给任务清单（「完成个人资料 ✓」）。

**常见坑**
- 全屏遮罩多步不可跳（流失）
- 引导完了还是不会用（没到 Aha）
- 每次登录都弹引导（烦）
- 指向不存在的控件（版本漂移）

**推荐模式**：Figma 首次进入、Notion 模板库、Linear checklist。

## 8 · 搜索 / 筛选

**核心目标**：快速命中，允许修正。

**UX 要点**
- 搜索框：显眼（导航中部或右）、占位符给示例、输入即联想（autocomplete）。
- 联想：最近搜索 / 热门 / 结果预览（图 + 标题 + 价格）。
- 结果页：结果数显式、高亮匹配词、无结果给建议（「尝试 X」或清空筛选）。
- 筛选：facet 化（品牌 / 价格 / 评分）、已选条件可单独移除、URL 可分享（筛选状态编码进 query）。
- 排序：默认相关度、可切价格 / 新品 / 评分。
- 移动端：搜索全屏展开、筛选抽屉。

**常见坑**
- 无联想（纯输入）
- 无结果页是死页
- 筛选不可分享（刷新丢状态）
- 搜索框藏在小图标后

**推荐模式**：Algolia instant search、Amazon 左 facet。

## 9 · 设置 / 偏好

**核心目标**：灵活但可找，默认值合理。

**UX 要点**
- 分组 sidebar（账户 / 通知 / 外观 / 隐私 / 高级），当前组高亮。
- 即时生效（toggle 立即应用），需保存的用明确「保存」按钮。
- 危险操作（删除账号 / 解绑）独立区 + 二次确认 + 输入确认。
- 默认值：选「最常用」非「最安全」（减少配置）。
- 帮助：每项配说明文案（i 图标 hover）。
- 搜索设置：大产品支持搜设置项（「暗模式」直达）。

**常见坑**
- 全部要求点「保存」（不知改了哪个）
- 危险操作无隔离（误触删号）
- 分组 >7（找不着）
- 无说明（术语用户不懂）

**推荐模式**：macOS 系统偏好、GitHub Settings、Slack 偏好。

## 10 · 社交 / 信息流

**核心目标**：粘性，但不制造焦虑。

**UX 要点**
- 流：卡片式、图视频优先、无限滚动（但给「已读完」终点）。
- 互动：点赞 / 评论 / 转发就近（卡片底部）、双击点赞（移动）。
- 发布：编辑器就近、支持草稿、图片预览可删。
- 通知：红点计数 + 抽屉式详情、可分类静音。
- 节奏控制：给「休息一下」提示、时间管理设置。
- 隐私：可见范围显式（公开 / 好友 / 私密）。

**常见坑**
- 无限滚动无终点（上瘾 + 焦虑）
- 通知无法关（疲劳）
- 默认公开（隐私事故）
- 信息流无「已读」边界（重复内容）

**推荐模式**：Twitter / Instagram 卡片、微信朋友圈隐私控制。

## 11 · 教育 / 学习

**核心目标**：坚持完成，降低挫败。

**UX 要点**
- 进度：课程进度条、章节完成态、连续打卡（streak）。
- 反馈：答题即时判分、错题解析、正反馈（鼓励语 / 动画）。
- 分段：单课 ≤10 分钟、视频分段 + 字幕。
- 练习：交互式（非纯视频）、即时可跑代码（编程类）。
- 路径：技能树可视化、前置依赖标注、可跳级测试免修。
- 社交：排行榜（可选 / 可关）、学习小组。

**常见坑**
- 纯视频无互动（无聊）
- 答错无解析（学不到）
- 进度不可见（无成就）
- streak 中断惩罚过重（弃用）

**推荐模式**：Duolingo 游戏化、Khan Academy 技能树、LeetCode 在线编辑。

## 12 · 长表单 / 数据录入

**核心目标**：填完不弃，数据干净。

**UX 要点**
- 分步：多步向导（进度条）+ 每步即时校验。
- 分组：逻辑分组（基本信息 / 联系方式 / 补充）。
- 默认值 + 预填：能推断的别让用户填。
- 即时校验：失焦校验非提交才报、对错视觉清晰。
- 智能输入：地址自动补全、日期 picker、电话格式化。
- 保存草稿：长表单可暂存防丢失。
- 必填标记：`*` 红色，可选字段尽量删。

**常见坑**
- 一页 50 字段（弃）
- 提交才校验（填完才发现错，重来怒）
- 无草稿（填一半误关全丢）
- 必填无标记

**推荐模式**：Typeform 对话式、Stripe 多步支付、政府表单分段。

## 13 · 实时协作 / 编辑器

**核心目标**：多人并发不冲突，操作流畅。

**UX 要点**
- 在场感知：他人光标 / 头像（彩色 + 名字）、选中区域高亮。
- 冲突处理：CRDT / OT 合并、绝不锁文件。
- 撤销：可靠 undo / redo、跨协作不串。
- 评论：选中内容就地评论、@提及通知、可解决（resolved）。
- 版本：历史版本可见可回滚。
- 同步状态：离线 / 同步中 / 已保存显式（别假「已保存」）。

**常见坑**
- 无他人光标（撞车编辑）
- 「已保存」但实际没同步
- 撤销把别人的改动也撤了
- 无离线降级（断网即崩）

**推荐模式**：Figma 多人光标、Google Docs 评论、Notion 协作。

## 14 · 通知 / 消息中心

**核心目标**：及时触达，不轰炸。

**UX 要点**
- 分级：紧急（系统告警）/ 重要（@提及）/ 常规（更新）/ 低（营销），分级送达（推送 / 红点 / 邮件）。
- 聚合：同类合并（「3 人赞了你的文章」非 3 条）。
- 可操作：通知内可直接处理（批准 / 回复），非仅跳转。
- 已读 / 未读：批量已读、按类型筛选。
- 控制：每类可独立开关推送、勿扰时段。
- 归档：旧通知归档非删除。

**常见坑**
- 全部推送（疲劳 → 全关）
- 无聚合（刷屏）
- 仅跳转无就地操作
- 无批量已读（逐条点）

**推荐模式**：GitHub 通知中心、Slack 分级通知、LinkedIn 聚合。

## 移动端独有补充

以上场景在移动端额外注意：

| 维度 | 要点 |
|------|------|
| 手势 | 左滑删除 / 长按多选 / 下拉刷新 / 边缘返回（别劫持系统返回）|
| 底栏 | Tab ≤5、主操作可 floating action button（FAB）|
| 抽屉 | 次级导航抽屉、筛选用 bottom sheet 非 modal |
| 输入 | 数字键盘 / 邮箱键盘按类型唤起、表单字段避免下拉（改 picker）|
| 安全区 | 避让刘海 / 底部 home indicator |
| 性能 | 首屏 <3s、骨架屏非 spinner、图片懒加载 |

## CLI / TUI 场景映射

| Web 场景 | CLI 对应 |
|---------|---------|
| Dashboard | `tool status` 概览 + `tool list` 表格 |
| 长表单 | 交互式 `tool init` 向导（见 [../cli/style.md](../cli/style.md) 模板 4）|
| 搜索筛选 | `tool list --filter x --sort y` flag 组合 |
| 设置 | `tool config set k v` + `tool config get` |
| 通知 | exit code + stderr `error:` 前缀（见 [../cli/style.md](../cli/style.md)）|
| 数据看板 | TUI 全屏 dashboard（见 [../tui/](../tui/INDEX.md)）|
