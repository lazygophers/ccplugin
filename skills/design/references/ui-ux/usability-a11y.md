# 可用性与无障碍（无关平台）

可用性 = 用户能否高效完成任务。无障碍 = 是否所有人（含残障）都能用。跨平台通用。原则见 [principles.md](principles.md)，IA 见 [information-architecture.md](information-architecture.md)，交互见 [interaction-design.md](interaction-design.md)。

## Nielsen 可用性启发式（10 条）

1. **系统状态可见**：用户随时知道在做什么
2. **贴近真实世界**：用用户语言，不搞内部术语
3. **用户可控与自由**：撤销 / 返回 / 取消
4. **一致与标准**：同概念同表现，遵循平台习惯
5. **防错**：设计上消除错误机会
6. **可识别优于回忆**：选项可见，不靠记忆
7. **灵活与效率**：新手引导 + 专家加速
8. **美学与极简**：只放任务相关信息，去冗余
9. **错误识别与恢复**：说人话的错误 + 修复路径
10. **帮助与文档**：需要时能找到帮助

每条拿来评审产品，逐条打分 + 具体改法。

## 无障碍基线（跨平台）

### 可感知

- 文字对比 ≥ 4.5:1（正文）/ ≥ 3:1（大字 / 图标）
- 不只靠颜色传信息（色盲），加图标 / 文字 / 纹理
- 文字可放大（系统字号 / 缩放）不塌布局

### 可操作

- 全键盘可达（Tab 顺序、快捷键）
- 焦点可见（`focus-visible` / 焦点边框）
- 触控目标够大（≥44px / 44pt / 48dp）
- 动效尊重 `prefers-reduced-motion` / 系统降低动效

### 可理解

- 语言简明、术语一致
- 错误信息说人话 + 修复建议
- 导航可预测、一致

### 健壮（兼容辅助技术）

- 语义化：Web 用语义 HTML + ARIA；App 用 `accessibilityLabel`；TUI 文本本身可读
- 控件有名称、角色、状态、值
- 动态变化通知辅助技术

## WCAG 对照（Web 主，精神通用）

| 原则 | AA 要求 |
|------|--------|
| 对比 | 正文 ≥4.5:1、大字 ≥3:1 |
| 键盘 | 全键盘可达，无键盘陷阱 |
| 焦点 | 可见，顺序有意义 |
| 文字缩放 | 200% 不丢功能 |
| 颜色 | 不作为唯一信息载体 |
| 动效 | 支持 prefers-reduced-motion |

## 响应式与场景适配（可用性延伸）

- 多设备 / 尺寸 / 横竖屏可用（见各媒介 scenes 文件）
- 亮 / 暗模式均可读
- 弱网 / 离线降级不假死

## 认知负荷

- 每屏一个主任务
- 分步而非一次铺开
- 渐进式披露：高级选项藏次级（非删除）
- 一致性降低学习成本（见 [principles.md](principles.md) 一致性）

## 可读性

- 行长 45–90 字符（西文）/ 中文适度
- 行高 1.5–1.7（正文）
- 段落短、小标题分段
- 字号阶梯清晰（见各媒介 style 文件）

## 测试方法

| 方法 | 发现 |
|------|------|
| 启发式走查 | 10 条逐条评 |
| 真人测试 | 真实任务、观察卡点 |
| 键盘测试 | 拔鼠标，能否完成全流程 |
| 屏幕阅读器 | VoiceOver / NVDA / TalkBack 跑一遍 |
| 自动扫描 | axe / Lighthouse / 平台审计工具 |
| 对比检查 | 取色器测比值 |

## 自检清单

- [ ] Nielsen 10 条无明显违反
- [ ] 全键盘可达、焦点可见
- [ ] 对比达标、不只靠颜色
- [ ] 动效尊重降低偏好
- [ ] 文字可放大不塌
- [ ] 屏幕阅读器可读
- [ ] 认知负荷克制（一屏一主任务）
