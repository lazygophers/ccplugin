# 交互设计（无关平台）

交互设计管「用户做事」：操作、状态、反馈、微交互、防错。跨平台通用。原则见 [principles.md](principles.md)，IA 见 [information-architecture.md](information-architecture.md)，可用性见 [usability-a11y.md](usability-a11y.md)。

## 交互模型基础

每个交互 = **触发** → **系统响应** → **状态变化** → **反馈**。

- 触发：用户动作（点击 / 输入 / 手势 / 键 / 命令）
- 响应：<100ms 感知即时；>1s 需进度；>10s 需估时 + 可取消
- 状态：动作前后系统状态明确（禁用、loading、完成、失败）
- 反馈：视觉 / 听觉 / 触觉告诉用户发生了什么

## 状态完整性

每个交互组件覆盖全套状态：

```
default → hover → focus → active/pressed → disabled
                                    ↘ loading / error / selected / empty
```

漏状态 = 半成品。设计组件时列全状态再实现。

### 关键状态

| 状态 | 必要性 | 说明 |
|------|--------|------|
| default | 必 | 默认态 |
| hover | 视平台 | 鼠标平台必有；触摸 / TUI 可省 |
| focus | 必 | 键盘焦点，必须可见 |
| active/pressed | 必 | 按下瞬间 |
| disabled | 必 | 不可用但可见 |
| loading | 必 | 异步等待 |
| error | 必 | 失败 + 原因 |
| empty | 必 | 无数据 + 引导 |
| selected | 视 | 选中态 |

## 反馈分层

| 层级 | 用什么 | 场景 |
|------|--------|------|
| 细节 | hover / press 态、微动效 | 即时确认操作 |
| 状态 | loading / 成功 / 失败 | 操作结果 |
| 系统 | Toast / 横幅 / 状态栏 | 跨区通知 |
| 阻断 | 对话框 / 确认 | 需停下处理 |

原则：**用最轻的反馈层**。能 Toast 别 Modal，能内联别弹窗。Modal 打断一切，克制用。

## 微交互

小动效增强反馈，但要克制：

- 禁用时长 >0.3s 的纯装饰动效
- 动效服务于状态变化（展开、进入、强调），不喧宾夺主
- 尊重 `prefers-reduced-motion`（见 [usability-a11y.md](usability-a11y.md)）

## 表单交互

- 标签在输入框上方（优于占位符——填后不消失）
- 即时校验（失焦 / 输入时），错误信息贴字段下方
- 必填标 `*`，可选标「（可选）」
- 键盘流畅：Tab 顺序、Enter 提交、聚焦滚动入视
- 错误恢复：保留用户已填，别清空重来

## 错误处理

三段式：**发生了什么 + 为什么 + 怎么修**。

- 错误说人话，给修复路径
- 有具体值（哪个字段、哪个文件、哪个限制）
- 批量错误一次报全（别让用户修一个再发现下一个）
- 危险操作可撤销优于确认对话框

## 防错

- 约束输入（格式、范围、类型）
- 危险操作二次确认或延时执行
- 默认非破坏性（不默认删 / 覆盖）
- 不可逆操作给冷却或撤销窗口

## 加载与等待

| 等待时长 | 反馈 |
|---------|------|
| <100ms | 无需 |
| 100ms–1s | 轻微态变化（禁用按钮） |
| 1–10s | spinner / 进度条 |
| >10s | 估时 + 可取消 + 后台跑 |

- 骨架屏：内容加载时保留布局防跳变
- 别假成功：异步失败要回头报，别 loading 消失就算成功

## 空状态

- 插画 / 图标 + 说明 + CTA
- 不只是「暂无数据」——告诉用户能做什么（创建、导入、刷新）

## 手势 / 键位 / 命令的可见性

- 不可见的交互（手势、快捷键、命令）用户发现不了
- 首次使用引导 / 悬浮提示 / 帮助页 / 底栏常驻
- 高频可见、长尾可检索（命令面板）

## 自检清单

- [ ] 状态全覆盖（含 empty / error / loading）
- [ ] 每个动作有反馈，时效达标
- [ ] 用最轻反馈层（少 Modal）
- [ ] 错误三段式 + 可恢复
- [ ] 危险操作防错
- [ ] 加载有合适反馈、无假成功
- [ ] 隐藏交互有可见暗示
