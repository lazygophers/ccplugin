# 配色可访问性（无关平台）

颜色要让所有人都能用——低视力、色盲、暗环境、强日光。理论见 [theory.md](theory.md)，模板见 [palette-templates.md](palette-templates.md)。

## 对比度标准（WCAG）

| 级别 | 正文（<18pt） | 大字（≥18pt 或 ≥14pt 粗） | UI 组件 / 图标 |
|------|--------------|--------------------------|----------------|
| AA | ≥ 4.5:1 | ≥ 3:1 | ≥ 3:1 |
| AAA | ≥ 7:1 | ≥ 4.5:1 | — |

**正文一律 AA（4.5:1）起步**，关键内容冲 AAA。

## 对比度计算

相对亮度公式：

```
L = 0.2126×R + 0.7152×G + 0.0722×B  （每通道 gamma 展开后的值）
对比度 = (L亮+0.05) / (L暗+0.05)
```

实操：别手算，用工具：

- WebAIM Contrast Checker
- 开发者工具取色器自带对比比
- 设计工具（Figma / Sketch）插件

## 检查清单

- [ ] 所有正文 ≥4.5:1
- [ ] 所有标题 / 大字 ≥3:1
- [ ] 按钮文字与按钮底 ≥4.5:1
- [ ] 图标与背景 ≥3:1
- [ ] 占位符文字 ≥4.5:1（常被忽略）
- [ ] 禁用态仍可辨（虽降对比，但可读）

## 色盲安全

色盲类型：

| 类型 | 影响 | 人口占比 |
|------|------|---------|
| 红绿色盲（Deuteranopia / Protanopia） | 红绿难辨 | 男性 ~8% |
| 蓝黄色盲（Tritanopia） | 蓝黄难辨 | 罕见 |
| 全色盲 | 无色感 | 极罕 |

### 色盲安全原则

- **禁只靠颜色传信息**：状态用颜色 + 图标 + 文字（如 ✕ 红 +「失败」）
- 数据可视化：用色盲安全色板（Okabe-Ito），避免红绿配
- 链接：颜色 + 下划线（不只靠蓝色）
- 必填：红星 +「必填」文字

### Okabe-Ito 色板（色盲安全，8 色）

```
橙:   #E69F00  天蓝: #56B4E9  绿:   #009E73  黄:   #F0E442
蓝:   #0072B2  朱红: #D55E00  品红: #CC79A7  黑:   #000000
```

分类编码（图表 / 标签）首选。任意两色在色盲下可辨。

## 暗模式

暗模式专项规则：

- **非亮色取反**：重新设计中性阶，降饱和、带蓝灰
- 主色提亮一档（暗底上 `#2563eb` 显沉，改 `#3b82f6` / `#60a5fa`）
- 文字别用纯白（`#ffffff` 刺眼），用 `#e6edf3` 类
- 背景别纯黑（`#000000`），用深蓝灰 `#0d1117` / `#161b22`
- 阴影失效：暗底上改用微亮边框 / 微亮区分层
- 图片 / 图标配暗底版本或半透明
- 降低饱和度：暗模式高饱和更刺眼

### 暗模式对比仍要达标

暗模式不豁免对比度。重新测文字与暗背景对比 ≥4.5:1。

## 高对比需求

- `prefers-contrast: more`（部分系统）→ 提供强化对比主题
- 黑底黄字 / 黄底黑字等极简高对比备用

## 动效与色

- `prefers-reduced-motion` 时动效退化为瞬切，颜色不变
- 闪烁内容频率 <3Hz（防光敏癫痫）

## 测试方法

| 方法 | 发现 |
|------|------|
| 对比检查工具 | 数值不达标 |
| 色盲模拟器（Coblis / Stark） | 色盲下信息丢失 |
| 灰度截图 | 去色后能否辨信息（终极色盲测试） |
| 强光 / 暗光实机 | 极端环境可读性 |
| 暗模式审计 | 暗模式对比达标 |

灰度测试：把界面去色截图，若信息仍可辨 = 色盲安全过关。

## 自检清单
