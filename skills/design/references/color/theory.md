# 色彩理论（无关平台）

配色的底层逻辑。懂理论才能自由调，不靠模板硬背。模板见 [palette-templates.md](palette-templates.md)，可访问性见 [accessibility.md](accessibility.md)。

## 色彩三要素

| 要素 | 说明 | 设计意义 |
|------|------|---------|
| 色相（Hue） | 颜色身份（红 / 蓝 / 绿） | 识别、分类编码 |
| 明度（Lightness） | 亮暗 | 层级、对比、可读性（最重要） |
| 饱和度（Saturation） | 鲜艳 / 灰度 | 情绪、强调（高=激烈，低=沉稳） |

**明度对比决定可读性**，色相饱和只影响情绪。对比不够时先调明度，别只换色。

## 色环与配色法

| 配色法 | 色环关系 | 效果 | 适用 |
|--------|---------|------|------|
| 单色（Monochromatic） | 同色相不同明度饱和 | 和谐、安全 | 品牌、克制设计 |
| 类似色（Analogous） | 色环相邻 ±30° | 柔和、自然 | 背景、氛围 |
| 互补色（Complementary） | 色环正对 | 强烈、跳眼 | 强调、CTA |
| 分裂互补 | 互补两侧 | 强烈但更平衡 | 主色 + 强调 |
| 三元色（Triadic） | 色环等距三角 | 活力、平衡 | 儿童产品、活力品牌 |
| 方案（Tetradic） | 色环矩形四角 | 丰富、难平衡 | 需经验 |

## 60-30-10 法则

配色比例分配：

- **60% 主色**（背景、大块）
- **30% 辅色**（次级、分区）
- **10% 强调色**（CTA、关键强调）

一个主色 + 一个强调色就够大多数产品。别超过 3 个显著色相。

## 色阶（Scale）

每个品牌色要有一组阶（5–10 档明度），不是单色：

- 浅阶：背景、悬浮态、标签底
- 中阶：主按钮、强调
- 深阶：hover、文字、标题

生成方法：HSL 中固定色相，明度从 95% → 20% 取 10 档，饱和略调。

## 中性色（灰阶）

中性色是 UI 主体（文字、背景、边框）。禁用纯灰（`#808080`），用**带色温的灰**：

- 冷灰：略加蓝（科技、专业）
- 暖灰：略加橙黄（亲和、人文）
- 中性灰：极淡色相

生成：主色 HSL 饱和降到 5–10%，明度分阶。

## 语义色

固定语义，跨主题一致：

| 语义 | 色相 | 用途 |
|------|------|------|
| success | 绿 | 完成、正向 |
| warning | 黄 / 橙 | 提醒、注意 |
| error | 红 | 失败、危险、删除 |
| info | 蓝 | 信息、中性提示 |

## 配色常见坑

- **彩虹编码有序数据**：彩虹色无感知顺序，误导。有序数据用单色明度阶
- **全饱和**：所有色都高饱和 = 互相喊叫，眼疲劳。强调色饱和、其余降
- **亮底亮字 / 暗底暗字**：对比不足。测对比度（[accessibility.md](accessibility.md)）
- **只靠颜色传信息**：色盲用户失信息。加图标 / 文字冗余
- **纯黑纯白**：`#000` on `#fff` 对比过强刺眼；暗模式用 `#0d1117` 而非纯黑

## 色彩与情绪

| 暖色（红橙黄） | 冷色（蓝绿紫） |
|----------------|----------------|
| 激进、活力、食欲 | 冷静、信任、专业 |
| 前进感（显近） | 后退感（显远） |
| 警示、紧急 | 安全、稳定 |

品牌情绪定主色色相区间，再细化。

## 自检清单

- [ ] 明度对比足够（不只换色相）
- [ ] 配色比例 60-30-10
- [ ] 每个品牌色有色阶非单色
- [ ] 中性色带色温非纯灰
- [ ] 语义色固定一致
- [ ] 不只靠颜色传信息
