# App-Design · 原生 App UI/UX 设计

原生 App 有平台习惯与物理约束（屏幕、手势、系统栏），不是「把网页塞进 App」。帮用户做平台选型、信息架构、导航、组件、手势层面的设计决策。

## 适用 / 不适用

适用：iOS / Android 移动 App、macOS / Windows / Linux 桌面应用、跨平台（Flutter / React Native / Electron）。

不适用：纯 Web 页面（→ html-design）、CLI（→ cli-design）、终端 UI（→ tui-design）。

## 核心硬门 · 三方向初稿（100% 必走）

任何新 App 视觉产出，先出**三个差异化方向的真实初稿**等用户选定。方向差异化在信息架构 / 导航模式 / 视觉风格 / 平台倾向上，不是同款换色。

## 决策路由

收到任务先定这几层（每层都帮用户选）：

| 决策层 | 选什么 |
|--------|--------|
| 平台 | iOS / Android / 桌面 / 跨平台——决定底层习惯 |
| 信息架构 | 层级深度、主要任务流程、内容优先级 |
| 导航模式 | 底部 Tab / 抽屉 / 顶栏 Segmented / 栈式推送 |
| 组件 | 平台原生组件 vs 自定义，状态完整性 |
| 手势 | 滑动 / 长按 / 拖拽 / 捏合——何时用、有无可见暗示 |
| 视觉风格 | 见设计原则段 |

## 平台习惯差异（选型硬约束）

### iOS vs Android

| 维度 | iOS | Android |
|------|-----|---------|
| 导航 | 底部 Tab + 栈推送；返回靠系统手势 | 顶栏汉堡或底部 Tab；返回靠系统返回键 |
| 触控目标 | ≥ 44pt | ≥ 48dp |
| 字体 | San Francisco（系统无衬线） | Roboto |
| 圆角 | 大圆角（12-16）持续流行 | Material 圆角阶（4/8/16） |
| 模态 | 从底部滑入 Sheet | 居中 Dialog 或 Bottom Sheet |
| 色彩 | 系统强调色跟随主题 | Dynamic Color 取壁纸 |
| 列表 | 右滑出操作 | 长按出菜单 / SwipeDismiss |

原则：**遵循平台习惯 > 跨平台一致**。用户在 iOS 期待 iOS 行为，强行统一反而别扭。跨平台框架（Flutter/RN）也尽量适配各自平台习惯。

### 桌面应用

- 多窗口 / 多面板（侧栏 + 主区 + 检视器），利用宽屏
- 菜单栏（macOS 顶部菜单 / Windows 应用内菜单）
- 键盘快捷键是一等公民（Cmd/Ctrl + 字母）
- 右键菜单（Context Menu）
- 拖拽（文件拖入拖出、面板重排）
- 高密度信息展示（小字号、紧凑间距可接受）

## 信息架构与导航

### 导航模式选择

| 模式 | 何时用 |
|------|--------|
| 底部 Tab（2-5 主区） | 主功能并列、需快速切换 |
| 侧栏抽屉 | 功能多、次级深、主区 ≤1 |
| 顶部 Segmented | 2-3 同级视图切换 |
| 栈式推送 | 线性流程（设置层级、详情展开） |
| 桌面侧栏 | 多模块后台、文档应用 |

### 任务流程设计

- 主流程 ≤3 步到达核心功能
- 次级功能藏在合理层级，别堆首页
- 空状态、首次启动引导、权限请求时机要设计
- 离线 / 弱网降级：哪些功能可用、怎么提示

## 组件选型（原生优先）

原生组件优先（免费获得无障碍 + 平台习惯 + 性能）。自定义只在原生不能满足品牌表达时。

| 组件族 | 关键点 |
|--------|--------|
| 导航栏 | 标题居中 / 左返回；行动按钮位置随平台 |
| 列表 / Collection | 行高、分隔、滑动操作、下拉刷新、上拉加载 |
| 表单 | 输入聚焦滚入可视区、键盘遮挡处理、实时校验 |
| 底部 Sheet | 确认操作、多选项；可拖拽高度 |
| Toast / Snackbar | 非阻断，带可选行动按钮（Android Snackbar） |
| 权限弹窗 | 用时请求而非启动全要；说明为什么需要 |
| 启动屏 | 极简，快速过渡到内容 |
| 空状态 | 插画 + 说明 + CTA，不只是「暂无数据」 |

### 状态完整性

每个交互组件覆盖：default / pressed / focused / disabled / loading / error / empty / selected。

## 手势设计

| 手势 | 用途 | 可见性 |
|------|------|--------|
| 点击 | 主操作 | 明确（按钮样） |
| 长按 | 次级菜单 / 进入编辑 | 需暗示或引导 |
| 滑动 | 切页 / 出操作 / 删除 | 需部分可见或提示 |
| 捏合 | 缩放 | 内容可缩时自然 |
| 拖拽 | 排序 / 移动 | 拖把手柄暗示 |

手势不可见 = 用户发现不了。关键手势首次使用要有引导或可见暗示。

## 视觉风格与配色

复用 `html-design` 的风格与配色方法论（色板构成、可访问性对比度、token 化），但适配 App 语境：

- 支持亮 / 暗模式（系统跟随 + 手动切换），暗模式重新调中性阶
- 强调色用于主操作与活跃态，克制使用
- 动效遵循平台（iOS 弹性缓动 / Material motion 时长阶）
- 图标用平台系统图标库（SF Symbols / Material Icons）优先，保证语义一致

## 原型产出

- 高保真原型可用 `html-design` 出 HTML mockup（快、跨平台预览）
- 或给原生实现指引：组件树、状态、流转图、标注（尺寸 / 间距 / 色值 / 字号 token）

## 自检清单

- [ ] 平台习惯对齐（不混用 iOS / Android 模式）
- [ ] 触控目标达标（44pt / 48dp）
- [ ] 主流程 ≤3 步
- [ ] 空状态 / 错误态 / 加载态有设计
- [ ] 亮 / 暗模式均可读（如要求）
- [ ] 关键手势有可见暗示
