# 组件设计与选择

组件是界面的原子。会选组件、会设计组件状态，UI 才稳定可维护。

## 组件选择原则

选组件先问三个问题：

1. **用户此刻要完成什么任务？** —— 任务决定组件，不是反过来
2. **这个组件的状态空间？** —— 默认 / hover / active / disabled / loading / error / empty，都要覆盖
3. **它在不同密度 / 尺寸下塌不塌？** —— 响应式不是事后补，是设计时就定

选错组件的典型：该用 Switch 的用了 Checkbox（即时生效 vs 提交生效）、该用 Segmented 的用了下拉（≤5 选项别藏起来）、该用 Stepper 的用了输入框（小范围整数）。

## 组件族速查

### 按钮（Action）

| 类型 | 用途 | 视觉 |
|------|------|------|
| 主按钮 | 每屏唯一的主操作 | 实心填充、品牌色、最强权重 |
| 次按钮 | 次要操作 | 描边 / 浅底 |
| 幽灵按钮 | 第三级、工具栏 | 无边框、hover 显底 |
| 危险按钮 | 删除等不可逆 | 红 / 橙填充 |
| 图标按钮 | 空间紧的操作 | 仅图标 + tooltip |

- 尺寸阶：sm / md / lg（高度 32 / 40 / 48）
- 状态：default / hover / active / focus-visible / disabled / loading
- loading 时禁用 + 转圈，文案不变
- 图标 + 文案：图标在左，垂直居中

### 表单输入

| 组件 | 选型边界 |
|------|---------|
| 文本输入 | 自由文本，单 / 多行 |
| 选择器 Select | ≥5 选项、动态数据源 |
| 分段控件 Segmented | 2-5 选项、常驻可见 |
| 单选 Radio | 2-5 选项、互斥、全可见 |
| 复选 Checkbox | 多选、独立开关 |
| 开关 Switch | 即时生效的二态（无需提交） |
| 滑块 Slider | 连续 / 大范围数值 |
| 步进器 Stepper | 小范围整数（数量） |
| 日期选择 | 日历 vs 输入，看是否跨月 |

表单设计要点：
- 标签放输入框上方（比占位符好——填了之后不消失）
- 校验即时（失焦 / 输入时），错误信息在字段下方紧贴
- 必填标星号，可选字段标「（可选）」减少认知
- 提交按钮在底部，主操作右（桌面）/ 全宽（移动）

### 导航

| 组件 | 场景 |
|------|------|
| 顶栏 Tab | ≤5 主区、同级并列 |
| 侧栏 | 多层级、后台、文档 |
| 抽屉 Drawer | 移动、空间紧 |
| 面包屑 | 深层级、回溯路径 |
| 底部 Tab | 移动 App 主导航 |
| 分页 / 无限滚动 | 列表浏览（目标导向用分页，探索用无限） |

当前位置必须明显——活跃态用色 + 字重，不只靠下划线。

### 数据展示

| 组件 | 选型 |
|------|------|
| 表格 | 结构化、需排序筛选、精确比对 |
| 列表 | 半结构、条目异构 |
| 卡片 | 视觉优先、多媒体、等权集合 |
| 树 | 层级、可折叠 |
| 网格 | 图片 / 等比内容 |

表格密度：行高 40（紧凑）/ 48（标准）/ 64（舒适）。数字右对齐、文字左对齐、状态用标签不用色块文字。

### 反馈

| 组件 | 何时 |
|------|------|
| Toast | 成功 / 轻提示，3-4 秒自消，不打断 |
| 横幅 Banner | 持续信息（系统通知、cookie），可关闭 |
| 对话框 Modal | 需确认的破坏性操作、阻断流程 |
| 抽屉 | 多字段表单、详情面板 |
| 内联信息 | 空状态、错误态、上下文提示 |
| 进度条 | 已知时长 |
| Spinner | 未知时长、短等待 |
| 骨架屏 | 内容加载，保留布局防跳变 |

**用 Modal 克制**：它打断一切。能 Toast 就别 Modal。

### 模态与浮层

- Modal：遮罩 + 居中 + ESC 关 + 点遮罩关（除非有未保存内容）+ 焦点陷阱
- Popover / Tooltip：指向触发元素、空间不足自动翻转、hover 延迟（进 200ms / 出 100ms）
- 抽屉：从边缘滑入，移动端全宽、桌面限宽 400

## 状态完整性

每个交互组件至少覆盖：

```
default → hover → focus → active → disabled
                                    ↘ loading / error / checked
```

漏状态 = 半成品。设计组件时列全状态再写。

## 组件 Token 化

组件值取自项目 token，不硬编码：

```css
.btn { padding: var(--space-2) var(--space-4); border-radius: var(--radius); font-size: var(--text-md); }
.btn--primary { background: var(--color-primary); }
.btn--lg { height: 48px; }
```

好处：换肤 / 密度切换 / 暗模式一处生效全局生效。详见 [style.md](style.md) 与 [layout.md](layout.md)。

## 可访问性基线

- 所有交互可键盘到达（Tab 顺序、focus-visible 可见）
- 图标按钮有 `aria-label`
- 表单 `<label>` 关联输入
- 模态开合管理焦点、`aria-modal`、ESC 关闭
- 动效尊重 `prefers-reduced-motion`
