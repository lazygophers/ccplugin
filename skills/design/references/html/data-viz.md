# 数据可视化模式

产出 standalone SVG（技术图表）或 HTML（信息图 / 数据图），最终导出 SVG / PNG。

## 子类与做法

| 子类 | 做法 |
|------|------|
| 架构图 / 组件关系 | 盒子 + 箭头，HTML+CSS 或内联 SVG；分层布局 |
| 流程图 / 状态机 | 节点 + 有向边；判定用菱形、开始 / 结束用圆角 |
| 时序图 | 泳道 + 生命线 + 消息箭头 |
| 信息图（高密度） | HTML 多栏排版，数据驱动，印刷级 |
| 数据图表 | D3 或手写 SVG；先定图表类型再画 |

## 图表类型选择

数据关系决定图表类型，不是反过来：

| 要表达 | 选 |
|--------|---|
| 随时间变化 | 折线 / 面积 |
| 对比大小 | 柱状（≤12 项）/ 条形（项名长） |
| 占比构成 | 饼图（≤5 类）/ 堆叠柱 / 树图 |
| 分布 | 直方图 / 箱线 |
| 相关性 | 散点 / 气泡 |
| 多维对比 | 雷达 / 热力 |
| 流向 | 桑基 / 和弦 |

规避：
- 饼图 > 5 类 → 改堆叠柱
- 3D 图表 → 永远平面（3D 扭曲感知）
- 双轴折线 → 只在两系列量纲差异极大时，否则误导

## 技术实现

### 内联 SVG（技术图首选）

```html
<svg viewBox="0 0 800 400" xmlns="http://www.w3.org/2000/svg">
  <defs>
    <marker id="arrow" markerWidth="10" markerHeight="10" refX="9" refY="3" orient="auto">
      <path d="M0,0 L0,6 L9,3 z" fill="#555"/>
    </marker>
  </defs>
  <g class="box"><rect .../><text .../></g>
  <line x1=".." y1=".." x2=".." y2=".." stroke="#555" marker-end="url(#arrow)"/>
</svg>
```

- 箭头统一用 `<marker>` def
- 描边粗细全局统一（1.5px 常用）
- 文字 `text-anchor` + `dominant-baseline` 对齐
- 分组用 `<g>`，便于样式与语义

### D3 / 手写（数据图）

- 数据驱动：先 `data.join()` 再绑元素
- 比例尺 `scaleLinear / scaleBand` 映射数据→像素
- 坐标轴用 `axisBottom / axisLeft`，别手摆
- 响应式：`viewBox` + `preserveAspectRatio`，或 ResizeObserver 重绘

## 配色（数据编码专用）

- **分类数据**：Okabe-Ito / Tab10（色盲安全）
- **顺序数据**：单色相明度渐变（浅→深 = 低→高）
- **发散数据**：双色相中间白（如红–白–蓝，中心 = 中性值）
- **禁彩虹色编码有序数据**：彩虹色无感知顺序，误导

文字标注：
- 直接标在数据上 > 单独图例（少一次视线跳转）
- 标签重叠时：错位 / 引线 / 仅标关键点
- 单位标在轴标题，不每条都重复

## 信息图排版

信息图 = 高密度可视化叙事，HTML 排版强于 SVG：

- 分栏 + 模块化（每个数据点一个独立视觉单元）
- 数字放大成视觉主角（72-120px）+ 一句解释
- 统一图标语言（线性 / 填充二选一，别混）
- 印刷级：CMYK 思维选色、出血位、300dpi 思维（虽然 HTML 输出 RGB，但配色克制）

## 导出

- 内联 SVG → 直接存 `.svg`（矢量、无损、可二次编辑）
- 需位图 → headless Chrome 按图表外接框截图 → PNG
- 批量图表 → 脚本遍历截图，见 [export.md](export.md)
