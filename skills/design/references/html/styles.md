# 风格与配色色系选型

帮用户在多风格 / 多配色方案中做选择，是本 skill 的核心能力之一。风格不是「好看就行」，是「匹配受众、场景、品牌情绪」。

## 选型流程

1. **先定语境**：受众是谁？场景是什么（营销 / 工具 / 内容 / 企业）？情绪关键词？
2. **出三个差异化方向**（三方向硬门）：每个方向一套「风格 + 配色 + 字体气质」的组合，不是同款换色
3. **并排对比**：让用户在真实初稿上选，不在抽象描述上选
4. **选定后固化**：把选定的配色 / 字号阶 / 圆角 / 阴影写成该项目的 token，后续组件复用

## 风格谱系（选方向时的参照库）

| 风格 | 气质 | 配色倾向 | 适用场景 |
|------|------|---------|---------|
| 极简留白 | 克制、高级 | 单色 + 大量负空间、低饱和 | 高端品牌、内容站、Apple 系 |
| 深空暗场 | 专业、沉浸 | `#0d1117` 底 + 高对比强调色 | 开发者工具、数据看板、技术演示 |
| 杂志编辑 | 权威、阅读感 | 衬线标题 + 米白底 + 墨色正文 | 报告、长文、出版物 |
| 玻璃拟物 | 现代、轻盈 | 半透明模糊 + 柔和渐变 | 消费 App、落地页 |
| Neo-brutalism | 张扬、年轻 | 高饱和色块 + 粗黑边 + 硬阴影 | 创意 agency、潮牌、Z 世代 |
| Material / 平面 | 清晰、规范 | 中性灰 + 品牌主色 + 层级阴影 | 企业工具、Android 系 |
| 暖色手绘 | 亲切、温度 | 暖调 + 不规则线条 + 插画 | 教育、社区、儿童 |

## 配色色系决策

### 选色板而非挑单色

一个项目要有一套色板，不是一两个颜色。色板构成：

- **主色**（1）：品牌识别、主按钮、关键强调
- **辅助色**（1-2）：次级强调、分类标签
- **中性色**（5-7 阶）：从最深文字到最浅背景的灰阶
- **语义色**（4）：success / warning / error / info
- **背景色**（2-3）：页面底、卡片底、悬浮态

### 色板推荐（感知均匀、可访问）

| 色板 | 特点 | 适用 |
|------|------|------|
| Okabe-Ito | 色盲安全、8 色 | 数据可视化、分类编码 |
| Tailwind default | 22 色 ×10 阶 | 通用 Web / App |
| Tab10 / Set2 | 经典分类色 | 图表 |
| 自建品牌色 | 用户给定主色 → HSL 拓展阶数 | 品牌项目 |

### 可访问性硬指标

- 文字与背景对比度 ≥ 4.5:1（WCAG AA，正文）/ ≥ 3:1（大字）
- 禁只靠颜色传递信息（色盲用户），加图标 / 纹理 / 标签冗余
- 暗色模式不是亮色取反：重新调中性阶，降低饱和，提高文字对比

### 配色生成技巧

- 用户给一个主色 → 用 HSL 旋转生成互补色 / 三元色 / 类似色
- 中性阶用主色低饱和 + 调明度生成（带色温的灰，比纯灰高级）
- 渐变用同色相不同明度，禁跨色相乱渐变

## 字体气质搭配

| 风格 | 标题 | 正文 |
|------|------|------|
| 极简 / 杂志 | 系统衬线 / 大字号无衬线 | 系统无衬线 |
| 暗场技术 | 等宽点缀 + 无衬线 | 无衬线 |
| 编辑权威 | 衬线 | 系统中文衬线 |
| 现代 App | 几何无衬线 | 无衬线 |

中文：用系统栈（`-apple-system, "PingFang SC", "Microsoft YaHei", sans-serif`），别引私有字体除非用户提供。

## 自适应场景（同项目多语境）

当同一产物要适配多种场景时，用 CSS 变量 + 媒体查询切换 token 集：

- 亮 / 暗模式：`[data-theme="dark"]` 覆写 `--bg / --text / --border`
- 桌面 / 移动：断点调字号阶、间距尺度、栅格列数
- 紧凑 / 舒适密度：`--spacing-unit` 缩放
- 品牌换肤：主色 token 一处改全局生效

详见 [layout-system.md](layout-system.md) 的响应式与密度切换。
