# CLI 媒介 · 场景自适应

同一 CLI 要在不同场景下都好用。场景轴决定何时切换行为。布局见 [layout.md](layout.md)，组件见 [components.md](components.md)；色彩 / 风格在姊妹 skill `design-color`。

## 场景轴速查

| 场景轴 | 切换什么 | 触发 |
|--------|---------|------|
| 终端 / 管道 / 文件 | 彩色、进度、格式 | TTY 检测 |
| 人 / 脚本 / 程序 | 输出格式（表格 / JSON / 静默） | `--json` / 检测 |
| 交互 / 非交互 | 提示确认 vs flag 直通 | TTY / `--yes` |
| CI / 本地 | 颜色、进度、退出码严格度 | `CI` env / `--no-color` |
| 新手 / 专家 | help 详略、向导 vs 直调 | `-h` / `--help` |
| 配置来源 | 默认值 | 优先级链 |

## TTY 检测（最关键场景）

管道 / 重定向 / CI 时自动调整：

- **关彩色**：输出到非 TTY 时去掉 ANSI 转义（否则管道到 `less` / 文件满是乱码）
- **关进度条**：非 TTY 时进度条无意义，改输出阶段日志
- **关交互**：非 TTY 时禁卡住等输入，fallback 到 flag 或默认值

检测：`isatty(stdout)`。例：`tool list | grep x` 时 `tool` 应关色。

## 输出格式分层

| 受众 | 格式 | 触发 |
|------|------|------|
| 人（终端） | 彩色 + 表格 + 进度 | 默认 TTY |
| 脚本（管道） | 纯文本无色 | 非自动关色 |
| 程序 | JSON / YAML | `--json` / `--output=json` |

## 交互场景

- 默认非交互（可脚本化）
- 危险操作（删除 / 覆盖 / 不可逆）要确认，或给 `--yes` / `-y` 跳过
- 向导模式（`tool init`）可选，但永远有非交互等价 flag 路径
- 非交互可跑：管道里禁卡住等输入

## 配置优先级链（场景 = 配置来源不同）

稳定优先级（高 → 低）：

```
命令行 flag > 环境变量 > 项目配置文件 > 全局配置文件 > 内置默认
```

- 每配置项五层都能设
- env 用统一前缀（`TOOL_DB_HOST`）
- 项目级覆盖全局
- `--config` 可指定文件路径

## CI 场景

- 检测 `CI=true` env → 强制无色、无进度、严格退出码
- 失败明确（日志可检索），批量报错（lint 类一次报全部）
- 幂等可重跑
