# TUI 媒介 · 布局与框架

TUI 布局在字符格栅上展开。先选框架 → 选布局模式 → 定栅格约束。场景见 [scenes.md](scenes.md)，组件见 [components.md](components.md)；色彩 / 风格在姊妹 skill `design-color`。

## 框架选型

| 框架 | 语言 | 特点 |
|------|------|------|
| Textual | Python | 现代异步、CSS 样式、组件丰富、易上手 |
| Bubble Tea | Go | Elm 架构（Model-Update-View）、轻快 |
| Ratatui | Rust | 性能、安全、细粒度控制 |
| ncurses | C/FFI | 老牌、底层、跨终端兼容最强 |
| Ink | JS/TS | React 模型写 CLI/TUI |

选型看：宿主语言生态、性能要求、团队熟悉度、是否需 CSS 样式层。

## 布局模式选择

| 布局 | 何时用 |
|------|--------|
| 单栏（列表 + 底栏） | 简单浏览、菜单 |
| 双栏 Master-Detail | 列表选条目、右侧看详情（邮件、文件） |
| 三栏（侧栏-主-检视） | 复杂应用、IDE、数据库工具 |
| 顶部 Tab 多视图 | 多独立功能、切换不丢上下文 |
| 底部命令栏 | 命令面板 + 交互混合（vim / emacs 风） |

## 栅格约束

- 最小尺寸假设：80×24（经典终端），设计在该尺寸不塌
- 响应式：宽屏展开多栏，窄屏折叠堆叠或隐藏次要栏
- 留边距（padding 1-2 字符），贴边拥挤
- 分隔线用 box-drawing：`─│┌┐└┘├┤┬┴┼`

## 布局骨架示例（双栏 Master-Detail）

```
┌─────────────┬──────────────────────────┐
│ Inbox       │  Subject: <selected>     │
│  > mail 1   │  From: ...               │
│    mail 2   │  ─────────────────────   │
│    mail 3   │  Body body body...       │
├─────────────┴──────────────────────────┤
│ j/k move  Enter open  q quit  ? help   │  ← 底栏（快捷键常驻）
└────────────────────────────────────────┘
```

原则：

- 边框区分区域
- 当前焦点栏边框 / 反色高亮
- 底栏常驻快捷键提示（发现性）

- [ ] 区域边框清晰
