# App 原生润色规则 + 预交付清单（Polish）

> 来源：`ui-ux-pro-max-skill/references/pro-rules.md`，中文化。
> **适用范围**：原生 / 移动 app UI（iOS / Android / React Native / Flutter）。
> 触发：native app UI 交付前，或用户反馈「UI 看起来不专业」但原因在 SKILL.md 优先级表里不明显时。
> Web / 桌面交互模式用 [[规则]]（stack-agnostic），不用本文件 — 下表假设触摸目标、安全区、平台手势约定，对桌面 web 不 1:1 适用。

---

## 一、图标与视觉元素

| 规则 | 标准 | 避免 | 为什么重要 |
|------|------|------|-----------|
| 禁 emoji 当结构图标 | 用矢量图标（Lucide / react-native-vector-icons / @expo/vector-icons） | 用 emoji 做导航、设置、系统控件 | emoji 依赖字体、跨平台不一致、无法用设计 token 控制 |
| 仅矢量资源 | 用 SVG 或平台矢量图标，缩放清晰、支持主题 | 模糊/像素化的 PNG 光栅图标 | 保证可缩放、清晰渲染、明暗模式自适应 |
| 交互态稳定 | 用颜色/透明度/阴影过渡做按压态，不变布局边界 | 改变布局的变换（移周边内容、引起抖动） | 防不稳定交互，保持移动端流畅动效/观感质量 |
| 品牌 logo 正确 | 用官方品牌资产，遵循其使用规范（间距/颜色/安全空间） | 猜 logo 路径、非官方改色、改比例 | 防 brand 误用，保证法律/平台合规 |
| 图标尺寸一致 | 图标尺寸定义为设计 token（如 icon-sm / icon-md=24pt / icon-lg） | 混用 20pt / 24pt / 28pt 等随意值 | 维持界面节奏与视觉层级 |
| 描边一致性 | 同一视觉层内用一致描边宽度（如 1.5px 或 2px） | 随意混用粗细描边 | 不一致描边降低观感精致度与一致性 |
| 实心 vs 线条纪律 | 每个层级用一种图标风格 | 同层级混用实心和线框图标 | 维持语义清晰与风格统一 |
| 触摸目标最小值 | 交互区最小 44×44pt（图标小则用 hitSlop 扩展） | 小图标无扩展点按区 | 满足无障碍与平台可用性标准 |
| 图标对齐 | 图标对齐文本基线，保持一致 padding | 图标错位或周围间距不一致 | 防微妙视觉失衡降低观感质量 |
| 图标对比度 | 遵循 WCAG：小元素 4.5:1，大 UI glyph 最低 3:1 | 低对比图标融入背景 | 保证明暗模式下的可访问性 |

## 二、交互（App）

| 规则 | 做 | 不做 |
|------|----|------|
| 点按反馈 | 80-150ms 内提供清晰按压反馈（ripple/opacity/elevation） | 点按时无视觉响应 |
| 动画时长 | 微交互约 150-300ms，用平台原生缓动 | 瞬切或慢动画（>500ms） |
| 无障碍焦点 | 读屏器焦点顺序匹配视觉顺序，标签具描述性 | 无标签控件或混乱焦点遍历 |
| 禁用态清晰 | 用 disabled 语义（`disabled`/原生 disabled props），降低强调，无点按动作 | 看着可点但不响应的控件 |
| 触摸目标最小值 | 点按区 ≥44×44pt（iOS）或 ≥48×48dp（Android），图标小则扩展命中区 | 极小点按目标或无 padding 的纯图标命中区 |
| 手势冲突预防 | 每区域一个主手势，避免嵌套点按/拖拽冲突 | 重叠手势导致误触 |
| 语义化原生控件 | 优先原生交互原语（`Button`/`Pressable`/平台等价物）带正确 a11y role | 用通用容器当主控件但无语义 |

## 三、明/暗模式对比

| 规则 | 做 | 不做 |
|------|----|------|
| 表面可读性（浅色） | 卡片/表面用足够 opacity/elevation 与背景清晰分离 | 过度透明表面模糊层级 |
| 文本对比（浅色） | 正文文本对比度 ≥4.5:1（对浅色表面） | 低对比灰色正文 |
| 文本对比（深色） | 深色表面主文本 ≥4.5:1，次文本 ≥3:1 | 深色模式文本融入背景 |
| 边框与分割线可见性 | 分隔符在两主题下均可见（不只浅色模式） | 单主题消失的主题性边框 |
| 状态对比对等 | 浅/暗主题下按压/聚焦/禁用态同样可辨 | 只为一个主题定义交互态 |
| Token 驱动主题 | 用语义化颜色 token 跨主题映射到表面/文本/图标 | 硬编码每屏 hex 值 |
| 遮罩与模态可读性 | 模态遮罩足够强以隔离前景内容（通常 40-60% 黑） | 弱遮罩让背景视觉竞争 |

## 四、布局与间距

| 规则 | 做 | 不做 |
|------|----|------|
| 安全区合规 | 所有固定头、标签栏、CTA 栏尊重上下安全区 | 固定 UI 放在刘海/状态栏/手势区下 |
| 系统栏间距 | 为状态/导航栏和手势 home indicator 加间距 | 可点内容与 OS chrome 碰撞 |
| 内容宽度一致 | 每设备类别（手机/平板）保持可预测的内容宽度 | 屏幕间混用随意宽度 |
| 8dp 间距节奏 | padding/gap/section 用一致 4/8dp 间距系统 | 无节奏的随机间距增量 |
| 可读文本度量 | 大设备上保持长文本可读（平板避免边到边段落） | 伤可读性的全宽长文本 |
| section 间距层级 | 按层级定义清晰垂直节奏档（如 16/24/32/48） | UI 层级相似但间距不一致 |
| 断点自适应槽距 | 大宽度/横屏时增大水平内边距 | 所有设备尺寸/方向用同一窄槽距 |
| 滚动与固定元素共存 | 加底部/顶部内容 inset，列表不被固定栏遮住 | 滚动内容被粘性头/尾遮住 |

## 五、预交付清单（canonical — 唯一权威清单）

交付 app UI 代码前，逐项验证。先走流程步骤，再走各分区勾选。

### 流程

- [ ] 实施前跑 `--domain ux "animation accessibility z-index loading"` 做验证
- [ ] 终检时过 [[规则]] §1–§3（CRITICAL + HIGH）
- [ ] 在 375px（小手机）和横屏下测试
- [ ] 开 **reduced-motion** 和 **Dynamic Type**/最大系统字号下验证行为
- [ ] 独立检查暗色模式对比（禁假设浅色值可继承）
- [ ] 确认所有触摸目标 ≥44pt 且无内容藏在安全区后

### 视觉质量

- [ ] 无 emoji 当图标（用 SVG）
- [ ] 所有图标来自一致图标族和风格
- [ ] 官方品牌资产，比例和安全空间正确
- [ ] 按压态视觉不移布局边界、不引起抖动
- [ ] 一致使用语义主题 token（无每屏硬编码颜色）

### 交互

- [ ] 所有可点元素提供清晰按压反馈（ripple/opacity/elevation）
- [ ] 触摸目标达最小尺寸（≥44×44pt iOS，≥48×48dp Android）
- [ ] 微交互时长在 150-300ms，原生感缓动
- [ ] 禁用态视觉清晰且不可交互
- [ ] 读屏器焦点顺序匹配视觉顺序，交互标签具描述性
- [ ] 手势区域避免嵌套/冲突交互（点按/拖拽/返回滑动手势冲突）

### 明/暗模式

- [ ] 主文本对比 ≥4.5:1（浅和暗模式均满足）
- [ ] 次文本对比 ≥3:1（浅和暗模式均满足）
- [ ] 分割线/边框和交互态在两模式下均可辨
- [ ] 模态/抽屉遮罩 opacity 足够强保前景可读（通常 40-60% 黑）
- [ ] 交付前两主题均测试（禁从单主题推断）

### 布局

- [ ] 头部、标签栏、底部 CTA 栏尊重安全区
- [ ] 滚动内容不被固定/粘性栏遮住
- [ ] 小手机、大手机、平板（竖屏 + 横屏）上验证
- [ ] 水平内边距/槽距按设备尺寸和方向正确自适应
- [ ] 4/8dp 间距节奏在组件、section、页面级维持
- [ ] 长文本度量在大设备上保持可读（无边到边段落）

### 无障碍

- [ ] 所有有意义的图片/图标有 a11y 标签
- [ ] 表单字段有标签、提示、清晰错误信息
- [ ] 颜色不是唯一指示器
- [ ] 支持 reduced-motion 和动态文本字号，不破布局
