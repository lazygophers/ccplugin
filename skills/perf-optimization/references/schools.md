# 三流派镜片(详解)+ 工具箱

按问题领域选镜片。SKILL.md 有对比表与症状映射表,这里是每个镜片的展开。

## 系统/后端镜片 —— USE 方法(排查第一步)

对**每一项**资源(CPU、内存、网络、磁盘、总线)逐一检查三指标:

- **U**tilization(使用率):资源繁忙时间占比
- **S**aturation(饱和度):无法处理的额外工作量,通常看队列长度
- **E**rrors(错误):错误事件计数

价值在于**完整性保证**——逼你不遗漏任何资源,避免"只看自己熟悉的指标"。利用率 100% 通常是瓶颈,I/O 资源 70%+ 常已是瓶颈。

补充手段:
- **火焰图**:看最宽平顶(x 轴=CPU 时间占比,非时间顺序,字母排序聚合)。
- **Off-CPU 分析**:很多问题不在 CPU 而在"等待"(阻塞/IO/锁),传统 CPU profiling 看不到,需追踪调度器事件补全。
- **TSA(线程状态分析)**:看线程在各状态(执行/可运行/等待)上花的时间,与 USE 互补。
- **工作负载特征化**:理解施加的负载本身(谁/为什么/是什么/如何变),有时最好的优化是消除不必要的工作。

## 前端/Web 镜片 —— 关键渲染路径 + 性能预算

渲染链:DOM → CSSOM → Render Tree → Layout → Paint。优化目标是最小化首次渲染前的关键资源数/字节数/路径长度(内联关键 CSS、async/defer 非关键 JS、优先首屏)。

**Core Web Vitals**(2024-2025 事实标准,均用 75 分位测量):
- **LCP**(最大内容绘制,加载)≤ 2.5s
- **INP**(交互到下次绘制,响应)≤ 200ms —— 2024 年 3 月正式取代 FID,FID 已于 2024 年 9 月废弃
- **CLS**(累积布局偏移,视觉稳定)≤ 0.1

其他参考阈值:TTFB ≤ 800ms、FCP ≤ 1.8s、TBT ≤ 200ms。

**性能预算**入 CI(如 JS < 170KB 压缩后,超预算构建失败)是防退化的关键。**RAIL 模型**(响应<100ms / 动画每帧<16ms / 最大化空闲 / 加载)从用户感知定义目标。

Souders 黄金法则:80-90% 响应时间在前端。注意:部分老规则(减少 HTTP 请求)在 HTTP/2 多路复用时代已弱化——区分"思维框架"(减少传输量/利用缓存仍成立)与"当前推荐实现"。

## 低延迟/高并发镜片 —— 机制同理心 + 单写者

- **False sharing(伪共享)**:两个无关变量落同一缓存行(x86 通常 64B)被不同线程写,等同写同一变量的竞争——"沉默的性能杀手"(看代码看不出来)。规避:cache line padding(注意 JIT 可能优化掉填充字段,需实测)。
- **单写者原则**:可扩展性最大限制是多写者竞争(实测两线程加锁自增比单线程慢约 394 倍)。让每个资源只被单一写者修改,业务逻辑跑单线程;无锁算法极难证明正确,只留给底层基础设施。多读者无妨(缓存一致性可广播只读副本)。
- **Disruptor**:预分配环形缓冲 + 序号 + memory barrier,GC 友好(immortal),三级流水线延迟比队列方案低 3 个数量级。队列差在 head/tail/size 写竞争 + 忽空忽满 + 产生垃圾。
- **数据导向设计(DOD,Acton)**:程序本质是数据变换;SoA 替代 AoS、提升内存连续性常带来 10x+。
- **GC**:分配率直接决定 GC 频率,降分配/对象复用优于调参;极端低延迟下关注 Time-To-Safepoint。
- **尾延迟度量(Tene)**:防协调遗漏,看 p99/p999/p9999,用 HDR Histogram。详见 `benchmarking.md`。

## 工具箱速查(按镜片选趁手工具)

工具不等于方法论——先定方法(该问系统什么),再抓工具。但选对工具能省时间:

| 镜片 | 度量/定位 | 负载/压测 | 分布/可视化 |
|------|-----------|-----------|-------------|
| 系统/后端 | `perf`、eBPF(`bpftrace`/BCC)、Off-CPU 分析、Netflix 60s checklist | —— | 火焰图(FlameGraph) |
| 前端/Web | Lighthouse、Chrome DevTools Performance、WebPageTest、PageSpeed Insights | —— | RUM(CrUX 真实用户)、瀑布图 |
| 低延迟/高并发 | JMH(Java 微基准)、async-profiler | wrk2 / k6(**开环**,防协调遗漏) | HDR Histogram(完整尾延迟分布) |
| 性能预算/CI | bundlesize、Lighthouse CI、SpeedCurve | —— | —— |
