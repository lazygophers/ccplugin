# 新平台画像模板(自扩展)

速查表外的平台(ProductHunt / B站动态 / 抖音文案 / 脉脉 / Discord / Threads / 微博 等), 先让用户填这张画像, 据此推导调性再改写。不要凭印象硬写——错配平台调性=全废。

## 画像填空表

```
平台名:
─────────────────────────────
① 受众画像: 谁在看 / 什么场景 / 什么诉求
② 内容形态: 帖 / 长文 / 短动态 / 视频文案 / 问答
③ 体量约束: 标题字数上限 / 正文长短 / 图片视频规格 / 标签数量
④ 调性定位: 技术真诚 / 种草共情 / 专业问答 / 轻社交 / 其它
⑤ 标题与开头偏好: 具体不党 vs 钩子痛点前置
⑥ 分发命中钥匙: 靠什么被推荐(算法权重 / 标签 / 圈子 / 转发 / 搜索)
⑦ CTA 得体形态: 该平台接受到哪一级(邀互动 / 轻引导 / 强转化)
⑧ 头号禁忌: 什么会被折叠 / 删 / 限流 / 反感
⑨ 违禁词与导流风险: 是否受广告法/平台敏感词约束, 站外导流是否禁止
⑩ AI 内容态度: 是否要求标注 / 打压 AI 空泛内容
```

## 据画像推导调性(决策启发式)

| 若画像显示… | 则按… 改写 |
| --- | --- |
| 受众是开发者/技术决策者 | 套技术阵营: show-don't-tell, demo+数据, CTA 只用 L1 邀互动, 营销腔即死 |
| 受众是大众消费者/找灵感 | 套种草阵营: PAS 痛点钩子, 第一人称体验, 极限词从严过滤 |
| 受众来求专业解答 | 套问答阵营: 先给结论/边界, 配数据引用, 利益相关前置披露 |
| 轻社交/同好圈子 | 真诚口语+发对圈子+提问引发评论, 禁通稿腔 |
| 平台禁站外导流(多数中文平台) | CTA 一律站内化, 删一切微信/二维码/外链 |
| 平台打压 AI 内容 | 强制走 `anti-ai-detection.md` 自检 + 注真实细节 |

## 通用底线(任何新平台都适用)

无论画像如何, 这几条不变:
1. 先给价值再谈自己(反硬广本质)。
2. 不刷量、不索要投票、不跨平台复制粘贴同一稿。
3. 不把未证实的算法降权数字当铁律。
4. 注入真实细节(去 AI 腔 + 建可信)。
5. 拿不准平台红线 → 让用户去该平台帮助中心/规则页核实, 不替用户赌。
