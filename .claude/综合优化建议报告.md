### P1-1: 实施智能并行任务调度
**维度**: 功能补全 + 性能提升

**问题描述**:
- 当前任务调度为手动串行执行
- `IMPROVEMENTS.md`中规划的"智能并行"特性未实现（0%完成度）
- 导致独立任务（如多文件分析）无法并行，浪费执行时间

**影响分析**:
- **效率损失**: 3个独立任务串行耗时15分钟，并行仅需5分钟
- **资源浪费**: CPU/GPU利用率低于30%
- **用户体验**: 等待时间过长，交互性差

**解决方案**:
1. **依赖分析引擎**:
   - 解析任务的输入输出声明
   - 构建DAG（有向无环图）
   - 识别可并行节点
2. **并行执行器**:
```python
class ParallelScheduler:
    def schedule(self, tasks: List[Task]) -> ExecutionPlan:
        dag = self._build_dag(tasks)
        levels = self._topological_sort(dag)
        return ExecutionPlan(parallel_groups=levels)
```
3. **并行度控制**: 限制最大并行数为2（符合CLAUDE.md规范）
4. **进度可视化**: 实时显示各并行任务状态

**工作量估算**: 12-16小时
- DAG算法实现: 6小时
- 执行器开发: 4小时
- 进度展示: 2小时
- 测试: 4小时

**收益预期**:
- 复杂任务执行时间减少40-60%
- CPU利用率提升至70%+
- 改善用户感知性能
