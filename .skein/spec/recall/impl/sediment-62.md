# 单变量轮模式

## 触发场景

优化循环（评分 → 改进 → 验证）—— 防止多维同改导致归因失效，无法确定哪项改动起效。

## 陷阱 / 正解

❌ 一轮改多个维度（如同时改触发词、结构、失败模式）  
根因：改后如果总分上升，无法确定是哪一项改动起效；如果下降，也不知道哪项需要回滚  
✅ 一轮只改一维度（或一紧密相关簇）

## 反例

❌ "本轮改触发词 + 结构 + 失败模式，希望总分 +5" → 改后总分 +3，不知道哪项起效哪项拖后腿  
✅ "本轮只改触发词，目标 Δ>0；下一轮改结构" → 每轮结果清晰可归因

## 案例

- optimize-any skill (已删除): 硬规 3 "单变量轮：一轮只改一维度（或一紧密相关簇），多维同改归因失效"
- plugin-dev optimize-rubric: "单变量轮: 一轮改一维度 (或一相关簇如 2/3/4 接线)"

## 适用

- 任何优化循环（评分 → 改进 → 验证 → ratchet）
- 实验性改动（需确定因果关系）
- A/B 测试场景
