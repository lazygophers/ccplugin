# CCPlugin Desktop - 测试指南

## 环境准备

### 1. 确认依赖已安装
```bash
# 检查Rust版本
rustc --version  # 应该 >= 1.90.0

# 检查Node.js版本
node --version   # 应该 >= v25.0.0

# 检查uv版本（用于Python命令）
uv --version
```

### 2. 启动开发服务器
```bash
cd desktop
npm run tauri dev
```

**预期结果**：
- ✅ Rust编译成功（可能需要1-2分钟首次编译）
- ✅ Vite开发服务器启动（端口1420）
- ✅ 应用窗口打开（1200x800）
- ✅ 显示侧边栏和顶部栏
- ✅ 默认显示Dashboard页面

## Phase 2: Python集成测试

### 测试7: Python命令调用（核心功能）

**前置条件**：
- 确保已安装uv：`uv --version`
- 确保网络连接正常

**操作步骤**：
1. 导航到Marketplace页面
2. 在输入框中输入插件名称（默认: `python`）
3. 点击"安装插件"按钮
4. 观察进度和结果

**测试清单**：
- [ ] 输入框可以输入文本
- [ ] 点击按钮后显示"执行中..."
- [ ] 进度条显示并更新（0% → 10% → 100%）
- [ ] 进度消息显示（"开始下载插件..." → "安装完成"）
- [ ] 最终显示执行结果（成功/失败）
- [ ] 标准输出显示安装日志
- [ ] 按钮恢复可点击状态

**预期结果（成功场景）**：
```
执行结果: ✅ 成功
标准输出:
Installing plugin python@ccplugin-market...
Plugin installed successfully to ~/.claude/plugins/cache/ccplugin-market/...
```

**预期结果（失败场景 - 插件已存在）**：
```
执行结果: ✅ 成功（可能显示已安装提示）
或
执行结果: ❌ 失败
错误输出: Plugin already installed
```

### 测试8: 更新插件

**操作步骤**：
1. 在输入框输入已安装的插件名称（如: `python`）
2. 点击"更新插件"按钮

**预期结果**：
- ✅ 进度显示正常
- ✅ 显示更新结果
- ✅ 如果已是最新版本，显示相应提示

### 测试9: 获取插件信息

**操作步骤**：
1. 输入插件名称
2. 点击"获取信息"按钮

**预期结果**：
- ✅ 显示插件详细信息
- ✅ 包含版本、描述、作者等metadata

### 测试10: 清理缓存

**操作步骤**：
1. 点击"清理缓存"按钮（红色按钮）
2. 等待执行完成

**预期结果**：
- ✅ 执行成功
- ✅ 显示清理结果
- ✅ 标准输出显示清理日志

### 测试11: 错误处理

**操作步骤**：
1. 输入不存在的插件名称（如: `nonexistent-plugin-xyz`）
2. 点击"安装插件"

**预期结果**：
- ✅ 显示错误状态（❌ 失败）
- ✅ 错误输出显示错误信息
- ✅ 错误信息清晰易懂
- ✅ 不会导致应用崩溃

### 测试12: 并发操作

**操作步骤**：
1. 输入插件名称
2. 快速连续点击"安装插件"按钮多次

**预期结果**：
- ✅ 按钮在执行期间被禁用
- ✅ 防止重复提交
- ✅ 只执行一次操作

# 生成覆盖率报告

# 或运行所有测试
pnpm test:all
