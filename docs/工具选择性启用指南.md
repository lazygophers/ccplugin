# Market Plugin 工具选择性启用指南

> 版本：0.1.0
> 适用场景：按需启用功能，减少工具列表复杂度

## 概述

Market Plugin 支持选择性启用工具，用户可以根据需求只启用部分功能，而不是全部 8 个工具。

## 设计原理

### 插件安装 vs 工具启用

| 层级 | 控制方式 | 粒度 |
|------|---------|------|
| **插件安装** | Claude Code 配置 | 整个插件 |
| **工具启用** | 环境变量 | 单个功能模块（2 个工具） |

**关键点**：
- ✅ 插件必须完整安装（代码文件）
- ✅ 工具可以选择性启用（运行时控制）
- ✅ 禁用的工具不会出现在 Claude 的工具列表中

### 工作原理

```
用户配置环境变量
    ↓
MarketConfig 读取配置
    ↓
MCP Server list_tools() 检查开关
    ↓
仅注册启用的工具
    ↓
Claude 只看到启用的工具
```

## 配置方法

### 方法 1：环境变量（推荐）

在启动 Claude Code 前设置：

```bash
# 仅启用记忆和知识库功能
export ENABLE_MEMORY=true
export ENABLE_CONTEXT=false
export ENABLE_TASK=false
export ENABLE_KNOWLEDGE=true

# 启动 Claude Code
claude
```

### 方法 2：在 plugin.json 中配置

编辑 `.claude-plugin/plugin.json`：

```json
{
  "mcpServers": {
    "market-server": {
      "command": "uv",
      "args": ["run", "python", "-m", "market.server"],
      "env": {
        "ENABLE_MEMORY": "true",
        "ENABLE_CONTEXT": "false",
        "ENABLE_TASK": "false",
        "ENABLE_KNOWLEDGE": "true",
        "LOG_LEVEL": "info"
      }
    }
  }
}
```

**优势**：配置持久化，无需每次设置环境变量。

### 方法 3：Shell 配置文件

添加到 `~/.bashrc` 或 `~/.zshrc`：

```bash
# Market Plugin 配置
export ENABLE_MEMORY=true
export ENABLE_CONTEXT=false
export ENABLE_TASK=true
export ENABLE_KNOWLEDGE=false
```

**优势**：全局生效，自动应用。

## 功能模块说明

### 模块 1：记忆管理（2 个工具）

**开关**：`ENABLE_MEMORY`

**包含工具**：
- `memory_store` - 存储记忆项
- `memory_search` - 搜索记忆

**使用场景**：
- ✅ 需要长期记忆功能
- ✅ 知识积累和检索
- ❌ 仅需临时上下文（可禁用）

### 模块 2：上下文管理（2 个工具）

**开关**：`ENABLE_CONTEXT`

**包含工具**：
- `context_save` - 保存会话上下文
- `context_retrieve` - 检索会话上下文

**使用场景**：
- ✅ 跨会话工作流
- ✅ 需要恢复中断的对话
- ❌ 单次对话场景（可禁用）

### 模块 3：任务管理（2 个工具）

**开关**：`ENABLE_TASK`

**包含工具**：
- `task_create` - 创建任务
- `task_list` - 列出任务

**使用场景**：
- ✅ 项目管理
- ✅ 待办事项追踪
- ❌ 不需要任务管理（可禁用）

### 模块 4：知识库管理（2 个工具）

**开关**：`ENABLE_KNOWLEDGE`

**包含工具**：
- `knowledge_add` - 添加知识
- `knowledge_search` - 搜索知识

**使用场景**：
- ✅ 文档和知识管理
- ✅ 语义搜索需求
- ❌ 不需要知识库（可禁用）

## 配置场景示例

### 场景 1：仅记忆和知识库（轻量级）

**需求**：只需要信息存储和检索，不需要任务管理。

```bash
export ENABLE_MEMORY=true
export ENABLE_CONTEXT=false
export ENABLE_TASK=false
export ENABLE_KNOWLEDGE=true
```

**结果**：4 个工具（memory_store, memory_search, knowledge_add, knowledge_search）

### 场景 2：全功能（默认）

**需求**：使用所有功能。

```bash
export ENABLE_MEMORY=true
export ENABLE_CONTEXT=true
export ENABLE_TASK=true
export ENABLE_KNOWLEDGE=true
```

**结果**：8 个工具（全部）

### 场景 3：仅任务管理

**需求**：仅作为任务管理工具使用。

```bash
export ENABLE_MEMORY=false
export ENABLE_CONTEXT=false
export ENABLE_TASK=true
export ENABLE_KNOWLEDGE=false
```

**结果**：2 个工具（task_create, task_list）

### 场景 4：会话助手

**需求**：需要上下文管理和记忆，不需要任务和知识库。

```bash
export ENABLE_MEMORY=true
export ENABLE_CONTEXT=true
export ENABLE_TASK=false
export ENABLE_KNOWLEDGE=false
```

**结果**：4 个工具（memory_*, context_*）

## 验证配置

### 1. 检查环境变量

```bash
echo $ENABLE_MEMORY
echo $ENABLE_CONTEXT
echo $ENABLE_TASK
echo $ENABLE_KNOWLEDGE
```

### 2. 在 Claude Code 中验证

启动 Claude Code 后，询问：

```
请列出当前可用的 market plugin 工具
```

Claude 只会列出启用的工具。

### 3. 查看日志（调试模式）

```bash
export LOG_LEVEL=DEBUG
uv run python -m market.server
```

日志会显示哪些工具被注册：

```
INFO - Market Server 初始化完成
DEBUG - 注册工具: memory_store
DEBUG - 注册工具: memory_search
DEBUG - 跳过工具: context_save (ENABLE_CONTEXT=false)
DEBUG - 跳过工具: context_retrieve (ENABLE_CONTEXT=false)
...
```

## 动态切换

### 重启生效

修改配置后，需要重启 MCP Server 才能生效：

```bash
# 1. 退出 Claude Code
# 2. 修改环境变量
export ENABLE_TASK=false

# 3. 重新启动 Claude Code
claude
```

### 配置文件优先级

```
plugin.json 中的 env 字段（最高）
    ↓
Shell 环境变量
    ↓
代码默认值（最低，全部 true）
```

## 性能影响

### 禁用工具的好处

| 方面 | 影响 |
|------|------|
| **启动速度** | 略微提升（减少工具注册） |
| **内存占用** | 略微降低（减少工具实例） |
| **Claude 响应** | 提升（工具列表更短，选择更快） |
| **用户体验** | 提升（减少无关工具干扰） |

### 建议

- ✅ **按需启用**：只启用实际使用的功能
- ✅ **简化列表**：减少 Claude 的工具选择负担
- ⚠️ **避免频繁切换**：确定配置后保持稳定

## 故障排查

### 问题：设置环境变量后工具仍然出现

**原因**：plugin.json 中的 env 覆盖了环境变量。

**解决**：
1. 检查 `.claude-plugin/plugin.json` 中的 `env` 字段
2. 如果有冲突，修改 plugin.json 或删除其中的 env 配置

### 问题：所有工具都不显示

**原因**：所有功能都被禁用。

**解决**：
```bash
# 至少启用一个功能
export ENABLE_MEMORY=true
```

### 问题：配置不生效

**排查步骤**：

1. 确认环境变量已设置：
```bash
env | grep ENABLE_
```

2. 确认 plugin.json 没有覆盖：
```bash
cat .claude-plugin/plugin.json | grep -A 10 '"env"'
```

3. 重启 Claude Code

## 总结

### 关键要点

| 问题 | 答案 |
|------|------|
| 可以部分启用工具吗？ | ✅ 是的，通过环境变量控制 |

| 如何配置？ | 环境变量或 plugin.json |

1. 评估实际需求
2. 选择需要的功能模块
3. 配置环境变量或 plugin.json
4. 重启 Claude Code
5. 验证工具列表
6. 根据使用情况调整
