# novelist

小说写作**全流程插件**。从爆款题材选择、世界观/人物/大纲设定, 到章节编写、一致性检查、校对、去 AIGC、重写——端到端一站式。中文细粒度目录结构, 编写前四要素确认, 生成后收尾链(检查→校对→去AIGC→定稿)+ 量化评分门控。

## 定位

```
不是「让 AI 写一段」, 而是「把一本书的完整生产线工程化」:
  设定(题材/世界观/人物/大纲) → 编写(四要素→生成) → 收尾(检查→校对→去AIGC→定稿)
每一环单一职责、独立可调; write 作为一站式入口把全链串起来。
```

核心理念: **书写 = 设计 + 生成 + 检查 + 校对 + 去AIGC 一体**, 任一环没过不算定稿。

## 快速上手

```
1. 新建小说       → /novelist-init  (建中文目录环境)
2. 核心设计       → /novelist-design (主线+主要人物+世界观, 一并设计)
3. 开写(一站式)  → /novelist-write  (前置就绪检查→四要素→生成→收尾链→定稿)
```

只用 `novelist-write` 就够——它会自动检查目录/设计是否就绪, 缺则引导补, 然后跑完整链。其余 skill 供深度单独使用。

## 目录结构(novelist-init 生成)

```
<小说名>/
├── 总览.md                  # 元信息 + 目录约定
├── 人物/<人物名>/           # 每人物独立: 简介.md / 经历.md / 关系.md
├── 世界观/                  # 地理 / 势力 / 规则(力量体系带代价) / 历史
├── 设定/                    # 物品 / 组织 / 术语, 一设定一文件
├── 大纲/                    # 总纲(核心冲突/结局) / 分卷
├── 情节/                    # 主线 / 支线 / 伏笔追踪(埋设→回收)
├── 章节/                    # 正文 + _索引.md
└── 元数据/                  # 进度 + 检查报告/ + 校对报告/
```

**目录归属隔离**: 一个目录只有一个 skill 独占写入, 互不踩踏(防多 skill 冲突)。

## Agents (4)

| Agent | 职责 | 边界 |
|---|---|---|

## 生成后收尾链(write/rewrite 核心)

每章生成后必过三环 + 评分门控才算定稿:

```
生成正文
  → a. 一致性检查 (novelist-check)   → 健康分 = 100−(致命×20+重要×5+建议×1)
  → b. 校对       (novelist-proofread) → 质量分 = 100−每千字错误×6
  → c. 去 AIGC    (novelist-humanize)  → 人味分(脚本算 tells+句长方差), 棘轮: 改后必须>改前
  → d. 定稿分门控  = 一致性×0.5 + 文字×0.2 + 人味×0.3
       🔴 ≥85 且一致性≥85 → 定稿; 否则退回最弱环
```

**重写棘轮**: rewrite 后定稿分必须 > 重写前那版, 否则重写无效。

## 编写前四要素(动笔硬闸门)

生成正文前必须确认(缺任一 🛑 STOP 不动笔):

1. **本期出场人物 + 故事梗概** — 谁出场、这章讲什么
2. **事件发生前** — 人物关系/世界观/设定当前状态
3. **事件发生后** — 同样三项变成什么(状态差 = 本章产出)
4. **结局** — 本章如何收尾(钩子/转折)

四要素把"事前→事后"状态差显式化, 既是生成边界, 也是一致性检查的对照基线。

## 评分机制(审查类量化)

只给**可量化**的审查环节加分(创作/设计类主观, 不强塞):

- 一致性 / 文字质量 / 定稿加权 = **纯算术**(skill 直接算)
- AI 味人味分 = **脚本算**(`skills/novelist-humanize/scripts/score_aitaste.py`, tells 正则 + 句长方差)

价值: 审查从定性→定量 → 阈值门控(达标才定稿)+ 棘轮(重写/去味必须分更高)。

## 从某章重写 / 连续多章

`novelist-rewrite` 三模式:
- **A 报告修复** — 按 check 报告定点修冲突
- **B 从第 N 章清空重写** — 破坏性, 先 `AskUserQuestion` 确认 + 备份到 `元数据/.废弃章节-<日期>/`, 再清空重写
- **C 连续多章** — 重写指定区间 [A, B]

均走收尾链 + 棘轮。

## 设计原则

- **单一职责 + 一站式编排**: 各 skill 职责清晰可单独调; write/rewrite 作为入口串全链, 用户不必逐个调
- **目录归属隔离**: 一目录一 skill 独占写入, 防冲突
- **自包含**: 脚本只在需要它的 skill 内(init / humanize 各自带), 无顶层共享目录
- **去 AIGC 锚定真人味**: 不为骗检测器(检测器误报率高), 锚定真·人味与可读性
