# novelist

小说写作**全流程插件**。从爆款题材选择、世界观/人物/大纲设定, 到章节编写、一致性检查、校对、去 AIGC、重写——端到端一站式。中文细粒度目录结构, 编写前四要素确认, 生成后收尾链(检查→校对→去AIGC→定稿)+ 量化评分门控。

## 定位

```
不是「让 AI 写一段」, 而是「把一本书的完整生产线工程化」:
  设定(题材/世界观/人物/大纲) → 编写(四要素→生成) → 收尾(检查→校对→去AIGC→定稿)
每一环单一职责、独立可调; write 作为一站式入口把全链串起来。
```

核心理念: **书写 = 设计 + 生成 + 检查 + 校对 + 去AIGC 一体**, 任一环没过不算定稿。

## 快速上手

```
1. 新建小说       → /novelist-init  (建中文目录环境)
2. 核心设计       → /novelist-design (主线+主要人物+世界观, 一并设计)
3. 开写(一站式)  → /novelist-write  (前置就绪检查→四要素→生成→收尾链→定稿)
```

只用 `novelist-write` 就够——它会自动检查目录/设计是否就绪, 缺则引导补, 然后跑完整链。其余 skill 供深度单独使用。

## 完整使用指南: 从零新建一本小说

下面是从空白到产出定稿章节的端到端流程。**最省事的走法**: 直接喊 `/novelist-write 写第1章`, write 会发现没目录/没设计, 自动倒序引导你补 init→design 再开写。下面按"显式分步"展开, 方便理解每一环。

### 第 1 步 · 初始化项目

```
/novelist-init 我的小说名 玄幻
```
- 在当前目录建 `我的小说名/`, 含全套中文子目录(人物/世界观/设定/大纲/情节/章节/元数据)+ 模板。
- 跑的是 `skills/novelist-init/scripts/init_novel.py`。已存在且非空会拒绝(防覆盖), 确认追加才加 `--force`。
- 产出后, `总览.md` 有目录约定表, 各目录已就位等填。

### 第 2 步 · 核心设计(主线 + 主要人物一起定)

```
/novelist-design 设计这本书的核心
```
- **主线剧情与主要人物必须一起设计**(主角欲望驱动主线冲突, 冲突反向筛选主角, 二者相互定义)。
- design 是编排者, 会依次:
  1. 立核心钩子(一句话核心冲突 + 题材基调 + 结局走向)→ 写 `大纲/总纲.md`
  2. (题材想兼顾市场)引用 `novelist-trending` 联网选爆款题材
  3. (选写法)引用 `novelist-craft` 按题材选叙事镜片(类型/结构/文学)
  4. 定 3-5 个主要人物 → 引用 `novelist-character` 落每人 简介/经历/关系 三文件
  5. 铺主线必需的世界观 → 引用 `novelist-worldview`(力量体系必带代价)
  6. 串主线三幕/分卷 → 引用 `novelist-outline`, 埋关键伏笔
  7. 闭环自检: 主角欲望 ↔ 主线冲突 ↔ 世界观规则 三者一致
- 产出: `大纲/总纲.md` + `人物/<主角>/...` + `世界观/规则.md` + `情节/主线.md` 等。

### 第 3 步 · 编写第 1 章(四要素 → 生成 → 收尾链)

```
/novelist-write 写第1章
```
write 自动跑完整链:
1. **前置就绪检查** — 目录、核心设计都在(第1-2步已就绪)→ 直接开写。
2. **🔴 四要素确认** — 逐项跟你敲定: (i)出场人物+梗概 (ii)事件前状态 (iii)事件后状态 (iv)结局。缺任一不动笔。
3. **生成正文** — 派 `chapter-writer` agent 按四要素 + craft 镜片写, 存 `章节/第001章-<标题>.md`。
4. **状态回写** — 把"事件后"的人物关系/世界观/伏笔变更同步回事实源。
5. **收尾链** — 检查(一致性健康分)→ 校对(文字质量分)→ 去AIGC(人味分, 棘轮)→ 定稿分门控(≥85 才定稿)。
6. 定稿 → 更新 `章节/_索引.md` + `元数据/进度.md`。

### 第 4 步 · 继续往下写

```
/novelist-write 写下一章        # 自动取索引最大号+1
```
- 第 2 章起, 动笔前先引用 `novelist-check` 拿"当前事实快照"(人物关系/世界观/未回收伏笔), 防设定漂移。
- 之后同第 1 章: 四要素 → 生成 → 收尾链 → 定稿。

### 第 5 步(按需) · 重写 / 修订

```
/novelist-rewrite 从第5章开始重写       # 模式B: 清空第5章及之后重写(破坏性, 先确认+备份)
/novelist-rewrite 第3到6章重写          # 模式C: 连续多章
/novelist-rewrite 按检查报告修复         # 模式A: 定点修冲突
```
- 重写同样走收尾链, 且有**棘轮**: 重写后定稿分必须 > 原版, 否则白改。

### 单独调用(深度使用)

任何 skill 都能单独喊, 不必走 write 编排:
```
/novelist-check 检查全书一致性
/novelist-proofread 校对第3章
/novelist-humanize 去掉第3章的 AI 味
/novelist-worldview 给魔法体系加个新规则
/novelist-character 完善反派的设定
```

### 一句话流程图

```
init(建目录) → design(主线+人物+世界观) → write(四要素→生成→检查→校对→去AIGC→定稿)
                                              ↑ 循环写每章 ↓
                                          rewrite(按需重写, 带棘轮)
```

## 目录结构(novelist-init 生成)

```
<小说名>/
├── 总览.md                  # 元信息 + 目录约定
├── 人物/<人物名>/           # 每人物独立: 简介.md / 经历.md / 关系.md
├── 世界观/                  # 地理 / 势力 / 规则(力量体系带代价) / 历史
├── 设定/                    # 物品 / 组织 / 术语, 一设定一文件
├── 大纲/                    # 总纲(核心冲突/结局) / 分卷
├── 情节/                    # 主线 / 支线 / 伏笔追踪(埋设→回收)
├── 章节/                    # 正文 + _索引.md
└── 元数据/                  # 进度 + 检查报告/ + 校对报告/
```

**目录归属隔离**: 一个目录只有一个 skill 独占写入, 互不踩踏(防多 skill 冲突)。

## Agents (4)

| Agent | 职责 | 边界 |
|---|---|---|
| chapter-writer | 执笔生成/重写章节正文 | 守四要素+规则, 不改设定 |
| continuity-auditor | 一致性审查 | 只读, 产冲突清单 |
| proofreader | 文字校对 | 只改文字层, 不动剧情 |
| humanizer | 去 AI 味改写 | 只改纹理, 不动剧情 |

## 生成后收尾链(write/rewrite 核心)

每章生成后必过三环 + 评分门控才算定稿:

```
生成正文
  → a. 一致性检查 (novelist-check)   → 健康分 = 100−(致命×20+重要×5+建议×1)
  → b. 校对       (novelist-proofread) → 质量分 = 100−每千字错误×6
  → c. 去 AIGC    (novelist-humanize)  → 人味分(脚本算 tells+句长方差), 棘轮: 改后必须>改前
  → d. 定稿分门控  = 一致性×0.5 + 文字×0.2 + 人味×0.3
       🔴 ≥85 且一致性≥85 → 定稿; 否则退回最弱环
```

**重写棘轮**: rewrite 后定稿分必须 > 重写前那版, 否则重写无效。

## 编写前四要素(动笔硬闸门)

生成正文前必须确认(缺任一 🛑 STOP 不动笔):

1. **本期出场人物 + 故事梗概** — 谁出场、这章讲什么
2. **事件发生前** — 人物关系/世界观/设定当前状态
3. **事件发生后** — 同样三项变成什么(状态差 = 本章产出)
4. **结局** — 本章如何收尾(钩子/转折)

四要素把"事前→事后"状态差显式化, 既是生成边界, 也是一致性检查的对照基线。

## 评分机制(审查类量化)

只给**可量化**的审查环节加分(创作/设计类主观, 不强塞):

- 一致性 / 文字质量 / 定稿加权 = **纯算术**(skill 直接算)
- AI 味人味分 = **脚本算**(`skills/novelist-humanize/scripts/score_aitaste.py`, tells 正则 + 句长方差)

价值: 审查从定性→定量 → 阈值门控(达标才定稿)+ 棘轮(重写/去味必须分更高)。

## 从某章重写 / 连续多章

`novelist-rewrite` 三模式:
- **A 报告修复** — 按 check 报告定点修冲突
- **B 从第 N 章清空重写** — 破坏性, 先 `AskUserQuestion` 确认 + 备份到 `元数据/.废弃章节-<日期>/`, 再清空重写
- **C 连续多章** — 重写指定区间 [A, B]

均走收尾链 + 棘轮。

## 设计原则

- **单一职责 + 一站式编排**: 各 skill 职责清晰可单独调; write/rewrite 作为入口串全链, 用户不必逐个调
- **目录归属隔离**: 一目录一 skill 独占写入, 防冲突
- **自包含**: 脚本只在需要它的 skill 内(init / humanize 各自带), 无顶层共享目录
- **去 AIGC 锚定真人味**: 不为骗检测器(检测器误报率高), 锚定真·人味与可读性
- **去 AIGC 接力生态**: humanize 运行时动态发现用户已安装的所有 humanizer 类 skill 依次执行, 再用内置补小说特化(不硬编码路径)
