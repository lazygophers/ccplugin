# novelist-worldview — 世界观与设定

把世界观拆成可独立演进、可逐项核对的文件, 让"规则"成为后续一致性检查的硬约束来源。

## 文件结构

| 路径 | 内容 | 一致性意义 |
|---|---|---|
| 世界观/地理.md | 区域、地标、气候、地图 | 人物移动/距离的约束 |
| 世界观/势力.md | 国家/组织/派系、势力范围、关系 | 阵营冲突的依据 |
| 世界观/规则.md | **力量/科技/魔法体系: 能力边界 + 代价** | 最关键——破例=最常见的设定崩坏 |
| 世界观/历史.md | 关键历史事件时间线 | 人物经历的时间锚点 |
| 设定/<名>.md | 其他设定: 物品/组织/术语, 一设定一文件 | 专有名词的唯一事实源 |

## 工作流(按序)

1. **定主题** — 用户要建/改哪块(地理/势力/规则/历史/某具体设定)。模糊 → 🔴 STOP, `AskUserQuestion` 确认范围再进 step 2。
2. **判新旧** — 读对应文件或 `设定/_索引.md`。索引文件不存在(首次) → 新建 `设定/_索引.md` 与 `世界观/_索引.md` 空模板再继续。已有则增量更新, 新设定则新建文件并登记。
3. **写设定 — 规则必带代价** — 按 `references/worldview-templates.md` 的模板写; 尤其 规则.md: 每种能力写清**能做什么 + 不能做什么 + 代价/限制**(内联最小模板: `| 能力名 | 可做什么 | 边界与代价 |`)。🔴 **任何能力缺代价/边界 → STOP, 追问后补全, 禁落盘「能力有、限制无」的规则**——无边界的力量是一致性灾难。
4. **登记索引** — 新增 设定/ 文件 → 更新 `设定/_索引.md`; 新增世界观块 → 确认 `世界观/_索引.md` 链接存在。
5. **回报 + 冲突扫描** — 若新设定与已有规则/历史/地理矛盾 → 🔴 STOP, 标出冲突点用 `AskUserQuestion` 请用户裁定改哪边, **禁自行让两处并存**; 建议改动后跑 `novelist-check`。

## 与其他 skill 的边界

- 本 skill **独占 世界观/ 与 设定/ 目录**。
- 被 `novelist-design` 引用: 核心设计阶段落地世界观骨架。
- `规则.md` 是 `novelist-check` 一致性审查的核心依据——写得越明确, 检查越有力。

## 失败模式(触发 → 一线修复 → 仍失败兜底)

| 触发 | 一线修复 | 仍失败兜底 |
|---|---|---|
| 主题模糊(不知建哪块) | `AskUserQuestion` 给四块+其他设定选项 | 用户仍不明 → 从 总览.md 题材反推建议 |
| 力量体系只写"能做什么"无代价 | 追问限制与代价, 补全边界 | 用户不定 → 文件标「代价待定」, 检查时视为高风险项 |
| 新设定与已有规则矛盾 | 标出矛盾请用户裁定 | 裁定后改唯一事实源, 禁两处并存 |
| 同一术语多处定义不一 | 合并到 设定/<名>.md 单一文件 | 引用处统一指向该文件 |

## ⛔ 反例黑名单(命中=错, 重来)

| # | 禁做 | 改为 |
|---|---|---|
| 1 | 力量体系只写能力不写代价/边界 | 每种能力必写限制与代价 |
| 2 | 同一设定散落多文件各说各话 | 一设定一文件, 唯一事实源 |
| 3 | 新设定与旧规则冲突却不标 | 主动扫冲突, 请用户裁定 |
| 4 | 写到 人物/ 或 章节/ 目录 | 只写 世界观/ 与 设定/ |
| 5 | 新增 设定/ 文件却不登记索引 | 必更新 设定/_索引.md |

> 被 novelist-design 引用落地世界观; 规则.md 是 novelist-check 的硬约束来源; 与 character/outline 目录互斥。
