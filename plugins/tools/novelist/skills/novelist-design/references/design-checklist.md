# 核心设计闭环校验清单

> novelist-design 第 6 步用。主角×主线×世界观三者必须互相咬合, 任一断裂 = 设计未成。

## 三角闭环(核心)

```
        主角欲望
        ╱      ╲
   驱动 ╱        ╲ 筛选
      ╱          ╲
  主线冲突 ──依赖── 世界观规则
```

- [ ] **主角欲望 → 主线冲突**: 主角想要的东西, 是否天然制造出贯穿全书的核心矛盾? (欲望够强、够具体、够难)
- [ ] **主线冲突 → 主角**: 这个冲突是否"只有这种性格的主角才会卷入/才会这样应对"? (冲突反向定义主角)
- [ ] **世界观规则 ↔ 主线**: 主线冲突所依赖的力量/势力/地理规则是否已定且自洽? 主角的解决手段是否在规则边界内(含代价)?

任一项答"否" → 调整, 直到三者闭环。反复对不上 → 硬停, 让用户拍板取舍。

## 设计产出清单

- [ ] 一句话核心冲突(写入 大纲/总纲.md)
- [ ] 结局走向(写入 大纲/总纲.md)
- [ ] 题材基调 → 据此用 novelist-craft 选定叙事镜片(类型/结构/文学)
- [ ] 3-5 个主要人物, 每个经 novelist-character 落三文件; 主角欲望已明确
- [ ] 主线必需的最小世界观, 经 novelist-worldview 落地(规则带代价)
- [ ] 主线展开为三幕/分卷节点, 经 novelist-outline 关联章节区间 + 埋关键伏笔

## 过度设定红线

- [ ] 世界观只建主线必需部分了吗? (开篇堆满全套设定 = 偏离主线, 砍)
- [ ] 主要人物 ≤ 5 个吗? (超了收敛为核心+配角, 配角后续单独 character)
- [ ] 镜片选定且全书统一吗? (混镜片导致写法漂移, 见 novelist-craft 维度5)
