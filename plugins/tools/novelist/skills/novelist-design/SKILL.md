# novelist-design — 核心设计(主要剧情 + 主要人物)

一本书的灵魂 = 主线冲突 × 主要人物。二者相互定义(主角的欲望生成冲突, 冲突筛选出主角), 故**必须在同一 skill 内一并设计**, 不可拆开各做各的。本 skill 统筹设计, 把细节落地委托给专责 skill。

## 设计顺序(核心骨架先行, 细节委托)

1. **前置检查** — 确认已 `novelist-init` 建好目录环境。无 → 🔴 STOP, 提示先跑 init, 禁无目录环境硬设计致产物无处落盘。
2. **立核心钩子** — 与用户敲定: 一句话核心冲突、题材基调、结局走向。题材方向若未定 / 想兼顾市场 → 先引用 `novelist-trending` 做选题(红蓝海判断 + 作者优势匹配, 联网取当下数据)。核心钩子格式: `[主角] 想要 [欲望/目标]，但 [障碍/对手] 阻拦，代价是 [选择/牺牲]`（一句话，禁写多句）。写入 `大纲/总纲.md` 的核心冲突与结局走向(也可直接引用 `novelist-outline` 来写)。
3. **定主要人物(3-5 个)** — 主角 + 关键对手/盟友。每个人物**引用 `novelist-character`** 落地其 简介/经历/关系 三文件。主角的核心欲望必须能驱动第 2 步的主线冲突。调用时传入: 人物名 + 核心欲望简述 + 与主角关系(主角/对手/盟友)。
4. **铺世界观底座** — 主线所依赖的最小世界观(规则/势力/地理)**引用 `novelist-worldview`** 落地, 只建主线必需的部分, 避免过度设定。调用时传入: 主线必需的模块列表(从规则/势力/地理中选)，模糊 → 🔴 STOP 与用户确认再委托。
5. **串主线结构** — **引用 `novelist-outline`** 把核心冲突展开为三幕/分卷的主线节点, 关联到章节区间, 埋下关键伏笔。调用时传入: 核心冲突句(来自 step2 格式) + 幕数(默认三幕)。
6. **一致性自检** — 按 `references/design-checklist.md` 逐项核对: 主角欲望 ↔ 主线冲突 ↔ 世界观规则三者闭环无矛盾 + 过度设定红线。(内联最小清单: ①主角欲望能直接推导出主线冲突吗？②世界观规则会影响冲突解法吗？③结局是否由主角欲望+障碍的合力决定，而非外部deus ex machina？三条全过→继续，任一不过→🔴 STOP 重审)有疑 → 建议跑 `novelist-check`。
7. **回报框架** — 给用户「主线一句话 + 主要人物表 + 三幕结构」概览, 提示用 `novelist-write` 开始编写。

## 引用关系(本 skill 是编排者)

| 设计部分 | 委托给 | 落盘目录 |
|---|---|---|
| 叙事镜片/创作手法选型 | novelist-craft | (不落盘, 给心智模型) |
| 主线/三幕/伏笔 | novelist-outline | 大纲/ 情节/ |
| 每个主要人物三文件 | novelist-character | 人物/ |
| 主线必需世界观 | novelist-worldview | 世界观/ 设定/ |

> **创作引擎**: 第 2-5 步设计主线与人物前, 引用 `novelist-craft` 按本书题材选定**叙事镜片**(类型/结构/文学), 用对应的人物塑造与情节推进心智模型指导设计, 保持全书写法一致。

> 本 skill **自己不直接写 人物/世界观/章节 目录**, 而是调用对应 skill, 避免与它们目录冲突。本 skill 的独有职责 = **统筹与闭环校验**(主角×主线×世界观三者一致)。

## 失败模式(触发 → 一线修复 → 仍失败兜底)

| 触发 | 一线修复 | 仍失败兜底 |
|---|---|---|
| 未 init, 无目录环境 | 提示先跑 novelist-init | 用户拒绝 → 不落盘, 仅口头给框架 |
| 主角欲望与主线冲突对不上 | 调整其一使二者互相驱动 | 反复对不上 → 🔴 STOP, 让用户拍板取舍 |
| 主要人物超 5 个, 框架发散 | 收敛到 3-5 个核心, 其余降为配角(后续单独 character) | 用户坚持多主角 → 标多线结构, 每线独立闭环 |
| 世界观越设越多, 偏离主线 | 只保留主线必需, 其余移交后续 worldview | —— |

## ⛔ 反例黑名单(命中=错, 重来)

| # | 禁做 | 改为 |
|---|---|---|
| 1 | 把主要剧情和主要人物拆成两次独立设计 | 二者同 skill 一并设计, 相互校验 |
| 2 | 本 skill 直接大写特写人物/世界观文件 | 引用 character/worldview 落地, 本 skill 只统筹 |
| 3 | 主角欲望与主线冲突各管各 | 必须互相驱动, 闭环 |
| 4 | 开篇就堆满全套世界观 | 只建主线必需, 避免过度设定 |
| 5 | 跳过 init 直接设计无处落盘 | 先确保目录环境存在 |

> 统筹层 skill: 引用 outline/character/worldview 落地, 自身专注主角×主线×世界观的一致性闭环。完成后交 novelist-write 编写。
