# novelist-check — 故事线一致性审查

把全书当成一个事实系统来核对: 任何"此处说 A、彼处说 ¬A"都是缺陷。本 skill 只读不改, 产出可执行的冲突清单, 供 novelist-rewrite 修复。

## 审查维度(六类冲突)

| # | 维度 | 典型缺陷 |
|---|---|---|
| 1 | 设定冲突 | 同一物品/术语/组织前后定义不一 |
| 2 | 人物矛盾 | 关系突变无铺垫、行为违背既定性格/动机、已死又出场 |
| 3 | 世界观违规 | 违反 规则.md 的力量边界/代价、势力格局自相矛盾 |
| 4 | 时间线错乱 | 事件顺序矛盾、人物年龄/经历与 历史.md 对不上 |
| 5 | 伏笔遗漏 | 情节/伏笔.md 中已过计划回收章仍「未回收」; 结尾仍有悬空伏笔 |
| 6 | 逻辑/合理性 | 因果断裂、动机不足、巧合过度 |

## 工作流(按序)

1. **定范围(免输入)** — 用户给了范围用之; **未给 → 默认全书自动审, 不追问**。可选: 第 N 章起 / 某人物线 / 某设定。
2. **派 continuity-auditor(只读)** — 用 `Agent` 工具(真实 tool_use)派 `continuity-auditor`, 6 字段 prompt: 目标(审 X 范围一致性) / 已知(事实源路径: 人物/世界观/设定/情节/章节) / 范围(只读, 禁改任何文件) / 输出格式(按 `references/conflict-rubric.md` 的单条冲突格式 + 六维核对清单) / 验收(覆盖六维、引用具体文件行) / 失败处理(信息不足标「需要:」)。
3. **汇总报告 + 一致性健康分** — 收 agent 结果, 落 `元数据/检查报告/<日期>-<范围>.md`, 按 `references/conflict-rubric.md` 的严重度分级(🔴致命/🟡重要/🟢建议)排序。**算量化健康分**(纯算术, 直接算): 健康分 = 100 −(致命×20 + 重要×5 + 建议×1), 下限 0。**🔴 门控: 有致命 或 分 < 85 = 未过**, 必须修复重检, 不进收尾链下一环。
4. **(被 write 引用时) 返回事实快照** — 若由 novelist-write 调用, 额外输出当前「人物关系 + 世界观规则 + 未回收伏笔」快照, 不必生成完整报告。
5. **导向修复** — 🔴 STOP: 有 🔴 致命冲突 → **必须先用 `novelist-rewrite` 修复才能继续编写**, 禁带着致命冲突往下写; 🟡 重要 → 建议本轮修复; 🟢 建议级 → 用户自行决定。

## 失败模式(触发 → 一线修复 → 仍失败兜底)

| 触发 | 一线修复 | 仍失败兜底 |
|---|---|---|
| 范围过大 agent 超时 | 分段审(按卷/按章区间)再合并 | 仍超时 → 只审用户最关心的人物/设定线 |
| continuity-auditor 标「需要: <缺事实源>」 | 补全对应设定文件后重派 | 设定本就缺 → 报告标「设定缺失」为冲突项 |
| 报告与用户认知不符(误报) | 复核证据行, 区分真冲突 vs 风格选择 | 用户判定误报 → 标注豁免, 不强改 |
| 规则.md 本身模糊导致无法判违规 | 提示先用 worldview 明确规则边界 | 暂不明确 → 标「规则待明确」高风险 |

## ⛔ 反例黑名单(命中=错, 重来)

| # | 禁做 | 改为 |
|---|---|---|
| 1 | 本 skill 直接改正文/设定 | 只读审查, 修复交 novelist-rewrite |
| 2 | main 自己通读判断, 不派 agent | 派 continuity-auditor(真实 Agent tool_use) |
| 3 | 口头说"已检查"但无 Agent 调用 | 宣称≠调用, 必有真实 tool_use |
| 4 | 报告只说"有矛盾"不给位置证据 | 每条带 文件:位置 + 证据 + 严重度 |
| 5 | 漏查伏笔回收 | 六维必含伏笔表核对 |

> 只读 skill; 派 continuity-auditor 审查; 被 novelist-write 引用取快照; 冲突清单交 novelist-rewrite 修复。
