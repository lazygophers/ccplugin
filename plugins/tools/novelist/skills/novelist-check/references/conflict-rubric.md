# 一致性冲突评级与报告格式

> continuity-auditor 产出、novelist-check 汇总时遵循。18 子项逐条核对。

## 严重度分级

| 级别 | 定义 | 处理 |
|---|---|---|
| 致命 | 读者会立刻察觉的硬矛盾(已死又出场、力量违规、时间线反转) | 必须 fix 或 rewrite 修复后才能继续 |
| 重要 | 影响连贯但非致命(关系突变缺铺垫、动机不足、伏笔过期未回收) | 必须修复(零容忍, 扣 5 分阻断满分) |
| 建议 | 风格/打磨级(可优化的巧合、可加强的铺垫) | **也阻断满分(零容忍, 扣 1 分)**, 须修后重检到 ==100 |

> 健康分 = 100 −(致命×20 + 重要×5 + 建议×1), 下限 0。**==100 才过(致命=重要=建议=0)**。建议级 不再「用户自行决定」, 任一 建议级 即 < 100 不过。

## 单条冲突报告格式

```
[级别] [子项编号] [维度] 一句话标题
- 子项: 如 2c 生死状态错乱
- 位置: 章节/第012章-xxx.md:行号  （或 设定文件路径）
- 证据: 此处写「A」 vs 世界观/规则.md:行 写「¬A」
- 影响: 为什么是冲突
- 建议(fix 路径): 单点改事实陈述(check fix) / 整段重写(派 rewrite 模式 A)
```

## check fix vs rewrite 模式 A 边界(冲突修复分流)

| 维度 | check mode=fix | rewrite 模式 A |
|---|---|---|
| 触发 | 单条冲突 / 单点可改 | 跨多章 / 结构性冲突清单 |
| 手段 | continuity-auditor 直接在受影响段落 Edit(文字级事实修正) | 派 chapter-writer 重写整段/整章 |
| 改动尺度 | 段内单点改几个事实陈述 | 整段重写 |
| 不做 | 不重写整段、不删章、不改结构 | — |

**判定规则**: 冲突可「改一处事实陈述」消除 → check fix; 需「重写段落衔接」→ rewrite 模式 A。
