# 一致性冲突评级与报告格式

> continuity-auditor 产出、novelist-check 汇总时遵循。

## 严重度分级

| 级别 | 定义 | 处理 |
|---|---|---|
| 🔴 致命 | 读者会立刻察觉的硬矛盾(已死又出场、力量违规、时间线反转) | 必须 rewrite 修复后才能继续 |
| 🟡 重要 | 影响连贯但非致命(关系突变缺铺垫、动机不足、伏笔过期未回收) | 建议本轮修复 |
| 🟢 建议 | 风格/打磨级(可优化的巧合、可加强的铺垫) | 用户自行决定 |

## 单条冲突报告格式

```
[级别] [维度] 一句话标题
- 位置: 章节/第012章-xxx.md:行号  （或 设定文件路径）
- 证据: 此处写「A」 vs 世界观/规则.md:行 写「¬A」
- 影响: 为什么是冲突
- 建议: 改哪边 / 怎么调和
```

## 六维核对清单

- [ ] 设定: 每个 设定/ 文件的定义在所有引用处一致
- [ ] 人物: 关系变化有铺垫、行为合性格动机、生死状态连贯
- [ ] 世界观: 力量使用不超 规则.md 边界且付出代价、势力格局自洽
- [ ] 时间线: 事件顺序、人物年龄/经历与 历史.md 对齐
- [ ] 伏笔: 情节/伏笔.md 逐条核对状态; 过计划回收章仍未回收 = 🟡; 结尾仍悬空 = 🔴
- [ ] 逻辑: 因果链完整、关键转折动机充分、无过度巧合
