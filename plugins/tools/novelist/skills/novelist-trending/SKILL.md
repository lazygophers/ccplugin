# novelist-trending — 爆款题材选择

帮作者回答"该写什么题材 / 现在能不能写 / 何时入场"。选题方法论稳定(怎么判断), 当下热点易变(谁在热)——本 skill 把二者分开: **判断逻辑写死在 skill, 实时数据靠 agentic protocol 联网取**。给镜片不替决策(同 `novelist-craft` 风格)。

## 主工作流(给定问题 → 收敛到候选题材)

按序执行, 每步标 `[入]→[出]`。下方四镜片 / 决策框架 / Agentic Protocol 是被本主干调用的**工具库**, 不单独跑。

1. **判问题类型** `[入]` 用户问题 `[出]` 三类之一 —— 纯方法论(怎么判红蓝海)→ step2a; 需当下数据(现在写什么火 / X 是红海吗 / 我想写 X 现在合适吗)→ step2b。
2. **取数或直答**
   - 2a **方法论直答** `[入]` 问题 `[出]` 判断框架 —— 用四镜片 + 决策框架答, 不联网, 到此结束。
   - 2b **联网取数** `[入]` 目标题材/方向 `[出]` 当月榜单 + 书荒呼声 + 扶持方向 + 行业增速 —— 必走下方 Agentic Protocol Step1-3, 禁凭语料编。
3. **四镜片分析** `[入]` 2b 联网事实 `[出]` 每题材的 生命周期段/拥挤度/供需缺口/跟风时机 —— 逐题材套镜片 1-4。
4. **决策框架收敛** `[入]` 候选题材 + 用户优势自评 `[出]` 热度 ∩ 优势交集, 排除「高热低驾驭」。
5. **输出候选表** `[入]` 3-4 结论 `[出]` 下方 Step3 表格, 由用户拍板, 禁替选。

## 第一原则: 当下热点必须联网取, 禁凭训练语料编

任何"现在 X 题材火 / Y 是蓝海"都是**时点变量**, 训练语料里的热点早已过时。硬停: 给具体题材建议前, **必须走下方 agentic protocol 联网取当下数据**, 禁直接报"现在流行 X"。方法论(本 skill)稳定, 实例每季度变。

## 选题方法论(四镜片, 稳定不过时)

### 镜片 1 — 平台与榜单逻辑(先搞清你在看哪张榜)

| 判断 | 启发式 |
|---|---|
| 分平台类型读榜 | 内容平台(起点=人工推荐+PK赛制, 上榜≈过编辑审美关) vs 算法平台(番茄=字节大数据, 起量≈过读者完读关)。同题材两类平台命运可能相反, 别跨平台迁移结论 |
| 分榜种读信号 | 新书榜读"趋势/增量"(机会窗口) vs 畅销/完结榜读"存量天花板"(多半已红海)。判蓝海看前者, 判拥挤度看后者 |
| 男频女频分开 | 男频锁"我能不能变强", 女频锁"她如何掌控人生"。爽点/节奏/命名两套体系, 禁跨频迁移 |
| 榜单配人工书单 | 纯算法榜被流量灌溉扭曲; 书荒广场/书单"求XX"的高频呼声更早暴露未满足需求 |

### 镜片 2 — 题材生命周期(识别自己在哪一段)

`蓝海开拓 → 验证爆发 → 跟风红海 → 同质衰退`

| 段 | 信号 | 动作 |
|---|---|---|
| 开拓期 | 现象级单品 + 榜上"孤例", 周围还没人跟 | 红利最厚风险最高, 值得赌但要快 |
| 红海期 | 标题/设定批量撞车, 平台拆分品类 | 你是后来者, 要么微创新要么换赛道 |
| 衰退期 | 能一眼说出"标准套路公式", 读者吐槽"又是这个" | 别入 |

> 当下哪个题材在哪一段 = 致命级 需联网取数(拉当月新书榜 + 书荒呼声 + 最近行业报告), 每次选题重跑"四段信号体检", 禁复用上本书的判断。

### 镜片 3 — 红海 vs 蓝海(三指标交叉, 别凭感觉)

- **拥挤度**: 数新书榜里目标题材占了几席, 占比高且雷同=红海。
- **供需缺口**: 真蓝海=书荒广场"求文"呼声高 × 榜上供给少(双条件, 缺一不是)。
- **平台投放意愿**: 区分"扶持新方向"(拆品类/专项征文=蓝海信号) vs "买量灌老题材"(战神/赘婿成本收入比约1:2=红海信号)。

### 镜片 4 — 跟风时机(入场早晚定生死)

- **跟的是品类需求, 不是那本书** — 网文一锤子买卖, 读者不为同款二次付费; 照抄已大爆的书最晚最死。承接它带火的固定读者群的"还想看什么变体"。
- **黄金窗口** = 已验证爆款 + 榜上同类还稀疏 + 书荒区还在求, 三者同时成立。
- **判定"已晚"红线** = 同质军团成型 / 影视短剧已批量跟拍(如赘婿剧一年27部)。
- **晚到出路** = 题材+一个新元素微创新, 或跨媒介先发(网文→短剧), 不硬挤原赛道。

## 选题决策框架(四镜片之上, 收敛到 1 个题材)

```
理想解 = 热门题材 ∩ 自己最擅长的细分      ← 不是二选一, 取交集
  ↓ 没有交集时
宁选「中热度 × 高驾驭」, 不选「高热度 × 低驾驭」  ← 不会写的爆款=陷阱
  ↓ 最后一步一票否决
可持续性: 撑不到完本 → 降级中长篇(<30万字也能出爆款, 还利改编)
```

- **作者优势优先**: 会写的冷门 > 不会写的爆款。先列"读过最多、不查资料就能写的 3 个题材"再从中挑。
- **平台模式定篇幅**: 免费推流(喜新厌旧)→控体量快节奏; 付费(有沉没成本)→可长线大世界观。
- **差异化硬定义**: 用户有需求 × 你能做 × 别人做不出, 三个 yes 才算切入点。微创新=熟悉设定+一个反转钩子; 或类型融合(刑侦+文物、规则怪谈+无限流)。

## Agentic Protocol — 实时题材数据怎么取

具体题材建议前**必须**执行(对应第一原则):

### Step 1 问题分类
| 类型 | 处理 |
|---|---|
| "现在写什么火/X 是红海吗"(需当下数据) | → 必走 Step 2 联网 |
| "选题方法论/怎么判断红蓝海"(纯框架) | → 直接用四镜片答 |
| "我想写 X, 现在合适吗"(混合) | → 先 Step 2 取 X 当下数据, 再用镜片分析 |

### Step 2 联网取数(⚠️ 必用 WebSearch/WebFetch, 禁跳过、禁编)
按四镜片反推搜索维度:
1. **拉当月各平台榜** — 起点/番茄/晋江/七猫 新书榜 + 畅销榜, 看目标题材占席与新晋异军。
2. **取供需缺口** — 搜书荒广场/书单"求XX类型"高频呼声 vs 实际供给。
3. **查平台扶持方向** — 近期平台征文公告/品类拆分/专项扶持(区分扶持新方向 vs 买量灌老题材)。
4. **看最近行业报告/增速** — 最新《网络文学发展报告》、艺恩短剧报告等的题材增速与红蓝海时点结论。
> 信息源黑名单: 知乎、微信公众号、百度百科。优先平台官方榜单规则、权威媒体(36氪/晚点/界面/光明网)、行业报告、作者访谈。

### Step 3 输出
联网事实 + 四镜片分析, 以下表格格式输出候选列表, **由用户拍板**, 禁替选:

| 题材 | 生命周期段 | 拥挤度 | 供需缺口 | 平台扶持 | 优势匹配(用户自评) | 综合信号 |
|---|---|---|---|---|---|---|
| (举例)都市重生 | 跟风红海 | 高(新书榜占比 N%) | 低 | 无专项 | — | ⚠️ 慎入 |
| (举例)文物鉴宝 | 验证爆发 | 中 | 高(书荒区求文频繁) | 有征文 | — | ✅ 窗口期 |

综合信号图例: ✅ 推荐窗口 / ⚠️ 谨慎 / ❌ 已晚。「优势匹配」列由用户填写, 不由 AI 替评。

## 失败模式(触发 → 一线修复 → 仍失败兜底)

| 触发 | 一线修复 | 仍失败兜底 |
|---|---|---|
| 无法联网取当下数据 | 明确告知用户"只能给方法论, 当下热点无法核实", 用四镜片给判断框架 | 禁拿训练语料里的旧热点冒充当下 |
| 用户目标函数不明(冲榜变现/影视改编/个人表达) | `AskUserQuestion` 问目标——目标不同, 方法论取舍方向翻转 | 默认"商业向", 标注可调 |
| 用户要的题材没有市场数据 | 用供需缺口 + 平台扶持两指标兜底判断 | 数据真空 → 标"信息不足", 降低结论置信度 |
| 用户坚持写不擅长的爆款题材 | 提示"不会写的爆款=陷阱(写长乏力)", 建议降级中长篇 | 用户坚持 → 标风险, 尊重创作选择不强拦 |

## 反例黑名单(命中=错, 重来)

| # | 禁做 | 改为 |
|---|---|---|
| 1 | 凭训练语料报"现在 X 题材火" | 必走 agentic protocol 联网取当下数据 |
| 2 | 替用户拍板写哪个题材 | 给候选+红蓝海+匹配度, 用户选 |
| 3 | 只看"当前榜单火"就推荐 | 数据滞后, 看趋势拐点/将热未热, 而非追已霸榜 |
| 4 | 推"高热度但用户不擅长"的题材 | 会写的冷门 > 不会写的爆款, 取热度∩优势交集 |
| 5 | 把蓝海等于"没人写的冷门" | 真蓝海=有人求×没人供; 纯冷门可能是无人问津 |
| 6 | 选题当成功充分条件 | 题材=必要非充分, 执行力/完本权重 > 选题精度 |
| 7 | 忽略政策红线选题 | 先过红线(涉政/涉黄/低俗全本下架), 现实题材是政策正向扶持方向 |

## 诚实边界

- 选题方法论解决"该不该写这个题材/何时入场", **不解决"能不能写好"**——题材是入场券, 留存靠作者独有的人物/情绪/世界观执行力。
- 当下热点是时点变量, 本 skill 的实例一律标"需联网取数"; 训练语料里的热点已过时, 不可作依据。
- 爆款不可精确复制——本框架提高"不踩坑、对得上优势、踩中将热赛道"的概率, 不承诺"按此必出爆款"。
- 数据有滞后(行业报告通常滞后约 1 年)与平台差异, 落地前务必按目标平台当下榜单二次校准。
