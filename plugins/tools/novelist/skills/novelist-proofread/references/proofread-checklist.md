# 文字校对清单

> novelist-proofread / proofreader 用。只在文字层工作, 不碰剧情设定。

## mode 入参(detect | fix)

proofreader 按 `mode` 入参决定行为:

| mode | 行为 | 是否改正文 |
|---|---|---|
| detect | 只读扫描, 逐项核对, 产出报告 | ❌ 不改正文 |
| fix(默认) | 产出报告 + 就地 Edit 修正 | ✅ 就地改 |

mode 解析: 用户输入首 token 为 `detect` / `fix` → 设 mode; 否则 `fix`(默认, 向后兼容)。

## 12 子项逐项核对

> 5 大维度 → 12 子项(3+3+2+2+2)。每子项独立报告条数 + detect/fix。

### 1 错别字(3)

- [ ] **1a 形近字** — 己/已、的/得/地、未/末
- [ ] **1b 音近字** — 坐/座、需/须、附/符
- [ ] **1c 多字 / 漏字** — 重衍字、脱漏关键字

### 2 语法(3)

- [ ] **2a 成分残缺** — 缺主/谓/宾
- [ ] **2b 搭配不当** — 动宾不合、修饰不当
- [ ] **2c 语序与关联词** — 语序错乱、关联词误用(虽…但 / 因为…所以)

### 3 标点(2)

- [ ] **3a 误用与缺失** — 逗号该句号、句末无标点
- [ ] **3b 中英标点混用与配对** — ,。 vs ,. 、引号/书名号配对

### 4 用词(2)

- [ ] **4a 啰嗦重复与口语书面混杂** — 同段重复词、文白夹杂
- [ ] **4b 生造词** — 自造词、不规范缩略

### 5 用字统一(2, 以基准为准)

- [ ] **5a 人物称呼/译名** — 前后一致(基准: 人物/_索引.md)
- [ ] **5b 术语/专有名词** — 用字一致(基准: 设定/_索引.md)

## 报告格式(每条)

```
[子项编号 类别] 位置: 章节/第NNN章-xxx.md:行号
  原文: <错处>
  改后: <修正>(fix 模式下已在正文就地改)
```

子项编号取上表 1a / 2c / 5a 等; 类别用维度名(错别字/语法/标点/用词/用字统一)。

## 边界红线(必守)

- 只改"文字对不对、顺不顺"; 不评判剧情合不合理。
- 疑似剧情/设定矛盾 → 不自行改, 记一条「建议跑 novelist-check」。
- 拿不准是错误还是风格 → 标出请人定, 不擅改。
- 方言/口头禅/有意的破碎句 = 风格, 保留, 禁当错别字抹平。
- 统一基准缺失 → 该子项(5a/5b)标「基准待定」, 不强统一。
- 🔴 **detect 模式禁改正文** — 只产报告; 改正文必须显式 mode=fix。

## 收尾

- detect 模式: 报告落盘即止, 正文不动。
- fix 模式: 正文就地修正 → 清单落 元数据/校对报告/<日期>-<范围>.md → 章节/_索引.md 对应章状态推进「已校对」。
