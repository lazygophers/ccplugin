# novelist-outline — 大纲与情节线

把"整体走向"与"逐节推进"分层管理, 并显式追踪伏笔, 为编写与一致性检查提供路线图。

## 文件结构

| 路径 | 内容 |
|---|---|
| 大纲/总纲.md | 核心冲突、三幕/分卷结构、结局走向 |
| 大纲/分卷.md | 每卷主题、起止章、卷末钩子 |
| 情节/主线.md | 主线节点 → 涉及章节 → 推进的核心冲突 |
| 情节/支线.md | 支线 → 涉及人物 → 涉及章节 → 与主线交汇点 |
| 情节/伏笔.md | **伏笔 → 埋设章 → 计划回收章 → 状态(未回收/已回收)** |

## 工作流(按序)

1. **定范围** — 用户要做哪层(总纲/分卷/主线/支线/伏笔)。模糊 → 🔴 STOP, `AskUserQuestion` 给五层选项, 确认后再进 step 2。
2. **对齐上游** — 读 总览.md、大纲/总纲.md; 若新规划与既定核心冲突/结局矛盾 → 🔴 STOP, 用 `AskUserQuestion` 请用户裁定改总纲还是改新节点, **禁落盘冲突版本**。
3. **写规划** — 按 `references/outline-templates.md` 的模板, 用表格落具体节点, 每节点关联到章节区间; 主线/支线标交汇点防脱节。
4. **维护伏笔表** — 新埋伏笔→登记并填「计划回收章」; 已回收→改状态。🔴 **新伏笔无计划回收章 → STOP, 追问回收点再落盘; 禁留空回收章**。这是 check 的关键输入。
5. **回报** — 给出结构概览, 提示编写可用 `novelist-write` 逐章推进。

## 与其他 skill 的边界

- 本 skill **独占 大纲/ 与 情节/ 目录**。
- 被 `novelist-design` 引用: 核心设计阶段产出主线总纲。
- `情节/伏笔.md`、`情节/主线.md` 是 `novelist-write`(编写依据) 与 `novelist-check`(回收核对) 的共同输入。

## 失败模式(触发 → 一线修复 → 仍失败兜底)

| 触发 | 一线修复 | 仍失败兜底 |
|---|---|---|
| 范围模糊 | `AskUserQuestion` 给五层选项 | 不明 → 默认从总纲入手 |
| 新规划与既定结局矛盾 | 标出请用户裁定改总纲还是改新节点 | 裁定前不落盘冲突版本 |
| 支线与主线脱节(无交汇) | 提示补交汇点或并为独立番外 | 用户坚持独立 → 标注「独立支线」 |
| 伏笔无计划回收章 | 追问预计回收点 | 暂不定 → 填「待定」, check 时高亮 |
| 伏笔已过计划回收章仍未标「已回收」 | 核实是否已回收: 是→改状态; 否→标「逾期」并提示补写 | novelist-check 也会捕获, 两处双保险 |

## ⛔ 反例黑名单(命中=错, 重来)

| # | 禁做 | 改为 |
|---|---|---|
| 1 | 大纲只有文字流水, 不关联章节 | 每节点关联章节区间, 用表格 |
| 2 | 埋伏笔不进 伏笔.md | 一律登记 + 填计划回收章 |
| 3 | 新规划与既定结局冲突却不标 | 主动对齐总纲, 请用户裁定 |
| 4 | 写到 章节/ 正文或 人物/ 目录 | 只写 大纲/ 与 情节/ |

> 被 novelist-design 引用产出主线; 伏笔/主线表供 write 与 check 共用; 与 character/worldview 目录互斥。
