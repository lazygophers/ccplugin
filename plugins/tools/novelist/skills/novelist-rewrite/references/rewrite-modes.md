# 重写三模式操作细则

> novelist-rewrite 用。模式 B/C 涉及删除 = 破坏性, 必须先确认、先备份。

## 模式判定

| 用户措辞 | 模式 |
|---|---|
| "按检查报告改" / "修复冲突" | A 定点修复 |
| "从第N章重写" / "第N章之后全删重写" / "推倒重写" | B 清空重写(破坏性) |
| "第A到B章重写" / "这几章重写" | C 连续多章 |

不明确 → `AskUserQuestion` 让用户选模式 + 范围。

## 模式 A — 定点修复

1. 读最新 元数据/检查报告/(无则先跑 novelist-check)。
2. 每条 🔴/🟡 冲突定位章节。
3. 逐条派 chapter-writer 重写受影响段落, prompt 附: 冲突点 + 正确事实(事实源) + 修复要求。
4. 回写状态变更 → 重跑 check 确认冲突清零。

## 模式 B — 从第 N 章清空重写(破坏性)

```
🔴 STOP 闸门顺序(不可乱序):
  ① AskUserQuestion 确认: "将删除第N章及之后共X章, 不可逆, 确认?" — 未确认不动手
  ② 备份: 待删章节移到 元数据/.废弃章节-<日期>/ (禁直接 rm)
  ③ 清空: 章节/_索引.md 移除第N章起的行
  ④ 回滚状态源: 人物关系/世界观/伏笔 回退到第 N-1 章末状态
     (依据第 N-1 章四要素的"事后状态")
  ⑤ 重写: 从第N章起逐章走 novelist-write 完整流程
```

## 模式 C — 连续多章(第 A-B 章)

1. 确认区间 [A, B]。
2. 区间内每章按序: 取原四要素(或与用户重定) → 派 chapter-writer 重写 → 回写状态。
3. 区间外不动; 重写完跑 novelist-check 核对区间与上下文衔接。

## 备份恢复

- 备份位置: 元数据/.废弃章节-<日期>/
- 删错/删多 → 从备份恢复; 备份也无 → 如实报告用户, 禁谎称已恢复。

## 收尾(三模式统一)

回写事实源 → 跑 novelist-check → 更新 章节/_索引.md 与 元数据/进度.md → 回报重写范围+状态变更+检查结果。
