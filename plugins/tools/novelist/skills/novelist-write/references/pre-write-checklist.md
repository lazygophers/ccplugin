# 编写前四要素确认清单

> novelist-write 第 3 步的硬闸门。四项全部确认才允许动笔。任一缺失 → 硬停。

逐项与用户确认, 把答案写进本章的临时笔记(或直接作为 chapter-writer 的 prompt 输入)。

## (i) 本期出场人物 + 故事梗概

- [ ] 本章出场人物清单(主/次)
- [ ] 每个出场人物本章的目标/动作
- [ ] 本章一句话梗概(讲什么事)
- [ ] 本章对应大纲/主线的哪个节点

## (ii) 事件发生「前」的状态(对照基线)

确认进入本章时的既有事实——必须与现有事实源一致(从 novelist-check 快照取):

- [ ] 出场人物**彼此关系**现状(谁和谁什么关系、是否已见过面)
- [ ] 相关**世界观/规则**现状(本章会用到的力量体系、势力格局)
- [ ] 相关**设定**现状(涉及的物品/组织/术语当前定义)

## (iii) 事件发生「后」的状态(本章真正产出 = 状态差)

确认本章结束时上述三项变成什么样——这是必须回写事实源的变更:

- [ ] 人物关系**变化**(新结识/反目/升温/死亡等)
- [ ] 世界观/规则**变化**(势力消长、新规则揭示)
- [ ] 设定**变化**(新物品出现、术语首次定义)

> 若 (ii) 与 (iii) 完全相同(无任何状态差), 本章可能是水章——提示用户确认是否值得写。

## (iv) 结局(本章收尾)

- [ ] 本章如何收束(高潮/转折/悬念钩子/平稳过渡)
- [ ] 是否埋新伏笔(→ 登记 情节/伏笔.md)或回收旧伏笔(→ 更新状态)
- [ ] 与下一章的衔接点

## 确认后

四项齐全 → 进入 novelist-write 第 4 步, 派 chapter-writer。
四项把"事前→事后"的状态差固化, 既是生成正文的边界, 也是后续 novelist-check 的对照基线。
