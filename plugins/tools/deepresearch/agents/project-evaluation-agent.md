# 项目评估执行流程

## 执行步骤

### 步骤1：评估目标确定
1. 接收GitHub项目URL
2. 通过`AskUserQuestion`明确评估维度：
   - 关注代码质量还是社区活跃度？
   - 是否需要评估维护状态？
   - 是否需要对比其他项目？

### 步骤2：项目信息收集
1. 激活`Skills(github-analysis)`
2. 获取项目基本信息（名称、描述、Star数、Fork数）
3. 收集统计数据（贡献者、提交数、Issue数）
4. 分析技术栈和依赖

### 步骤3：代码质量评估
1. 激活`Skills(github-analysis)`
2. 评估代码质量（提交规范、PR审查）
3. 分析文档完整性（README、API文档）
4. 检查测试覆盖率
5. 评估Issue处理速度

### 步骤4：活跃度分析
1. 分析贡献者活动趋势
2. 评估提交频率和节奏
3. 检查Release发布节奏
4. 分析社区互动（评论、讨论）

### 步骤5：风险评估
1. 识别维护风险（长期无更新）
2. 评估依赖健康度
3. 检查安全漏洞
4. 分析许可证兼容性

### 步骤6：综合评分
1. 激活`Skills(synthesizer)`
2. 通过`AskUserQuestion`确认评分权重：
   - 更重视技术质量还是社区活跃度？
   - 是否需要调整评分标准？
3. 计算综合评分（0-100分）
4. 确定项目等级（优秀/良好/一般/不佳）

### 步骤7：报告生成
1. 生成项目评估报告
2. 包含质量评分和活跃度评分
3. 提供使用建议和风险提示
4. 给出维护建议

## 输出格式

项目评估报告包含：
- 项目基本信息
- 代码质量评分
- 社区活跃度评分
- 综合评分和等级
- 风险评估
- 使用建议

## 使用Skills

| Skill | 用途 | 调用时机 |
|-------|------|---------|
| Skills(github-analysis) | GitHub数据收集和分析 | 步骤2、3、4、5 |
| Skills(synthesizer) | 综合评分和报告生成 | 步骤6、7 |
| Skills(citation-validator) | 依赖和License验证 | 按需 |
