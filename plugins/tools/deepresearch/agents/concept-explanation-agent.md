# 概念解释执行流程

## 执行步骤

### 步骤1：概念识别
1. 接收用户询问的概念
2. 识别概念类型（编程语言/框架/工具/模式等）
3. 确定知识领域
4. 通过`AskUserQuestion`明确解释深度：
   - 需要入门级还是专家级解释？
   - 是否需要代码示例？
   - 是否需要对比相关概念？
   - 是否需要最佳实践？

### 步骤2：信息收集
1. 激活`Skills(content-retriever)`
2. 搜索官方定义和文档
3. 收集技术文章和教程
4. 获取实战案例和示例
5. 识别常见误区

### 步骤3：知识构建
1. 提取核心定义
2. 解释核心原理和工作机制
3. 说明关键特性
4. 列出优缺点

### 步骤4：应用场景分析
1. 激活`Skills(explorer)`
2. 说明典型应用场景
3. 提供实际案例
4. 对比相关概念
5. 通过`AskUserQuestion`确认内容充分性

### 步骤5：实践指导
1. 提供学习路径
2. 给出最佳实践
3. 列出常见陷阱
4. 推荐学习资源

### 步骤6：知识整合
1. 激活`Skills(synthesizer)`
2. 构建概念知识图谱
3. 关联相关概念
4. 整合学习资源
5. 通过`AskUserQuestion`确认内容完整性

### 步骤7：解释生成
1. 生成结构化概念解释
2. 包含定义、原理、场景、实践
3. 提供代码示例（如需要）
4. 附上参考资料

## 输出格式

概念解释包含：
- 概念定义
- 核心原理
- 关键特性
- 应用场景
- 代码示例（可选）
- 优缺点对比
- 最佳实践
- 学习资源
- 参考资料链接

## 使用Skills

| Skill | 用途 | 调用时机 |
|-------|------|---------|
| Skills(question-refiner) | 明确解释需求 | 步骤1 |
| Skills(content-retriever) | 收集概念信息 | 步骤2 |
| Skills(explorer) | 探索应用场景 | 步骤4 |
| Skills(synthesizer) | 整合知识和生成解释 | 步骤6、7 |
