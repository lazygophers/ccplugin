# 技术选型执行流程

## 执行步骤

### 步骤1：需求明确
1. 接收选型需求描述
2. 通过`AskUserQuestion`明确需求细节：
   - 应用场景（Web/移动/桌面/后端/数据等）
   - 技术约束（语言、框架、平台）
   - 性能要求（响应时间、并发量）
   - 团队技术栈和经验
   - 预算和时间限制

### 步骤2：候选方案收集
1. 激活`Skills(content-retriever)`
2. 搜索相关技术方案
3. 收集官方文档和最佳实践
4. 获取社区反馈和案例
5. 通过`AskUserQuestion`确认候选方案是否完整

### 步骤3：多维度对比
1. 功能特性对比
2. 性能基准对比
3. 学习曲线对比
4. 生态系统对比（库、工具、社区）
5. 成本分析（开发、部署、维护）

### 步骤4：风险和限制评估
1. 激活`Skills(explorer)`
2. 识别技术风险
3. 评估供应商锁定风险
4. 分析长期维护风险
5. 检查安全合规性

### 步骤5：加权评分
1. 激活`Skills(synthesizer)`
2. 通过`AskUserQuestion`确认评分权重：
   - 功能优先级？
   - 性能重要性？
   - 团队熟悉度权重？
   - 社区活跃度权重？
3. 为每个方案打分
4. 生成排序结果

### 步骤6：推荐方案
1. 提供Top3推荐方案
2. 说明推荐理由
3. 列出各方案优缺点
4. 提供实施建议

### 步骤7：报告生成
1. 生成选型报告
2. 包含对比矩阵
3. 提供决策建议
4. 附上迁移路径（如适用）

## 输出格式

技术选型报告包含：
- 需求总结
- 候选方案列表
- 多维度对比表
- 风险评估
- 推荐方案（Top3）
- 实施建议

## 使用Skills

| Skill | 用途 | 调用时机 |
|-------|------|---------|
| Skills(question-refiner) | 明确选型需求 | 步骤1 |
| Skills(content-retriever) | 收集候选方案信息 | 步骤2、3 |
| Skills(explorer) | 探索风险和限制 | 步骤4 |
| Skills(synthesizer) | 评分和推荐生成 | 步骤5、6、7 |
