# 架构评审执行流程

## 执行步骤

### 步骤1：评审范围确定
1. 接收架构文档或代码路径
2. 通过`AskUserQuestion`明确评审范围：
   - 评审整个系统还是特定模块？
   - 关注点：性能/安全/可维护性/扩展性？
   - 是否需要对比参考架构？
   - 是否提供重构建议？

### 步骤2：架构理解
1. 激活`Skills(local-code-analysis)`
2. 分析架构文档
3. 识别架构模式（分层/微服务/事件驱动等）
4. 理解设计决策和权衡
5. 绘制架构视图

### 步骤3：模块划分评估
1. 激活`Skills(local-code-analysis)`
2. 评估模块职责划分
3. 检查模块边界清晰度
4. 分析模块间耦合度
5. 评估内聚性

### 步骤4：依赖关系分析
1. 分析依赖方向和层次
2. 检查循环依赖
3. 评估依赖复杂度
4. 识别关键依赖路径

### 步骤5：设计原则遵循度
1. 激活`Skills(citation-validator)`
2. 检查SOLID原则遵循
3. 评估设计模式使用
4. 验证最佳实践应用
5. 通过`AskUserQuestion`确认评审标准

### 步骤6：风险和限制识别
1. 识别单点故障
2. 评估扩展性限制
3. 检查安全风险
4. 分析性能瓶颈
5. 评估可测试性

### 步骤7：报告生成
1. 激活`Skills(synthesizer)`
2. 生成评审报告
3. 提供改进建议
4. 标注风险等级
5. 给出重构路线图（如需）

## 输出格式

架构评审报告包含：
- 架构概览
- 模块划分评估
- 依赖关系分析
- 设计原则遵循度
- 风险评估
- 改进建议
- 重构路线图（可选）

## 使用Skills

| Skill | 用途 | 调用时机 |
|-------|------|---------|
| Skills(local-code-analysis) | 架构和依赖分析 | 步骤2、3、4 |
| Skills(citation-validator) | 设计原则验证 | 步骤5 |
| Skills(synthesizer) | 评审报告生成 | 步骤7 |
| Skills(explorer) | 探索重构方案 | 按需 |
