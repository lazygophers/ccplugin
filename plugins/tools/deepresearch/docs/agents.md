# 代理系统

Deep Research 插件提供六个专业代理，覆盖不同研究场景。

## 代理列表

| 代理 | 描述 | 适用场景 |
|------|------|----------|
| `本地代码分析专家` | 深度分析本地代码库 | 代码审查、重构分析 |
| `GitHub项目研究专家` | 研究开源项目 | 技术选型、竞品分析 |
| `依赖包分析专家` | 分析第三方依赖 | 安全审计、依赖优化 |
| `关键词探索专家` | 深度主题研究 | 概念学习、领域研究 |
| `架构分析专家` | 分析系统架构 | 架构评估、设计评审 |
| `技术方案搜索专家` | 搜索对比技术方案 | 技术选型、方案对比 |

## 本地代码分析专家

### 职责

- 分析代码结构和质量
- 识别代码模式
- 发现潜在问题
- 提供优化建议

### 使用示例

```
分析 src/ 目录的代码质量
```

## GitHub项目研究专家

### 职责

- 分析开源项目结构
- 研究项目实现方式
- 评估项目质量
- 提供学习建议

### 使用示例

```
研究 facebook/react 项目的架构设计
```

## 依赖包分析专家

### 职责

- 分析依赖关系
- 检查安全漏洞
- 评估依赖质量
- 提供优化建议

### 使用示例

```
分析项目的 npm 依赖安全性
```

## 关键词探索专家

### 职责

- 深度研究主题
- 收集多源信息
- 整合研究发现
- 生成研究报告

### 使用示例

```
研究微服务架构的最佳实践
```

## 架构分析专家

### 职责

- 分析系统架构
- 评估架构决策
- 识别架构风险
- 提供改进建议

### 使用示例

```
分析当前项目的架构合理性
```

## 技术方案搜索专家

### 职责

- 搜索技术方案
- 对比方案优劣
- 提供选型建议
- 生成对比报告

### 使用示例

```
对比 REST 和 GraphQL 的优劣
```
