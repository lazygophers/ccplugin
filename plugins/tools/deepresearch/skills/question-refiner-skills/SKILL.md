## 问题优化器

这个技能负责将用户提出的模糊或宽泛的研究问题转化为结构化的、可执行的研究计划。

### 激活条件
当用户提出以下内容时自动激活：
- 模糊的研究问题（"研究人工智能"）
- 缺乏明确方向的问题
- 过于宽泛的主题（"技术发展"）
- 需要进一步细化的研究需求

### 优化流程

#### 1. 初步分析
```markdown
- 识别问题类型（学术、技术、市场、个人等）
- 确定核心关键词和概念
- 评估问题的复杂度和范围
```

#### 2. 关键维度提问
针对每个研究问题，系统会询问以下5-6个关键维度：

**研究范围**
- 具体关注哪些子领域？
- 地理和时间范围？
- 重点关注哪些方面？

**输出要求**
- 需要什么格式的输出？
- 需要多详细的程度？
- 是否需要特定数据或分析？

**受众分析**
- 报告给谁看？
- 他们的背景和需求是什么？
- 需要什么级别的技术深度？

**时间约束**
- 有严格的时间限制吗？
- 需要快速概览还是深入研究？
- 可以接受多长时间？

**质量要求**
- 是否需要权威来源？
- 数据准确性要求？
- 需要验证吗？

#### 3. 生成结构化提示
基于回答生成包含以下要素的结构化提示：
```markdown
# 研究主题
## 具体范围
- [具体子领域1]
- [具体子领域2]
## 时间范围
- [起始时间] 至 [结束时间]
## 地理范围
- [地区/国家]
## 受众
- [目标受众描述]
## 输出要求
- [格式要求]
- [详细程度]
- [特殊要求]
## 质量标准
- [来源要求]
- [验证要求]
```

### 最佳实践

#### 问题识别
- 区分事实性问题 vs 观点性问题
- 识别是否有足够的公开信息
- 评估是否需要专业权限或付费资源

#### 范围控制
- 避免范围过大导致无法完成
- 避免范围过小失去研究价值
- 保持适当的粒度

#### 受众导向
- 根据受众调整技术深度
- 考虑受众的背景知识
- 确保输出对受众有价值

### 常见问题处理

**问题过于宽泛**
- 提供具体的子领域选择
- 建议聚焦当前相关度高的方面

**问题过于具体**
- 识别是否可以适当扩展
- 建议相关的补充方向

**缺乏明确目标**
- 通过追问明确研究目的
- 区分信息收集和分析需求

### 输出示例

#### 输入
```
我想研究人工智能在医疗领域的应用
```

#### 输出
```markdown
# 人工智能在医疗领域的应用研究

## 具体范围
- 诊断辅助系统
- 医学影像分析
- 药物研发
- 临床决策支持

## 时间范围
- 2020年 至 2024年

## 地理范围
- 全球市场，重点关注美国、欧洲、中国

## 受众
- 医疗行业投资者
- 医疗科技公司管理层
- 医疗政策制定者

## 输出要求
- 综合分析报告（20-30页）
- 市场规模和增长预测
- 技术成熟度评估
- 主要厂商分析
- 投资机会和建议

## 质量标准
- 权威行业报告
- 学术研究论文
- 上市公司财报
- 专家访谈
```
