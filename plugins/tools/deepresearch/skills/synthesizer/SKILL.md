## 研究合成器

这个技能负责将分散的研究发现整合为统一、连贯的综合报告，支持多种输出格式以满足不同的使用场景。

### 激活条件

当需要整合以下内容时自动激活：
- 多个智能体的研究发现汇总
- 不同来源的信息整合
- 多角度的观点综合
- 时间序列的趋势分析
- 跨领域的知识连接

### 核心合成流程

#### 1. 信息分类与标记
```markdown
- 按主题分类：相关主题的信息聚合
- 按来源标记：标明信息来源和质量等级
- 按时间排序：建立时间发展脉络
- 按重要性标记：区分核心信息和补充信息
```

#### 2. 矛盾识别与处理
```markdown
- 事实冲突：不同来源的矛盾信息
- 观点对立：不同专家的不同看法
- 时效差异：信息的更新时间不同
- 视角差异：不同利益相关者的看法
```

#### 3. 价值提炼
```markdown
- 关键发现：最具价值的核心信息
- 重要趋势：发展方向和变化规律
- 隐含模式：表面信息下的深层规律
- 洞见生成：基于数据的创新见解
```

### 输出格式选择

#### 可用输出格式

| 格式类型 | 文件路径 | 适用场景 | 目标受众 | 特点 |
|---------|---------|---------|---------|------|
| 执行摘要 | formats/executive-summary.md | 高层决策、快速汇报 | 管理层、决策者 | 简洁、重点突出、可操作 |
| 技术报告 | formats/technical-report.md | 深度分析、技术评审 | 技术团队、开发者 | 详细、结构化、完整 |
| 学术论文 | formats/academic-paper.md | 学术研究、理论分析 | 研究者、学者 | 严谨、引用规范、深度 |
| 演示文稿 | formats/presentation.md | 会议分享、团队汇报 | 团队、客户 | 可视化、简洁、互动 |
| 博客文章 | formats/blog-post.md | 知识分享、技术传播 | 广泛读者 | 通俗易懂、案例丰富 |
| 对比表格 | formats/comparison-table.md | 方案对比、选型决策 | 决策者、评估者 | 一目了然、多维对比 |
| 清单报告 | formats/checklist.md | 质量检查、审计结果 | 审核者、QA | 结构化、可检查 |
| llms.txt | formats/llms-txt.md | API文档、项目概览 | LLM、开发者工具 | LLM友好、标准化、精确 |

#### 输出格式选择策略

**基于研究目的**：
```markdown
- 决策支持 → 执行摘要 / 对比表格
- 深度分析 → 技术报告 / 学术论文
- 知识分享 → 博客文章 / 演示文稿
- 质量检查 → 清单报告
- 方案评估 → 对比表格 / 执行摘要
- API文档 → llms.txt
- 项目概览 → llms.txt
```

**基于受众类型**：
```markdown
- 管理层/决策者 → 执行摘要、对比表格
- 技术团队 → 技术报告、清单报告
- 研究者 → 学术论文
- 广泛受众 → 博客文章、演示文稿
- LLM/AI Agents → llms.txt
- 开发者工具 → llms.txt
```

**基于内容复杂度**：
```markdown
- 简单单一主题 → 执行摘要、清单报告
- 中等复杂度 → 博客文章、对比表格
- 高复杂度/多维度 → 技术报告、学术论文
- API/项目文档 → llms.txt
```

### 输出格式选择流程

#### 步骤1：分析研究特征
```markdown
1. 识别研究目的和核心问题
2. 确定目标受众和使用场景
3. 评估内容复杂度和信息量
4. 明确时间约束和交付要求
```

#### 步骤2：筛选合适格式
```markdown
1. 基于研究目的筛选候选格式
2. 基于受众类型进一步过滤
3. 基于内容复杂度确定最终候选列表（2-3个）
```

#### 步骤3：用户确认
```markdown
1. 通过 AskUserQuestion 提供候选格式选项
2. 说明每种格式的特点和适用场景
3. 获取用户选择或接受推荐
```

**AskUserQuestion 示例**：
```
问题：请选择研究报告的输出格式

选项：
- 执行摘要（推荐）：适合快速汇报和决策支持，1-2页
- 技术报告：适合深度技术分析，包含完整细节，10-20页
- 对比表格：适合方案对比和选型决策，一目了然

默认选择：执行摘要
```

#### 步骤4：应用格式模板
```markdown
1. 加载选定的格式模板（见 formats/ 目录）
2. 根据模板结构填充研究内容
3. 确保符合格式规范和质量标准
```

### 合成质量标准

#### 完整性
- 覆盖所有关键研究维度
- 包含主要的发现和分析
- 提供充分的背景信息
- 保持内容的平衡性

#### 一致性
- 消除信息矛盾和冲突
- 确保逻辑连贯性
- 保持概念的一致使用
- 统一表述风格

#### 深度
- 提供深入的因果分析
- 展现多角度的理解
- 包含前瞻性思考
- 体现创新性洞见

#### 实用性
- 提供可行的建议
- 包含具体的行动计划
- 考虑实际实施条件
- 关注实际应用价值

### 整合技术

#### 矛盾处理技术
```markdown
- 优先采用权威来源信息
- 标注不确定性和争议点
- 提供多方观点对比
- 寻求更高层次的一致性
```

#### 叙事构建技术
```markdown
- 建立清晰的故事线
- 设置合理的章节结构
- 使用过渡性语言连接
- 保持读者阅读流畅性
```

#### 价值提升技术
```markdown
- 突出最重要的发现
- 强化关键洞见
- 提供实用的建议
- 展示研究的独特价值
```

### 最佳实践

#### 合成前准备
- 确保所有信息经过验证
- 标记关键信息和补充信息
- 建立信息分类体系
- 设计报告结构框架

#### 合成过程控制
- 按逻辑顺序逐步整合
- 定期检查一致性
- 及时处理发现的矛盾
- 保持整体的平衡性

#### 质量检查
- 检查信息覆盖的完整性
- 验证逻辑的连贯性
- 确认深度的适当性
- 评估实用性程度

#### 优化改进
- 根据反馈调整结构
- 精简冗余内容
- 强化关键部分
- 提升整体可读性
