## llms.txt 输出格式

### 适用场景
- API文档生成
- 项目文档概览
- 技术站点索引
- LLM上下文提供
- 开发者文档入口

### 目标受众
- LLM/AI Agents
- 开发者
- 技术文档系统
- 开发环境工具

### 格式特点
- **长度**：1-5页
- **风格**：结构化、LLM友好、精确格式
- **重点**：关键信息、文档链接、清晰层级

## llms.txt 规范

### 基本结构

llms.txt文件必须按顺序包含以下部分：

1. **H1标题**（必需）：项目或站点名称
2. **引用块**（推荐）：简短摘要，包含理解文件所需的关键信息
3. **详细信息**（可选）：除标题外的任何markdown内容
4. **文件列表**（可选）：H2分隔的markdown链接列表

### 格式规范

```markdown
# [项目/站点名称]

> [简短摘要 - 包含理解项目所需的关键信息]

[详细信息 - 可选，任何非标题的markdown内容]

## [分类名称]

- [链接标题](URL): [可选的链接描述]
- [链接标题](URL): [可选的链接描述]

## Optional

- [次要信息链接](URL): [描述]
```

**特殊说明**：
- `## Optional` 章节有特殊含义 - 可在需要缩短上下文时跳过
- 所有链接必须是有效的markdown超链接格式
- 链接描述应为简洁的说明性文字

## 质量标准

### 内容要求
- **摘要必须精准**：引用块是LLM理解内容的关键，必须包含核心信息
- **链接必须有效**：所有URL必须是实际可访问的有效链接
- **描述必须简洁**：每个链接描述应在1-2句话内
- **层级必须清晰**：使用H2标题明确分类

### 格式要求
- 严格遵循llms.txt规范的四个部分结构
- H1只能用于标题，H2用于分类
- 链接格式必须为`[title](url): description`
- Optional章节必须命名为"Optional"

### LLM友好性
- 使用清晰、精确的语言
- 避免歧义和未解释的术语
- 提供足够的上下文信息
- 包含理解和使用的必要信息

### 验证方法
```bash
# 使用llms_txt2ctx工具验证
llms_txt2ctx your-file.txt

# 检查生成的上下文文件
cat llms-ctx.txt

# 用LLM测试理解效果
```

## 最佳实践

### 摘要编写
```markdown
好的摘要：
> FastHTML是一个Python库，集成Starlette、Uvicorn、HTMX和FastTags，
> 用于创建服务端渲染的超媒体应用。注意：不兼容FastAPI语法，
> 不支持React/Vue/Svelte。

不好的摘要：
> FastHTML是一个很棒的Web框架。
```

### 链接组织
```markdown
- 将最重要、最常访问的链接放在前面
- 按使用频率排序，而非按字母顺序
- 相关链接放在同一分类下
- 使用Optional存放次要信息
```

### 描述编写
```markdown
好的描述：
- [快速开始](URL): 5分钟快速上手，包含安装和Hello World

不好的描述：
- [快速开始](URL): 点击查看
```

### 自动生成
```python
# 从研究数据生成llms.txt
def generate_llms_txt(research_data):
    output = []
    output.append(f"# {research_data['title']}\n")
    output.append(f"> {research_data['summary']}\n")
    # ... 添加其他部分
    return '\n'.join(output)
```
