# 来源验证器（Source Validator）

## 核心职责

对研究中的所有信息源进行A-E级质量评分和链式验证。

## 激活时机

信息检索后的质量检查阶段，过滤低质量来源。

## A-E 评级系统

### A级（9.0-10分）- 最高可信度

**特征**：
- 同行评审论文（Nature、Science、IEEE）
- 权威机构报告（WHO、FDA、NASA）
- 系统综述和Meta分析

**验证**：
- DOI存在且可验证
- 期刊影响因子>5
- 引用次数>100（发表1年+）

---

### B级（7.0-8.9分）- 高可信度

**特征**：
- 行业分析报告（Gartner、IDC、McKinsey）
- 官方技术文档（React Docs、AWS Docs）
- 临床指南和标准文档

**验证**：
- 来源机构权威性
- 作者专家背景
- 发布时间<2年

---

### C级（5.0-6.9分）- 中等可信度

**特征**：
- 专业技术博客（Martin Fowler、Dan Abramov）
- 会议记录和演讲稿
- 案例研究和白皮书

**验证**：
- 作者有相关领域经验
- 内容有引用支持
- 无明显商业偏见

---

### D级（3.0-4.9分）- 低可信度

**特征**：
- 个人博客（无专家背景）
- 营销材料和产品宣传
- 预印本论文（未经评审）

**验证**：
- 信息仅供参考
- 需交叉验证
- 标注不确定性

---

### E级（0-2.9分）- 不可信

**特征**：
- 传言和小道消息
- 纯个人观点（无依据）
- 过时信息（>5年且已被推翻）

**处理**：
- 自动过滤
- 不纳入最终报告
- 如果唯一来源，标注高风险

---

## 链式验证

### 1. 交叉验证

同一事实需要≥2个独立来源（B级以上）确认。

### 2. 时效性检查

```
技术类：发布时间<2年
学术类：发布时间<5年
政策类：需为当前有效版本
```

### 3. 利益冲突检测

```
商业来源：标注赞助商和利益相关方
作者关系：声明与研究对象的关系
资金来源：注明研究资金来源
```

### 4. 事实核查

```
统计数据：验证原始数据源
引用声明：检查原文是否支持引用观点
专家观点：确认专家身份和领域
```

---

## 执行流程

```
1. 初步评级：基于来源类型自动评分
2. 深度验证：检查DOI、作者背景、引用次数
3. 交叉验证：查找独立来源确认
4. 时效性检查：验证发布时间和当前有效性
5. 利益冲突检测：识别潜在偏见
6. 最终评分：综合评定A-E级别
7. 过滤处理：过滤<5.0分的来源
```

---

## 质量指标

```
总分 = 权威性(40%) + 时效性(25%) + 准确性(20%) + 独立性(15%)
```

**权威性**：来源机构、作者背景、同行认可
**时效性**：发布时间、更新频率、当前有效性
**准确性**：事实核查、数据验证、引用正确性
**独立性**：无利益冲突、客观中立、无商业偏见

## 使用示例

```python
# 验证单个来源
source = {
    "title": "React性能优化指南",
    "url": "https://react.dev/learn/optimizing-performance",
    "author": "React Team",
    "date": "2024-01-15"
}

rating = validate_source(source)
# 输出：{"grade": "B", "score": 8.5, "reasons": ["官方文档", "时效性强"]}
```
