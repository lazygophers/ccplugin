## 图思维控制器

这个技能负责在整个研究过程中应用图思维框架，确保研究路径的高效性和结果的准确性。

### 激活条件
当需要进行复杂的多步推理和路径优化时自动激活：
- 需要同时处理多个研究方向
- 需要在信息不确定时动态调整
- 需要权衡深度与广度的关系
- 需要识别和填补知识缺口

### 核心操作

#### Generate(k) - 生成k个并行研究路径
```markdown
- 功能：从当前节点生成k个新的研究分支
- 目标：探索不同的研究角度和可能性
- 参数：k（并行路径数量，通常4-8）
- 输出：k个新的研究节点
```

#### Aggregate(k) - 合并k个发现
```markdown
- 功能：将k个相关发现整合为一个综合报告
- 目标：从多个视角形成完整理解
- 参数：k（要合并的发现数量）
- 输出：综合分析报告
```

#### Refine(1) - 改进单个发现
```markdown
- 功能：对特定发现进行深度优化
- 目标：提高信息质量和准确性
- 参数：目标发现的标识符
- 输出：优化后的发现
```

#### Score - 质量评分
```markdown
- 功能：评估发现的质量（0-10分）
- 目标：指导资源分配和路径选择
- 评分维度：权威性、时效性、相关性、准确性
- 输出：质量分数和改进建议
```

#### KeepBestN(n) - 保留最佳n个节点
```markdown
- 功能：筛选并保留最有价值的n个节点
- 目标：控制信息过载，聚焦高质量内容
- 参数：n（保留数量）
- 输出：筛选后的高质量节点集合
```

### 研究策略模式

#### 平衡搜索策略（推荐用于大多数场景）
```markdown
1. Generate(5): 生成5个并行研究路径
2. Score: 评估每个路径的初始质量
3. KeepBestN(3): 保留质量最高的3个路径
4. Refine(3): 对每个保留路径进行深度优化
5. Aggregate(3): 合并为最终综合报告
```

#### 深度优先策略（推荐用于技术深度研究）
```markdown
1. Generate(3): 生成3个主要研究方向
2. Score: 评估并选择最有价值的主路径
3. Generate(3): 从最佳路径继续深入3个子方向
4. Repeat: 重复3-4次达到所需深度
5. Aggregate(n): 合并所有深度发现
```

#### 广度优先策略（推荐用于市场调研和概览）
```markdown
1. Generate(8): 生成大量并行路径（8-10个）
2. KeepBestN(5): 筛选出5个最有前景的方向
3. Generate(2): 对每个方向扩展2个子方向
4. Score: 评估扩展后的信息质量
5. Aggregate(5): 整合所有广度发现
```

### 研究路径管理

#### 路径初始化
```markdown
- 输入：结构化研究提示
- 处理：提取核心概念和关键词
- 输出：初始研究路径图
```

#### 动态调整
```markdown
- 监控：每个路径的信息收集进度
- 评估：实时质量和相关性
- 调整：增加资源或终止低效路径
- 优化：根据新信息调整研究方向
```

#### 路径终止条件
```markdown
- 信息饱和：无法获取更多有价值信息
- 质量下降：新增信息质量低于阈值
- 时间限制：达到预定研究时长
- 目标达成：覆盖所有关键维度
```

### 信息质量评估算法

#### 权威性评分（权重40%）
```markdown
- 作者/机构权威性（20%）
- 出版/发布渠道权威性（15%）
- 引用和影响力（5%）
```

#### 时效性评分（权重25%）
```markdown
- 信息发布时间（15%）
- 数据更新频率（10%）
- 相关性随时间变化（0%）
```

#### 相关性评分（权重20%）
```markdown
- 主题匹配度（10%）
- 问题针对性（5%）
- 信息完整性（5%）
```

#### 准确性评分（权重15%）
```markdown
- 信息一致性验证（8%）
- 数据来源可靠性（7%）
```

#### 深度评分（权重0%）
```markdown
- 分析深度影响质量分数但不作为独立维度
- 通过Refine操作专门提升深度
```

```markdown
## 活跃路径
- [路径1]: 活跃, 质分8.5, 发现12个
- [路径2]: 活跃, 质分7.2, 发现8个
- [路径3]: 活跃, 质分6.8, 发现6个

## 已完成路径
- [路径4]: 已完成, 质分9.1, 发现15个
- [路径5]: 已完成, 质分7.8, 发现10个

## 待处理路径
- [路径6]: 等待评估, 发现5个
- [路径7]: 需要深化, 发现3个

## 质量分布
- 9-10分: 15个发现
- 7-8分: 25个发现
- 5-6分: 12个发现
- <5分: 8个发现（已标记为低质量）
```

### 优化策略

#### 资源分配优化
```markdown
- 高质量路径（>8分）：分配70%资源
- 中等质量路径（6-8分）：分配25%资源
- 低质量路径（<6分）：分配5%资源，考虑终止
```

#### 时间分配优化
```markdown
- 初始探索：20%时间
- 深度研究：60%时间
- 结果整合：20%时间
```

#### 风险管理
```markdown
- 信息不足风险：预留30%时间用于补充搜索
- 偏见风险：确保多源交叉验证
- 时效性风险：优先关注最新信息
```

### 最佳实践

#### 路径选择
- 优先选择覆盖关键研究维度的路径
- 确保不同路径之间有合理的互补性
- 避免过度集中在单一视角

#### 质量控制
- 定期评估和调整质量阈值
- 对重要信息进行多重验证
- 及时清理低质量信息

#### 效率优化
- 根据信息质量动态调整资源分配
- 及时终止低效的研究路径
- 利用并行处理提高效率

#### 结果整合
- 确保不同路径的发现能够有效整合
- 识别和解决信息冲突
- 保持整体叙事的一致性
