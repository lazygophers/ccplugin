# Source Quality

## Rating

- A: Official docs, standards, peer-reviewed papers, authoritative reports
- B: Maintainer blogs, release notes, reputable engineering blogs
- C: Community posts, issues, forum discussions, personal case studies
- D: Old posts, marketing material, secondary summaries
- E: Unverifiable claims, rumors, outdated info

## Validation Rules

- Require at least one A/B source for key claims.
- If only C/D/E sources exist, mark as uncertain.
- Note conflicts explicitly and describe why they differ.
- Always include source date and context.
