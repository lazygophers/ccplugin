## 问题优化技能使用示例

### 示例1：学术研究优化

#### 输入
```
/refine-question 我想研究机器学习
```

#### 对话过程
```
系统：您想研究机器学习的哪个方面？
用户：我想研究深度学习在医疗诊断中的应用

系统：您希望研究的时间范围是什么？
用户：2018年到2023年

系统：目标受众是谁？
用户：医学研究人员和AI开发者

系统：需要什么样的输出格式？
用户：学术论文格式，30-40页

系统：信息来源有什么要求？
用户：主要来自学术论文和临床试验
```

#### 输出
```markdown
# 深度学习在医疗诊断中的应用研究

## 具体研究范围
### 核心研究领域
- 医学影像诊断（CT、MRI、X光）
- 病理图像分析
- 生理信号监测
- 临床决策支持系统

### 技术维度
- 深度学习架构（CNN、Transformer等）
- 数据预处理和增强
- 模型训练和优化
- 可解释性方法

### 时间范围
- 2018年1月 至 2023年12月
- 重点：2020-2023年的最新进展

### 地理范围
- 全球研究，重点关注：
  - 北美（美国、加拿大）
  - 欧洲（英国、德国、法国）
  - 中国

## 研究目标
### 主要研究问题
1. 深度学习在医疗诊断中的准确性和可靠性如何？
2. 不同深度学习架构在医疗诊断中的表现对比？
3. 医疗AI系统的可解释性和临床实用性如何？

### 次要研究问题
1. 医疗AI系统的监管和伦理问题？
2. 临床实施的技术挑战和解决方案？
3. 未来发展趋势和潜在突破点？

## 受众分析
### 目标受众
- 医学研究人员
- AI算法工程师
- 医疗机构技术负责人
- 政策制定者

### 需求分析
- 学术研究人员：需要理论基础和实证结果
- 技术工程师：需要技术实现细节
- 医疗管理者：需要临床应用指导
- 政策制定者：需要监管建议

## 输出要求
### 格式要求
- 30-40页学术论文
- 包含摘要、引言、文献综述、方法、结果、讨论、结论
- 学术参考文献格式（APA或IEEE）
- 图表和数据可视化

### 详细程度
- 理论深度：详细的技术原理和数学基础
- 实证分析：详细的实验设计和结果
- 案例研究：具体的应用案例分析
- 文献综述：全面的文献覆盖和分析

### 特殊要求
- 包含可解释性分析
- 提供临床实施指南
- 标注技术难点和解决方案

## 质量标准
### 来源要求
- 权威期刊论文（影响因子 > 3.0）
- 临床试验数据
- 行业白皮书和报告
- 高质量会议论文

### 验证要求
- 同行评审文献优先
- 多中心研究数据
- 可重复的实验结果
- 专家评审验证

### 时效性要求
- 最新文献：近2年
- 经典文献：近5年
- 预测数据：近1年
```

### 示例2：技术研究优化

#### 输入
```
/refine-question 我想研究云原生
```

#### 对话过程
```
系统：您想研究云原生的哪个具体方面？
用户：微服务架构和服务网格

系统：您的主要应用场景是什么？
用户：电商平台的高可用架构

系统：技术深度要求如何？
用户：需要深入的技术实现细节

系统：有什么特定的技术栈要求？
用户：Kubernetes、Istio、Go语言

系统：研究的时间范围？
用户：近1年的最新发展和最佳实践
```

#### 输出
```markdown
# 云原生微服务架构与服务网格技术研究

## 具体研究范围
### 技术架构
- 微服务设计模式
- 服务网格（Service Mesh）
- 容器编排（Kubernetes）
- API网关和流量管理

### 核心技术栈
- Kubernetes 1.20+
- Istio 1.8+
- Go 1.18+
- Prometheus + Grafana
- ELK Stack

### 应用场景
- 电商平台架构
- 高可用性和弹性设计
- 分布式事务处理
- 监控和可观测性

### 时间范围
- 2023年1月 至 2023年12月
- 重点：2023年最佳实践和最新趋势

## 研究目标
### 主要研究问题
1. 微服务架构在高并发电商平台中的最佳实践？
2. 服务网格在流量管理和故障恢复中的作用？
3. 云原生架构的性能优化和成本控制策略？

### 次要研究问题
1. 微服务治理和监控体系设计？
2. 服务网格的安全性和合规性？
3. 多云环境下的架构挑战和解决方案？

## 受众分析
### 目标受众
- 系统架构师
- DevOps工程师
- 技术团队负责人
- 技术决策者

### 需求分析
- 架构师：需要架构设计和决策依据
- 工程师：需要技术实现细节
- 管理者：需要ROI和风险评估
- 决策者：需要技术选型建议

## 输出要求
### 格式要求
- 技术白皮书格式
- 包含架构图、代码示例、配置文件
- 技术规范和最佳实践
- 实施指南和注意事项

### 详细程度
- 架构设计：详细的架构图和组件说明
- 代码实现：核心代码片段和实现细节
- 配置管理：完整配置文件和参数说明
- 性能测试：详细的性能数据和优化建议

### 特殊要求
- 包含实际案例研究
- 提供配置模板
- 标注技术难点和解决方案
- 性能基准和测试方法

## 质量标准
### 来源要求
- 官方文档和技术白皮书
- 开源项目最佳实践
- 行业技术博客和文章
- 技术会议演讲和论文

### 验证要求
- 生产环境实践经验
- 性能测试数据
- 社区验证的解决方案
- 专家评审建议

### 时效性要求
- 最新版本文档
- 近6个月的最佳实践
- 当前行业趋势
```

### 最佳实践总结

#### 优化成功案例特征
```markdown
1. **问题清晰化**
   - 从模糊到具体
   - 从宽泛到聚焦
   - 从主观到客观

2. **范围合理化**
   - 大小适中
   - 边界清晰
   - 可执行性强

3. **目标明确化**
   - 问题具体
   - 期望明确
   - 标准可量化

4. **受众针对性**
   - 了解受众背景
   - 满足受众需求
   - 符合受众期望
```

#### 常见错误和解决方案
```markdown
1. **过于宽泛**
   - 错误：研究人工智能
   - 解决：聚焦具体应用领域

2. **缺乏焦点**
   - 错误：研究所有技术
   - 解决：选择关键技术栈

3. **目标不明确**
   - 错误：了解市场情况
   - 解决：设定具体研究问题

4. **受众不清晰**
   - 错误：给所有人看
   - 解决：明确目标受众群体
```

### 使用建议

#### 何时使用问题优化
- 研究主题模糊时
- 研究范围过大时
- 研究目标不明确时
- 需要细化研究方向时

#### 如何获得最佳效果
- 详细回答每个问题
- 提供相关背景信息
- 明确具体需求
- 考虑实际约束条件

#### 后续步骤
- 使用优化后的提示进行深度研究
- 制定详细的研究计划
- 执行多智能体研究
- 验证和整合研究结果
