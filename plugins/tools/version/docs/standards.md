# 版本规范

Version 插件遵循的语义化版本规范。

## 版本号格式

采用 Semantic Versioning 标准，格式为 `X.Y.Z.W`：

```
X.Y.Z.W
│ │ │ │
│ │ │ └── Build: 构建版本
│ │ └──── Patch: 补丁版本
│ └────── Minor: 次版本
└──────── Major: 主版本
```

## 版本号含义

| 部分 | 名称 | 何时增加 |
|------|------|----------|
| **X** | Major | 不兼容的 API 变更或重大功能 |
| **Y** | Minor | 向后兼容的新功能 |
| **Z** | Patch | Bug 修复和性能优化 |
| **W** | Build | 完成任务和小改进 |

## 版本更新规则

### Major 版本

**何时增加**：

- 不兼容的 API 修改
- 架构重构
- 重大功能变更

**示例**：

```
1.x.x.x → 2.0.0.0
```

### Minor 版本

**何时增加**：

- 向后兼容的新功能
- 功能增强
- 新增 API

**示例**：

```
1.2.x.x → 1.3.0.0
```

### Patch 版本

**何时增加**：

- Bug 修复
- 性能优化
- 安全补丁

**示例**：

```
1.2.3.x → 1.2.4.0
```

### Build 版本

**何时增加**：

- 完成单个任务
- 代码小改进
- 文档更新

**示例**：

```
1.2.3.4 → 1.2.3.5
```

## 版本文件

### .version 文件

版本号存储在项目根目录的 `.version` 文件中：

```
1.2.3.4
```

### Git 集成

插件会检测 `.version` 文件的 Git 提交状态：

- 已提交：版本已保存
- 未提交：版本变更未保存
