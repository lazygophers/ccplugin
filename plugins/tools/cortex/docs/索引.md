# 索引

本文回答：cortex 文档总目录在哪里？读什么、按什么顺序读。
适用读者：第一次接触 cortex 的用户、想做贡献的开发者、做故障排查的运维。

## 入门路线

按顺序读完即可上手：

1. `快速上手.md` — 5 分钟从安装到第一次落档。
2. `安装与配置.md` — 详细安装、vault 路径解析、`config.json` schema、环境变量。
3. `知识库结构.md` — 4 个 preset 的目录布局, 共享根 `_meta` / `_templates` / `log` / `folds` 说明。
4. `Skills 详解.md` — 11 个 skill 一一展开 (用途 / 触发短语 / 示例 / 失败处理)。

## 进阶路线

深入理解 cortex 的工作机制：

5. `Hooks 机制.md` — `SessionStart` / `Stop` / `SubagentStop` / `PostCompact` 四个 hook 的协议与启发式落档判定。
6. `模板与美化.md` — 6 个模板与 13 类 callout, 何时回退 HTML grid。
7. `Lint 规则.md` — 17 条 vault lint 规则逐条解释 + `--fix` 行为。
8. `重构与归档.md` — `refactor` 4 子操作 (rename / merge / split / fold) 用法与 backup 路径。
9. `周期任务.md` — `install_cron.sh` 用法与 cron / launchd / GitHub Actions 三种 snippet。

## 维护与排错

10. `故障排查.md` — 常见 6+ 症状到原因再到修复的对照表。
11. `架构设计.md` — 数据流、单向依赖、hook 时序图、MCP 主 CLI 兜底。
12. `设计决策.md` — 关键 ADR (D1-D7 + research-driven §10 patches) 中文化解读。

## 贡献者文档

13. `贡献指南.md` — 添加新 skill / 新 lint 规则 / 新 preset 的步骤、测试约定、GLM 自检要求。

## 速查

| 想加新 skill | `贡献指南.md` |
