# 周期任务

本文回答：cortex 推荐的 3 个周期任务是什么、`install_cron.sh` 怎么用、在 macOS launchd / Linux cron / GitHub Actions 上分别怎么写。
适用读者：想让 cortex 在后台自动维护 vault 的运维。

## 推荐 3 个任务

来源 `prd.md §7 M6`。

| 任务              | 频率           | 命令                                                              | 说明                          |
| ----------------- | -------------- | ----------------------------------------------------------------- | ----------------------------- |
| lint --fix        | daily 01:00    | `bash ~/.claude/plugins/marketplaces/ccplugin-market/plugins/tools/cortex/scripts/cron/lint.sh` | autofix 循环修复 17 类规则 |
| dashboard refresh | daily 02:30    | `bash ~/.claude/plugins/marketplaces/ccplugin-market/plugins/tools/cortex/scripts/cron/dashboard.sh` | 重渲 index.md / hot.md / canvas |
| log fold          | weekly Sun 02:00 | `bash ~/.claude/plugins/marketplaces/ccplugin-market/plugins/tools/cortex/scripts/cron/fold.sh` | log 滚动归档到 folds/         |

三个任务都通过 wrapper bash 调 `claude -p "/cortex:<name>"` 触发, 需先装 Claude Code CLI (官方安装渠道)。

## install_cron.sh 用法

cortex 不自动写用户 crontab / launchd, **仅打印 snippet 供用户复制**。

```bash
bash ~/.claude/plugins/marketplaces/ccplugin-market/plugins/tools/cortex/scripts/install_cron.sh cron      # 默认: Linux/macOS cron
bash ~/.claude/plugins/marketplaces/ccplugin-market/plugins/tools/cortex/scripts/install_cron.sh launchd   # macOS launchd plist
bash ~/.claude/plugins/marketplaces/ccplugin-market/plugins/tools/cortex/scripts/install_cron.sh gha       # GitHub Actions workflow
```

输出直接复制到对应位置即可。

## Linux / macOS cron

`bash install_cron.sh cron` 输出：

```bash
# ===== cortex cron snippet (Linux/macOS cron) =====
# 复制以下行到 'crontab -e':

# daily lint with autofix at 01:00
0 1 * * * /usr/bin/env python3 "~/.claude/plugins/marketplaces/ccplugin-market/plugins/tools/cortex/scripts/lint/run.py" --vault "${VAULT}" --fix > "$HOME/.cache/cortex/lint-$(date +%Y%m%d).json" 2>&1

# weekly log fold at Sunday 02:00
0 2 * * 0 /usr/bin/env python3 "~/.claude/plugins/marketplaces/ccplugin-market/plugins/tools/cortex/scripts/refactor/fold.py" --vault "${VAULT}" --days 7 --apply >> "$HOME/.cache/cortex/fold.log" 2>&1

# daily dashboard refresh at 02:30 (claude CLI required)
30 2 * * 0 claude -p "/cortex:dashboard refresh all" >> "$HOME/.cache/cortex/dashboard.log" 2>&1
```

注意：

- `~/.claude/plugins/marketplaces/ccplugin-market/plugins/tools/cortex` / `${VAULT}` 在脚本生成时已被替换成实际路径。
- 用 `crontab -e` 粘贴, `crontab -l` 验证。
- macOS 默认 cron 受 SIP 限制, 建议用 launchd。

## macOS launchd

`bash install_cron.sh launchd` 输出 plist：

```xml
<?xml version="1.0" encoding="UTF-8"?>
<plist version="1.0">
<dict>
  <key>Label</key>
  <string>dev.lazygophers.cortex.daily-lint</string>
  <key>ProgramArguments</key>
  <array>
    <string>/usr/bin/env</string>
    <string>python3</string>
    <string>~/.claude/plugins/marketplaces/ccplugin-market/plugins/tools/cortex/scripts/lint/run.py</string>
    <string>--vault</string>
    <string>${VAULT}</string>
    <string>--fix</string>
  </array>
  <key>StartCalendarInterval</key>
  <dict>
    <key>Hour</key><integer>1</integer>
    <key>Minute</key><integer>0</integer>
  </dict>
  <key>StandardOutPath</key>
  <string>${HOME}/.cache/cortex/lint.log</string>
  <key>StandardErrorPath</key>
  <string>${HOME}/.cache/cortex/lint.err</string>
</dict>
</plist>
```

操作：

```bash
bash install_cron.sh launchd > ~/Library/LaunchAgents/dev.lazygophers.cortex.daily-lint.plist
launchctl load ~/Library/LaunchAgents/dev.lazygophers.cortex.daily-lint.plist
launchctl list | grep cortex   # 验证
```

派生 fold / dashboard plist：

- `Label` → `dev.lazygophers.cortex.weekly-fold` / `daily-dashboard`
- `StartCalendarInterval` 加 `Weekday=0` (周日), 改 `Hour` / `Minute`
- `ProgramArguments` 替换为 fold.py / claude CLI

## GitHub Actions

`bash install_cron.sh gha` 输出 workflow：

```yaml
name: cortex-cron

on:
  schedule:
    - cron: "0 1 * * *" # daily lint
    - cron: "0 2 * * 0" # weekly fold
  workflow_dispatch:

jobs:
  lint:
    if: github.event.schedule == '0 1 * * *'
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with: { python-version: "3.11" }
      - name: cortex lint
        run: python3 lint/run.py --vault . --fix > lint-report.json
      - uses: actions/upload-artifact@v4
        with: { name: lint-report, path: lint-report.json }

  fold:
    if: github.event.schedule == '0 2 * * 0'
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with: { python-version: "3.11" }
      - name: cortex fold
        run: python3 refactor/fold.py --vault . --days 7 --apply
      - name: commit folds
        run: |
          git config user.name "cortex-bot"
          git config user.email "bot@example.com"
          git add folds/ log/
          git commit -m "chore(cortex): weekly fold" || true
          git push
```

保存为 `.github/workflows/cortex:cron.yml`, 要求 vault 是 GitHub repo, secrets 配置 `OBSIDIAN_API_KEY` 等 (若 lint 需走 REST)。

## 选哪个

| 场景                           | 推荐                             |
| ------------------------------ | -------------------------------- |
| macOS 个人电脑, vault 在本地   | launchd                          |
| Linux 服务器 / WSL2            | cron                             |
| vault 在 GitHub repo, 想自动化 | GitHub Actions                   |
| 容器化环境 (Docker / k8s)      | 用容器内 cron 或外部调度器调容器 |

## 验证

```bash
# 手动跑一次确认
python3 ~/.claude/plugins/marketplaces/ccplugin-market/plugins/tools/cortex/scripts/lint/run.py --vault $(jq -r .vault ~/.cortex/config.json) --fix
python3 ~/.claude/plugins/marketplaces/ccplugin-market/plugins/tools/cortex/scripts/refactor/fold.py --vault $(jq -r .vault ~/.cortex/config.json) --days 7
claude -p "/cortex:dashboard refresh all"
```

确认无报错再设进 cron / launchd / GHA。

## 日志位置

| 任务      | 日志                                 |
| --------- | ------------------------------------ |
| lint      | `~/.cache/cortex/lint-YYYYMMDD.json` |
| fold      | `~/.cache/cortex/fold.log`           |
| dashboard | `~/.cache/cortex/dashboard.log`      |

GHA 场景通过 `actions/upload-artifact@v4` 把报告上传为 artifact。
