# 架构设计

本文回答：cortex 的数据流、模块依赖、hook 与 skill 的时序、MCP 主 CLI 兜底的结构、Smart Connections 三级回退。
适用读者：想理解 cortex 内部、做贡献或迁移到自己工具链的开发者。

## 数据流总览

```
              ┌────────────────────────────────┐
              │   Claude Code 会话 (用户视角)    │
              └──────────────┬─────────────────┘
                             │
        ┌────────────────────┼─────────────────────┐
        ↓                    ↓                     ↓
   SessionStart          用户输入触发           Stop / SubagentStop / PostCompact
        │                    │                     │
   ┌────▼─────┐         ┌────▼─────┐         ┌─────▼──────┐
   │ AGENT.md │         │  Skill   │         │  save_     │
   │ 注入正文 │         │ (11 个)  │         │  session   │
   └────┬─────┘         └────┬─────┘         └─────┬──────┘
        │                    │                     │
        ↓                    ↓                     ↓
   ┌─────────────────────────────────────────────────┐
   │                vault 操作层                      │
   │  ┌──────────┐  ┌──────────┐  ┌─────────────┐    │
   │  │ MCP 主路 │→ │ obsidian │→ │ ripgrep兜底 │    │
   │  │ (95%)    │  │ CLI 备用 │  │ (degraded)  │    │
   │  └──────────┘  └──────────┘  └─────────────┘    │
   └────────────────────┬────────────────────────────┘
                        ↓
                 Obsidian vault (用户笔记)
```

## 模块依赖 (单向)

```
plugin.json
   │
   ├─→ hooks/        (session_start.sh / stop.sh / post_compact.sh)
   │     └─→ hooks/_lib/  (resolve_vault.sh / save_session.py)
   │
   ├─→ skills/       (11 个 SKILL.md)
   │     └─→ allowed-tools 调 mcp__obsidian__* / Bash / Read / Write
   │
   ├─→ templates/    (6 个 .md)
   │
   ├─→ presets/      (4 个 _structure.json + seed/)
   │
   ├─→ lint/         (rules.json + run.py)
   │
   ├─→ refactor/     (rename.py / merge.py / split.py / fold.py)
   │
   └─→ scripts/      (install_cron.sh)
```

约束：

- 不依赖 ccplugin 主仓的 `lib/`, 自包含。
- 仅用 Python stdlib + bash, 无 pip 依赖。
- skill 内部不直接调 hooks 模块, 反之亦然。

## Hook 时序图

### SessionStart

```
CC 启动会话
  │
  ↓
plugin.json hooks.SessionStart.matcher 命中 (startup|resume|clear)
  │
  ↓
fork session_start.sh (timeout=1500ms)
  │
  ├─ resolve_vault → 5 段优先级
  │     ├─ 命中 → 继续
  │     └─ 未命中 → exit 0 (沉默)
  │
  ├─ python3 内嵌 → 读 AGENT.md + hot.md (≤5KB) + 计 index.md 行数
  │
  └─ stdout: v2 wrapped JSON {hookSpecificOutput.additionalContext}
        │
        ↓
   CC 注入到 system prompt
```

### Stop / SubagentStop

```
会话结束 / 子 agent 结束
  │
  ↓
fork stop.sh (async=true, timeout=5000ms)
  │
  ├─ 解析 stdin v2 JSON: transcript_path, stop_hook_active, hook_event_name
  │
  ├─ stop_hook_active=true? → exit 0 (防循环)
  │
  ├─ resolve_vault → 未命中 exit 0
  │
  ├─ transcript 文件存在? → 否 exit 0
  │
  ├─ python3 save_session.py --vault X --transcript Y --reason {stop|subagent-stop}
  │     │
  │     ├─ 启发式: min_lines / min_chars / skip_if_only_questions
  │     │     └─ 未达阈值 → rc=2 → 静默
  │     │
  │     ├─ 选目录: 项目特定 (30_domains) vs 通用 (10_concepts)
  │     ├─ 套模板: concept / entity / source
  │     ├─ 注 block-id: ^cortex-<sha8>
  │     ├─ 同步 index.md / hot.md
  │     └─ stdout: 落档绝对路径
  │
  └─ rc=0 → emit v2 wrapped JSON ("📝 cortex 已落档 ...")
```

### PostCompact

差异：跳过启发式 (`--force`), 强制落档为 `wiki/log/YYYY-MM/DD-HHMM-compact.md`。

## MCP 主 CLI 兜底 (3 级)

| 操作 | L1: MCP | L2: obsidian CLI | L3: 直接文件 |
|------|---------|------------------|--------------|
| 读文件 | `mcp__obsidian__obsidian_get_file_contents` | (不适用) | `Read` 工具 |
| 写新文件 | `mcp__obsidian__obsidian_append_content` | (不适用) | `Write` 工具 |
| 改既有文件 | `mcp__obsidian__obsidian_patch_content` | (不适用) | `Edit` 工具 |
| 列目录 | `mcp__obsidian__obsidian_list_files_in_dir` | (不适用) | `Glob` |
| 搜内容 | `mcp__obsidian__obsidian_simple_search` | (不适用) | `ripgrep` |
| 复杂搜索 | `mcp__obsidian__obsidian_complex_search` | (不适用) | `ripgrep` 多 pattern |
| canvas | (不支持) | `obsidian` CLI 生成 `.canvas` | 静态 JSON 模板 |
| Bases | (不支持) | `obsidian` CLI 生成 `.base` | Dataview 兜底 |

## Smart Connections 三级回退 (cortex-search)

cortex-search 五级回退中, Smart Connections 处于第 3 级：

```
1. hot.md (热点缓存, ≤200 行)
     ↓ 未命中
2. index.md (顶层索引, 解析 wikilink)
     ↓ 未命中
3. Smart Connections REST (GET /search?q=...&top_k=10, 端口 27124)
     ↓ 不可用 (插件未启 / 索引未建)
4. MCP obsidian_simple_search (全文 grep)
     ↓ 未命中
5. ripgrep (vault 目录下原始 grep)
     ↓ 还是没结果
   返回 "vault 无相关条目"
```

每级独立判失败, 失败即落下一级, 不互相阻塞。

## 设计原则速查

| 原则 | 体现 |
|------|------|
| 不依赖 lib/ | 纯 bash + stdlib |
| MCP 主 CLI 兜底 | mcp__obsidian__* 覆盖 95% CRUD; canvas/bases 才回退 CLI |
| callout 替代 HTML grid | dashboard.md 仅在 KPI 卡片用 grid |
| Hook v2 wrapped JSON | hookSpecificOutput.{hookEventName,additionalContext} |
| 不写 noop hook | 教训自 commit 07e713d4 |
| 显式 vs 自动 skill 二元 | disable-model-invocation 标记副作用大的 |
| 默认 dry-run | lint / refactor / fold 都需 --apply |

## 失败语义

cortex 所有 hook 与 skill 遵循：

- **永不阻断会话** — hook exit 0, 失败仅写 `~/.cache/cortex/*.log`。
- **degraded 优先** — MCP 不可用时回退到本地工具, 而非报错退出。
- **dry-run 优先** — 副作用大的操作 (refactor / fold / lint --fix) 默认 dry-run。
- **backup 强制** — 任何 `--apply` 操作都先 backup 到 `.cortex-backup/`。
