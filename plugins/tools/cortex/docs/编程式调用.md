# 编程式调用 + cron

cortex v2 提供 4 个 bash wrapper (`scripts/cron/*.sh`) 通过 `claude --bare -p` 编程式调用本插件 skill, 实现 daily lint / weekly fold / weekly dashboard 等周期维护。

## 1. claude CLI 关键 flag

| Flag | 作用 | cron 必用 |
|------|------|-----------|
| `-p, --print` | non-interactive 单次执行 | ✓ |
| `--bare` | 关 SessionStart hook + 全局注入 (caveman / cortex 自身), 避免污染 | ✓ |
| `--no-session-persistence` | 不写 transcript 到 `~/.claude/projects/`, 防膨胀 | ✓ |
| `--settings <path>` | 显式 settings (推荐 `~/.claude/settings.glm-4.5-flash.json` 廉价 model) | ✓ |
| `--output-format stream-json` | NDJSON 流, 边跑边过滤 | ✓ |
| `--verbose` | 提升事件密度, 失败时回放友好 | ✓ |
| `--allowed-tools "Bash Read Glob Grep"` | 收紧权限, 只读类 cron 用 | ✓ |
| `--strict-mcp-config --mcp-config <file>` | 限定 MCP server (e.g. /dev/null 全清) | 推荐 |
| `--max-budget-usd 0.30` | 美元上限, 防 LLM 跑飞 | ✓ |
| `--session-id $(uuidgen)` | 显式 session id, 便于追溯 | 可选 |
| `--exclude-dynamic-system-prompt-sections` | 移除 cwd/git/env, 提高 prompt cache hit | 推荐 |

完整 flag 表见 `research/01-claude-code-programmatic.md §A`。

## 2. 退出码不可信

`claude` 退出码 `0` ≠ "任务成功"。例:
- `--disallowed-tools Bash` 让 LLM 弃用工具仍 exit 0
- 需必查 `result.is_error` 字段才能确定

cortex 的 `run.sh` 已封装:

```bash
is_error=$(jq -r '.is_error' "$RESULT_FILE")
[[ "$is_error" == "true" ]] && exit 1
```

## 3. 三个标准 cron 任务

### 3.1 daily lint (01:00)

```bash
bash ${CLAUDE_PLUGIN_ROOT}/scripts/cron/lint.sh \
  --vault /path/to/vault \
  --lang zh-CN
```

只读, 输出 lint summary 到 `~/.cache/cortex/cron/lint-<YYYY-MM-DD>.json`。

### 3.2 weekly fold (Sun 02:00)

```bash
bash ${CLAUDE_PLUGIN_ROOT}/scripts/cron/fold.sh --vault /path/to/vault
```

允许写, 滚动 log/ → folds/。

### 3.3 weekly dashboard (Sun 02:30)

```bash
bash ${CLAUDE_PLUGIN_ROOT}/scripts/cron/dashboard.sh --vault /path/to/vault
```

刷新既有 dashboard 页 (Bases 或 Dataview 自动选择)。

## 4. 通用 wrapper run.sh

```bash
bash ${CLAUDE_PLUGIN_ROOT}/scripts/cron/run.sh <job-name> -- <prompt> [extra claude flags]
```

提供:
- 锁文件 (`/tmp/cortex-<job>.lock`, flock 或 PID lockfile)
- 超时 (`CORTEX_TIMEOUT=300` 默认)
- 日志 (`~/.cache/cortex/cron/<job>-<YYYY-MM-DD>.log` 与 `.err` 与 `.json`), 7 天轮转
- 退出码: 0 成功 / 1 一般失败 / 2 锁忙 / 3 超时 / 4 配置错

环境变量:
- `CORTEX_VAULT` (必需)
- `CORTEX_LANG` (可选)
- `CORTEX_SETTINGS` (默认 `~/.claude/settings.glm-4.5-flash.json`)
- `CORTEX_TIMEOUT` (默认 300, fold/dashboard 设 600)
- `CORTEX_DRY_RUN=1` 仅打印命令不执行

## 4.5 cron 路径解析 (marketplace vs 源码)

cron daemon 与登录 shell 环境隔离, **不会**继承 `$CLAUDE_PLUGIN_ROOT`。snippet 必须把 `PLUGIN_ROOT` 解析为绝对路径字符串后再硬编码进 crontab/plist。

`scripts/install_cron.sh` 解析顺序 (高到低):

1. `--plugin-root <path>` 命令行覆盖
2. `$CORTEX_INSTALL_PATH` 环境变量
3. **marketplace 主路径** (默认, 见下)
4. **marketplace XDG 兜底路径**
5. `$CLAUDE_PLUGIN_ROOT` (CC 主线调用 install_cron.sh 时)
6. fallback: 脚本所在源码路径 + stderr 警告 "未检测到 marketplace 安装"

跨平台 marketplace 默认安装位置:

| 平台 | 主路径 | XDG 兜底 |
|------|--------|----------|
| macOS / Linux | `~/.claude/plugins/marketplaces/ccplugin-market/plugins/tools/cortex/` | `~/.config/claude/plugins/marketplaces/ccplugin-market/plugins/tools/cortex/` |
| Windows | `%USERPROFILE%\.claude\plugins\marketplaces\ccplugin-market\plugins\tools\cortex\` | — |

为何不能用源码路径:

- 源码路径 (`~/persons/.../ccplugin/plugins/tools/cortex`) 仅本机开发时存在
- 用户通过 marketplace 安装时**没有**源码目录, cron 会因路径无效失败
- cron daemon 跑 wrapper (`scripts/cron/*.sh`) 时需要稳定的 marketplace 绝对路径

调用示例:

```bash
# 默认 (走 marketplace)
bash scripts/install_cron.sh cron

# 显式覆盖 (e.g. 自定义部署)
bash scripts/install_cron.sh --plugin-root /opt/cortex cron

# 通过环境变量
CORTEX_INSTALL_PATH=/opt/cortex bash scripts/install_cron.sh cron

# 查看完整优先级说明
bash scripts/install_cron.sh --help
```

GHA snippet 例外: CI 环境路径不固定, 使用 `${GITHUB_WORKSPACE}/plugins/tools/cortex/...` 占位, 由 Actions runner 注入。用户若 checkout 到子目录需自行调整。

## 5. cron 注册 (cortex-cron skill)

cortex-install 末尾 `[6/6]` 询问注册; 也可后续主动:

```text
"register cron"
→ cortex-cron skill 检测平台 (launchd/cron/gha) → dry-run 给用户看 → 确认后落盘
```

平台映射:
- macOS → `~/Library/LaunchAgents/dev.lazygophers.cortex.<job>.plist`
- Linux → `crontab -l` 追加行 (备份原 crontab) 或 systemd user timer
- CI (`$CI=true`) → 输出 `.github/workflows/cortex-<job>.yml` 让用户复制, 不真写

卸载:

```text
"uninstall cortex cron"
→ cortex-cron uninstall all
```

## 6. 常见故障

| 症状 | 原因 | 处理 |
|------|------|------|
| exit 0 但 vault 没变化 | LLM 跑了但工具被拒 | 查 `~/.cache/cortex/cron/<job>.err`; 调 `--allowed-tools` |
| exit 2 (锁忙) | 上次 cron 没退出 | `rm /tmp/cortex-<job>.lock` 后排查 |
| exit 3 (超时) | LLM thinking 长 | 增大 `CORTEX_TIMEOUT` 或缩小任务 |
| exit 1 + is_error=true | API 错 / max-budget 超 | 看 `.json` 里的 `result` 字段 |
| 没有 `.json` 输出 | jq 解析失败 | 看 `.err` 里的 stderr |
| keychain 弹窗 (macOS) | 没用 `--bare` 或 settings 缺 `apiKeyHelper` | 强制 `--bare` + settings 配 `ANTHROPIC_API_KEY` |

## 7. 安全

- cron 任务**永不**用 `--dangerously-skip-permissions`
- 写型任务白名单严格 (`Read Write Edit Glob`, 不含 `Bash` 任意命令)
- 不动 user `~/.claude/settings.json`; 仅写 LaunchAgents/crontab/`.github/workflows`
- 卸载脚本可一键回退
