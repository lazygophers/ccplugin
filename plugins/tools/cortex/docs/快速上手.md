# 快速上手

本文回答：怎样用 5 分钟从零开始, 跑通 cortex 的安装、初始化、首次搜索、首次落档。
适用读者：第一次接触 cortex 的 Claude Code 用户。

## 前置条件

- macOS / Linux (Windows 用 WSL2)
- Python 3.11+ (检查：`python3 --version`)
- Bash 4+
- 已安装 Claude Code CLI 与 ccplugin marketplace
- 一个 Obsidian vault (没有也没关系, cortex-install 会创建)

## 第 1 步：安装插件

通过 ccplugin marketplace：

```bash
# 在 Claude Code 会话中
/plugin install cortex
```

或手动 clone 后从 `plugins/tools/cortex/` 软链到本地 marketplace。

## 第 2 步：声明 vault 位置 (可选)

cortex 按 5 段优先级解析 vault, 最简单是设环境变量：

```bash
export OBSIDIAN_VAULT="$HOME/persons/knowledge/obsidian"
```

未设置时 cortex 自动尝试 `~/persons/knowledge/obsidian`, 再扫 `~/Documents` / `~/Library` 找唯一 `.obsidian/` 目录。详见 `安装与配置.md`。

## 第 3 步：初始化 vault

在 Claude Code 会话中, 用自然语言触发 `cortex-install`：

```text
"安装 cortex"
"init vault"
```

cortex 会：

1. 检测 vault 路径是否命中。
2. 创建共享根 (`_meta/` `_templates/` `index.md` `hot.md`) + 顶层 (`知识库/` `记忆/` `仪表盘/` `归档/`)。
3. 创建知识库 4 子目录 (`项目/` `领域/` `日记/日/` `收件箱/`)。
4. 写入 seed 文件 (各目录 `_index.md`)。
5. 不覆盖既有文件。

## 第 4 步：诊断

`cortex-doctor` 是显式触发 (`disable-model-invocation: true`), 必须明确请求：

```text
"诊断 cortex"
"cortex doctor"
```

会跑 13 项体检 (vault / MCP / CLI / REST / SC / rg / lint / backlink ...) 并产报告。失败项给修复建议。

## 第 5 步：首次搜索

cortex-search 自动触发, 四级回退 (P9 重排, MCP first)：`mcp__obsidian__obsidian_simple_search` → `mcp__obsidian__obsidian_complex_search` → `bash ~/.cortex/scripts/search.sh` → ripgrep。

```text
"搜知识库 auth middleware"
"obsidian 里有没有关于事件驱动的笔记"
```

返回引用 (file:line + `obsidian://` URI), 仅查不写。

## 第 6 步：首次落档

会话结束时自动触发 (Stop hook 启发式判定), 或显式：

```text
"归档"
"save this to 知识库"
```

cortex-save 会：

- 启发式选目录 (项目特定 → `知识库/项目/<host>/<org>/<repo>/`, local 项目 → `知识库/项目/local/<basename>/`; 通用 → `知识库/领域/`)。
- 套模板 (concept / entity / source ...)。
- 注入 block-id (`^cortex-<sha8>`)。
- 同步 `index.md` 与 `hot.md`。
- 重名不覆盖。

## 验证：跑一次完整流程

```text
你：搜知识库看看有没有关于"事件驱动架构"的笔记
Claude：(触发 cortex-search) 找到 3 条 ...

你：(讨论一阵)
你：把刚才的结论归档到 cortex
Claude：(触发 cortex-save) 已落档 知识库/领域/事件驱动架构.md ...
```

## 知识库按需检索约定 (hook 每轮注入)

cortex 在 `user_prompt_submit` hook 每轮注入软约定: **AI 执行过程中需要外部信息时, 知识库是第一信息源** (非每轮强制先搜)。

| 触发场景 (需要查时才走) | 无需查的场景 |
|---|---|
| 不熟悉的概念/API/选型 | 改本地代码逻辑 |
| 历史决策/踩坑 | 读项目文件 |
| 项目专有约定 | 执行明确指令 |
| 用户偏好/记忆 | 写显而易见的代码 |
| 跨会话上下文 | 纯对话 |

**需要查时优先级**:

| 级 | 工具 | 用途 |
|---|---|---|
| 1 | `bash ~/.cortex/scripts/search.sh --query "<词>"` | 知识库 首选, 6 层并行 (Omnisearch / Obsidian REST / hot / index / SC / rg) + 拆词回退 |
| 2 | `mcp__obsidian__obsidian_simple_search` | 知识库 补充, Obsidian 内置索引 |
| 3 | `mcp__obsidian__obsidian_complex_search` | JsonLogic 按 tag/path/frontmatter 过滤 |
| 4 | Read / Grep / Glob | 本地代码 (知识库 无命中且问题是项目内的) |
| 5 | WebSearch / WebFetch / context7 / octocode | 外部 (知识库 + 本地都无命中) |

**禁忌**:

- 涉及历史决策 / 项目约定 / 用户偏好时直接 WebSearch / 训练记忆答 — 应先 知识库
- 用 qmd MCP 替代 obsidian MCP (qmd 索引不全)
- 绕过 search.sh 用 Bash rg / Grep 搜 vault 内容 (rg 是 search.sh 第 6 层)

### 召回率提升 — aliases / keywords

`ingest_remote` / `save` 落档时自动写入 (启发式抽取):

```yaml
aliases: [中文同义, 英文缩写, 全称]   # ≥ 3, 中英对 + 缩写↔全称
keywords: [文件名, 函数名, 术语]      # ≥ 5, 含 path stem / repo meta / 代码标识符 / heading
```

CLI override:

```bash
bash ~/.cortex/scripts/save.sh ... --aliases=认证,auth,RBAC --keywords=jwt,oauth,middleware
```

## 远程 ingest (github / website)

```bash
# github/gitlab repo 整库 ingest
bash ~/.cortex/scripts/ingest_remote.sh https://github.com/foo/bar
# → 落 知识库/项目/github.com/foo/bar/

# website 整站 ingest (sitemap.xml 自动发现, depth ≤ 3)
bash ~/.cortex/scripts/ingest_remote.sh https://example.com --depth=3
# → 落 知识库/项目/example.com/_site/<slug>/

# dry-run 仅预览路径不写盘
bash ~/.cortex/scripts/ingest_remote.sh <url> --dry-run
```

## 批量增量更新

```bash
# 全扫所有项目, 仅刷变动 (git pull diff / website hash 对比)
bash ~/.cortex/scripts/refresh_projects.sh

# 限定单项目
bash ~/.cortex/scripts/refresh_projects.sh --scope=github.com/foo/bar

# dry-run
bash ~/.cortex/scripts/refresh_projects.sh --dry-run
```

每周一 03:00 自动跑 (install_cron.sh 注册的 weekly cron)。

## 评分字段 (强制 frontmatter)

知识库 `.md` 必含 4 字段 (0.0-10.0 浮点 + 1 enum):

- `score` — 内容质量 (覆盖度 + 深度 + 准确性)
- `confidence` — AI 自信度
- `source_credibility` — 源可信度 (host 白名单查表, 见 `skills/cortex-ingest/references/extract.md §3.1`)
- `maturity` — `draft | review | stable | deprecated`

记忆 `.md` (L0-L4) 必含 2 字段:

- `importance` — 重要程度
- `confidence` — 可信度

### AI 自评

`ingest_remote` / `save` 落档时自动按启发式填值。可用 CLI flag 覆盖:

```bash
bash ~/.cortex/scripts/save.sh ... --score=8.5 --confidence=9.0 --source-credibility=7.5 --maturity=stable
```

### digest 双路评分调整

每天 03:00 cron 跑 `cortex-digest evolution` 时:

- 使用信号 (sessions 召回 + vault 反向 wikilink) → `importance` ↑ (log10 自然衰减)
- 反馈信号 (用户 "不对/错了" → `confidence` -= 1.0; "对的/正确" → += 0.5)

### refresh_projects maturity 重评

每周一 03:00 跑 `refresh_projects` 时, 按内容 hash 变化重评 `maturity` (stable→review→draft 链); `score` / `confidence` / `source_credibility` 不动。

### 一次性迁移 (老 vault)

旧 vault `score: 1-5 整数` / `patterns.md confidence: 0-1` 需要一次性升级:

```bash
bash ~/.cortex/scripts/migrate.sh --to=v2 --dry-run   # 预演 (推荐先跑)
bash ~/.cortex/scripts/migrate.sh --to=v2             # 实落 (自动 tar.gz backup)
```

backup 落 `/tmp/cortex-migration-backup-<YYYYMMDD-HHMMSS>.tar.gz`。回滚:

```bash
tar xzf /tmp/cortex-migration-backup-<TS>.tar.gz -C <vault 父目录>
```

### 一次性 v3 migration — 老 .md 补 aliases / keywords

老 vault 缺 aliases/keywords frontmatter (PR3 后新落档自动写, 但老文件需批量补):

```bash
bash ~/.cortex/scripts/migrate.sh --to=v3 --dry-run   # 预演
bash ~/.cortex/scripts/migrate.sh --to=v3             # 实落 (自动 tar.gz backup)
```

启发式抽取 (不调 AI): 中英对 + 缩写↔全称 / path stem / 代码标识符 / heading 等。
backup 落 `/tmp/cortex-migration-backup-v3-<TS>.tar.gz`。

旧文件首次 `refresh_projects` 或 `ingest_remote` 重 ingest 也会自动补, migration v3 是批量入口。
