# 快速上手

本文回答：怎样用 5 分钟从零开始, 跑通 cortex 的安装、初始化、首次搜索、首次落档。
适用读者：第一次接触 cortex 的 Claude Code 用户。

## 前置条件

- macOS / Linux (Windows 用 WSL2)
- Python 3.11+ (检查：`python3 --version`)
- Bash 4+
- 已安装 Claude Code CLI 与 ccplugin marketplace
- 一个 Obsidian vault (没有也没关系, cortex-install 会创建)

## 第 1 步：安装插件

通过 ccplugin marketplace：

```bash
# 在 Claude Code 会话中
/plugin install cortex
```

或手动 clone 后从 `plugins/tools/cortex/` 软链到本地 marketplace。

## 第 2 步：声明 vault 位置 (可选)

cortex 按 5 段优先级解析 vault, 最简单是设环境变量：

```bash
export OBSIDIAN_VAULT="$HOME/persons/knowledge/obsidian"
```

未设置时 cortex 自动尝试 `~/persons/knowledge/obsidian`, 再扫 `~/Documents` / `~/Library` 找唯一 `.obsidian/` 目录。详见 `安装与配置.md`。

## 第 3 步：初始化 vault

在 Claude Code 会话中, 用自然语言触发 `cortex-install`：

```text
"安装 cortex"
"init vault"
```

cortex 会：

1. 检测 vault 路径是否命中。
2. 创建共享根 (`_meta/` `_templates/` `index.md` `hot.md`) + 顶层 (`知识库/` `记忆/` `仪表盘/` `归档/`)。
3. 创建知识库 4 子目录 (`项目/` `领域/` `日记/日/` `收件箱/`)。
4. 写入 seed 文件 (各目录 `_index.md`)。
5. 不覆盖既有文件。

## 第 4 步：诊断

`cortex-doctor` 是显式触发 (`disable-model-invocation: true`), 必须明确请求：

```text
"诊断 cortex"
"cortex doctor"
```

会跑 13 项体检 (vault / MCP / CLI / REST / SC / rg / lint / backlink ...) 并产报告。失败项给修复建议。

## 第 5 步：首次搜索

cortex-search 自动触发, 五级回退：hot → index → Smart Connections → MCP simple_search → ripgrep。

```text
"搜知识库 auth middleware"
"obsidian 里有没有关于事件驱动的笔记"
```

返回引用 (file:line + `obsidian://` URI), 仅查不写。

## 第 6 步：首次落档

会话结束时自动触发 (Stop hook 启发式判定), 或显式：

```text
"归档"
"save this to 知识库"
```

cortex-save 会：

- 启发式选目录 (项目特定 → `知识库/项目/<host>/<org>/<repo>/`, local 项目 → `知识库/项目/local/<basename>/`; 通用 → `知识库/领域/`)。
- 套模板 (concept / entity / source ...)。
- 注入 block-id (`^cortex-<sha8>`)。
- 同步 `index.md` 与 `hot.md`。
- 重名不覆盖。

## 验证：跑一次完整流程

```text
你：搜知识库看看有没有关于"事件驱动架构"的笔记
Claude：(触发 cortex-search) 找到 3 条 ...

你：(讨论一阵)
你：把刚才的结论归档到 cortex
Claude：(触发 cortex-save) 已落档 知识库/领域/事件驱动架构.md ...
```

## 远程 ingest (github / website)

```bash
# github/gitlab repo 整库 ingest
bash ~/.cortex/scripts/ingest_remote.sh https://github.com/foo/bar
# → 落 知识库/项目/github.com/foo/bar/

# website 整站 ingest (sitemap.xml 自动发现, depth ≤ 3)
bash ~/.cortex/scripts/ingest_remote.sh https://example.com --depth=3
# → 落 知识库/项目/example.com/_site/<slug>/

# dry-run 仅预览路径不写盘
bash ~/.cortex/scripts/ingest_remote.sh <url> --dry-run
```

## 批量增量更新

```bash
# 全扫所有项目, 仅刷变动 (git pull diff / website hash 对比)
bash ~/.cortex/scripts/refresh_projects.sh

# 限定单项目
bash ~/.cortex/scripts/refresh_projects.sh --scope=github.com/foo/bar

# dry-run
bash ~/.cortex/scripts/refresh_projects.sh --dry-run
```

每周一 03:00 自动跑 (install_cron.sh 注册的 weekly cron)。
