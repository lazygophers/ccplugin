# 贡献指南

本文回答：怎么给 cortex 加新 skill / 新 lint 规则 / 新 preset, 测试约定是什么, GLM 自检怎么跑。
适用读者：想给 cortex 做贡献的开发者、想 fork 改造 cortex 的进阶用户。

## 仓库结构

```
plugins/tools/cortex/
├── .claude-plugin/plugin.json    # 插件清单 (hooks 注册)
├── README.md                     # 项目速览
├── AGENT.md                      # SessionStart 注入正文模板
├── docs/                         # 中文文档 (本目录)
├── hooks/                        # 4 个 hook 入口 + _lib/
├── skills/                       # 11 个 SKILL.md
├── templates/                    # 6 个模板
├── presets/                      # 4 个 preset (LYT / Zettel / PARA / blank)
├── lint/                         # rules.json + run.py
├── refactor/                     # rename.py / merge.py / split.py / fold.py
├── scripts/                      # install_cron.sh
└── styles/                       # CSS snippets (Obsidian)
```

## 工作流

```bash
# 1. fork + clone
git clone https://github.com/<you>/ccplugin.git
cd ccplugin/plugins/tools/cortex

# 2. 跑基线检查
uv run scripts/check.py -d plugins/tools/cortex
# 必须通过, 警告 ≤1 (允许 main.py 的 1 条 warning)

# 3. 改代码

# 4. 跑相关测试
uv run pytest lib/tests              # 主仓测试
python3 lint/run.py --vault /tmp/test-vault  # 手动验证 lint

# 5. AI 可读性自检 (commands/skills/agents 改动必跑)
claude --settings ~/.claude/settings.glm-4.7-flash.json \
  -p "$(cat skills/cortex-X/SKILL.md)" \
  --output-format stream-json | jq -r '...'
```

## 加新 Skill

### 步骤

1. 决定是否真需要新 skill。先看 11 个现有 skill 是否能覆盖。
2. 在 `skills/` 下建子目录, 名字与 frontmatter `name` 字段一致 (防漂移)。
3. 写 `skills/<name>/SKILL.md`：

```markdown
---
name: cortex-<verb>
description: <一句话用途>。Triggers on "<短语1>", "<短语2>", "<中文短语>".
allowed-tools: Bash Read Write Glob mcp__obsidian__obsidian_X mcp__obsidian__obsidian_Y
---

# cortex-<verb>

<详细描述, 200-400 行>

## 触发场景

## 流程

## 失败处理

## 示例
```

4. 必要时在 frontmatter 加：
   - `disable-model-invocation: true` — 副作用大或耗时长, 强制显式触发
   - `argument-hint: "<arg>"` — 提示参数格式

5. 更新 `AGENT.md` 的 skill 表格 + `README.md` 速览。

### 约束

- `description` 长度 ≤ 1024 字符。
- 11 个 (现有) + 你的新 skill 总 description ≤ 1500 字符 (description 池软上限 1536)。
- `allowed-tools` 用**空格**分隔 (skill 语法; commands 用逗号但本插件无 commands)。
- 触发短语必须中英双语覆盖 (cortex 面向中文用户)。

### 验证

```bash
# AI 可读性自检
claude --settings ~/.claude/settings.glm-4.7-flash.json \
  -p "<复述 skills/cortex-<verb>/SKILL.md 全文>" \
  --output-format stream-json | jq -r 'select(.type == "result" and .subtype == "success") | .result'
```

返回结果应包含：用途、触发短语、关键流程。否则 description 太模糊, 重写。

## 加新 Lint 规则

### 步骤

1. 编辑 `lint/rules.json`, 加新条目：

```json
{
	"id": "<verb>-<noun>",
	"severity": "error|warn",
	"description": "<中文一句话>",
	"autofix": false
}
```

2. 在 `lint/run.py` 中实现检查函数 `check_<id_normalized>(vault, file_path) -> list[Issue]`。
3. 若 `autofix: true`, 实现 `fix_<id_normalized>(vault, file_path, issue) -> bool`。
4. 在 `docs/Lint 规则.md` 加该规则的逐条说明 (sev / autofix / 示例 / 建议)。
5. 加测试用例 `lint/tests/test_<id>.py` (pytest)。

### autofix 判定原则

- 仅当**机械变换无歧义**时才 autofix (e.g. 字段缺省、格式不一致)。
- 涉及内容语义 (孤儿页是否归档、alias 冲突哪个保留) 一律 manual。

## 加新 Preset

### 步骤

1. 在 `presets/` 下建目录, 例如 `presets/johnny-decimal/`。
2. 写 `_structure.json`：

```json
{
  "preset": "johnny-decimal",
  "version": "1.0",
  "description": "Johnny.Decimal — 10 区域 × 10 类别 × 100 ID。",
  "directories": ["10-19_admin", "20-29_finance", ...],
  "seed_files": [
    {"src": "seed/10-19_admin/_index.md", "dst": "10-19_admin/_index.md"}
  ]
}
```

3. 在 `presets/johnny-decimal/seed/` 下放种子文件。
4. 共享根 (`_meta` / `_templates` / `index.md` / `hot.md` / `log` / `folds`) 不在 preset 内重复, cortex-install 统一注入。
5. 在 `docs/知识库结构.md` 加新 preset 章节。
6. 更新 `docs/索引.md` 与 `README.md` (若改了"4 个 preset"措辞)。

## 加新 Hook (谨慎)

cortex 故意不注册 PreToolUse / PostToolUse / Notification 等 hook (D5: 不写 noop hook)。

新加 hook 必须证明：

1. 有实际行为 (不只 log)。
2. 性能开销可接受 (timeout 合理设置)。
3. 失败不阻断会话 (`exit 0` + log)。

## 提交规范

```bash
git commit -m "$(cat <<'EOF'
feat(cortex): add johnny-decimal preset

- 新增 presets/johnny-decimal/_structure.json
- 6 个 seed _index.md
- docs/知识库结构.md 加 §Preset 5

测试: pytest lint/tests + cortex-install /tmp/jd-vault --preset johnny-decimal
EOF
)"
```

约束：

- commit message 中文 OK, 关键字 `feat:` / `fix:` / `chore:` / `docs:` 用英文。
- 一个 commit 一个目的, 不混杂。
- 跨 plugin 改动分开 commit。

## 测试约定

| 改动类型  | 必跑                                                  |
| --------- | ----------------------------------------------------- |
| skill     | GLM 自检 + 手动会话验证                               |
| lint rule | `pytest lint/tests/test_<id>.py`                      |
| preset    | `cortex-install /tmp/test-vault --preset <name>` 手动 |
| hook      | 新建 vault, 跑会话, 看 `~/.cache/cortex/*.log`        |
| template  | `cortex-new <type> '<title>'` 手动验证                |
| refactor  | dry-run + apply 都跑, 看 `.cortex-backup/`            |

## 文档要求

任何改动都要更新：

- `README.md` 速览表格 (若改了能力清单)
- `AGENT.md` skill 表 (若改了 skill 列表)
- `docs/<对应章节>.md`
- 必要时更新 `docs/索引.md` 速查

## GLM 自检详解

为什么用 GLM-4.5-flash：模拟"另一个 LLM"是否能正确理解 skill description。

```bash
# 待测试: skills/cortex-X/SKILL.md
claude --settings ~/.claude/settings.glm-4.7-flash.json \
  -p "<待测试的内容>" \
  --output-format stream-json | jq -r 'select(.type == "result" and .subtype == "success") | .result'
```

判定通过：

1. 返回非空。
2. 返回正确复述了 skill 的用途、触发短语、关键流程。
3. 没把"显式触发"误解为"自动触发"。

不通过 → 改 description, 重测。

## 行为准则

- 不引入 emoji 除非用户已用过同款 (本插件 README 仅有 1 个 📝, 限于落档通知)。
- 中文文档全中文 (技术术语保留英文标识符)。
- 文件名 ASCII 慎用空格 (Obsidian + git 兼容性)。
- 保留 `@desktop` 风格的事件驱动思路供其他插件参考。
