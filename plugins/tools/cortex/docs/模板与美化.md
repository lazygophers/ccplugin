# 模板与美化

本文回答：cortex 提供 6 个什么模板、Obsidian callout 13 类各有什么用、什么时候必须回退到嵌入 HTML。
适用读者：用 cortex-new 建笔记的人、想自定义模板的进阶用户、关注双端 (Obsidian + GitHub) 渲染的贡献者。

## 6 个模板

位于 `templates/`, cortex-install 时拷贝/软链到 `<vault>/_templates/`。

| 模板 | 用途 | frontmatter type |
|------|------|------------------|
| `concept.md` | 抽象概念 (架构思想、模式、术语) | `concept` |
| `entity.md` | 具体实体 (人、工具、库、服务) | `entity` |
| `domain.md` | 项目域 (按 git remote 路径) | `domain` |
| `dashboard.md` | Bases / Dataview 仪表盘 | `dashboard` |
| `question.md` | 待解决问题, 答完转 concept | `question` |
| `source.md` | 外部素材 (文章/书/视频) | `source` |

## 模板共有结构

```markdown
<!-- cortex template: <type> -->
---
type: <type>
title: {{TITLE}}
aliases: []
tags: [<type>]
created: {{CREATED}}
updated: {{UPDATED}}
status: seed       # 仅 concept/question
related: []
sources: []
up: ""             # 上位概念
---

# {{TITLE}}

> [!abstract]+ 一句话定义
> ...
```

`{{TITLE}}` / `{{CREATED}}` / `{{UPDATED}}` 由 cortex-new 替换。

## Callout 13 类白名单

cortex 强制使用 Obsidian 标准 callout (lint rule `callout-unknown-type` 限制), 13 类：

| 类型 | 语义 | 视觉 |
|------|------|------|
| `note` | 一般备注 | 蓝 |
| `abstract` / `summary` / `tldr` | 摘要 | 蓝 |
| `info` | 信息 | 蓝 |
| `todo` | 待办 | 蓝 |
| `tip` / `hint` / `important` | 提示 | 青 |
| `success` / `check` / `done` | 成功 | 绿 |
| `question` / `help` / `faq` | 问题 | 黄 |
| `warning` / `caution` / `attention` | 警告 | 橙 |
| `failure` / `fail` / `missing` | 失败 | 红 |
| `danger` / `error` | 危险 | 红 |
| `bug` | bug | 红 |
| `example` | 示例 | 紫 |
| `quote` / `cite` | 引用 | 灰 |

折叠语法：

```markdown
> [!warning]+ 默认展开
> 内容

> [!warning]- 默认折叠
> 内容
```

## 优先 callout, 慎用 HTML

设计哲学："callout 替代 HTML grid" (见 `../README.md#设计哲学`)。原因：

- callout 在 Obsidian + GitHub 双端都能渲染。
- HTML 在 GitHub Markdown 中受限 (style 属性会被剥离), 在 Obsidian 中需启用 "Render HTML in markdown"。

### 何时回退 HTML

仅在 callout 无法表达时, 例如**多列 KPI 卡片**：

```html
<div style="display:grid;grid-template-columns:repeat(auto-fit,minmax(160px,1fr));gap:12px">
  <div style="padding:10px;border-left:3px solid #3b82f6;background:#f0f7ff">
    <strong>总条目</strong><br/><span style="font-size:1.4em">N</span>
  </div>
  <div style="padding:10px;border-left:3px solid #10b981;background:#f0fdf4">
    <strong>本月新增</strong><br/><span style="font-size:1.4em">N</span>
  </div>
</div>
```

`dashboard.md` 模板里有完整范例。其余场景一律 callout。

## frontmatter properties

Obsidian 1.4+ 把 frontmatter 升级为 properties, 类型化字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| `type` | text | 必填; lint rule `fm-missing-type` 检查 |
| `title` | text | 显式标题, 与 H1 一致 (`title-h1-mismatch` 检查) |
| `aliases` | list | 跨页搜索别名; lint `duplicate-alias` 防冲突 |
| `tags` | list | 标签 |
| `created` / `updated` | datetime | ISO 8601 |
| `related` | list (link) | wikilink 列表; 渲染为可点击 |
| `sources` | list (link) | 来源 wikilink |
| `up` | link | 上位概念页 |

填 wikilink 列表时用 `["[[note-a]]", "[[note-b]]"]`。

## Bases 集成 (Obsidian 1.7+)

`dashboard.md` 模板预留两套查询块：

1. Bases (`.base` 文件) — 1.7+ 核心功能, 类型化数据库视图。
2. Dataview — 兜底, 兼容老版本。

cortex-dashboard 检测 vault 是否启用 Bases:

```bash
test -d "$VAULT/.obsidian/bases" && USE_BASES=true
```

## 模板自定义

直接编辑 `<vault>/_templates/<type>.md` 即可。约束：

- 保留 `<!-- cortex template: <type> -->` 首行注释 (cortex-new 据此识别)。
- 保留 `{{TITLE}}` / `{{CREATED}}` / `{{UPDATED}}` 占位。
- 不要删 `type` frontmatter 字段。

## 美化原则

- **首段 callout**：用 `[!abstract]+ 一句话定义` 开头, 让笔记可脱离上下文阅读。
- **示例 callout**：用 `[!example]` 列 2-3 个具体场景。
- **陷阱 callout**：用 `[!warning]- 已知陷阱` 折叠, 减少视觉噪音。
- **对比表格**：3 行以上的对比一律用表格而非 bullet。
- **代码块标语言**：` ```bash ` / ` ```python ` / ` ```json `, 不裸 ` ``` `。
