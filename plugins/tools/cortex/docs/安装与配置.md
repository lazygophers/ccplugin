# 安装与配置

本文回答：怎样把 cortex 装到 Claude Code、vault 路径如何被解析、`config.json` 字段含义、有哪些环境变量影响行为。
适用读者：刚装好 Claude Code 想接入 Obsidian 的用户、需要在 CI/cron 中跑 cortex 的运维。

## 安装方式

### 方式 A：通过 ccplugin marketplace (推荐)

```bash
# 在 Claude Code 会话内
/plugin install cortex
```

安装完成后, 推荐再跑一次插件根的一键安装脚本生成共享配置与 wrapper:

```bash
cd <marketplace>/plugins/tools/cortex
./install.sh                              # 交互模式, 提示 vault / lang / settings
./install.sh --non-interactive --vault /path/to/vault   # 非交互
```

一键安装做了三件事:

1. 写 `~/.cortex/config.json` (vault / lang / settings / install_path)
2. 生成 `~/.cortex/scripts/*.sh` wrapper 套件 24 个:
   - 10 slash: lint / dashboard / doctor / init / promote / forget / digest / recall / refactor / ingest
   - 3 shell: install_cron / config / update
   - 11 CLI: save / search / deep_search / ingest_url / ingest_file / ingest_remote / refresh_projects / memory / ledger / session / html_render
3. (可选) 调 `install_cron.sh` 安装周期任务 snippet (3 daily + 1 weekly: lint / dashboard / digest 每日 + refresh_projects 每周一)

**入口稳定**: 安装后所有日常操作 (lint / 更新 / 健康检查) 走 `~/.cortex/scripts/*.sh`, 即使 marketplace 升级或重新安装 cortex, wrapper 仍指向新的绝对路径——只要重跑一次 `./install.sh` 即可刷新。

### 方式 B：本地源码

```bash
git clone https://github.com/lazygophers/ccplugin.git
cd ccplugin/plugins/tools/cortex
# 把本目录注册到本地 marketplace (参见 ccplugin 主仓 docs/plugin-development.md)
```

### 方式 C：直接软链

```bash
ln -s $(pwd)/plugins/tools/cortex ~/.claude/plugins/cortex
```

注意：`plugin.json` 中的 hooks 路径用 `~/.claude/plugins/marketplaces/ccplugin-market/plugins/tools/cortex` 占位, 软链方式也能正确解析。

## vault 路径解析

cortex 在 hook 启动与 skill 调用时统一走 `hooks/_lib/resolve_vault.sh::resolve_vault`, 严格按以下 5 段优先级, 任一命中即停止：

| 优先级 | 来源                                                   | 示例                                                           |
| ------ | ------------------------------------------------------ | -------------------------------------------------------------- |
| 1      | `$(jq -r .vault ~/.cortex/config.json)` 环境变量                             | `export OBSIDIAN_VAULT=/path/to/vault`                         |
| 2      | `$XDG_CONFIG_HOME/cortex/config.json` 中 `.vault` 字段 | `~/.cortex/config.json`                                 |
| 3      | `~/.cortex/config.json` 中 `.vault` 字段        | 兜底, 与 2 同语义                                              |
| 4      | 默认 `~/persons/knowledge/obsidian`                    | 作者个人约定                                                   |
| 5      | auto-detect                                            | 扫 `~/Documents`、`~/Library` 找**唯一**含 `.obsidian/` 的目录 |

未命中时 hook 沉默退出 (退出码 0, stdout 空), 不阻断会话。skill 则报错"vault 未解析"。

诊断命令：

```bash
"诊断 cortex"   # 触发 cortex-doctor, 报告每段命中情况
```

## config.json schema

路径：`~/.cortex/config.json` (或 `$XDG_CONFIG_HOME/cortex/config.json`)。

```json
{
	"vault": "/Users/me/Documents/obsidian-vault",
	"lint": {
		"ignore": ["dead-wikilink"],
		"fix_on_save": false
	},
	"save": {
		"min_lines": 30,
		"min_chars": 800,
		"skip_if_only_questions": true
	},
	"smart_connections": {
		"enabled": true,
		"rest_port": 27124
	}
}
```

字段说明：

| 字段                          | 类型     | 默认    | 说明                                        |
| ----------------------------- | -------- | ------- | ------------------------------------------- |
| `vault`                       | string   | (无)    | vault 绝对路径; 优先级 2/3                  |
| `lint.ignore`                 | string[] | `[]`    | 跳过这些 lint rule id                       |
| `lint.fix_on_save`            | bool     | `false` | cortex-save 写完立即跑 lint --fix           |
| `save.min_lines`              | int      | `30`    | Stop hook 启发式: transcript 少于此行不归档 |
| `save.min_chars`              | int      | `800`   | 启发式: transcript 字符少于此不归档         |
| `save.skip_if_only_questions` | bool     | `true`  | 启发式: 全是疑问句不归档                    |
| `smart_connections.enabled`   | bool     | `true`  | cortex-search 是否走 SC REST                |
| `smart_connections.rest_port` | int      | `27124` | Smart Connections REST 端口                 |

不存在 `config.json` 时全部走默认值。

## 环境变量

| 变量                 | 用途                                                          |
| -------------------- | ------------------------------------------------------------- |
| `OBSIDIAN_VAULT`     | vault 路径优先级 1; 一切问题先设这个                          |
| `OBSIDIAN_API_KEY`   | obsidian-local-rest-api 鉴权; MCP 不走 stdio 而走 REST 时必需 |
| `XDG_CONFIG_HOME`    | 影响 `config.json` 查找路径                                   |
| `CLAUDE_PLUGIN_ROOT` | hook 内自动注入, 指向插件根; 不要手动覆盖                     |
| `CORTEX_DEBUG`       | 设为 `1` 让 hook 打印更多日志到 `~/.cache/cortex/*.log`       |

## MCP 配置 (强制 — AI 交互式 vault 写依赖)

cortex AI 交互层 (skills / agents) 走 **L1 = `mcp__obsidian__*`** 强制契约 (详见 `AGENT.md §协作约定 3`)。MCP 未注册 → AI 硬拒 vault 写, 必须 `AskUserQuestion` 单次授权 (本会话有效不写盘) 才能用 L2 obsidian CLI / L3 直写兜底。

注册官方 `mcp-obsidian` (Python, install.sh 末段引导):

```bash
claude mcp add obsidian uvx mcp-obsidian \
  -e OBSIDIAN_API_KEY=<your-key> \
  -e OBSIDIAN_HOST=127.0.0.1 \
  -e OBSIDIAN_PORT=27123
```

session_start hook 探活: 跑 `claude mcp list`, 找不到 obsidian 条目则回退检 `~/.claude.json:.mcpServers`。

要求 Obsidian 已装 `Local REST API` 社区插件 (端口默认 27123 / TLS 27124)。

例外: Stop hook / cron / python CLI (非 AI 上下文) 不走 MCP, 直接文件 IO。

## 验证安装

依次跑：

```bash
ls $CLAUDE_PLUGIN_ROOT/plugins/tools/cortex/.claude-plugin/plugin.json   # 文件存在
echo $(jq -r .vault ~/.cortex/config.json)                                                     # 路径正确
```

在会话内：

```text
"诊断 cortex"
```

13 项体检全绿 (或仅 SC 失败但有 fallback) 即视为安装成功。

## 卸载

```bash
/plugin uninstall cortex
```

cortex 不修改 vault 内已生成的文件, 卸载后用户笔记保留。
