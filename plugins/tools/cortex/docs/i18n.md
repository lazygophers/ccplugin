# i18n — vault 多语言机制

cortex v2 把 vault 目录与文件名按 vault 的 `lang` 渲染, 让中/英/日 (与用户自定义 lang) 用户都能拿到自然命名的 vault, 同时核心基础设施 (frontmatter 字段、`_meta/`、`MOC`、git remote 路径等) 保持 ASCII 一致, 跨 lang 数据兼容。

## 1. 一个 vault, 一个主语言

- vault 主语言记录在 `_meta/version.json:.lang` (例: `zh-CN` / `en` / `ja`)
- 业务目录命名 (`概念/` vs `concepts/`) 由 `locales/<lang>.yml:dirs.<key>` 渲染
- **不**支持一个 vault 内多 lang 子树共存; 切语言后既有目录保留 (lint 仅 info 提示), 通过 `cortex-refactor migrate-locale` 主动迁移

## 2. locale 文件查找顺序

按优先级 (高优先覆盖低优先, 同 key 的 deep-merge):

```
1. <vault>/locales/<lang>.yml         # vault 级 (最高)
2. ~/.config/cortex/locales/<lang>.yml # 用户级
3. <plugin>/locales/<lang>.yml         # 插件内置 (最低)
```

修改 vault `locales/<lang>.yml` 不影响其他 vault; 修改 `~/.config/cortex/locales/` 影响该用户所有 vault。

## 3. fallback 链

每个 key 单独 fallback (不整文件回退):

```
zh-CN  →  zh  →  en  →  KeyError
ja     →        en  →  KeyError
xx-YY  →  xx   →  en  →  KeyError
```

例: `zh-CN.yml:dirs.concepts` 缺失 → 回退 `en.yml:dirs.concepts`, 而非整个文件用 en。

## 4. 不翻译白名单 (locales 不允许覆盖)

- 基础设施: `_meta` `_templates` `locales` `sessions` `index` `hot` `log` `folds`
- 子结构: `_domain` `_index`
- LYT 通行术语: `MOC`
- 时间格式: `YYYY-MM` · `YYYY-MM-fold-NNN` · `DD-HHMM`
- block-id: `^cortex-<sha8>`
- frontmatter 字段名: `type` `title` `aliases` `tags` `created` `updated` `preset` `cli` `cli_session` `lang`
- git remote 路径: `<host>/<org>/<repo>/`

## 5. SessionStart 注入语言

`session_start.sh` 输出 `additionalContext` 时, 头部固定一行:

```text
## Cortex 已连接 (lang=zh-CN, vault=/Users/.../obsidian, preset=lyt)
```

后续协作约定从 `prompts.search_first` 等键渲染。用户不改 prompt 就能自动多语化。

## 6. 切换语言

### 6.1 仅改 lang 标记

```text
"切换 vault 语言到 en"
→ cortex-locale skill 写 _meta/version.json:.lang = "en"
```

效果:

- 改 `_meta/version.json:.lang`
- **不**重命名既有目录
- 此后 cortex-new / cortex-install 走新 lang
- lint i18n-002 报告"目录不在 dirs map" (warn, 不阻塞)

### 6.2 一次性 rename 既有目录

```bash
python3 ~/.claude/plugins/marketplaces/ccplugin-market/plugins/tools/cortex//refactor/migrate_locale.py \
  --vault <path> --from zh-CN --to en --apply
```

- 默认 dry-run 先看计划, 再 `--apply`
- git repo 走 `git mv` 保留历史
- 全 vault wikilink 路径前缀替换 (`[[概念/X]]` → `[[concepts/X]]`)
- 写 `_meta/migrations/<ts>-migrate-locale.json` 记录

## 7. lint i18n 规则

| ID                             | 触发                               | 严重度 | 处理                           |
| ------------------------------ | ---------------------------------- | ------ | ------------------------------ |
| i18n-frontmatter-lang-mismatch | 页 frontmatter `lang` ≠ vault.lang | warn   | 手工改 / cortex-translator     |
| i18n-path-not-in-locale        | 顶层业务目录不在 dirs map          | warn   | cortex-refactor migrate-locale |

`--lang <code>` 参数可临时覆盖 vault.lang 做交叉检查 (cron 跨 lang 审计常用)。

## 8. 用户自定义 lang

不需要插件发版, 只要:

```bash
mkdir -p ~/.config/cortex/locales
cat > ~/.config/cortex/locales/de.yml <<EOF
meta:
  lang: de
  fallback: en
dirs:
  concepts: konzepte
  entities: entitäten
  ...
EOF
# 在某 vault:
echo '{"lang":"de","preset":"lyt"}' > <vault>/_meta/version.json
```

cortex-doctor 可验证 fallback 链与 dirs map 完整性。
