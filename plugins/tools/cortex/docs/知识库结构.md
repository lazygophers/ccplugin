# 知识库结构

本文回答：cortex 在 vault 里铺了什么目录、4 个 preset 各自适合什么场景、共享根目录文件有什么用。
适用读者：第一次选 preset 的用户、想从一个 preset 迁到另一个的进阶用户。

## 共享根 (所有 preset 一致)

无论选哪个 preset, cortex 都会在 vault 根创建以下结构：

```
<vault>/
├── _meta/
│   ├── cortex-version.md     # 写入 cortex 版本与 preset, 升级时检查
│   └── presets.md            # 当前 preset 与历史切换记录
├── _templates/
│   ├── concept.md            # 6 个模板软链/拷贝自插件 templates/
│   ├── entity.md
│   ├── domain.md
│   ├── dashboard.md
│   ├── question.md
│   └── source.md
├── index.md                  # 顶层索引: 列每个业务子目录入口
├── hot.md                    # 热点缓存: 最近 10 条变更, ≤200 行 (lint 限制)
├── log/                      # 日志: 按月分子目录
│   └── YYYY-MM/
│       └── DD-HHMM-<slug>.md
└── folds/                    # 滚动归档区: cortex-fold 把 log 老条目滚到这
    └── YYYY-MM-fold-NNN.md
```

设计要点：

- `_meta/` 只放 cortex 自己用的元数据, 用户不直接编辑。
- `_templates/` 让 Obsidian 的 "Templates" 核心插件能直接选用。
- `index.md` 与 `hot.md` 是 SessionStart hook 注入的关键来源 (见 `Hooks 机制.md`)。
- `log/` 每月一子目录, 文件名 `DD-HHMM-<slug>.md` 保证按时间排序。
- `folds/` 由 `cortex-fold` (默认 dry-run, `--apply` 才落盘) 滚动 7 天前的 log。

## Preset 1: LYT (默认)

Linking Your Thinking by Nick Milo, 8-bucket + MOC 主导。

```
<vault>/
├── 00_MOC/                  # Map of Content: home / topics-moc / projects-moc
├── 10_concepts/             # 抽象概念
├── 20_entities/             # 具体实体 (人/工具/库/服务)
├── 30_domains/              # 项目域 (按 git remote 路径)
│   └── github.com/<org>/<repo>/
├── 40_sources/              # 外部素材 (文章/书/视频)
├── 50_questions/            # 待解决问题, 答完转 concept
├── 60_dashboards/           # Bases / Dataview 仪表盘
├── 70_fleeting/             # 闪念: 进 inbox, 待处理
└── 80_archive/              # 归档: 不再活跃但有保留价值
```

适合：跨多项目知识汇聚、长期个人 PKM。

## Preset 2: Zettelkasten

扁平 + UID, 每篇单一原子概念。

```
<vault>/
├── zettels/                 # 主体: YYYYMMDDHHMM-<slug>.md, frontmatter 必含 uid
├── structure-notes/         # 结构笔记: 高层导航
├── inbox/                   # 待消化
└── references/              # 引用源
```

特点：

- 文件名固定 `YYYYMMDDHHMM-<slug>.md`, frontmatter `uid` 字段必填。
- 不分类 by topic, 仅靠 wikilink 与 structure-notes 浮现结构。
- `cortex-new` 在 zettel preset 下自动加 `uid` 字段。

适合：原子化笔记 + 长期沉淀, 不喜欢分类层级的人。

## Preset 3: PARA (Tiago Forte)

按行动距离组织。

```
<vault>/
├── 1_projects/              # 有截止时间, 有目标
├── 2_areas/                 # 持续维护的责任区
├── 3_resources/             # 主题参考资料
└── 4_archive/               # 完成或失活
```

特点：

- 项目状态变化时频繁移动文件 (1 ↔ 4)。
- `cortex-refactor` 的 `rename` 子命令会修所有反链。

适合：以执行为中心的工作流 (GTD 用户)。

## Preset 4: blank

最小骨架, 只有共享根。

```
<vault>/
├── _meta/ _templates/ index.md hot.md log/ folds/   # 共享根
# (无业务目录)
```

适合：

- 已有自定义结构的 vault, 只想接入 cortex 协作约定与 hooks。
- 不想被 preset 影响目录命名。

## 选 Preset 的速查

| 你的场景 | 选 |
|----------|---|
| 新 vault, 想要长期 PKM | LYT |
| 喜欢扁平 + UID | Zettelkasten |
| 项目执行导向 | PARA |
| 已有 vault 不想动结构 | blank |

## 路径与命名规则 (prd §3.2.7)

cortex 强制 (lint rule `path-naming-violation` 检查)：

- 文件名只用 `[\w一-鿿\-\.]`, 不允许空格、`/`、`\`、`:`。
- 中文文件名 OK (Obsidian 与 git 都兼容)。
- domain 路径必须是 `30_domains/<host>/<org>/<repo>/` 三层。
- log 文件名固定 `DD-HHMM-<slug>.md` (cortex-save 自动生成)。
- block-id 必须 `^cortex-<sha8>` (lint `block-id-duplicate` 强制唯一)。

## 切换 preset

cortex **不自动**切换 preset。手动流程：

1. 备份 vault (`cp -r vault vault.backup`)。
2. 编辑 `_meta/presets.md` 添加切换记录。
3. 跑 `cortex-refactor rename` 子命令把现有目录映射到新 preset。
4. 跑 `cortex-lint --fix` 修复 frontmatter `preset` 字段。
5. 重启 Claude Code 让 SessionStart 重新注入。
