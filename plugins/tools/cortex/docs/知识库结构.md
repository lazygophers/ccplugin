# 知识库结构

本文回答：cortex 在 vault 里铺了什么目录、共享根目录文件有什么用、知识库 4 子目录如何路由。
适用读者：第一次接入 cortex 的用户。

## 共享根

cortex 在 vault 根创建以下结构：

```
<vault>/
├── _meta/
│   ├── version.json         # vault 元数据 (lang / 时间戳)
│   └── memory-policy.yaml   # L0-L4 记忆策略 + cron 配置
├── _templates/
│   ├── concept.md            # 6 个模板软链/拷贝自插件 templates/
│   ├── entity.md
│   ├── domain.md
│   ├── dashboard.md
│   ├── question.md
│   └── source.md
├── index.md                  # 顶层索引: 列每个业务子目录入口
└── hot.md                    # 热点缓存: 最近 10 条变更, ≤200 行 (lint 限制)
```

设计要点：

- `_meta/` 只放 cortex 自己用的元数据, 用户不直接编辑。
- `_templates/` 让 Obsidian 的 "Templates" 核心插件能直接选用。
- `index.md` 与 `hot.md` 是 SessionStart hook 注入的关键来源 (见 `Hooks 机制.md`)。
- `log/` 每月一子目录, 文件名 `DD-HHMM-<slug>.md` 保证按时间排序。

## 知识库 4 子目录

```
<vault>/
├── 知识库/项目/             # git repo + 本地项目 (host/org/repo 三级)
│   ├── github.com/<org>/<repo>/
│   ├── gitlab.com/<org>/<repo>/
│   └── <rel-host>/<rel-org>/<rel-repo>/  # 本地 (相对 $HOME 路径拆段, 不足 3 段补 _local)
├── 知识库/领域/             # 用户/AI 自决域名 (创作/学习/工作/技术/生活/金融/未分类/...)
│   └── <域>/<子主题>?/<topic>.md
├── 知识库/日记/日/<YYYY-MM>/<YYYY-MM-DD>.md   # 仅日维度, 周/月/年 已废弃
├── 知识库/收件箱/             # 落档兜底入口, 等 digest 分发 (无类目/未决条目)
├── _assets/dashboards/        # Bases / Dataview 仪表盘
└── 归档/                       # 归档 (日记季度桶: 归档/日记/<YYYY-QN>.md)
```

适合：跨多项目知识汇聚、长期个人 PKM。

## 路径与命名规则 (prd §3.2.7)

cortex 强制 (lint rule `path-naming-violation` 检查)：

- 文件名只用 `[\w一-鿿\-\.]`, 不允许空格、`/`、`\`、`:`。
- 中文文件名 OK (Obsidian 与 git 都兼容)。
- project / domain (alias) 路径必须是 `知识库/项目/<host>/<org>/<repo>/` 三层 (本地走相对 `$HOME` 路径策略: `~/persons/lyxamour/ccplugin/` → `知识库/项目/persons/lyxamour/ccplugin/`; 不足 3 段补 `_local`)。
- log 文件名固定 `DD-HHMM-<slug>.md` (cortex-save 自动生成)。
- block-id 必须 `^cortex-<sha8>` (lint `block-id-duplicate` 强制唯一)。
