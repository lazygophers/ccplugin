# 故障排查

本文回答：cortex 常见 6+ 症状的可能原因与修复路径。
适用读者：装好 cortex 但发现某些功能不工作的用户。

## 诊断三板斧

任何问题先跑：

```bash
# 1. 看 vault 是否命中
echo "$(jq -r .vault ~/.cortex/config.json)"
ls "$(jq -r .vault ~/.cortex/config.json)" 2>/dev/null | head

# 2. 跑 cortex-doctor (会话内)
"诊断 cortex"

# 3. 看 hook 日志
ls -la ~/.cache/cortex/
tail -50 ~/.cache/cortex/session_start.log
tail -50 ~/.cache/cortex/stop.log
```

## 症状 1: SessionStart 无注入

会话开始后看不到"## Cortex 已连接"段。

| 可能原因                    | 排查                                                                              | 修复                                                   |
| --------------------------- | --------------------------------------------------------------------------------- | ------------------------------------------------------ |
| vault 路径未命中 5 段优先级 | 看 `~/.cache/cortex/session_start.log` 是否有 `vault not resolved`                | 设 `export OBSIDIAN_VAULT=/path`                       |
| hook 超时 (1500ms)          | 看 log 是否有 timeout                                                             | vault 太大时 `index.md` / `hot.md` 读取慢; 缩 `hot.md` |
| `plugin.json` 未加载        | `/plugin list` 看 cortex 是否启用                                                 | 重装 `/plugin install cortex`                          |
| AGENT.md 缺失               | `ls ~/.claude/plugins/marketplaces/ccplugin-market/plugins/tools/cortex/AGENT.md` | 重装                                                   |

## 症状 2: skill 报"MCP 不可用"

cortex-search / cortex-save 等报 obsidian MCP tool 不存在。

| 可能原因                     | 排查                     | 修复                                               |
| ---------------------------- | ------------------------ | -------------------------------------------------- |
| `~/.mcp.json` 未配 obsidian  | `cat ~/.mcp.json`        | 加 `obsidian-mcp-server` 条目 (见 `安装与配置.md`) |
| Obsidian 未启 Local REST API | 在 Obsidian 内启用插件   | 装 "Local REST API" 插件并启用                     |
| 端口冲突                     | `lsof -i:27123 -i:27124` | 改 Obsidian 插件端口或 kill 冲突进程               |
| OBSIDIAN_API_KEY 未设        | `echo $OBSIDIAN_API_KEY` | 在 Obsidian REST 插件设置里复制 key, export        |
| Claude Code 启动时 MCP 已挂  | 看 `/mcp` 命令输出       | 重启 CC, 或 `/mcp restart obsidian`                |

cortex 在 MCP 不可用时**会自动 degraded**:

- cortex-search 走 ripgrep 兜底
- cortex-save 直接 `Write` 文件 (跳过 MCP append)

degraded 模式仍可用, 只是性能与索引同步差。

## 症状 3: Stop 后没落档

会话结束后 知识库/日记/日/ 没有新文件。

| 可能原因                  | 排查                                                                   | 修复                                                       |
| ------------------------- | ---------------------------------------------------------------------- | ---------------------------------------------------------- |
| 启发式判定为非平凡 (rc=2) | `tail ~/.cache/cortex/stop.log` 看是否有 `heuristic threshold not met` | 显式说"归档"强制触发 cortex-save                           |
| transcript 不可读         | log 含 `transcript missing`                                            | 偶发, CC 未 flush; 重试或显式归档                          |
| stop_hook_active=true     | log 含 `skip`                                                          | 上一次 hook 还没结束; 多发生在嵌套 sub-agent               |
| vault 未命中              | 同症状 1                                                               | 同症状 1                                                   |
| transcript 全是 CRUD      | 启发式认为平凡                                                         | 调 `~/.cortex/config.json` 的 `save.min_lines` 阈值 |

调阈值示例：

```json
{
	"save": {
		"min_lines": 10,
		"min_chars": 200,
		"skip_if_only_questions": false
	}
}
```

## 症状 4: auto-commit 与 OGit 冲突

cortex 写入后 git 报"already committed" 或重复 commit。

| 可能原因                         | 排查                                                 | 修复                                                        |
| -------------------------------- | ---------------------------------------------------- | ----------------------------------------------------------- |
| OGit 未被检测到                  | `ls $(jq -r .vault ~/.cortex/config.json)/.obsidian/plugins/obsidian-git/` | 路径标准, 应自动检测; 若仍冲突看 `~/.cache/cortex/save.log` |
| OGit + cortex 都开了 auto-commit | 看 vault 提交历史                                    | 关一边: cortex 默认会让步, OGit 设里关 auto-commit 也行     |

cortex 检测逻辑：

```bash
test -d "$VAULT/.obsidian/plugins/obsidian-git" && AUTO_COMMIT=false
```

命中即关 auto-commit, OGit 接管。

## 症状 5: lint --fix 未改盘

跑 `cortex-lint --fix` 报告说"修了 N 处"但文件没变。

| 可能原因                      | 排查                           | 修复                                                         |
| ----------------------------- | ------------------------------ | ------------------------------------------------------------ |
| autofix 仅对 6 条规则生效     | 看报告 `fixed` 字段对应的 rule | 其他规则 (rule 3/4/5/7/10/12/13) 用 cortex-refactor 协助     |
| 文件被其他进程占用 (Obsidian) | Obsidian 是否打开该文件        | 关掉 Obsidian 再跑, 或忽略 (cortex 写入会触发 Obsidian 重载) |
| 权限问题                      | `ls -l <file>`                 | `chmod u+w <file>`                                           |
| backup 失败回滚了             | 看 `~/.cache/cortex/lint.log`  | 检查 `<vault>/.cortex-backup/` 是否有空间写入                |

autofix 范围 (见 `Lint 规则.md`)：rule 1 (fm-missing-type), 2 (fm-missing-created), 6 (hot-too-long), 8 (index-missing-section), 9 (title-h1-mismatch), 11 (block-id-duplicate)。

## 症状 6: canvas 文件空 / 无节点

cortex-canvas 生成的 `.canvas` 打开是空白。

| 可能原因                           | 排查                                         | 修复                                                             |
| ---------------------------------- | -------------------------------------------- | ---------------------------------------------------------------- |
| obsidian CLI 不可用且 topic 无匹配 | log 有 "no nodes matched"                    | 用 cortex-search 先确认 vault 内有相关页                         |
| Breadcrumbs 插件未启               | 节点无 frontmatter `up`/`down`               | 启用 Breadcrumbs 插件并填 frontmatter, 或 cortex 降级到静态 JSON |
| JSON Canvas schema 错              | 用 `python3 -m json.tool <file>.canvas` 验证 | 删了重生                                                         |

## 症状 7: 中文文件名乱码 / 链接失效

某些情况下中文文件名在 git 显示乱码或 wikilink 失效。

| 可能原因                        | 修复                                                  |
| ------------------------------- | ----------------------------------------------------- |
| git 未配 `core.quotepath false` | `git config core.quotepath false`                     |
| macOS HFS+ NFD vs Linux NFC     | 统一用 NFC, Obsidian 默认 NFC, macOS 上跨设备同步注意 |
| 文件名含空格                    | 改用 `-` 或下划线                                     |

## 症状 8: hot.md 越来越长

`hot.md` 超 200 行, lint 报 `hot-too-long`。

修复：

```text
"知识库 audit --fix"   # autofix rule 6, 自动截断到 200 行, 多余存 归档/
```

## 症状 9: git clone 失败 (ingest_remote)

`bash ~/.cortex/scripts/ingest_remote.sh https://github.com/foo/bar` 返回 `{"ok": false, "error": "git clone failed"}`。

排查:

1. 网络: `curl -I https://github.com` 检通
2. 私有 repo: 远程仅支持公开 repo (MVP), 私有走 `git clone` 后调 `ingest.sh <local-path>`
3. 仓库不存在: 检 URL 拼写 (404)
4. 超时: 默认 timeout 60s, 大 repo 用 `--depth=N` 限历史 (默认 shallow)

## 症状 10: website crawl rate-limit / sitemap 缺失

`ingest_remote.sh https://example.com` 返回 0 pages 或 `429 Too Many Requests`。

排查:

1. sitemap.xml 缺失: `curl https://example.com/sitemap.xml` 检 200; 缺则走递归 `<a href>` BFS (慢, 不全)
2. 429 限流: 等 retry-after 后重跑; MVP 不支持自动 retry
3. robots.txt disallow: 检 `https://example.com/robots.txt`

## 症状 11: refresh_projects hash 比对失败

`refresh_projects.sh` 报旧项目 `content_hash missing → 视为首次全量`。

原因: PR1 之前 ingest 的项目 frontmatter 缺 `last_commit_sha` / `content_hash` 字段, refresh 视为首次走全量 ingest pipeline。

解决: 自动行为, 首次 refresh 后写入字段, 下次起增量。

## MCP 不可达 (L1/L2 失败)

症状: AI 调 `mcp__obsidian__obsidian_simple_search` 报 "MCP server not available" 或类似错误。

排查:

1. Obsidian 已启动? Local REST API plugin 已装 + enable? (端口 27123/27124)
2. `claude mcp list | grep obsidian` 检 MCP 注册
3. 未注册 → 触发 `AskUserQuestion` 单次授权, 走 L3 fallback `bash ~/.cortex/scripts/search.sh --query <q>`

L3 走 CLI 内部 hot.md / index.md / ripgrep 兜底, 与 P9 之前的 search.sh 行为一致。

## 召回率低 (搜不到已存在的内容)

症状: `mcp__obsidian__obsidian_simple_search` 返 0 hits, 但文件实际在 vault。

可能原因:

1. 关键词不在 frontmatter `title` / `tags` / 正文 — 该页缺 `aliases` / `keywords` 字段 (P9 召回率字段)
2. ingest 时未跑 P9 + 召回率字段 — 走 `bash ~/.cortex/scripts/refresh_projects.sh --scope=<host/org/repo>` 重 ingest
3. 老文件无 aliases/keywords — 需 migration (本任务未实现, 等用户调 `ingest_remote` 覆盖)

调试: 用 L2 `mcp__obsidian__obsidian_complex_search` 按 path glob 过滤已知子目录, 验证文件确实在 vault, 再回头看 frontmatter 是否缺 aliases/keywords。

## qmd MCP 误用

症状: AI 用 qmd MCP search 而非 obsidian MCP, 命中结果与 cortex vault 不一致。

原因: hook 提示软标 "优先 obsidian, 非 qmd" (D2 决策不强制禁), AI 多 MCP 时偶尔选错。

解决:

- 用户明确指示 "用 `mcp__obsidian__obsidian_simple_search`" 强制纠正
- 或 disable qmd MCP server: `claude mcp remove qmd`
- 检查 hook 提示: `cat /dev/null | python3 ~/.claude/plugins/marketplaces/ccplugin-market/plugins/tools/cortex/scripts/hooks/user_prompt_submit.sh` 输出应含 "优先 obsidian, **非 qmd**"


## 评分字段缺失 (lint rule 23 warn 大量)

**症状**: `bash ~/.cortex/scripts/lint.sh` 报千+ warn `frontmatter-required-scores`

**原因**: 旧 vault 文件缺 `score` / `confidence` / `source_credibility` / `maturity` 4 字段, 或记忆缺 `importance` / `confidence` 2 字段。

**解决**:

1. 一次性 migrate (推荐): `bash ~/.cortex/scripts/migrate.sh --to=v2`
2. 后续 `ingest_remote` / `save` 落档自动写入, 不会再缺
3. lint `--fix` 也可加 stub 但不智能, 推荐 `migrate.sh`

## 评分字段范围越界

**症状**: lint warn `score=11.5 非合法 0.0-10.0 浮点`

**原因**: 手写或第三方工具写入越界值

**解决**: `lint --fix` 自动 clamp 到 `[0, 10]`; 或 `migrate.sh` 一并处理

## migrate.sh 失败

**症状**: `bash ~/.cortex/scripts/migrate.sh` 报错

**排查**:

1. **磁盘空间**: backup tar.gz 至少需 vault 同等空间
2. **权限**: vault 路径可读写
3. **PyYAML 依赖**: `pip3 install --user pyyaml` (install.sh 应已装)
4. **回滚**: `tar xzf /tmp/cortex-migration-backup-<TS>.tar.gz -C <vault 父目录>` 覆盖
