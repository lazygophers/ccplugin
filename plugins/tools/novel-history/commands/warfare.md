# 设计战争策略

帮助作者快速设计或完善战争和军事相关情节。

## 适用场景

| 场景 | 示例 |
|------|------|
| **战役设计** | `/novel-history warfare 设计一场以少胜多的经典战役` |
| **兵法运用** | `/novel-history warfare 围魏救赵的具体战术实施` |
| **攻城战** | `/novel-history warfare 设计一场三个月的围城攻防` |
| **军队体系** | `/novel-history warfare 构建一支十万人大军的编制` |

## 执行流程

### 1. 确认需求方向

通过提问明确：
- **战争规模**：小规模遭遇战/中等战役/大规模会战/灭国之战
- **战争类型**：野战/攻城/守城/水战/山地战
- **双方实力**：兵力对比、兵种构成、将领能力
- **战场地形**：平原/山地/河流/城池/关隘
- **期望结果**：完胜/惨胜/失败/相持

### 2. 生成战争方案

根据需求激活 `Skills(warfare-strategy)` 并生成：

#### 战前态势
- 双方兵力部署和行军路线
- 后勤补给状况和时间窗口
- 情报侦察和战前谋划
- 各将领的战术倾向分析

#### 战役过程
- 初期接触和试探交锋
- 战术运用和兵种配合
- 关键转折点和意外因素
- 决战时刻的临场决断
- 追击或撤退的处理

#### 战后影响
- 伤亡统计和战果评估
- 对战略格局的影响
- 论功行赏和政治后果
- 对后续剧情的推动作用

### 3. 合理性验证

检查战争情节的合理性：
- 战术是否符合当时的军事技术水平
- 后勤补给是否能支撑军队规模
- 地形是否合理影响了战术选择
- 伤亡比例是否在合理范围内

### 4. 输出战役文档

以结构化格式输出，包含战场地图描述、时间线和关键节点。

## 设计原则

- **地形为先**：先设计地形再推演战术
- **后勤约束**：军队规模受后勤制约
- **将领特色**：每个将领有独特的指挥风格
- **战争残酷**：真实展现战争的代价
