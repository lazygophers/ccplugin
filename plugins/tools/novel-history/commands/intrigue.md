# 设计权谋情节

帮助作者快速设计或完善权谋斗争情节。

## 适用场景

| 场景 | 示例 |
|------|------|
| **派系设计** | `/novel-history intrigue 设计三方势力的朝堂格局` |
| **阴谋布局** | `/novel-history intrigue 设计一个连环计夺嫡剧情` |
| **宫廷斗争** | `/novel-history intrigue 后宫争宠的层层反转` |
| **权力博弈** | `/novel-history intrigue 皇权与相权的终极对决` |

## 执行流程

### 1. 确认需求方向

通过提问明确：
- **博弈类型**：朝堂派系/宫廷后宫/夺嫡之争/地方割据
- **参与方数量**：两方对立/三方博弈/多方混战
- **主角定位**：主导者/参与者/幕后操盘/被卷入者
- **期望结果**：大获全胜/惨胜/两败俱伤/暂时休战

### 2. 生成权谋方案

根据需求激活 `Skills(political-intrigue)` 并生成：

#### 势力格局
- 各派系的核心人物和诉求
- 派系间的利害关系图
- 各方的明面实力与暗中底牌
- 可被利用的关键弱点

#### 阴谋设计
- 完整的计策链（起手→布局→引爆→收网）
- 每一步的执行细节和所需条件
- 可能出现的意外和应变预案
- 关键反转点的设计

#### 角色行为
- 每个关键角色在博弈中的行为逻辑
- 角色的信息差（谁知道什么）
- 角色的情感变化和心理活动
- 背叛、被骗、觉醒等关键时刻

### 3. 逻辑验证

检查权谋情节的合理性：
- 每个人的行为是否符合其立场和性格
- 信息流动是否合理（不该知道的不能知道）
- 阴谋是否有合理的破绽（不能万无一失）
- 最终结果是否推动了整体剧情

### 4. 输出情节文档

以结构化格式输出，包含时间线、势力图和关键节点。

## 设计原则

- **利益为本**：所有权谋行为都要有利益动机
- **智力对等**：对手也要有高光表现
- **代价真实**：胜利也要付出代价
- **层层剥笋**：真相要一层层揭开
