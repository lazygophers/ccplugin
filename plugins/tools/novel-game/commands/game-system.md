# 设计游戏系统

帮助作者快速设计或完善游戏世界的核心系统。

## 适用场景

| 场景 | 示例 |
|------|------|
| **游戏类型** | `/novel-game game-system 设计一款全息MMORPG` |
| **职业体系** | `/novel-game game-system 八大职业体系设计` |
| **等级系统** | `/novel-game game-system 200级等级系统和转生机制` |
| **经济系统** | `/novel-game game-system 游戏内经济循环设计` |

## 执行流程

### 1. 确认需求方向

通过提问明确：
- **系统类型**：游戏类型定位/职业体系/等级系统/数值平衡/经济循环/完整系统
- **游戏风格**：全息网游/虚拟现实/传统端游/手游竞技
- **创新程度**：经典框架微调/大幅创新/完全原创
- **已有设定**：与已有世界观的衔接要求

### 2. 生成系统方案

根据需求激活 `Skills(game-system)` 并生成：

#### 游戏框架
- 游戏类型和核心玩法定位
- 世界观背景和基调
- 核心循环（战斗-成长-探索）
- 与现实世界的关联设定

#### 职业设计
- 完整的职业分类和定位
- 每个职业的核心机制和特色
- 转职/觉醒路线设计
- 职业间的配合与克制关系

#### 等级与数值
- 等级上限和经验曲线
- 属性成长和分配方式
- 战力评估体系
- 突破/转生等进阶机制

#### 经济体系
- 货币种类和汇率
- 产出和消耗的平衡
- 交易系统设计
- 防通胀和金币沉淀机制

### 3. 一致性验证

检查系统的合理性：
- 职业、等级、数值三者是否平衡
- 经济系统是否能自循环
- 是否为主角的"外挂"留足空间
- 系统是否支撑足够的剧情冲突

### 4. 输出设定文档

以结构化格式输出完整设定，便于后续查阅和扩展。

## 设计原则

- **可玩性**：系统设计要像真正的游戏一样有深度和乐趣
- **合理外挂**：主角的系统优势要有合理的解释
- **版本驱动**：通过游戏更新创造新的剧情契机
- **社交核心**：游戏系统要促进角色间的互动和冲突
