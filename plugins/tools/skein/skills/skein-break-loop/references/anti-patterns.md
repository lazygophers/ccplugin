# skein-break-loop 反例目录

check 反复不过、进入根因复盘时的禁止行为与纠正动作。

## ⛔ 反例

| 禁 | 改为 |
|---|---|
| 第 3 轮继续同层盲改 | STOP 定点循环, 先跨维度定位 root cause |
| 只查实现层 (③) 就下结论 | 5 维度全扫, 表层多轮不过强制上溯 ①② |
| 根因无证据凭空断言 | 每维度给 file:line + 报错原文 + 归因理由 |
| 需求/设计错却硬留 exec 重修 | ①② 超 exec 范围 → STOP 转人工退回 planning |
| 复盘完自动写规则 | 走 skein-memory sediment 判定门 (判定→审批→写盘) |
| 一次性 bug 硬凑成沉淀契约 | 过「排除」项, 一次性 → drop 跳过 |
| 新造一套沉淀机制 | 复用 memory.py sediment, 禁另起炉灶 |
| STOP 只丢一句"过不了" | 附根因报告 (维度 + 证据 + 退回建议) 交用户裁 |
