# Issue Guidelines

## Title Format

Use:

```text
[<type>] <subject>
```

Types:

- `[bug]`: Bug report
- `[feat]`: Feature request
- `[docs]`: Documentation issue
- `[question]`: Question
- `[discussion]`: Discussion

## Labels

Priority:

- `P0`: urgent, handle immediately
- `P1`: high priority
- `P2`: medium priority
- `P3`: low priority

Type:

- `bug`: bug report
- `enhancement`: feature improvement
- `documentation`: docs-related
- `help wanted`: needs external help
- `good first issue`: suitable for new contributors

Status:

- `status: in progress`: work in progress
- `status: blocked`: blocked
- `status: review needed`: needs review

## Creation Checklist

- [ ] Existing issues searched.
- [ ] Issue type selected.
- [ ] Bug report includes description, reproduction steps, expected behavior, actual behavior, and environment.
- [ ] Feature request includes feature description, problem background, proposed solution, and alternatives.
- [ ] Title is concise and under 100 characters.
- [ ] Screenshots or logs are included when useful.
- [ ] Labels, milestone, and assignee are added only when known.

## Lifecycle

1. Create issue with template.
2. Add labels and milestone.
3. Assign owner when clear.
4. Link implementation PR.
5. Update status labels as work progresses.
6. Close with reason after resolution or rejection.
