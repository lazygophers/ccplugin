# Subtask 五要素

任何 subtask (含 implement.md checklist 项 / dispatch 派发单 / parent 拆 child 时的工作描述) 必须填全五要素, 缺一不派 / 不执行 / 不写。

## 五要素

| 要素 | 一句话定义 | 必须有 | 反例 |
| --- | --- | --- | --- |
| **目标** | 完成什么 (动词 + 对象 + 结果) | 可证伪的结果对象 | "优化登录" / "完善 UI" |
| **产出** | 交付什么 | 具体物 (diff/报告/输出/截图) | "代码改完" / "测试通过" |
| **验证** | 如何判定完成 | 命令优先, 次选人工对比 | "看一下" / "感觉对" |
| **资源** | 独占什么 | 文件 glob / 端口 / 配置 / 槽位 | "代码库" (太宽) |
| **依赖** | 前置是什么 | 显式 subtask id / 文件 / 无 | "前面那个" |

## 目标写法

| 不行 | 行 |
| --- | --- |
| 优化 X 性能 | X 接口 p95 ≤ 200ms (现 800ms), QPS ≥ 500 |
| 修复登录 bug | 当 token 过期返回 401 而非 500, 测试 `tests/auth/test_expiry.py::test_expired` 通过 |
| 完成 UI | 新页面 `/dashboard` 加载完成时间 ≤ 2s, 含 N/M/O 三个组件, screenshot 与 design.figma 第 X 帧一致 |

## 产出格式

| 任务类型 | 产出 |
| --- | --- |
| 代码改动 | 具体文件路径 + 函数名 / diff 范围 |
| 调研 / 分析 | 报告路径 + 章节清单 (结论 + 证据 + 来源) |
| 配置 / 脚本 | 改动后文件 + 验证命令输出 |
| UI / 视觉 | 截图 / 录屏 (路径) |
| 工具执行 | 命令输出 / 退出码 / 日志摘要 (路径) |

## 验证可执行优先

```
不行: "测试通过"
行: pytest tests/auth/ -k expiry → 退出码 0 且 captured stdout 含 "test_expired PASSED"
```

```
不行: "lint 干净"
行: ruff check src/auth/ → 退出码 0
```

```
不行: "看起来对"
行: 对比 screenshots/before.png 与 screenshots/after.png; 按钮位置差 ≤ 2px (用 pixelmatch < 10 像素)
```

## 资源标注

资源类型与互斥规则见 `shared-resources.md`。subtask 行简洁: `资源: packages/api/src/auth/**, .env.test, port 5432`。

并行检查:
- 两 subtask 资源集合**不交**且**不依赖**同一前置 → 可并行
- 任一相交 → 串行 (用依赖标注明顺序)

## 依赖标注

| 形式 | 含义 |
| --- | --- |
| `依赖: 无` | 可立即启动 |
| `依赖: S1` | 等 S1 done |
| `依赖: S1, S2` | 等 S1 + S2 都 done |
| `依赖: S1.<产出文件>` | 等 S1 产出特定文件存在 (细粒度) |

禁: "依赖: 前面的", "依赖: 后端先做"。

## 自检

- [ ] 目标含可证伪结果对象
- [ ] 产出有具体路径 / 命令 / 截图位置
- [ ] 验证可执行 (复制粘贴能跑)
- [ ] 资源覆盖所有写入点
- [ ] 依赖用 ID / 文件指代, 无模糊词
