# axis-deep-dive — 12 骨架轴深挖问题清单 + 历史盲点实例

> 主 skill `trellisx-grill` 第 2 步逐轴 grill 时的深挖弹药库。每轴给 3-5 个追问 + 一条项目实证盲点。**按需查, 非每轴每问必过** —— 动态裁剪见主 skill §可扩展骨架轴。

## A 目标

- 工件解决的核心问题一句话能说清吗? 说不清 = 目标未封闭。
- deliverable 是名词 (可交付物) 还是动词 (做某事)? 动词式 ("实现登录") 无验收锚点。
- 若目标达成, 怎么证明? 拿不出证据 = 目标虚。
- **盲点**: 开放式 PRD "实现 X 功能" 无 deliverable 矩阵, planning 全程发散。

## B 产出

- 交付物清单化了吗? 每项可独立验收吗?
- 产出格式指定了吗 (文件/路径/schema)? 模糊格式 → 下游没法对接。
- "完成" 的定义是什么? 谁判定完成?
- **盲点**: 交付物写 "优化性能", 无基线数字 + 目标数字 = 无法验收。

## C 验证

- 每条产出有可执行断言吗 (grep / test / task.py / 退出码)?
- 铁律要求验证, 正文却把断言落 reference → 易被跳过。断言必须在主文可直接跑。
- 验证覆盖 happy path + 边界 + 失败路径三类吗?
- **盲点**: 声称 "行为闭环" 但通篇无一条可执行命令, 验证形同虚设。

## D 资源

- 改哪些文件? 列全了吗?
- 多 agent / 多 subtask 的文件集有重叠吗? 重叠 = 互相覆盖风险。
- 谁独占写? 并行还是串行? 顺序敏感步骤串行化了吗?
- **盲点**: 并行组文件集重叠未串行化 → 后写覆盖先写, 静默丢改动。

## E 依赖

- 前后序画清了吗 (依赖箭头)? 并行组标依赖了吗?
- 有共享文件 / 共享环境 / 阻塞点吗?
- 循环依赖有吗? A 等 B, B 等 A = 死锁。
- **盲点**: 并行组不标依赖箭头 → 乱序执行, 前置未就绪就跑。

## F 失败模式 (dim3)

- 每步失败怎么办? 有 if-then 三段式 (触发 / 一线修复 / 仍失败兜底)?
- 只写正向流程 = dim3 硬伤。每个 STOP 点有兜底吗?
- 失败可重入吗? 半成功状态怎么恢复?
- **盲点**: 只写 "跑 X 生成 Y", 不写 "X 失败 / Y 不存在 / X 部分成功" 分支。

## G 检查点 (dim4)

- 关键决策前有 🔴 / STOP / CHECKPOINT 视觉标记吗?
- 靠 "必须 / 应该" 措辞代替标记 → LLM 扫标记优先于语义, 会漏。
- 检查点是真停 (AskUserQuestion) 还是假停 (纯文本问)?
- **盲点**: "建议先确认" 式软措辞不构成检查点, model 直接冲过。

## H 触发准确性

- should-trigger 词列了吗? should-not-trigger 对列了吗?
- 触发词与邻居 skill 重叠吗? 重叠 → 误抢路由。
- disable-model-invocation / user-invocable 设置对吗?
- **盲点**: 缺 should-not-trigger 对, 触发词与邻居撞车 → 该调 A 却调了 B。

## I token 生命周期

- 工件 + references 总行数多少? auto-trigger 还是手动?
- auto-trigger skill 堆 references → 多 skill session 旧 skill 被静默踢出, 无错误信息。
- references 是懒加载还是全量注入? 全量 → context 爆。
- **盲点 (最易漏, 杀伤最大)**: auto-trigger + 大 references, 零可见度故障。

## J 自举 / 矛盾

- 工件规则适用于自身吗 (自举一致)?
- 同名 skill + agent 走两条互斥路径吗? → routing 死结。
- 上下级规则 (root CLAUDE.md vs 本 skill) 打架吗?
- **盲点 (最易漏)**: skill 说走 A, 配套 agent 说走非 A → 每次 routing 都撞。

## K 诚实边界

- 局限 / 降级 / dry_run 显式标注了吗?
- 有摘樱桃吗 (用 "诚实" 框架选择性呈现: 弃用论据但保结论)?
- 声称的能力有实证还是拍脑袋?
- **盲点**: "已充分测试" 无 test-prompts, dim8 凭空打分。

## L 反例黑名单 (dim9)

- 有 "不要做什么" 清单吗? 只写应做 = dim9 缺失。
- 反例是抽象口号还是具体可命中的反模式?
- 红灯 / 危险动作单独成章了吗?
- **盲点**: 通篇 "应该做 X", 无一条 "禁做 Y", LLM 无负向约束。

## 调度类工件加轴 (task-tree / scheduling / config hook)

- **调度正确性**: 拓扑序对吗? 资源上限 (并发数) 设了吗? 饿死 / 优先级反转有吗?
- **hook 副作用**: hook 幂等吗? 失败阻塞主流程吗? 触发时机 (Pre/Post) 对吗?
