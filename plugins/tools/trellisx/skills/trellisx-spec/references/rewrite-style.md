# 命令式重写范本

spec 条款必须是命令式 + 可机器验证。这是写新条款 / REWRITE 现有条款的强制范本。

## 命令式词汇表

| 用 | 不用 |
| --- | --- |
| MUST / MUST NOT | should / shouldn't / would |
| 禁 / 严禁 | 不应 / 最好别 / 尽量别 |
| 必须 / 必填 / 必带 | 建议 / 通常 / 推荐 |
| ≥ N / ≤ N / = N | 大概 N / 差不多 N |
| 立即 / 同步 | 尽快 / 适时 |
| 必走 X / 必经 Y | 一般走 X / 通常经 Y |

## 条款结构

```
- <主体> <命令式动词> <对象> <(可选) 验证>
```

示例:

```
- 改 ≥ 2 层 (e.g. controller + service) 必须先列每层契约边界, 缺一不改
- 跨层调用必须走显式接口, 禁直接读对方实现字段
- 验证: `grep -r '<对方实现字段>' src/ | grep -v <允许目录>` 必须 0 行
```

## 旧 → 新对照表

### 例 1: 跨层

旧 (描述式):
```
建议在改动多层架构时, 通常应该考虑跨层影响, 以避免引入耦合问题。
```

新 (命令式):
```
- 改 ≥ 2 层 (e.g. controller + service) 必须先列每层契约边界, 缺一不改
- 跨层调用必须走显式接口, 禁直接读对方实现字段
- 验证: `grep -rE '<对方实现字段>' src/ | grep -v <允许目录>` 必须 0 行
```

### 例 2: 代码重用

旧:
```
重用代码前应该先调研项目中是否已有类似实现, 尽量避免重复造轮子。
```

新:
```
- 写新函数前必须 grep 项目同名 / 同语义函数; 命中则用旧, 禁新写
- grep 模式: `<具体词列表>` 在 `src/**` `packages/**`
- 若旧实现 ≥ 3 调用站点, 必须改造旧实现而非旁路新写
```

### 例 3: 测试

旧:
```
测试通常需要覆盖主要路径, 边界条件也最好测一下。
```

新:
```
- 任何新公共函数必须含 ≥ 3 测试: 主路径 / 边界值 / 错误处理
- 边界值必含: 空输入 / 单元素 / 最大值
- 验证: `pytest tests/<module>/ --cov=<module> --cov-fail-under=80` 退出码 0
```

### 例 4: 性能

旧:
```
接口性能应该尽量优化, 减少不必要的开销。
```

新:
```
- 公共接口 p95 必须 ≤ 200ms (现状基线 <X>ms, 由 `<basename>` 跑)
- 单实例 QPS ≥ 500 (压测: `wrk -t4 -c100 -d30s <url>`)
- 不达标 PR 禁合, 必须先优化或调整 SLA
```

## 验证条款写法

可机器验证 > 可执行命令 > 文件 / 行存在 > 输出对比 > 人工对比。

| 类型 | 写法 |
| --- | --- |
| 命令退出码 | `<cmd>` 退出码必须 0 |
| 命令 stdout 含 | `<cmd>` stdout 必须含 "<str>" |
| 命令 stdout 不含 | `<cmd>` stdout 必须不含 "<str>" |
| 文件存在 | `<path>` 必须存在 |
| grep 0 行 | `grep -r '<pat>' <dir>` 必须 0 行 |
| 行数比例 | `grep <pat>` 行数 / 总行数 ≥ N% |
| schema 校验 | `jq <expr> <file>` 退出码 0 |

## 规则结构 (整文件层)

每个 spec 文件首段 ≤ 3 行说明:

```
何时被读: <场景 1> / <场景 2>
谁读: <main / sub-agent / agent-team / workflow>
不遵守的代价: <具体后果>
```

例:

```
何时被读: sub-agent dispatch 改跨层代码时
谁读: trellis-implement sub-agent
不遵守的代价: 跨层耦合泄漏, 下次跨层改动连锁失败, 测试无法独立
```

## 反例 (描述性垃圾)

不可出现:
- "应该考虑 X"
- "尽量做到 Y"
- "通常情况下..."
- "在大多数情况下..."
- 长段叙述无 bullet / 无验证
- 空泛的"提高质量" / "优化体验"

## 自检 grep

```bash
# 描述式残留必须 0
grep -nE '建议|可以|通常|尽量|应该|考虑|大概|差不多|尽快|适时' .trellis/spec/

# 命令式比例
total=$(grep -cE '^[-*]' .trellis/spec/**/*.md | awk -F: '{s+=$2} END {print s}')
must=$(grep -cE 'MUST|MUST NOT|禁|必须|严禁|必填|必带|必走|必经' .trellis/spec/**/*.md | awk -F: '{s+=$2} END {print s}')
echo "命令式比例: $(( must * 100 / total ))%  (达标 ≥ 60%)"
```
