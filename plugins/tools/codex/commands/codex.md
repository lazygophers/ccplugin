请按以下流程执行：

1. 先完成任务理解与拆分，且使用 Agents(codex:codex) 来执行具体的任务
2. 严格拆分任务、确保任务
	1. 相互无依赖
	2. 并行任务不允许同时处理同一个模块
	3. 最小化、独立、可执行
	4. 每个子任务都有明确的：目标、约束、验收标准。
	5. 如果对于任务有问题或拆分任务的是出现问题，请通过 `AskUserQuestion` 询问我
	6. 对于不明确的需求，或需要持续优化的任务清单的任务，可以先拆分第一阶段的任务，然后再根据结果逐步拆分，完善
	7. 允许运行过程中增加、删除、更新任务清单，当时需要先通过 `AskUserQuestion` 和我确认
	8. 不要过度设计，不要扩大任务，不要做要求之外的事情
3. 使用 `AskUserQuestion` 向用户确认执行流程
	1. 需要有明确的任务依赖和执行顺序的展示(不要使用 `mermaid`)
	2. 每个任务都需要有简要的描述
	3. 任务需要更新单任务清单
4. 对每个 Codex 执行任务，都交给单独的 Agents(codex) 执行
	1. 并行任务最多 5 个，如果超过需要等待某一个完成后才可以继续下一个
	2. 有依赖时严格串行
	3. 如果某一个任务执行失败或不符合预期，需要整体重新规划，并会到步骤 2 重新开始

**注意**

1. 严禁模糊不清直接处理任务，必须是明确的，可量化的
2. 只允许 Agents(codex:codex) 一个 agent 调度
3. 对于 `codex` 的调用，只允许使用 Agents(codex:codex)
