# Skills(task:deep-iteration) - 深度迭代规范

## 概述

深度迭代是 MindFlow 的核心质量保障机制。与普通迭代（完成功能即止）不同，深度迭代通过多轮递进式研究和优化，确保最终结果完全符合预期。

| 维度 | 普通迭代 | 深度迭代 |
|------|---------|---------|
| 目标 | 完成功能 | 完美满足预期 |
| 策略 | 失败重试 | 质量递进提升 |
| 验收 | 功能通过 | 质量达标+最佳实践 |
| 研究 | 浅层 | 深度(论文/案例/专家) |
| 终止 | 功能完成 | 用户完全满意 |

## 四个核心机制

1. **深度理解**：deepresearch agent查论文/博客/开源 → 对比3-5方案 → 确定最佳实践 → 量化标准
2. **质量递进**：按迭代轮次递增阈值（详见 [flows/verify.md](../loop/flows/verify.md) §质量评分），质量门控7维度
3. **深度分析**：失败时5 Why根因分析 → 查类似案例 → 对比修复方案 → 最优策略+预防
4. **持续改进**：通过验收后仍对比最佳实践 → 识别高价值优化点 → 自动触发新迭代

## 触发条件（任一）

第1轮迭代 | 高复杂度(>60) | 失败≥2次 | 质量<阈值-10 | 用户要求

## 终止条件（全部满足）

所有验收标准通过 + 质量≥阈值 + 遵循最佳实践 + 用户确认 + 达最小迭代

## 复杂度评估

4维度(各25分)：文件变更数 + 技术栈新颖度 + 任务类型 + 质量要求

| 分级 | 分数 | 最小迭代 |
|------|------|---------|
| Simple | 0-30 | 1轮 |
| Moderate | 31-60 | 2轮 |
| Complex | 61-100 | 3轮 |

## Loop 集成

深度迭代增强 Loop 各阶段行为，非独立模块：

| 阶段 | 增强行为 |
|------|---------|
| 初始化 | 评估复杂度 → 配置 min_iterations + quality_threshold |
| 深度研究 | 触发条件满足时在计划前执行 |
| 计划设计 | 研究发现注入上下文 + 推荐方案优先 |
| 结果验证 | 质量门控(分数≥阈值) + 持续改进(高价值优化点) |
| 失败调整 | 失败2次→5 Why深度根因分析→应用推荐修复 |

## 最佳实践

**必须**：首轮启用研究、每轮提高阈值、失败时分析根因、通过后仍检查优化空间
**禁止**：浅尝辄止、重复低质量迭代、跳过深度研究、过早终止

详细实现 → [implementation.md](implementation.md)
