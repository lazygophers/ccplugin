# 深度迭代详细说明

## 深度迭代 vs 普通迭代

| 维度 | 普通迭代 | 深度迭代 |
|------|---------|---------|
| 目标 | 完成功能 | 完美满足预期 |
| 策略 | 失败重试 | 质量递进提升 |
| 验收 | 功能通过 | 质量达标+最佳实践 |
| 研究 | 浅层 | 深度(论文/案例/专家) |
| 终止 | 功能完成 | 用户完全满意 |

## 四阶段流程

1. **深度理解**：调用deepresearch查找最新方案→对比3-5种→确定最佳实践→量化标准。将研究结果融入planner prompt。
2. **质量递进**：阈值60→75→85→90(保持)。质量门控7维度(functionality/test_coverage/code_quality/performance/maintainability/security/best_practices)，低于阈值→needs_improvement。
3. **深度分析**：失败时调用deepresearch执行5Why分析→查找类似案例→对比修复方案→推荐最优策略。结果注入adjuster。
4. **持续改进**：passed后仍检查优化空间→高价值优化询问用户(纳入/记录技术债/完成)。

## 终止条件（全部满足）

功能完整 + 质量≥阈值 + 遵循最佳实践 + 用户满意 + 达到最小迭代

## 触发深度研究条件（任一）

高复杂度 | 失败≥2次 | 质量<阈值-10 | 用户要求 | 无现成方案

## 最佳实践

**Do**：首轮启用研究、每轮提高阈值、失败深度分析、通过后仍检查优化、记录到memory
**Don't**：浅尝辄止、重复低质量迭代、跳过研究、忽视最佳实践、过早终止
