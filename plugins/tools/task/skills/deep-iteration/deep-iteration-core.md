# Skills(task:deep-iteration) - 深度迭代核心

通过多次迭代深度研究找到最优方案。详细流程见[deep-iteration-details.md](deep-iteration-details.md)。

## 与普通迭代对比

| 维度 | 普通 | 深度 |
|------|------|------|
| 目标 | 完成功能 | 完美满足预期 |
| 策略 | 失败重试 | 质量递进提升 |
| 验收 | 功能通过 | 质量+最佳实践 |
| 研究 | 浅层 | 论文/案例/专家 |
| 终止 | 功能完成 | 用户满意 |

## 核心机制

1. **深度理解**：deepresearch agent查论文/博客/开源 → 对比3-5方案 → 确定最佳实践
2. **质量递进**：迭代1→60分(Foundation) | 迭代2→75分(Enhancement) | 迭代3→85分(Refinement) | 迭代4+→90分(Excellence)
3. **深度分析**：失败时5 Why根因分析 → 查类似案例 → 对比修复方案 → 最优策略+预防
4. **持续改进**：通过验收后仍对比最佳实践 → 识别优化点 → 自动触发新迭代

## 触发条件

第1轮迭代 | 高复杂度 | 失败2次 | 质量<阈值-10 | 用户要求

## 终止条件（全部满足）

所有标准通过 + 质量达标 + 遵循最佳实践 + 用户确认 + 达最小迭代次数

## 复杂度评估

4维度(各25分)：文件变更数 + 技术栈新颖度 + 任务类型 + 质量要求

| 分级 | 分数 | 最小迭代 |
|------|------|---------|
| Simple | 0-30 | 1轮 |
| Moderate | 31-60 | 2轮 |
| Complex | 61-100 | 3轮 |
