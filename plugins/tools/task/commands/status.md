# 查看任务状态

实时查看当前 loop 的执行状态和进度。

## ✋ 前置条件

**必须有正在执行的 task:loop**，否则无法使用 `/status`：

- ✅ **可用场景**：Loop 执行过程中（步骤 1-6 任意阶段）
- ✅ **可用场景**：Loop 已暂停（状态为 `paused`）
- ❌ **不可用**：无活跃或暂停的 Loop 任务时

**检查方式**：TaskList 查询状态为 `in_progress` 或 `paused` 且类型为 `loop` 的主任务

## 📋 快速参考

| 信息类别 | 包含内容 | 数据来源 |
|---------|---------|---------|
| 📊 主任务概览 | 任务标题、当前步骤、迭代次数 | TaskGet(main_task) |
| 📦 子任务进度 | 状态、进度百分比、依赖关系 | TaskList + TaskGet |
| 👥 Team 状态 | Team ID、活跃 Agents 数量 | TaskGet(metadata.team_id) |
| 🔄 活跃 Agents | Agent 名称、执行任务、运行时长 | 后台任务列表 |
| 📈 迭代统计 | 当前迭代、停滞次数、用户指导次数 | TaskGet(metadata) |

**核心原则**：只读操作，不影响任务执行

## ⚠️ 核心原则

1. **只读查询**：不修改任何任务状态，不影响执行流程
2. **实时性**：显示当前最新的任务状态和进度
3. **层次清晰**：从主任务 → 子任务 → Agents 逐层展示
4. **快速扫描**：使用表格和符号，便于快速理解
5. **可追溯**：显示历史迭代统计和 Anchor 版本

## 查询流程

### 1️⃣ 获取主任务信息

**工具**：TaskList, TaskGet

**操作**：
1. TaskList 查询 `in_progress` 或 `paused` 且类型为 `loop` 的主任务
2. TaskGet 获取主任务详细信息
3. 提取关键字段：
   - 任务标题和描述
   - 当前步骤（1-6）
   - 当前迭代次数
   - 停滞次数
   - 用户指导次数
   - Team ID
   - Anchor 版本

### 2️⃣ 获取子任务状态

**工具**：TaskList, TaskGet

**操作**：
1. TaskList 查询所有子任务
2. 按状态分类统计：
   - `completed` - 已完成 ✅
   - `in_progress` - 执行中 🔄
   - `failed` - 失败 ❌
   - `pending` - 待执行 ⏸️
   - `blocked` - 阻塞中 🚫
   - `skipped` - 已跳过 ⏭️
3. 提取依赖关系（blockedBy）

### 3️⃣ 获取 Team 和 Agents 状态

**工具**：TaskGet, TaskList（后台任务）

**操作**：
1. 检查 Team 是否存在（metadata.team_id）
2. 如果存在 Team：
   - 查询活跃的 Agents 数量
   - 获取每个 Agent 的：
     * Agent 名称
     * 执行的任务 ID
     * 运行时长
     * 当前状态

## 适用场景

| 场景 | 说明 |
|------|------|
| 📊 **日常监控** | 定期查看任务进度，了解执行情况 |
| 🔍 **调试排查** | 任务失败或异常时，查看详细状态 |
| 📝 **进度汇报** | 向团队或用户汇报任务进展 |
| ⏸️  **暂停后查看** | 暂停后查看当前状态，决定是否恢复 |
| 🤔 **决策辅助** | 根据当前状态决定是否使用 /add、/pause、/cancel |

## 注意事项

- **频率控制**：建议间隔 1 分钟以上再次查询，避免频繁调用
- **执行中查询**：查询不会中断任务执行，可以随时查看
- **状态延迟**：Agent 状态可能有 1-2 秒延迟
- **暂停状态**：暂停状态下，Team 和 Agents 仍然保留，但不执行任务
