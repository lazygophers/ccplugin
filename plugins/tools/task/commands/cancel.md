# 取消当前任务

立即终止当前正在执行的 loop 流程。

## ✋ 前置条件

**必须有正在执行的 task:loop**，否则无法使用 `/cancel`：

- ✅ **可用场景**：Loop 执行过程中（步骤 1-6 任意阶段）
- ❌ **不可用**：无活跃 Loop 任务时

**检查方式**：TaskList 查询状态为 `in_progress` 且类型为 `loop` 的主任务

## 📋 快速参考

| 步骤 | 核心操作 | 工具 | 输出 |
|------|---------|------|------|
| 1️⃣ 停止执行 | 终止所有 Agents | TaskStop | - |
| 2️⃣ 状态保存 | 标记任务状态 | TaskUpdate | completed/cancelled/pending |
| 3️⃣ 清理团队 | 删除团队资源 | TeamDelete | - |
| 4️⃣ 输出报告 | 汇总取消结果 | - | 已完成/已取消/未开始 |

**核心原则**：安全取消，保留已完成工作，不回退代码变更

## ⚠️ 核心原则

1. **安全取消**：不丢弃已完成的工作
2. **状态保留**：任务状态记录在 TaskList 中
3. **代码保护**：已提交的代码变更不回退
4. **清理顺序**：TaskStop → TaskUpdate → TeamDelete
5. **可追溯**：记录取消时间和原因

## 取消流程

### 1️⃣ 停止执行

**工具**：TaskStop

**操作**：
- 终止所有正在运行的 Agents
- 中断当前迭代循环
- 不再调度新的子任务

### 2️⃣ 状态保存

**工具**：TaskUpdate

**状态分类**：
- **已完成任务** → 保持 `completed` 状态
- **执行中任务** → 标记为 `cancelled`（记录时间戳和原因）
- **待执行任务** → 保持 `pending` 状态

**原则**：取消不等于丢弃，保留已完成的工作

### 3️⃣ 清理团队

**前置条件**：✓ 已执行 TaskStop（由步骤1完成）

**工具**：TeamDelete

**操作**：删除团队资源（如果存在 team）

**⚠️ 清理顺序**：TaskStop（步骤1）→ TaskUpdate（步骤2）→ TeamDelete（步骤3）

### 4️⃣ 输出取消报告

**输出格式**：
```
⚠️ [取消] 任务已终止

✅ 已完成：T1, T2
❌ 已取消：T3（执行中被终止）
⏸️  未开始：T4, T5

💡 可通过 /loop 重新开始
```

## 适用场景

- 用户主动中断（Ctrl+C 或明确取消指令）
- 发现任务规划错误，需要重新规划
- 遇到无法解决的阻塞，需要用户介入

## 不适用场景

- 任务即将完成（建议等待完成）
- 只需要调整部分内容（使用 `/add` 补充说明）
- 需要修复失败任务（让 Loop 自动迭代修复）
