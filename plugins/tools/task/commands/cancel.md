# 取消当前任务

立即终止当前正在执行的 Loop 或 Exec 流程。

## 取消流程

### 1. 停止执行

- 终止所有正在运行的 Agent（通过 TaskStop）
- 中断当前迭代循环
- 不再调度新的子任务

### 2. 状态保存

取消不等于丢弃，保留已完成的工作：
- 已完成的子任务状态保持 `completed`
- 执行中的子任务标记为 `cancelled`
- 待执行的子任务保持 `pending`
- 通过 `TaskUpdate` 更新所有任务状态

### 3. 输出取消报告

```
[取消] 任务已终止

已完成：T1, T2
已取消：T3（执行中被终止）
未开始：T4, T5

可通过 /exec 恢复执行未完成的任务
```

### 4. 恢复选项

取消后用户可以：
- `/exec` — 恢复执行剩余的子任务
- `/plan` — 重新规划未完成的部分
- `/loop` — 用新的目标重新开始迭代

## 注意事项

- 取消操作是安全的，不会丢弃已完成的工作
- 已提交的代码变更不会被回退
- 任务状态保留在 TaskList 中，支持后续恢复
