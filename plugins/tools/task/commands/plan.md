# 任务规划建议

生成任务分解和执行方案，但**不执行**，仅提供规划建议作为决策参考。

## ✋ 前置条件

**无前置条件**：可以在任何时候使用 `/plan`

- ✅ **可用场景**：任何时候（无需活跃的 loop）
- ✅ **典型用途**：决策前评估、方案对比、复杂度估算

## 📋 快速参考

| 步骤 | 核心操作 | 工具 | 输出 |
|------|---------|------|------|
| 1️⃣ 信息收集 | 收集项目信息 | Read/Glob/Grep/Agent(Explore) | 项目上下文 |
| 2️⃣ 任务分解 | 生成分解方案 | 分析+设计 | 任务列表+DAG |
| 3️⃣ 估算评估 | 预估时间和风险 | 分析 | 时间估算+风险点 |
| 4️⃣ 输出建议 | 格式化输出 | - | Markdown规划方案 |

**核心原则**：只建议不执行，提供决策参考，不创建任务

## ⚠️ 核心原则

1. **只建议不执行**：生成规划方案，不创建任务，不执行代码
2. **决策参考**：帮助用户评估可行性、复杂度、工作量
3. **可对比**：可多次执行，对比不同方案
4. **无副作用**：不修改项目文件，不创建任务记录
5. **快速迭代**：基于用户反馈快速调整方案

## 📊 与 loop 的区别

| 对比项 | plan | loop |
|--------|------|------|
| 执行任务 | ❌ 不执行 | ✅ 执行 |
| 创建任务 | ❌ 不创建 TaskCreate | ✅ 创建任务 |
| 修改文件 | ❌ 不修改 | ✅ 修改代码 |
| 创建 Team | ❌ 不创建 | ✅ 创建（如需要） |
| 输出 | 规划建议（Markdown） | 任务执行结果 |
| 用途 | 决策参考 | 实际执行 |
| 可重复 | ✅ 可多次执行对比方案 | ⚠️  需谨慎（会修改代码） |

## 执行流程

### 1️⃣ 信息收集

**目标**：收集足够的项目信息以支持任务规划。

**执行方式**：与 loop 的步骤 1 相同

**操作**：
- 使用 Read/Glob/Grep 读取项目文件、文档、CLAUDE.md
- 使用 Agent(Explore) 深度分析代码结构
- 收集：目标、约束、上下文、文件路径、测试覆盖

### 2️⃣ 任务分解

**目标**：将任务分解为原子子任务，建立依赖关系，但**不创建任务**。

**执行方式**：与 loop 的步骤 2 类似，但不调用 TaskCreate

**核心原则**：MECE、可交付原子化、可量化可验证、依赖闭环

**输出内容**：
- 任务列表（包含 ID、描述、agent、skills、files、依赖）
- DAG 执行流程图
- 并行策略（哪些任务可并行）

### 3️⃣ 估算评估

**目标**：评估任务复杂度、预估时间、识别风险。

**评估维度**：
- **时间估算**：每个子任务的预估时间
- **总时长**：考虑并行后的总时长
- **复杂度**：技术难度评估（简单/中等/困难）
- **风险点**：依赖风险、技术风险、时间风险
- **关键路径**：识别影响总时长的关键任务

## 注意事项

- **不创建任务**：plan 不会调用 TaskCreate，不会在 TaskList 中留下记录
- **不执行代码**：plan 不会修改任何项目文件
- **可重复执行**：可以多次执行 plan，对比不同的方案
- **估算误差**：时间估算仅供参考，实际执行可能有 ±20% 误差
- **需要执行**：如果满意方案，需要手动执行 `/loop` 来实际执行任务
- **信息收集**：plan 会读取项目文件以生成更准确的建议
- **基于上下文**：plan 的建议质量取决于项目信息的完整性
