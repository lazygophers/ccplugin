# 审查任务完成质量

使用 `reviewer` Agent 对已完成的任务进行全面审查。

## 审查流程

### 1. 确定审查范围

- 指定任务名称：审查特定子任务
- "全部"：审查所有已完成的子任务

### 2. 执行审查

对每个待审查的子任务：

**功能验证**
- 运行所有相关测试
- 验证输出是否符合预期
- 检查边界条件处理

**代码质量**
- 运行 lint/format 工具
- 检查安全性（OWASP Top 10）
- 检查性能隐患
- 检查代码规范遵循度

**验收标准**
- 逐项检查每个验收条件
- 标注通过/未通过及原因

### 3. 输出审查报告

```markdown
# 审查报告

## 总体：[通过/有条件通过/不通过]

### T1: [标题] - 通过
- [x] 验收标准 1
- [x] 验收标准 2
- 代码质量：良好

### T2: [标题] - 不通过
- [x] 验收标准 1
- [ ] 验收标准 2：[失败原因]
- 代码质量：需修复
- 问题：[具体问题描述，附文件路径和行号]

## 必须修复
1. [问题描述]

## 建议改进
1. [改进建议]
```

### 4. 后续行动

- **通过**：输出完成总结，清理 TaskList 中的已完成条目
- **有条件通过**：列出建议改进项，由用户决定是否修复
- **不通过**：列出必须修复项，建议使用 `/loop` 迭代修复

### 5. 清理

审查通过后自动执行清理：
- 清理 TaskList 中已完成的任务条目
- 确保不留残余的临时状态
