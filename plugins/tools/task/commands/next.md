# 建议下一步行动

基于项目当前状态，智能推荐下一步应该做什么，帮助快速找到最重要的工作。

## ✋ 前置条件

**无前置条件**：可以在任何时候使用 `/next`

- ✅ **可用场景**：任何时候（无需活跃的 loop）
- ✅ **典型用途**：不知道接下来做什么，需要快速找到重点工作

## 📋 快速参考

| 步骤 | 核心操作 | 分析维度 | 输出 |
|------|---------|---------|------|
| 1️⃣ 项目扫描 | 分析项目状态 | 未完成工作、技术债、测试覆盖 | 项目健康度 |
| 2️⃣ 优先级排序 | 应用优先级矩阵 | 紧急度×重要度 | 排序列表 |
| 3️⃣ 推荐行动 | 生成 Top 3-5 建议 | 可执行性、影响力 | 行动建议 |
| 4️⃣ 输出建议 | 格式化输出 | - | Markdown 建议列表 |

**核心原则**：智能分析，优先级排序，快速决策

## ⚠️ 核心原则

1. **数据驱动**：基于项目实际状态（代码、测试、文档）分析
2. **优先级明确**：使用紧急/重要矩阵排序
3. **可执行**：每个建议都提供具体的执行命令
4. **聚焦重点**：只推荐 Top 3-5，避免选择困难
5. **上下文感知**：考虑项目阶段、团队状态、技术栈

## 分析维度

### 📊 项目健康度评估

| 维度 | 检查项 | 数据来源 |
|------|--------|---------|
| **代码质量** | 覆盖率、复杂度、重复代码 | 测试报告、lint 结果 |
| **技术债** | TODO、FIXME、已知问题 | Grep 搜索、Issue 列表 |
| **文档完整性** | README、API 文档、注释 | 文档文件扫描 |
| **安全性** | 依赖漏洞、敏感信息泄露 | 依赖扫描、代码审查 |
| **性能** | 性能测试、瓶颈分析 | 性能报告 |

### 🎯 优先级矩阵（紧急×重要）

```
高重要度 │ 🔴 修复安全漏洞        │ 🟡 架构重构
        │ 🔴 修复生产 Bug        │ 🟡 补充文档
────────┼───────────────────────┼─────────────────
低重要度 │ 🟢 代码格式化          │ 🟢 优化注释
        │ 🟢 更新依赖版本        │ 🟢 重命名变量
        └───────────────────────┴─────────────────
          高紧急度               低紧急度
```

**优先级判断**：
- 🔴 **高优先级**（紧急+重要）：立即执行，影响核心功能或安全
- 🟡 **中优先级**（不紧急但重要）：计划内执行，影响长期质量
- 🟢 **低优先级**（紧急或重要其一）：有时间再做，影响较小

## 执行流程

### 1️⃣ 项目扫描

**工具**：Read, Glob, Grep, Bash

**扫描内容**：

1. **未完成的工作**
   - Grep 搜索 `TODO`、`FIXME`、`HACK`、`XXX`
   - 统计数量和分布

2. **代码质量**
   - 运行测试（如果存在）
   - 检查覆盖率报告
   - 运行 lint 工具（如果配置）

3. **技术债**
   - 识别高复杂度文件（行数 > 500、CC > 10）
   - 检测重复代码

4. **文档完整性**
   - 检查 README.md 是否存在
   - 检查 docs/ 目录
   - 统计代码注释率

5. **安全性**
   - 检查敏感信息（密钥、密码）
   - 检查依赖版本（是否过时）

### 2️⃣ 优先级排序

**排序算法**：

```python
def calculate_priority(item):
    # 基础分数
    score = 0

    # 紧急度评分（0-10）
    if item.type == "security":
        urgency = 10  # 安全问题最紧急
    elif item.type == "production_bug":
        urgency = 9   # 生产 Bug 次之
    elif item.type == "technical_debt":
        urgency = 5   # 技术债中等
    else:
        urgency = 3   # 其他较低

    # 重要度评分（0-10）
    if item.impact == "core_functionality":
        importance = 10  # 影响核心功能
    elif item.impact == "user_experience":
        importance = 8   # 影响用户体验
    elif item.impact == "maintainability":
        importance = 6   # 影响可维护性
    else:
        importance = 4   # 其他

    # 综合评分
    score = urgency * 0.6 + importance * 0.4

    # 考虑工作量（预估时间越短，优先级越高）
    if item.estimated_time < 30:  # 小于30分钟
        score += 1

    return score
```

**排序后分类**：
- Top 1-3：🔴 高优先级（立即执行）
- Top 4-5：🟡 中优先级（短期执行）
- 其他：🟢 低优先级（长期规划）

### 3️⃣ 生成推荐

**推荐格式**：每个推荐包含

- **优先级标识**：🔴/🟡/🟢
- **标题**：简洁描述（≤ 10 字）
- **原因**：为什么要做（1-2 句）
- **位置**：文件路径或模块
- **预估时间**：分钟/小时
- **操作命令**：可直接执行的 `/loop` 或 `/plan` 命令

## 适用场景

| 场景 | 说明 |
|------|------|
| 🤔 **不知道做什么** | 项目有很多待办，不确定从哪里开始 |
| 📊 **定期检查** | 每周/每天定期检查项目状态，找出最重要的工作 |
| 🎯 **快速决策** | 时间有限，需要快速找到最重要的工作 |
| 👥 **团队协作** | 向团队展示当前的优先级和待办事项 |
| 🔍 **项目诊断** | 发现项目中的问题和改进点 |

## 📊 与其他 Commands 的关系

| Command | 关系 | 说明 |
|---------|------|------|
| **plan** | 互补 | next 推荐"做什么"，plan 规划"怎么做" |
| **loop** | 后续 | next 推荐后，使用 loop 执行 |
| **suggest** | 互补 | next 聚焦当前任务，suggest 聚焦长期改进 |

**典型工作流**：
```
/next → 找到最重要的工作
   ↓
/plan → 规划如何执行
   ↓
/loop → 实际执行
```

## 注意事项

- **数据依赖**：next 的建议质量取决于项目信息的完整性（测试报告、lint 配置等）
- **主观判断**：优先级排序有一定主观性，可能与实际需求有偏差
- **动态变化**：项目状态会变化，建议定期执行 next 更新优先级
- **不执行任务**：next 只提供建议，不会执行任何代码或创建任务
- **预估误差**：时间估算仅供参考，实际执行可能有偏差
- **聚焦当前**：next 聚焦当前最紧急的工作，不涵盖长期规划（使用 suggest）
