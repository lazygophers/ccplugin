# 深度迭代模式使用指南

## 概述

深度迭代模式确保任务通过**多次迭代 + 深度研究**，结果**完全符合预期**。

## 使用方式

```bash
claude /loop "实现用户认证功能"
```

## 执行流程

```
Initialization → PromptOptimization → 复杂度评估
  → (DeepResearch?) → Planning → Execution
  → Verification → QualityGate
  → (Adjustment?) → Cleanup → End
```

## 质量递进

| 轮次 | 等级 | 阈值 | 验收标准 |
|------|------|------|---------|
| 1 | Foundation | 60 | 功能实现，测试通过 |
| 2 | Enhancement | 75 | 边界处理，错误处理，性能优化 |
| 3 | Refinement | 85 | 代码质量，可维护性，文档完善 |
| 4+ | Excellence | 90 | 最佳实践，可扩展性，安全性 |

## 深度研究触发

| 触发条件 | 说明 |
|---------|------|
| 复杂度 >8 | 自动启动研究 |
| 复杂度 6-8 | 询问用户是否研究 |
| 失败 ≥2 次 | 询问用户 |

## 终止条件

满足**所有**条件时终止：
1. 所有验收标准通过（对照 prompt.md）
2. 质量分数 ≥ 当前迭代阈值
3. 用户确认结果符合预期

## 预期收益

| 维度 | 普通迭代 | 深度迭代 |
|------|---------|---------|
| 最终质量 | 60-75 分 | 85-95 分 |
| 返工率 | 30-50% | 10-20% |
| 最佳实践对齐 | 部分 | 100% |
