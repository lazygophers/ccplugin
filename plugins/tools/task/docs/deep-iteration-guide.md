# 深度迭代模式使用指南

## 概述

深度迭代模式确保任务通过**多次迭代 + 深度研究**，结果**完全符合预期**。

## 快速开始

### 使用方式（默认启用）

```bash
claude /loop "实现用户认证功能"
```

深度迭代模式已默认启用，无需额外配置。

### 执行流程

```
初始化（深度迭代模式）
  ↓
深度研究（第1轮）→ 查找最佳方案
  ↓
计划设计 → 质量目标：60分
  ↓
任务执行
  ↓
结果验证 → 质量：68分 ⚠ 不达标（< 75）
  ↓
失败调整 → 深度分析
  ↓
计划设计 → 质量目标：75分
  ↓
任务执行
  ↓
结果验证 → 质量：78分 ✓ 未达最小迭代（2 < 3）
  ↓
用户选择："继续优化"
  ↓
计划设计 → 质量目标：85分
  ↓
任务执行
  ↓
结果验证 → 质量：87分 ✓
  ↓
持续改进 → 识别优化点
  ↓
用户选择："完成，结果已满意"
  ↓
全部完成
```

## 核心特性

### 1. 质量递进

| 轮次 | 等级 | 阈值 | 验收标准 |
|------|------|------|---------|
| 1 | Foundation | 60 | 功能实现，测试通过 |
| 2 | Enhancement | 75 | 边界处理，错误处理，性能优化 |
| 3 | Refinement | 85 | 代码质量，可维护性，文档完善 |
| 4+ | Excellence | 90 | 最佳实践，可扩展性，安全性 |

### 2. 深度研究触发

| 触发条件 | 说明 |
|---------|------|
| 第 1 轮迭代 | 了解最佳方案 |
| 失败 2 次以上 | 根本原因分析 |
| 质量不达标 | 分数 < 阈值 - 10 |
| 复杂任务 | planner 识别 |

### 3. 深度终止条件

满足**所有**条件时终止：
1. ✓ 所有验收标准通过
2. ✓ 质量分数 ≥ 当前阈值
3. ✓ 遵循最佳实践
4. ✓ 用户确认"完全符合预期"
5. ✓ 达到最小迭代次数（3 轮）
