# Agent/Skill 公共约定

编写或修改 agent/skill 时必须遵循的公共约定。

## 必需约束（每个 agent 必须包含）

| 约束 | 标准措辞 | 适用范围 |
|------|---------|---------|
| 最大回合 | `≤N 轮工具调用，超出时{降级行为}` | 所有 agent |
| 项目范围 | `所有文件操作必须限定在 project_root 目录内` | explore/plan/verify/adjust |
| 输出前缀 | `所有输出必须包含前缀：[flow·{task_id}·{state}]` | 所有 agent |

## 推荐约束（按角色选用）

## 必需章节

每个 agent .md 必须包含：

1. **核心职责**（3-4 条，编号列表）
2. **约束**（含最大回合 + 角色特定约束）
3. **边界情况**（3-5 条，处理回合耗尽/数据缺失/异常输入）
4. **输出**（返回值结构 + 前缀要求）

可选章节：
- **反例**（plan/verify 建议添加，帮助模型理解拒绝条件）

## maxTurns 参考值

| Agent | maxTurns | 理由 |
|-------|----------|------|
| explore | 15 | 需扫描多文件 |
| plan | 20 | 含 10 次验证迭代 |
| verify | 10 | 逐条检查标准 |
| adjust | 8 | 分析 + 用户交互 |
| done | 5 | 简单汇总 |

## Checklist

新增/修改 agent 前检查：
- [ ] 包含最大回合约束
- [ ] 包含边界情况处理
- [ ] 输出前缀格式正确
- [ ] 项目范围约束（如适用）
- [ ] model 字段与复杂度匹配
