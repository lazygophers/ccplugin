# 设计角色秘密

帮助作者设计悬疑推理角色的秘密、动机和心理画像。

## 适用场景

| 场景 | 示例 |
|------|------|
| **动机设计** | `/novel-mystery secrets 为凶手设计三层动机` |
| **嫌疑人群像** | `/novel-mystery secrets 设计6个各怀秘密的嫌疑人` |
| **秘密揭露** | `/novel-mystery secrets 规划角色秘密的揭露顺序` |
| **心理画像** | `/novel-mystery secrets 为侦探角色构建心理特征` |

## 执行流程

### 1. 确认需求方向

通过提问明确：
- **设计目标**：单一角色/嫌疑人群像/真凶设计/侦探设计
- **角色定位**：主角/真凶/嫌疑人/受害者/配角
- **秘密类型**：犯罪动机/身份伪装/过往秘密/关系秘密
- **已有设定**：与已有角色和剧情的关联

### 2. 生成设计方案

根据需求激活 `Skills(character-secrets)` 并生成：

#### 单一角色设计
- 表面身份与隐藏身份
- 公开目标与真实动机
- 秘密内容与隐藏方式
- 压力下的行为模式
- 秘密被揭露时的反应

#### 嫌疑人群像设计
- 每个嫌疑人的秘密和动机
- 嫌疑人之间的关系网络
- 嫌疑指向的排列顺序
- 各角色在调查中的态度变化

#### 真凶设计
- 犯罪动机的深层逻辑
- 作案手法与心理状态
- 伪装策略与破绽设计
- 最终暴露的触发条件

### 3. 一致性验证

检查角色设计的合理性：
- 动机是否足以驱动犯罪行为
- 秘密与角色性格是否匹配
- 伪装是否留有可追溯的破绽
- 揭露节奏是否合理

### 4. 输出角色档案

以结构化格式输出角色设计方案。

## 设计原则

- **人人有秘密**：每个重要角色都应该有自己隐藏的一面
- **动机为王**：再精妙的诡计也需要合理的动机支撑
- **破绽留痕**：角色的伪装要有可追溯的裂缝
- **同理共情**：读者不需认同角色，但要能理解其逻辑
