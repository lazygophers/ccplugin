# 设计情节发展

帮助作者设计故事情节的发展走向，包括主线推进、支线编排和节奏控制。

## 适用场景

| 场景 | 示例 |
|------|------|
| **主线设计** | `/novel plot 主线` |
| **支线编排** | `/novel plot 感情支线` |
| **单卷规划** | `/novel plot 第一卷` |
| **高潮设计** | `/novel plot 大战高潮` |
| **过渡章节** | `/novel plot 过渡 第50-60章` |
| **解决卡文** | `/novel plot 接下来写什么` |

## 执行流程

### 1. 了解当前状态

收集必要信息：
- **已有设定**：世界观、角色、力量体系
- **已写内容**：当前进度和已发生的事件
- **当前问题**：卡在哪里、想解决什么
- **目标方向**：希望情节往哪个方向发展

### 2. 情节设计

根据需求类型提供不同层次的设计：

#### 主线情节
- **核心事件链**：推动故事前进的关键事件序列
- **因果逻辑**：每个事件的起因和结果
- **悬念布局**：伏笔埋设和揭示时机
- **转折设计**：意料之外、情理之中的转折点

#### 支线情节
- **支线目的**：丰富世界观/角色成长/调节节奏
- **与主线关系**：如何与主线交织和呼应
- **收束时机**：支线何时开始、何时收束

#### 冲突设计
- **外部冲突**：主角与对手/环境的对抗
- **内部冲突**：主角内心的挣扎和选择
- **冲突升级**：逐步提升冲突强度的阶梯

#### 节奏控制
- **张弛有度**：紧张段落与舒缓段落的交替
- **爽点密度**：每N章至少一个小爽点
- **钩子设计**：章节末尾的悬念和期待感

### 3. 章节级情节安排

将情节设计落实到具体章节：
- 每章的核心事件和目的
- 章节之间的衔接和过渡
- POV（视角）切换建议
- 字数和节奏建议

### 4. 可行性检查

验证情节设计：
- 是否符合已有设定和角色性格
- 逻辑是否自洽，有无漏洞
- 节奏是否合理，有无拖沓或跳跃
- 是否具备足够的阅读驱动力

## 情节设计原则

- **因果驱动**：事件之间要有因果关系，避免巧合堆砌
- **角色驱动**：情节由角色的选择和行动推动
- **递进升级**：冲突和挑战逐步升级，避免重复
- **伏笔回收**：埋下的伏笔要及时回收，避免烂尾
- **读者预期管理**：适度满足预期，适度打破预期

## 网文节奏参考

| 阶段 | 章节范围 | 重点 |
|------|---------|------|
| 黄金三章 | 1-3章 | 建立期待，展示主角，引出核心冲突 |
| 开局展开 | 4-30章 | 世界观铺开，第一个小高潮 |
| 第一卷高潮 | 25-40章 | 第一个大冲突的解决 |
| 过渡期 | 新卷开头 | 新地图/新势力/实力提升 |

## 注意事项

- 避免长期无进展的水文
- 每一章都要有存在的意义
- 高潮前要有足够的铺垫
- 注意伏笔的管理，避免遗忘
