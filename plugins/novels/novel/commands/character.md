# 创建或完善角色设定

帮助作者创建丰满立体的角色，或对已有角色进行补充完善。

## 适用场景

| 场景 | 示例 |
|------|------|
| **创建主角** | `/novel character 主角` |
| **设计反派** | `/novel character 反派 大boss` |
| **完善配角** | `/novel character 完善师父角色` |
| **角色关系** | `/novel character 关系图谱` |

## 执行流程

### 1. 确认角色定位

通过提问明确：
- **角色类型**：主角/反派/配角/龙套
- **故事定位**：在故事中承担的功能（导师、对手、搭档等）
- **登场时机**：何时出场，出场持续多久
- **已有设定**：用户已经想好的部分

### 2. 生成角色档案

#### 基础信息
- 姓名、年龄、外貌特征
- 身份/职业/势力归属
- 实力定位（在力量体系中的位置）

#### 性格内核
- **核心性格**：用3个关键词概括
- **性格成因**：过去经历如何塑造了这个性格
- **外在表现**：日常行为习惯、说话方式
- **内在矛盾**：性格中的对立面（如外冷内热）

#### 背景故事
- 出身背景与成长经历
- 关键转折事件
- 与其他角色的历史关联
- 隐藏的秘密或伏笔

#### 成长弧线（主要角色）
- **起点状态**：故事开始时的状态
- **成长驱动**：推动角色改变的核心动力
- **关键节点**：3-5个成长转折点
- **终点状态**：故事结束时的状态

#### 角色标签
- 口头禅或标志性动作
- 独特的技能或特长
- 弱点或软肋
- 读者记忆点

### 3. 角色关系梳理

如果已有其他角色，分析并建议：
- 与已有角色的关系定义
- 潜在的冲突点和合作点
- 关系发展的可能走向

### 4. 角色验证

检查角色设定的合理性：
- 性格和行为是否一致
- 动机是否充分
- 是否与其他角色有足够的区分度
- 是否具备读者吸引力

## 角色设计原则

- **功能明确**：每个角色都要有存在的理由
- **性格鲜明**：避免脸谱化，但要有辨识度
- **内在逻辑**：行为要符合角色性格和动机
- **服务剧情**：角色为故事服务，而非反过来

## 注意事项

- 主角需要有明确的成长空间
- 反派需要有合理的动机，避免为恶而恶
- 配角要有自己的人生，不只是工具人
- 女性角色避免花瓶化，赋予独立人格
