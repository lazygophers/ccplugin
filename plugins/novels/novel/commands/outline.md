# 管理小说大纲

帮助作者创建和维护小说的多层级大纲体系，从总纲到章节级细纲。

## 适用场景

| 场景 | 示例 |
|------|------|
| **创建总纲** | `/novel outline 创建` |
| **分卷大纲** | `/novel outline 第一卷` |
| **细化章节** | `/novel outline 细化 第1-10章` |
| **调整大纲** | `/novel outline 调整 删掉副本线` |
| **查看大纲** | `/novel outline 查看` |

## 执行流程

### 1. 确认操作类型

- **创建**：从零开始构建大纲
- **细化**：将粗纲细化为章节级
- **调整**：修改已有大纲
- **查看**：展示当前大纲状态

### 2. 大纲层级体系

#### 总纲（故事骨架）

整部小说的核心框架：
- **一句话简介**：故事的核心卖点
- **主题**：故事要表达的核心思想
- **主线脉络**：起→承→转→合的主线走向
- **分卷规划**：每卷的核心事件和目标
- **结局方向**：故事的最终走向
- **预估总字数**：整体篇幅规划

#### 分卷大纲（卷级规划）

每一卷的详细规划：
- **卷名和主题**：本卷核心内容
- **起止章节**：章节范围和字数预估
- **核心事件**：本卷的关键事件列表
- **角色动态**：本卷登场/退场/成长的角色
- **伏笔管理**：本卷埋设和回收的伏笔
- **卷末状态**：本卷结束时的故事状态

#### 章节细纲（章节级规划）

每章的具体安排：
- **章节标题**（可选）
- **核心事件**：本章发生什么
- **场景列表**：具体场景和地点
- **角色出场**：本章出场角色
- **章末钩子**：本章结尾的悬念
- **预估字数**：2000-4000字/章

### 3. 大纲调整

当需要修改大纲时：
- 评估调整的影响范围
- 检查与其他情节线的关联
- 确认伏笔和因果关系不受破坏
- 提供调整前后的对比

### 4. 大纲输出

支持多种输出格式：
- **树状结构**：层级清晰的大纲树
- **时间线**：按时间顺序排列的事件
- **角色线**：按角色视角梳理的事件线

## 大纲管理原则

- **灵活不僵化**：大纲是指南而非枷锁，允许在写作中调整
- **粗细得当**：总纲宜粗，近期章节宜细
- **伏笔追踪**：记录每个伏笔的埋设和计划回收时机
- **节奏可视**：在大纲中标注节奏变化（高潮/过渡/日常）

## 网文大纲模板

```
# 《书名》总纲

## 基本信息
- 类型：
- 预估字数：
- 更新频率：

## 一句话简介
（核心卖点）

## 主线脉络
1. 开局（1-X章）：
2. 发展（X-Y章）：
3. 高潮（Y-Z章）：
4. 结局（Z-末章）：

## 分卷规划
### 第一卷：卷名
- 章节：1-N
- 核心：
- 关键事件：

### 第二卷：卷名
...

## 伏笔清单
| 伏笔 | 埋设章节 | 计划揭示 | 状态 |
|------|---------|---------|------|
```

## 注意事项

- 写作过程中大纲会不断调整，这是正常的
- 远期大纲保持模糊即可，避免过度规划
- 注意单卷的字数控制，避免某一卷过于臃肿
- 伏笔清单需要定期维护，防止遗漏
