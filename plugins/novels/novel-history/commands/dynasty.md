# 设计朝代背景

帮助作者快速设计或完善历史架空世界的朝代背景。

## 适用场景

| 场景 | 示例 |
|------|------|
| **创建朝代** | `/novel-history dynasty 创建一个融合唐宋特征的架空朝代` |
| **社会制度** | `/novel-history dynasty 设计科举和选官制度` |
| **经济体系** | `/novel-history dynasty 构建商帮和贸易体系` |
| **文化风俗** | `/novel-history dynasty 设计婚丧嫁娶礼仪规制` |

## 执行流程

### 1. 确认需求方向

通过提问明确：
- **朝代原型**：参考哪个真实朝代/多朝代融合/完全架空
- **时代阶段**：开国/盛世/中兴/衰败/乱世
- **核心矛盾**：皇权相权/文武对立/民族冲突/阶层矛盾
- **已有设定**：与已有世界观的衔接要求

### 2. 生成朝代方案

根据需求激活 `Skills(dynasty-setting)` 并生成：

#### 政治制度
- 中央官制和权力架构
- 地方行政区划和管理
- 选官和考核制度
- 皇位继承规则

#### 社会阶层
- 各阶层的权利和义务
- 阶层流动的途径和限制
- 等级标识（服饰、住宅、出行）
- 核心矛盾和冲突点

#### 经济体系
- 土地制度和赋税体系
- 商业格局和贸易路线
- 货币制度和金融雏形
- 官营与民营的博弈

#### 文化风俗
- 礼仪制度和日常规范
- 节日庆典和宗教信仰
- 饮食起居和建筑风格
- 文艺特色和学术风气

### 3. 一致性验证

检查设定的合理性：
- 政治制度与时代阶段是否匹配
- 经济水平与社会发展是否对应
- 文化风俗与政治环境是否协调
- 是否为主角的活动留足空间

### 4. 输出设定文档

以结构化格式输出完整设定，便于后续查阅和扩展。

## 设计原则

- **历史质感**：架空要有真实历史的厚重感
- **内在矛盾**：好的朝代设定自带故事冲突
- **细节真实**：日常细节要贴合时代气质
- **留有余地**：不要一开始就把所有设定写完
