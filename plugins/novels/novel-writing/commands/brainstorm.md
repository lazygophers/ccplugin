# 快速启动故事构思

通过结构化的创意激发流程，帮助作者从零开始构建故事核心概念。

## 适用场景

| 场景 | 示例 |
|------|------|
| **新故事起步** | `/novel-writing brainstorm 都市修仙` |
| **灵感发散** | `/novel-writing brainstorm 废土+机甲` |
| **题材探索** | `/novel-writing brainstorm 系统流` |
| **融合创新** | `/novel-writing brainstorm 玄幻+科幻` |

## 执行流程

### 1. 解析用户输入

从用户提供的关键词中提取：
- **核心题材**：主要类型（玄幻、都市、科幻、历史等）
- **元素标签**：附加元素（系统、重生、穿越、末世等）
- **风格倾向**：轻松、热血、暗黑、治愈等

如果用户未提供关键词，通过提问引导：
1. 你想写什么类型的故事？
2. 有没有特别想融入的元素？
3. 目标读者群体是？

### 2. 生成故事核心概念

基于输入生成 3-5 个差异化的故事概念，每个概念包含：

- **一句话概念**：用一句话概括故事核心卖点
- **世界观速写**：50字以内的世界观描述
- **主角画像**：主角的核心特质和起始状态
- **核心冲突**：故事的主要矛盾
- **独特卖点**：区别于同类作品的差异点
- **爽点设计**：3个核心爽点方向

### 3. 深化选定概念

用户选择感兴趣的概念后，进一步展开：
- 扩展世界观设定（力量体系、社会结构）
- 设计主线剧情走向（开局→发展→高潮）
- 规划前30章节奏（黄金三章+后续钩子）
- 列出潜在的配角和势力

### 4. 输出构思文档

整理为结构化的构思文档，方便后续使用其他命令深化。

## 构思原则

- **读者视角优先**：每个概念都要有明确的阅读驱动力
- **差异化**：避免过度同质化，突出新意
- **可执行性**：概念要能落地为具体的故事线
- **黄金三章意识**：开局设计要能快速抓住读者

## 注意事项

- 构思阶段不追求完美，重在激发创意
- 每个概念简洁有力，避免过度展开
- 关注网文市场的热门元素和读者偏好
- 鼓励元素融合创新，但避免过于猎奇
