# 设计宗门势力

帮助作者设计宗门门派或构建完整的修仙界势力格局。

## 适用场景

| 场景 | 示例 |
|------|------|
| **创建宗门** | `/novel-xuanhuan sect 设计一个剑修宗门` |
| **势力格局** | `/novel-xuanhuan sect 构建修仙界势力版图` |
| **宗门内部** | `/novel-xuanhuan sect 设计宗门内部架构` |
| **势力关系** | `/novel-xuanhuan sect 正邪两道关系设定` |
| **家族设定** | `/novel-xuanhuan sect 设计修仙世家` |

## 执行流程

### 1. 确认设计范围

通过提问明确：
- **设计目标**：单个宗门/势力格局/完整修仙界
- **宗门定位**：主角宗门/敌对势力/中立组织
- **世界风格**：传统正邪对立/灰色地带/多方博弈
- **已有设定**：与已有修炼体系和角色的关联

### 2. 激活技能生成方案

激活 `Skills(sect-faction)` 并根据方向生成：

#### 单个宗门设计
- 宗门名称、等级、传承历史
- 核心功法和镇宗之宝
- 内部架构（管理层、弟子层级）
- 宗门特色和文化传统
- 当前状态和面临的挑战

#### 势力格局构建
- 各大势力分布和实力对比
- 势力间的同盟、敌对、竞争关系
- 核心冲突点和利益纠葛
- 格局变化的潜在触发因素

#### 资源与经济设计
- 灵脉、矿脉等核心资源分布
- 秘境和试炼之地设定
- 坊市、拍卖会等经济体系
- 资源争夺引发的冲突

### 3. 整合验证

- 宗门实力与修炼体系的匹配
- 势力格局的合理性和稳定性
- 是否为故事冲突提供足够空间
- 主角在格局中的成长路径

## 设计原则

- **生态合理**：势力分布要形成合理的金字塔结构
- **动态平衡**：格局要有变化的可能，不能铁板一块
- **利益驱动**：每个势力的行为要有利益逻辑
- **故事空间**：格局要能承载足够长的故事线
