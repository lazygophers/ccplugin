# 设计修炼体系

帮助作者快速设计或完善修仙世界的修炼体系。

## 适用场景

| 场景 | 示例 |
|------|------|
| **设计境界** | `/novel-xuanhuan cultivation 设计九境修炼体系` |
| **功法设定** | `/novel-xuanhuan cultivation 功法等级划分` |
| **灵根体系** | `/novel-xuanhuan cultivation 灵根资质分类` |
| **突破机制** | `/novel-xuanhuan cultivation 天劫和突破条件` |

## 执行流程

### 1. 确认需求方向

通过提问明确：
- **体系类型**：境界划分/功法等级/灵根资质/突破机制/完整体系
- **世界风格**：传统仙侠/暗黑修仙/科技修仙/混合类型
- **创新程度**：经典框架微调/大幅创新/完全原创
- **已有设定**：与已有世界观的衔接要求

### 2. 生成体系方案

根据需求激活 `Skills(cultivation-system)` 并生成：

#### 境界设计
- 完整的境界层级（大境界、小阶段）
- 每个境界的标志性特征和能力
- 境界间的质变节点
- 对应的寿命和人数分布

#### 功法设计
- 功法等级体系（与境界对应）
- 不同属性功法的特色
- 功法修炼要求和禁忌
- 特殊功法的设定（禁术、合击功法等）

#### 灵根资质
- 灵根分类和稀有度
- 资质对修炼速度的影响
- 特殊体质设定
- 逆天改命的可能性（为主角留下空间）

#### 突破机制
- 各境界突破条件
- 天劫体系设定
- 辅助突破的资源（丹药、灵泉等）
- 失败后果和补救措施

### 3. 一致性验证

检查体系的合理性：
- 境界、功法、灵根三者是否匹配
- 修炼资源与经济体系是否关联
- 是否为主角成长留足空间
- 是否有足够的冲突驱动点

### 4. 输出设定文档

以结构化格式输出完整设定，便于后续查阅和扩展。

## 设计原则

- **逻辑自洽**：体系内部不能有矛盾
- **层次分明**：每个境界要有质的区别
- **服务故事**：体系为剧情服务，不是为了设定而设定
- **留有余地**：不要一开始就把所有境界写完，留空间给后续展开
