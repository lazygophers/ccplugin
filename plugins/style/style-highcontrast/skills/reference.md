# High Contrast - 高对比无障碍设计风格规范

高对比设计遵循 WCAG AAA 标准，强调清晰、可读和包容性，确保所有用户都能访问内容。

## 核心特征

**对比度标准**：
```
AAA 级别要求：
- 正文文本：7:1 以上
- 大文本（18pt+）：4.5:1 以上
- UI 组件：3:1 以上
```

**颜色系统**：
```css
/* 高对比黑白 */
--text-primary: #000000;    /* 纯黑 */
--bg-primary: #ffffff;      /* 纯白 */
--text-secondary: #333333;
--bg-secondary: #f0f0f0;

/* 强调色（高对比） */
--accent: #0000ff;          /* 纯蓝 */
--warning: #ff0000;         /* 纯红 */
--success: #008000;         /* 纯绿 */
```

## 实现要点

**对比度检查工具**：
```
使用 WebAIM Contrast Checker
输入 RGB 值，得到对比度比例
```

**CSS 变量**：
```css
:root {
  --fg: #000000;
  --bg: #ffffff;
  --border: #333333;
  --focus: #0000ff;
}
```

**焦点指示器**：
```css
:focus-visible {
  outline: 3px solid #0000ff;
  outline-offset: 2px;
}
```

| 灰 + 白 | 3.5:1 | 仅 A |

## DO & DON'T

✅ **DO**:
- 坚持 WCAG AAA 对比度
- 清晰的焦点指示器
- 键盘导航完整
- 色觉缺陷友好

❌ **DON'T**:
- 仅依赖颜色表达信息
- 闪烁或动画刺激
- 小于 12px 的文本
- 跳过 alt 文本

## 可访问性清单

- [ ] 对比度 ≥7:1（文本）
- [ ] 焦点指示器清晰
- [ ] 键盘导航完整
- [ ] 颜色不是唯一信息
- [ ] 动画可关闭
- [ ] 字体可读
- [ ] 链接可识别

## 文本可读性

```css
/* 确保足够的行高 */
line-height: 1.5;

/* 合理的文本宽度 */
max-width: 80ch;

/* 足够的字体大小 */
font-size: 16px;
```
