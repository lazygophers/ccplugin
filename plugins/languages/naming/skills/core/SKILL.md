# 命名规范指南

## 相关 Skills

适用于所有语言插件的命名场景：
- **Skills(python:core)** / **Skills(golang:naming)** / **Skills(typescript:core)** 等 - 各语言特定命名规则

## Red Flags：AI 常见误区

| AI 可能的理性化解释 | 实际应该检查的内容 |
|---------------------|-------------------|
| "简短名称更高效" | ✅ 名称是否清晰表达意图？ |
| "data/temp/obj 足够明确" | ✅ 是否使用了具体的描述性名称？ |
| "ID 全大写更规范" | ✅ 是否使用 `Id`（PascalCase）？ |
| "status 和 state 一样" | ✅ 是否统一使用 `state`？ |
| "字符串时间更易读" | ✅ 时间字段是否使用时间戳？ |

## 核心原则

### 1. 清晰性

- 名称应该清楚表达其含义
- 避免使用过于简洁的缩写（除非是广为人知的术语）
- 使用完整的单词组合

```
✓ user_name, get_user_by_id, calculate_total_price
✗ u, gubi, calc_tp
```

### 2. 一致性

- 项目内保持命名风格统一
- 相同概念使用相同命名方式
- 遵循语言社区的标准

### 3. 可读性

- 避免过长的名称（通常 < 40 字符）
- 使用有意义的词汇
- 优先考虑团队可读性

### 4. 避免混淆

- 不要使用相似的名称
- 不要使用容易误解的缩写
- 避免歧义的单数/复数形式

## 命名清单

使用前检查：

- [ ] 名称是否清晰表达意图？
- [ ] 是否遵循语言社区规范？
- [ ] 名称长度是否合理（< 40 字符）？
- [ ] 是否与项目内其他命名一致？
- [ ] 避免了歧义吗？
- [ ] 是否需要代码注释来解释这个名称？（如果是，考虑重新命名）
- [ ] 避免了过度缩写吗？
- [ ] 避免了容易混淆的相似名称吗？

## 常见命名错误

| 错误命名 | 正确命名 | 说明 |
|---------|---------|------|
| `status` | `state` | 使用 state 表示状态 |
| `ID` | `Id` | ID 应使用 PascalCase |
| `Users` | `UserList` | 使用清晰的集合名称 |
| `id` | `uid`/`userId` | 简单的 `id` 易混淆 |
| `FLAG_ENABLE` | `IS_ENABLED` | 布尔值使用 `IS_` 前缀 |
| `data` | `user_data` | 避免过于泛化的名称 |
| `temp` | `cache` | 避免 temp 等临时性命名 |
| `obj` | 具体类型名 | 避免过于抽象的名称 |
| `created_time` | `created_at` | 时间字段使用 `_at` 后缀 |
| 字符串时间 | 时间戳 | 时间必须使用时间戳格式 |

## 执行过程检查清单

### 命名创建检查
- [ ] 名称清晰表达意图
- [ ] 遵循语言社区规范
- [ ] 名称长度合理（< 40 字符）
- [ ] 与项目内其他命名一致
- [ ] 避免歧义
- [ ] 不需要代码注释来解释名称
- [ ] 避免过度缩写
- [ ] 避免容易混淆的相似名称

### 特定类型命名检查
- [ ] 状态使用 `state` 而非 `status`
- [ ] ID 使用 `Id` 而非 `ID`
- [ ] 集合使用清晰名称（如 `UserList` 而非 `Users`）
- [ ] 简单 `id` 改为 `uid`/`userId` 等明确名称
- [ ] 布尔值使用 `IS_`/`is_` 前缀
- [ ] 时间字段使用 `_at` 后缀（如 `created_at`）
- [ ] 时间必须使用时间戳格式

### 避免错误命名检查
- [ ] 避免过于泛化的名称（如 `data` 改为 `user_data`）
- [ ] 避免临时性命名（如 `temp` 改为 `cache`）
- [ ] 避免过于抽象的名称（如 `obj` 改为具体类型名）

## 完成后检查清单

### 命名质量检查
- [ ] 名称清晰易懂，无需额外解释
- [ ] 名称符合语言和项目规范
- [ ] 名称一致性良好
- [ ] 避免了所有常见命名错误

### 团队可读性检查
- [ ] 团队成员能快速理解名称含义
- [ ] 名称符合团队约定
- [ ] 命名风格与现有代码一致
