# Mermaid 用户旅程图 (User Journey)

用户旅程图用于描述用户完成特定任务的步骤和体验，展示用户与系统的交互过程。

## 基本语法

```mermaid
journey
    title 我的一天
    section 上班
      泡茶: 5: 我
      上楼: 3: 我
      工作: 1: 我, 猫
    section 下班
      下楼: 5: 我
      坐下: 5: 我
```

## 结构说明

### 标题

```mermaid
journey
    title 用户购物流程
```

### 章节

```mermaid
journey
    title 用户购物流程
    section 浏览商品
      打开应用: 5: 用户
      搜索商品: 4: 用户
      查看详情: 4: 用户
    section 下单
      加入购物车: 5: 用户
      填写地址: 3: 用户
      支付: 4: 用户, 系统
```

### 任务

```
任务名称: 评分: 参与者1, 参与者2
```

## 任务评分

评分范围为 1-5，表示用户满意度：

| 分数 | 含义 |
| --- | --- |
| 1 | 非常不满意 |
| 2 | 不满意 |
| 3 | 一般 |
| 4 | 满意 |
| 5 | 非常满意 |

## 参与者

### 单一参与者

```mermaid
journey
    title 登录流程
    section 输入凭证
      打开登录页: 5: 用户
      输入用户名: 4: 用户
      输入密码: 3: 用户
```

### 多参与者

```mermaid
journey
    title 购物流程
    section 下单
      选择商品: 5: 用户
      提交订单: 4: 用户
      处理订单: 4: 系统
      发送确认: 5: 系统, 用户
```

参与者用逗号分隔，表示多个参与者共同参与该步骤。

## 完整示例

### 示例一：用户注册流程

```mermaid
journey
    title 用户注册流程
    section 填写信息
      打开注册页: 5: 用户
      输入邮箱: 4: 用户
      设置密码: 3: 用户
      填写个人资料: 3: 用户
    section 验证
      发送验证邮件: 4: 系统
      点击验证链接: 5: 用户
      验证成功: 5: 系统
    section 完成
      跳转首页: 5: 用户, 系统
      显示欢迎信息: 5: 系统
```

### 示例二：在线购物体验

```mermaid
journey
    title 在线购物体验
    section 发现商品
      浏览首页: 4: 用户
      搜索商品: 5: 用户
      查看详情: 4: 用户
    section 购买决策
      比较价格: 3: 用户
      查看评价: 4: 用户
      加入购物车: 5: 用户
    section 结算
      填写地址: 3: 用户
      选择支付: 4: 用户
      确认订单: 5: 用户
    section 收货
      物流跟踪: 4: 用户, 系统
      确认收货: 5: 用户
      评价商品: 4: 用户
```

### 示例三：技术支持流程

```mermaid
journey
    title 技术支持流程
    section 提交工单
      登录系统: 4: 用户
      选择问题类型: 5: 用户
      描述问题: 3: 用户
      上传附件: 4: 用户
    section 处理工单
      分配工程师: 4: 系统
      分析问题: 3: 工程师
      提供解决方案: 4: 工程师
    section 解决
      用户确认: 5: 用户
      关闭工单: 5: 系统, 工程师
      满意度调查: 4: 用户
```

## 最佳实践

### 章节划分

- 按时间顺序组织
- 每个章节代表一个阶段
- 章节名称简洁明了

### 任务描述

- 使用动词开头
- 描述具体行为
- 保持简洁

### 评分原则

- 基于用户体验客观评分
- 痛点步骤给予低分
- 愉悦步骤给予高分

### 参与者定义

- 明确区分用户和系统
- 多角色协作时都列出
- 名称保持一致
