## 核心理念

Markdown 文档编写追求**清晰、简洁、结构化**，通过统一的规范和最佳实践，帮助作者创建高质量的技术文档。

**三个支柱**：

1. **清晰表达** - 内容表达清晰，避免歧义
2. **结构合理** - 标题层级合理，内容组织有序
3. **格式统一** - 格式风格一致，易于阅读维护

## 版本与环境

- **Markdown 版本**：CommonMark 规范 + GFM 扩展
- **推荐工具**：VS Code + Markdown 插件
- **渲染器**：支持 GFM 的渲染器
- **Lint 工具**：markdownlint

## 核心约定

### 强制规范

- ✅ 文件使用 UTF-8 编码
- ✅ 一级标题（#）每个文件仅一个
- ✅ 标题层级不跳级（# → ## → ###）
- ✅ 代码块必须指定语言标记
- ✅ 链接文字必须描述目标内容
- ✅ 图片必须包含 alt 文本
- ✅ 列表项前后保持格式一致
- ✅ 表格必须有表头行

### 文档结构

```
文档结构示例
├── 标题（#）
├── 简介/摘要
├── 目录（可选）
├── 主要内容（##）
│   ├── 子章节（###）
│   └── 详细内容（####）
├── 参考资料
└── 版本信息
```

## 最佳实践概览

**标题格式**

```markdown
# 文档标题

## 主要章节

### 子章节

#### 详细内容
```

**代码块格式**

````markdown
```python
def hello():
    print("Hello, World!")
```
````

````

**链接格式**

```markdown
[链接描述](https://example.com)

![图片描述](./images/example.png)
````

**表格格式**

```markdown
| 列1  | 列2  | 列3  |
| ---- | ---- | ---- |
| 内容 | 内容 | 内容 |
```

## 文件行数规范

- **最大行数**：500 行
- **推荐行数**：200~400 行
- **超过建议**：考虑拆分为多个文档

## 格式规范速查

### 标题规范

| 规则     | 说明             |
| -------- | ---------------- |
| 一级标题 | 文档标题，仅一个 |
| 二级标题 | 主要章节         |
| 三级标题 | 子章节           |
| 四级标题 | 详细内容         |
| 标题前后 | 空一行           |

### 列表规范

| 类型       | 格式                 |
| ---------- | -------------------- |
| 无序列表   | `-` 或 `*` 开头      |
| 有序列表   | `1.` 开头            |
| 嵌套列表   | 缩进 2 或 4 空格     |
| 列表项间距 | 子项不空行，组间空行 |

### 代码块规范

| 规则     | 说明             |
| -------- | ---------------- |
| 语言标记 | 必须指定语言     |
| 围栏符号 | 使用三个反引号   |
| 代码缩进 | 保持原有缩进     |
| 前后空行 | 代码块前后空一行 |

### 链接规范

| 类型     | 格式              |
| -------- | ----------------- |
| 行内链接 | `[文字](URL)`     |
| 引用链接 | `[文字][ref]`     |
| 图片     | `![alt](URL)`     |
| 锚点     | `[文字](#anchor)` |

## 禁止行为

- ❌ 标题层级跳跃
- ❌ 代码块无语言标记
- ❌ 链接文字为"点击这里"
- ❌ 图片无 alt 文本
- ❌ 过长的段落（>5 句）
- ❌ 表格缺少表头
- ❌ 混用列表符号
