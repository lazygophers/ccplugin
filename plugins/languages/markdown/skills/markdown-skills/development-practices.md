# Markdown 开发实践指南

## 写作风格

### 语言风格

1. **简洁明了**
    - 使用简短的句子
    - 避免冗长和复杂的表达
    - 一个段落表达一个主题

2. **主动语态**
    - 优先使用主动语态
    - 明确动作执行者
    - 增强表达力度

3. **一致性**
    - 保持术语使用一致
    - 保持人称使用一致
    - 保持时态使用一致

### 术语规范

1. **术语定义**
    - 首次使用时提供定义
    - 使用标准术语
    - 避免自造术语

2. **缩写使用**
    - 首次使用时展开缩写
    - 常见缩写可省略展开
    - 提供缩写表（如需要）

3. **中英文混排**
    - 中英文之间加空格
    - 数字与中文之间加空格
    - 专有名词使用正确大小写

## 格式规范

### 标题格式

1. **标题层级**

    ```markdown
    # 一级标题 - 文档标题

    ## 二级标题 - 主要章节

    ### 三级标题 - 子章节

    #### 四级标题 - 详细内容
    ```

2. **标题规则**
    - 标题前后空一行
    - 标题不以标点结尾
    - 标题简洁明确

3. **标题编号**
    - 一般不使用编号
    - 长文档可使用编号
    - 编号格式：`1. `、`1.1. `

### 段落格式

1. **段落规则**
    - 段落之间空一行
    - 每段不超过 5 句
    - 每句不超过 40 字

2. **换行规则**
    - 句子之间不换行
    - 列表项可换行
    - 代码块独立成段

3. **空行规则**
    - 标题前后空一行
    - 代码块前后空一行
    - 列表组之间空一行

### 列表格式

1. **无序列表**

    ```markdown
    - 项目一
    - 项目二
      - 子项目一
      - 子项目二
    - 项目三
    ```

2. **有序列表**

    ```markdown
    1. 第一步
    2. 第二步
    3. 第三步
    ```

3. **列表规则**
    - 同级列表符号一致
    - 嵌套缩进 2 或 4 空格
    - 列表项结尾不加标点

### 代码格式

1. **行内代码**

    ```markdown
    使用 `code` 表示行内代码
    ```

2. **代码块**

    ````markdown
    ```python
    def example():
        return "Hello"
    ```
    ````

3. **代码规则**
    - 必须指定语言标记
    - 保持原有缩进
    - 添加必要注释

### 表格格式

1. **基本表格**

    ```markdown
    | 列1 | 列2 | 列3 |
    | --- | --- | --- |
    | 内容 | 内容 | 内容 |
    ```

2. **对齐方式**

    ```markdown
    | 左对齐 | 居中 | 右对齐 |
    | :--- | :---: | ---: |
    | 内容 | 内容 | 内容 |
    ```

3. **表格规则**
    - 必须有表头行
    - 列数保持一致
    - 避免过宽表格

### 链接格式

1. **行内链接**

    ```markdown
    [链接描述](https://example.com)
    ```

2. **引用链接**

    ```markdown
    [链接描述][ref]

    [ref]: https://example.com
    ```

3. **链接规则**
    - 链接文字描述目标
    - 避免裸露 URL
    - 检查链接有效性

### 图片格式

1. **基本图片**

    ```markdown
    ![图片描述](./images/example.png)
    ```

2. **带链接图片**

    ```markdown
    [![图片描述](./images/example.png)](https://example.com)
    ```

3. **图片规则**
    - 必须包含 alt 文本
    - 使用相对路径
    - 图片大小适中

## 内容组织

### 文档结构

1. **标准结构**

    ```markdown
    # 文档标题

    简介或摘要

    ## 主要内容

    ### 子章节

    ## 参考资料

    ## 版本信息
    ```

2. **README 结构**

    ```markdown
    # 项目名称

    简介

    ## 特性

    ## 安装

    ## 使用

    ## 配置

    ## 贡献

    ## 许可证
    ```

3. **API 文档结构**

    ```markdown
    # API 名称

    概述

    ## 端点

    ## 参数

    ## 请求示例

    ## 响应示例

    ## 错误码
    ```

### 章节组织

1. **章节长度**
    - 每节 50-200 行
    - 超过 200 行考虑拆分
    - 保持章节平衡

2. **章节顺序**
    - 从概述到详细
    - 从简单到复杂
    - 从常用到不常用

3. **章节标题**
    - 简洁明确
    - 概括内容
    - 吸引读者

## 文档维护

### 版本控制

1. **版本标注**
    - 文档开头标注版本
    - 记录更新日期
    - 说明变更内容

2. **变更日志**
    - 记录重要变更
    - 说明变更原因
    - 提供迁移指南

### 文档审查

1. **内容审查**
    - 检查内容准确性
    - 验证代码示例
    - 确认链接有效

2. **格式审查**
    - 检查 Markdown 语法
    - 验证格式规范
    - 确认风格一致

3. **可读性审查**
    - 评估可读性
    - 检查逻辑清晰
    - 确认易于理解

### 文档更新

1. **更新时机**
    - 功能变更时
    - 问题修复时
    - 定期审查时

2. **更新原则**
    - 及时更新
    - 保持一致
    - 记录变更

## 常见问题

### 格式问题

1. **标题层级跳跃**
    - 问题：# 直接跳到 ###
    - 解决：补充中间层级

2. **代码块渲染错误**
    - 问题：代码块显示异常
    - 解决：检查语言标记和缩进

3. **表格对齐问题**
    - 问题：表格列不对齐
    - 解决：检查分隔符和内容

### 内容问题

1. **内容过时**
    - 问题：信息不再准确
    - 解决：及时更新内容

2. **内容缺失**
    - 问题：缺少必要信息
    - 解决：补充完整内容

3. **内容冗余**
    - 问题：重复表达
    - 解决：精简合并内容
