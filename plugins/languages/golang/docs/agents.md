# 代理系统

Golang 插件提供四个专业代理，覆盖开发、测试、调试和性能优化场景。

## 代理列表

| 代理 | 描述 | 适用场景 |
|------|------|----------|
| `dev` | Golang 开发专家 | 新功能开发、架构设计 |
| `test` | 测试专家 | 单元测试、集成测试 |
| `debug` | 调试专家 | 问题诊断、错误修复 |
| `perf` | 性能优化专家 | 性能分析、优化建议 |

## dev - Golang 开发专家

### 职责

- 开发新功能和模块
- 设计代码架构
- 实现业务逻辑
- 代码重构和优化

### 技能

- `core` - Golang 核心规范
- `error` - 错误处理规范
- `libs` - 常用库规范
- `naming` - 命名规范
- `structure` - 项目结构规范

### 使用示例

```
请帮我实现一个 HTTP API 服务，支持用户 CRUD 操作
```

## test - 测试专家

### 职责

- 编写单元测试
- 编写集成测试
- 测试覆盖率优化
- 测试策略设计

### 技能

- `testing` - 测试策略
- `core` - Golang 核心规范

### 使用示例

```
为用户服务编写表驱动测试
```

## debug - 调试专家

### 职责

- 诊断运行时错误
- 分析 goroutine 问题
- 定位性能瓶颈
- 解决并发问题

### 技能

- `error` - 错误处理规范
- `concurrency` - 并发编程规范
- `core` - Golang 核心规范

### 使用示例

```
排查 goroutine 泄漏问题
```

## perf - 性能优化专家

### 职责

- 性能分析和测量
- 识别性能瓶颈
- 提供优化建议
- 内存优化

### 技能

- `core` - Golang 核心规范
- `concurrency` - 并发编程规范

### 使用示例

```
优化 JSON 序列化性能
```

## 调用方式

代理会在相关场景自动激活，也可以显式请求：

```
使用 Golang 开发专家帮我设计一个微服务架构
```
