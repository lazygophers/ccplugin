所有 Go 代码必须遵守以下 Skills 规范：

- Skills(core) - 核心规范：强制约定、代码格式
- Skills(error) - 错误处理规范
- Skills(libs) - 优先库规范：stringx/candy/osx
- Skills(naming) - 命名规范
- Skills(structure) - 项目结构规范
- Skills(testing) - 测试规范
- Skills(concurrency) - 并发规范
- Skills(lint) - Lint 规范
- Skills(tooling) - 工具规范

如果涉及到 Golang Template，请遵守 Skills(gtpl-skills) 的规范。
每一个 `*.go` 文件都不得超过500行，推荐200~400行
