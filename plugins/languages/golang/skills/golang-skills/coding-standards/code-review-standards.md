# Golang 代码审查规范

## 核心原则

### ✅ 必须遵守

1. **代码质量** - 确保代码符合编码规范
2. **功能正确** - 确保代码实现正确的功能
3. **测试覆盖** - 确保代码有充分的测试
4. **文档完整** - 确保文档和注释完整
5. **性能考虑** - 确保代码性能合理

### ❌ 禁止行为

- 审查不仔细
- 审查延迟
- 审查时只看表面
- 审查时忽略测试
- 审查时忽略文档

## 审查流程

### 审查步骤

1. **理解需求** - 理解代码要实现的功能
2. **阅读代码** - 仔细阅读代码实现
3. **运行测试** - 运行测试确保功能正确
4. **检查清单** - 按照审查清单检查
5. **提供反馈** - 提供建设性的反馈
6. **确认修复** - 确认作者已修复问题

## 常见问题

### 命名问题

```go
// ❌ 错误 - 命名不清晰
func getData(id int64) (*User, error) {
    return User.Where("id", id).First()
}

// ✅ 正确 - 命名清晰
func GetUserById(id int64) (*User, error) {
    return User.Where("id", id).First()
}
```

### 错误处理问题

```go
// ❌ 错误 - 错误处理不完整
if err != nil {
    return err
}

// ✅ 正确 - 错误处理完整
if err != nil {
    log.Errorf("err:%v", err)
    return err
}
```

### 测试覆盖问题

```go
// ❌ 错误 - 测试覆盖不足
func TestGetUser(t *testing.T) {
    user, err := GetUserById(1)
    if err != nil {
        t.Error(err)
    }
}

// ✅ 正确 - 测试覆盖充分
func TestGetUserById(t *testing.T) {
    tests := []struct {
        name    string
        id      int64
        wantErr bool
    }{
        {
            name:    "valid user",
            id:      1,
            wantErr: false,
        },
        {
            name:    "user not found",
            id:      999,
            wantErr: true,
        },
    }

    for _, tt := range tests {
        t.Run(tt.name, func(t *testing.T) {
            user, err := GetUserById(tt.id)
            if (err != nil) != tt.wantErr {
                t.Errorf("GetUserById() error = %v, wantErr %v", err, tt.wantErr)
                return
            }
            if !tt.wantErr && user == nil {
                t.Error("GetUserById() returned nil user")
            }
        })
    }
}
```
