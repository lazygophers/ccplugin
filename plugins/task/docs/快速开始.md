## 安装

### 环境要求

- Python ≥ 3.10
- uv 包管理器
- Claude Code CLI

### 安装步骤

```bash
# 1. 进入插件目录
cd plugins/task

# 2. 创建虚拟环境
uv venv

# 3. 激活虚拟环境
source .venv/bin/activate  # Linux/macOS
# 或
.venv\Scripts\activate  # Windows

# 4. 安装插件（包含开发依赖）
uv pip install -e ".[dev]"

# 5. 验证安装
uv run python -m task --help
```

## 配置

### Claude Code 插件配置

在 `~/.claude/settings.json` 或项目的 `.claude-plugin/plugin.json` 中添加：

```json
{
  "mcpServers": {
    "task": {
      "command": "uv",
      "args": ["run", "python", "-m", "task"],
      "env": {
        "LOG_LEVEL": "INFO",
        "PYTHONUNBUFFERED": "1"
      }
    }
  }
}
```

### 环境变量（可选）

```bash
# 指定工作空间根目录
export TASK_WORKSPACE_ROOT=/path/to/your/project

# 指定数据库 URL
export TASK_DATABASE_URL=sqlite:///path/to/tasks.db

# 设置日志级别
export LOG_LEVEL=DEBUG
```

```
添加依赖：任务 tk-aaa 依赖任务 tk-bbb

```

## 编程接口（可选）

如果需要在 Python 代码中使用：

```python
from task import get_workspace, TaskRepository

# 获取工作空间
workspace = get_workspace()

# 获取数据库管理器
db = workspace.get_database_manager()

# 获取会话
session = db.get_session()

# 使用 Repository
task_repo = TaskRepository(session)

# 创建任务
task = task_repo.create({
    "id": "tk-custom001",
    "title": "自定义任务",
    "task_type": "task",
    "priority": 2,
    "status": "open",
})

# 提交
session.commit()

# 清理
session.close()
workspace.cleanup()
```

## 故障排查

### MCP 服务器无法启动

**症状**：Claude Code 报告 "task server failed to start"

**解决方法**：

1. 检查 Python 版本：
   ```bash
   python --version  # 应 ≥ 3.10
   ```

2. 检查依赖安装：
   ```bash
   cd plugins/task
   uv pip install -e ".[dev]"
   ```

3. 检查配置文件：
   ```bash
   cat ~/.claude/settings.json
   ```

4. 查看日志：
   ```bash
   export LOG_LEVEL=DEBUG
   # 重启 Claude Code
   ```

### 工具未显示

**症状**：调用工具时提示 "tool not found"

**解决方法**：

1. 重启 Claude Code
2. 检查插件路径配置
3. 验证插件已加载：
   ```
   列出所有可用工具
   ```

### 数据库错误

**症状**：操作失败，提示数据库错误

**解决方法**：

1. 检查工作空间目录权限
2. 检查数据库健康：
   ```
   显示工作空间信息
   ```

3. 重新初始化：
   ```bash
   rm -rf .task_data
   # 在 Claude Code 中
   初始化工作空间
   ```

### 导入错误

**症状**：`ModuleNotFoundError: No module named 'task'`

**解决方法**：

```bash
# 确保在虚拟环境中
source .venv/bin/activate

# 重新安装
uv pip install -e ".[dev]"
```

## 常见问题

**Q: 如何备份任务数据？**

A: 任务数据存储在 `.task_data/tasks_*.db` 文件中，直接复制该文件即可备份。

**Q: 可以在多个项目间共享任务吗？**

A: 不可以。每个工作空间的任务是独立的，这样设计是为了确保数据隔离和安全性。

**Q: 如何导出任务列表？**

A: 当前版本暂不支持导出功能，将在 v0.3.0 中添加。

**Q: 支持团队协作吗？**

A: 当前版本是单机使用。团队协作功能（远程同步）计划在未来版本中添加。
