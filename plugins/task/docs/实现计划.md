## 风险管理

### 高风险项

| 风险 | 描述 | 影响 | 概率 | 缓解措施 |
|------|------|------|------|----------|
| **依赖图性能** | 大规模依赖图查询慢 | 高 | 中 | 使用 NetworkX 优化，添加缓存 |
| **SQLAlchemy 学习曲线** | ORM 复杂度高 | 中 | 中 | 阅读文档，参考最佳实践 |
| **状态一致性** | 并发更新导致状态不一致 | 高 | 低 | 使用事务，添加锁机制 |

### 中风险项

| 风险 | 描述 | 影响 | 概率 | 缓解措施 |
|------|------|------|------|----------|
| **测试覆盖率不足** | 边界情况未覆盖 | 中 | 中 | 提前编写测试，定期审查 |
| **文档滞后** | 代码更新但文档未更新 | 中 | 中 | 代码提交时同步更新文档 |
| **迁移失败** | 数据库迁移出错 | 中 | 低 | 提供回滚脚本，充分测试 |

**1. 类型检查**

```bash
# 必须通过 mypy 严格模式
uv run mypy src/ --strict
```

**2. 代码格式**

```bash
# 使用 black 格式化
uv run black src/

# 使用 ruff 检查
uv run ruff check src/ --fix
```

**3. 测试覆盖率**

```bash

uv run pytest --cov=src/task --cov-report=term-missing --cov-fail-under=80
```

## 开发规范

### Git 工作流

**分支策略**：

```
main (稳定版本)
  ├── develop (开发分支)
  │   ├── feature/v0.2.0-storage (v0.2.0 功能)
  │   ├── feature/v0.3.0-dependency (v0.3.0 功能)
  │   └── bugfix/issue-123 (Bug 修复)
  └── release/v0.2.0 (发布分支)
```

**提交规范**：

```
<type>: <subject>

<body>

🤖 Generated with [Claude Code](https://claude.com/claude-code)

Co-Authored-By: Claude <noreply@anthropic.com>
```

**类型**：
- `feat`: 新功能
- `fix`: Bug 修复
- `docs`: 文档更新
- `test`: 测试相关
- `refactor`: 重构
- `perf`: 性能优化
- `chore`: 杂项

### 代码审查标准

**自我审查清单**：

1. **功能**：
   - [ ] 实现符合需求
   - [ ] 边界情况处理
   - [ ] 错误处理完善

2. **代码质量**：
   - [ ] 命名清晰（函数、变量、类）
   - [ ] 单一职责原则
   - [ ] DRY 原则（无重复代码）
   - [ ] 注释清晰必要

3. **类型安全**：
   - [ ] 所有函数有类型注解
   - [ ] 所有类属性有类型注解
   - [ ] 无 `Any` 类型（除非必要）
   - [ ] 通过 mypy --strict

4. **测试**：
   - [ ] 单元测试覆盖 ≥ 80%
   - [ ] 集成测试覆盖关键路径
   - [ ] 边界情况测试

5. **文档**：
   - [ ] Docstring 完整
   - [ ] README 更新
