# Task Plugin MCP 工具参考

> 版本：0.2.0 | 工具数量：15 | 通信协议：stdio | 最后更新：2025-12-29

## 工具总览

Task Plugin 通过 **MCP (Model Context Protocol)** 提供 15 个工具，使用 **stdio** 通信方式（标准输入输出流）。

### 分类索引

**任务操作（7个）**：
- [task_create](#task_create) - 创建任务
- [task_list](#task_list) - 列出任务
- [task_show](#task_show) - 查看详情
- [task_update](#task_update) - 更新任务
- [task_close](#task_close) - 关闭任务
- [task_reopen](#task_reopen) - 重新打开
- [task_delete](#task_delete) - 删除任务

**依赖管理（3个）**：
- [task_dep_add](#task_dep_add) - 添加依赖
- [task_dep_remove](#task_dep_remove) - 移除依赖
- [task_dep_list](#task_dep_list) - 列出依赖

**查询工具（3个）**：
- [task_ready](#task_ready) - 就绪任务（无阻塞）
- [task_blocked](#task_blocked) - 阻塞任务
- [task_stats](#task_stats) - 统计信息

**工作空间（2个）**：
- [workspace_init](#workspace_init) - 初始化工作空间
- [workspace_info](#workspace_info) - 工作空间信息

## 任务操作

### task_create

创建新任务。

**参数**：
- `title` (string, required): 任务标题
- `description` (string): 任务描述
- `task_type` (string): 类型（bug/feature/task/epic/chore，默认 task）
- `priority` (integer): 优先级（0=Critical, 1=High, 2=Medium, 3=Low, 4=Backlog，默认 2）
- `assignee` (string): 负责人
- `tags` (string[]): 标签列表

**返回**：
```json
{
  "id": "tk-abc123",
  "title": "实现用户认证",
  "status": "open",
  "priority": 2
}
```

---

### task_list

列出任务，支持多维度过滤。

**参数**：
- `status` (string): 状态过滤（open/in_progress/blocked/deferred/closed）
- `task_type` (string): 类型过滤（bug/feature/task/epic/chore）
- `priority` (integer): 优先级过滤（0-4）
- `assignee` (string): 负责人过滤
- `limit` (integer): 返回数量（默认 20）
- `brief` (boolean): 简化输出（默认 false）

**返回**：
```json
[
  {
    "id": "tk-abc123",
    "title": "修复登录 Bug",
    "status": "in_progress",
    "priority": 0,
    "assignee": "alice"
  }
]
```

---

### task_show

查看任务详情，包括依赖关系。

**参数**：
- `task_id` (string, required): 任务 ID
- `brief` (boolean): 简化输出
- `brief_deps` (boolean): 简化依赖信息

**返回**：
```json
{
  "id": "tk-abc123",
  "title": "实现用户认证",
  "description": "添加 JWT 认证功能",
  "status": "in_progress",
  "priority": 1,
  "dependencies": [...],
  "dependents": [...]
}
```

---

### task_update

更新任务属性（部分更新）。

**参数**：
- `task_id` (string, required): 任务 ID
- `status` (string): 更新状态
- `priority` (integer): 更新优先级
- `assignee` (string): 更新负责人
- `title` (string): 更新标题
- `description` (string): 更新描述

**返回**：
```json
{
  "id": "tk-abc123",
  "action": "updated",
  "changes": ["status: open -> in_progress"]
}
```

---

### task_close

关闭任务。

**参数**：
- `task_id` (string, required): 任务 ID
- `reason` (string): 关闭原因（默认 "Completed"）
- `brief` (boolean): 简化输出

**返回**：
```json
{
  "id": "tk-abc123",
  "action": "closed",
  "reason": "Completed"
}
```

---

### task_reopen

重新打开已关闭的任务。

**参数**：
- `task_id` (string, required): 任务 ID
- `reason` (string): 重新打开原因
- `brief` (boolean): 简化输出

**返回**：
```json
{
  "id": "tk-abc123",
  "action": "reopened",
  "status": "open"
}
```

---

### task_delete

删除任务。

**参数**：
- `task_id` (string, required): 任务 ID

**返回**：
```json
{
  "id": "tk-abc123",
  "action": "deleted"
}
```

## 依赖管理

### task_dep_add

添加任务间的依赖关系。

**参数**：
- `task_id` (string, required): 任务 ID
- `depends_on_id` (string, required): 依赖的任务 ID
- `dep_type` (string): 依赖类型（blocks/related/parent-child/discovered-from，默认 blocks）

**依赖类型**：
- `blocks`: 硬阻塞（未完成时自动设为 blocked）
- `related`: 软关联（仅表示关联）
- `parent-child`: 层级关系（Epic 包含子任务）
- `discovered-from`: 发现关系（工作时发现的新任务）

**返回**：
```json
{
  "task_id": "tk-abc123",
  "depends_on_id": "tk-def456",
  "dep_type": "blocks",
  "action": "dependency_added"
}
```

---

### task_dep_remove

移除任务间的依赖关系。

**参数**：
- `task_id` (string, required): 任务 ID
- `depends_on_id` (string, required): 依赖的任务 ID

**返回**：
```json
{
  "task_id": "tk-abc123",
  "depends_on_id": "tk-def456",
  "action": "dependency_removed"
}
```

---

### task_dep_list

列出任务的依赖关系。

**参数**：
- `task_id` (string, required): 任务 ID

**返回**：
```json
{
  "task_id": "tk-abc123",
  "dependencies": [
    {
      "id": "tk-def456",
      "title": "数据库迁移",
      "type": "blocks",
      "status": "closed"
    }
  ],
  "dependents": [
    {
      "id": "tk-ghi789",
      "title": "前端集成",
      "type": "blocks",
      "status": "open"
    }
  ]
}
```

## 查询工具

### task_ready

查找可以开始工作的任务（无未完成的阻塞依赖）。

**参数**：
- `limit` (integer): 返回数量（默认 10）
- `priority` (integer): 优先级过滤
- `assignee` (string): 负责人过滤
- `unassigned` (boolean): 仅未分配任务
- `brief` (boolean): 简化输出

**返回**：
```json
[
  {
    "id": "tk-abc123",
    "title": "修复登录 Bug",
    "priority": 0,
    "status": "open"
  }
]
```

---

### task_blocked

查找被阻塞的任务及其阻塞原因。

**参数**：
- `brief` (boolean): 简化输出
- `brief_deps` (boolean): 简化依赖信息

**返回**：
```json
[
  {
    "id": "tk-abc123",
    "title": "前端集成",
    "status": "blocked",
    "blocked_by": [
      {
        "id": "tk-def456",
        "title": "API 设计",
        "status": "in_progress"
      }
    ]
  }
]
```

## 工作空间

### workspace_init

初始化工作空间（创建数据库和目录结构）。

**参数**：
- `workspace_root` (string): 工作空间根目录（默认当前目录）

**返回**：
```json
{
  "action": "initialized",
  "workspace_root": "/path/to/project",
  "database_path": "/path/to/project/.task_data/tasks.db"
}
```

**注意**：SessionStart hook 会自动初始化，通常无需手动调用。

---

**参数**：无

```json
{

  "database_path": "/path/to/project/.task_data/tasks.db",
  "database_exists": true,
  "database_healthy": true,
  "current_revision": "head"
}
```

## 使用示例

### 基本工作流

```python
# 1. 创建任务
task_create(
    title="实现用户认证",
    task_type="feature",
    priority=1,
    assignee="alice"
)

# 2. 列出开放任务
task_list(status="open")

# 3. 查看详情
task_show(task_id="tk-abc123")

# 4. 开始工作
task_update(task_id="tk-abc123", status="in_progress")

# 5. 完成任务
task_close(task_id="tk-abc123")
```

### 依赖管理

```python
# 添加阻塞依赖
task_dep_add(
    task_id="tk-abc123",
    depends_on_id="tk-def456",
    dep_type="blocks"
)

# 查看依赖关系
task_dep_list(task_id="tk-abc123")

# 查找就绪任务
task_ready(limit=5)

# 查找阻塞任务
task_blocked()
```

## 技术细节

### 通信协议

- **协议**：Model Context Protocol (MCP)
- **传输**：stdio（标准输入输出流）
- **数据格式**：JSON-RPC
- **SDK**：`mcp` Python 包

### 启动配置

```json
{
  "mcpServers": {
    "task-server": {
      "command": "uv",
      "args": ["run", "python", "-m", "task.server"],
      "env": {
        "LOG_LEVEL": "info",
        "PYTHONUNBUFFERED": "1"
      }
    }
  }
}
```

### 数据存储

- **引擎**：SQLAlchemy 2.0
- **数据库**：SQLite（本地文件）
- **位置**：`{workspace}/.task_data/tasks.db`
- **迁移**：Alembic

## 错误处理

### 常见错误

| 错误代码 | 说明 | 解决方法 |
|---------|------|----------|
| `TASK_NOT_FOUND` | 任务不存在 | 检查任务 ID |
| `CIRCULAR_DEPENDENCY` | 循环依赖 | 移除导致循环的依赖 |
| `INVALID_PRIORITY` | 无效的优先级 | 使用 0-4 的整数 |
| `INVALID_TASK_TYPE` | 任务类型无效 | 使用 bug/feature/task/epic/chore |
| `WORKSPACE_NOT_INITIALIZED` | 工作空间未初始化 | 运行 workspace_init |
