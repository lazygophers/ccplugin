# Task Plugin 使用示例

## 完整工作流示例

### 场景：实现用户认证功能

#### 1. 需求分析与任务规划

**用户操作**：
```
请帮我规划"实现用户认证功能"的任务
```

**Claude 响应**（调用 task-planner agent）：
```markdown
📋 任务规划完成

## 任务概览
- 总任务数：5
- 优先级分布：P0=1, P1=2, P2=2
- 预计周期：2-3 周

## 任务列表

✅ 已创建以下任务：

1. **tk-001**: 设计认证架构（P1, epic）
   - 依赖：无
   - 标签：architecture, auth

2. **tk-002**: 实现用户注册 API（P1, feature）
   - 依赖：tk-001
   - 标签：backend, api, auth

3. **tk-003**: 实现用户登录 API（P0, feature）
   - 依赖：tk-002
   - 标签：backend, api, auth

4. **tk-004**: 添加密码重置功能（P2, feature）
   - 依赖：tk-002
   - 标签：backend, api, auth

5. **tk-005**: 编写认证测试用例（P2, task）
   - 依赖：tk-003, tk-004
   - 标签：testing, auth

## 执行顺序建议
1. tk-001（架构设计）→ 2. tk-002（注册）→ 3. tk-003（登录）+ tk-004（重置密码）→ 4. tk-005（测试）

## 关键路径
tk-001 → tk-002 → tk-003 → tk-005（影响整体完成时间）
```

---

## 高级用法

### 1. 按标签过滤任务

```
列出所有后端相关的待处理任务
```

Claude 调用：
```
task_list(status="open", brief=true)
```
然后过滤 tags 包含 "backend" 的任务。

---

### 2. 查看任务依赖关系

```
显示 tk-003 的所有依赖
```

Claude 调用：
```
task_show(task_id="tk-003")
task_dep_list(task_id="tk-003")
```

---

### 3. 批量更新任务

```
将所有测试相关的待处理任务优先级设为 P2
```

Claude 会：
1. 调用 `task_list` 获取所有任务
2. 过滤 tags 包含 "testing" 且 status="open" 的任务
3. 对每个任务调用 `task_update(task_id=..., priority=2)`

## 常见问题

### Q: 如何处理被阻塞的任务？

**A**: 使用 `task_blocked` 查看阻塞任务，然后优先完成阻塞依赖：
```
/task-blocked（查看被阻塞的任务）
```

### Q: 如何重新排列任务优先级？

**A**: 使用 `task_update` 修改 priority：
```
将 tk-001 的优先级改为 P0
```

### Q: 如何删除错误创建的任务？

**A**: 使用 `task_delete`：
```
删除任务 tk-999
```

### Q: 如何查看团队成员的任务？

**A**: 使用 `task_list` 的 assignee 过滤：
```
列出 Alice 负责的所有任务
```
