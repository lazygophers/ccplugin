### 代码风格规则

#### 1. 行长度

**规则**：最大 100 字符

```python
# ✅ 好
result = some_function(
    arg1="value1",
    arg2="value2",
    arg3="value3",
)

# ❌ 差
result = some_function(arg1="value1", arg2="value2", arg3="value3", arg4="value4")
```

#### 2. 导入顺序

**规则**：标准库 → 第三方库 → 本地模块

```python
# ✅ 好
import os
from typing import Optional

from pydantic import BaseModel
from sqlalchemy import Column

from task.types import TaskStatus

# ❌ 差
from task.types import TaskStatus
import os
from pydantic import BaseModel
```

#### 3. 空行使用

**规则**：
- 顶级函数/类之间：2 个空行
- 类方法之间：1 个空行
- 函数内逻辑块之间：1 个空行

```python
# ✅ 好
class TaskManager:
    def __init__(self):
        self.tasks = []

    def create_task(self, title: str) -> Task:
        # 验证输入
        if not title:
            raise ValueError("Title is required")

        # 创建任务
        task = Task(title=title)
        self.tasks.append(task)

        return task
```

#### 4. 字符串引号

**规则**：优先使用双引号 `"`

```python
# ✅ 好
title = "Task title"
description = """
Multi-line
description
"""

# ❌ 差
title = 'Task title'
```

## 类型系统

### 强类型要求（严格执行）

**规则**：
- ✅ **所有函数必须有类型注解**（参数 + 返回值）
- ✅ **所有类属性必须有类型注解**
- ✅ **禁止使用 `Any` 类型**（除非确实无法确定）
- ✅ **使用 `Optional[T]` 明确可选类型**
- ✅ **使用 Pydantic BaseModel 进行数据验证**

### 类型注解示例

#### 1. 函数类型注解

```python
# ✅ 好：完整的类型注解
async def create_task(
    title: str,
    description: str = "",
    priority: Priority = Priority.MEDIUM,
    tags: list[str] | None = None,
) -> Task:
    """创建任务"""
    ...

# ❌ 差：缺少类型注解
async def create_task(title, description="", priority=2):
    ...

# ❌ 差：使用 Any
from typing import Any

    ...
```

#### 2. 类属性类型注解

```python
# ✅ 好：Pydantic 模型（推荐）
class Task(BaseModel):

    description: str = ""
    priority: Priority = Priority.MEDIUM

@dataclass
```

### Pydantic 模型规范

**推荐**：所有数据模型使用 Pydantic

```python
from datetime import datetime
from pydantic import BaseModel, Field, field_validator

class Task(BaseModel):
    """任务模型"""

    # 字段定义（带描述和约束）
    id: str = Field(..., description="任务唯一标识")
    title: str = Field(..., min_length=1, max_length=200, description="任务标题")
    description: str = Field(default="", max_length=10000, description="任务描述")
    priority: Priority = Field(default=Priority.MEDIUM, description="优先级")
    tags: list[str] = Field(default_factory=list, max_items=20, description="标签列表")
    created_at: datetime = Field(default_factory=datetime.utcnow, description="创建时间")

    # 验证器
    @field_validator("tags")
    @classmethod
    def validate_tags(cls, v: list[str]) -> list[str]:
        """验证标签格式"""
        # 过滤空标签、转小写、去重
        return list(set(tag.lower().strip() for tag in v if tag.strip()))

    @field_validator("title")
    @classmethod
    def validate_title(cls, v: str) -> str:
        """验证标题非空"""
        if not v.strip():
            raise ValueError("任务标题不能为空")
        return v.strip()

    # 模型配置
    model_config = {
        "frozen": False,  # 是否不可变
        "validate_assignment": True,  # 赋值时验证
        "use_enum_values": True,  # 枚举使用值而非名称
    }
```

## 命名约定

### 通用规则

**遵循 PEP 8**：

| 类型 | 约定 | 示例 |
|------|------|------|
| **模块** | snake_case | `task_manager.py` |
| **类** | PascalCase | `TaskManager`, `DependencyResolver` |
| **函数/方法** | snake_case | `create_task()`, `get_by_id()` |
| **变量** | snake_case | `task_id`, `priority_value` |
| **常量** | UPPER_SNAKE_CASE | `DEFAULT_PRIORITY`, `MAX_TITLE_LENGTH` |
| **私有成员** | _leading_underscore | `_internal_method()`, `_cache` |
| **类型别名** | PascalCase | `TaskID`, `MetadataDict` |

### 具体规范

#### 1. 类命名

```python
# ✅ 好：清晰、具体、PascalCase
class TaskManager:
    """任务管理器"""

class DependencyResolver:
    """依赖解析器"""

class TaskRepository:
    """任务仓库"""

# ❌ 差：缩写、不清晰
class TM:  # 太短
class Manager:  # 太通用
class task_manager:  # 不符合规范
```

#### 2. 函数/方法命名

```python
# ✅ 好：动词开头、描述清晰
async def create_task(title: str) -> Task:
    """创建任务"""

async def get_task_by_id(task_id: str) -> Task | None:
```

### 测试结构

**AAA 模式**：Arrange - Act - Assert

```python
async def test_create_task_success():
    """测试创建任务成功"""

    # Arrange（准备）
    manager = TaskManager()
    title = "测试任务"
    priority = Priority.HIGH

    # Act（执行）
    task = await manager.create_task(
        title=title,
        priority=priority,
    )

    # Assert（断言）
    assert task.id.startswith("tk-")
    assert task.title == title
    assert task.priority == priority
    assert task.status == TaskStatus.OPEN
```

## 提交规范

### 提交消息格式

**结构**：

```
<type>: <subject>

<body>

🤖 Generated with [Claude Code](https://claude.com/claude-code)

Co-Authored-By: Claude <noreply@anthropic.com>
```

### 类型（Type）

| 类型 | 说明 | 示例 |
|------|------|------|
| **feat** | 新功能 | `feat: 添加任务依赖管理` |
| **fix** | Bug 修复 | `fix: 修复状态转换验证错误` |
| **docs** | 文档更新 | `docs: 更新 API 参考文档` |
| **test** | 测试相关 | `test: 添加依赖解析器单元测试` |
| **refactor** | 重构 | `refactor: 重构 Repository 查询逻辑` |
| **perf** | 性能优化 | `perf: 优化任务列表查询性能` |
| **chore** | 杂项 | `chore: 更新依赖版本` |

### 主题（Subject）

**规则**：
- 使用简体中文
- 使用祈使句（"添加"而非"添加了"）
- 不超过 50 字符
- 不以句号结尾

```
# ✅ 好
feat: 添加任务依赖管理功能
fix: 修复循环依赖检测错误
docs: 更新快速开始文档

# ❌ 差
feat: 添加了任务依赖管理功能  # 不用过去式
fix: bug修复  # 不具体
docs: update quickstart doc  # 应该中文
```

### 正文（Body）

**规则**：
- 详细说明变更内容
- 说明变更原因
- 说明影响范围


### 常见问题

**Q: 如何处理长行？**

A: 使用括号分隔多行：

```python
# ✅ 好
result = some_function(
    arg1="value1",
    arg2="value2",
    arg3="value3",
)

# ✅ 好
long_string = (
    "This is a very long string "
    "that spans multiple lines "
    "for better readability."
)
```

**Q: 何时使用 `type: ignore`？**

A: 仅在确实无法修复类型错误时使用，并添加注释说明原因：

```python
# ✅ 好
result = some_library_function()  # type: ignore  # Library has no type stubs

# ❌ 差
result = some_library_function()  # type: ignore
```
