# 任务文件示例

本目录包含 `@.claude/task/` 中各个 Markdown 文件的格式示例和最佳实践。

## 文件说明

| 文件 | 描述 | 用途 |
|------|------|------|
| **[todo.md](todo.md)** | 待完成任务示例 | 展示如何创建和组织任务 |
| **[in-progress.md](in-progress.md)** | 进行中任务示例 | 展示如何追踪任务进展 |
| **[done.md](done.md)** | 已完成任务示例 | 展示如何记录完成的任务 |
| **[archive.md](archive.md)** | 任务归档示例 | 展示如何组织和归档历史任务 |

## 快速参考

### 任务优先级

- **P0** - 紧急，当天处理
- **P1** - 高优先级，本周处理
- **P2** - 普通优先级，本月处理
- **P3** - 低优先级，有时间再处理

### 任务分类

- **feature** - 新功能开发、功能增强
- **bug** - 缺陷修复、问题解决
- **refactor** - 代码重构、性能优化
- **docs** - 文档编写、API 文档
- **test** - 测试相关工作
- **config** - 配置和环境设置

### 任务生命周期

```
1. 创建任务
   ↓ 在 todo.md 中添加任务

2. 开始任务
   ↓ 移动到 in-progress.md，记录开始时间

3. 更新进展
   ↓ 定期更新进度百分比和具体描述

4. 完成任务
   ↓ 移动到 done.md，记录完成时间和交付物

5. 归档任务
   ↓ 移动到 archive/ 相应的项目/模块文件
```

## 使用指南

### 查看示例

1. **了解格式** - 打开相应的示例文件（todo.md、in-progress.md 等）
2. **复制模板** - 根据格式在自己的项目中创建任务
3. **遵循规范** - 参考字段说明，确保信息完整

### 创建任务

创建新任务时，复制示例中的模板，填入以下信息：

```markdown
### [P{N}] TASK-{编号} {任务标题}

- **类别**: {功能/文档/重构/bug}
- **描述**: {详细说明}
- **验收标准**:
  - [ ] 标准 1
  - [ ] 标准 2
- **子任务** (可选):
  - [ ] 子任务 1
  - [ ] 子任务 2
```

**字段说明**：
- **任务编号** (TASK-001 等) - 唯一标识号，便于追踪和引用
- **优先级** [P0/P1/P2/P3] - P0 紧急，P1 高，P2 普通，P3 低
- **类别** - feature/bug/refactor/docs/test/config
- **描述** - 详细说明背景、目标、需求
- **验收标准** - 具体可验证的完成条件（使用复选框）
- **子任务** - 可选，拆分为更小的可执行步骤

### 更新进展

进行中的任务需要定期更新进展和验收标准完成情况：

```markdown
- **进展**: {百分比} {具体描述}
- **完成的验收标准**:
  - [x] 标准 1 - 已完成
  - [ ] 标准 2 - 进行中
```

例如：
- `50% 已完成数据模型设计，正在实现 API`
- `75% 完成核心功能，剩余单元测试`
- 同时更新对应的验收标准复选框

### 记录完成

任务完成时，记录以下信息：

```markdown
- **说明**: {实现方式或解决方案}
- **验收标准完成情况**:
  - [x] 标准 1 - 说明完成方式
  - [x] 标准 2 - 说明完成方式
- **关键交付物**: {列出主要成果}
- **关联提交**: {commit hash 或 PR}
```

确保所有验收标准都已标记为完成，这证明了任务的完整性。

## 最佳实践

✅ **DO**

- 为每个任务分配唯一编号（TASK-001, TASK-002 等）
- 使用清晰的任务标题，包含优先级标记 [P0/P1/P2/P3]
- 提供具体的验收标准，使用复选框跟踪完成状态
- 对于复杂任务，拆分为可执行的子任务
- 提供详细的描述，解释背景、目标和需求
- 定期更新进行中任务的进展和验收标准状态
- 任务完成时，确保所有验收标准都已标记为完成
- 定期将完成的任务移动到 done.md 并归档

❌ **DON'T**

- 创建过于宽泛的验收标准（应该具体可验证）
- 创建过于宽泛的任务（应该拆分）
- 忽视进度更新（让任务长期滞留）
- 混淆任务文件之间的内容（避免重复）
- 删除任务历史（应该归档而不是删除）
- 重复使用相同的任务编号
- 任务完成后仍标记为进行中

## 文件大小参考

| 文件 | 推荐大小 | 说明 |
|------|----------|------|
| **todo.md** | <5KB | 待办任务队列，保持简洁 |
| **in-progress.md** | <2KB | 当前任务，通常只有 1-3 个 |
| **done.md** | <10KB | 最近完成的任务（5-10 个） |
| **archive/\*.md** | <100KB | 历史任务存档 |

当文件超过推荐大小时，考虑进行归档或清理。
