# 添加新任务

在 `@.claude/task/todo.md` 中快速添加新任务。

## 工作流程

### 第 1 步：打开任务文件

编辑 `@.claude/task/todo.md`

### 第 2 步：添加新任务

在相应的 **## [功能模块名称]** 部分下添加：

```markdown
### [P{N}] TASK-{编号} {任务标题}

- **类别**: {功能/文档/重构/bug/测试}
- **描述**: {详细说明任务的背景、目标和范围}
- **验收标准**:
  - [ ] 标准 1 - 具体可验证的要求
  - [ ] 标准 2 - 具体可验证的要求
- **子任务** (可选):
  - [ ] 子任务 1
  - [ ] 子任务 2
```

### 优先级参考
- `P0` - 紧急（当天完成）
- `P1` - 高优先级（本周完成）
- `P2` - 普通优先级（本月完成）
- `P3` - 低优先级（有时间再处理）

### 类别说明
- `feature` - 新功能开发
- `bug` - 缺陷修复
- `refactor` - 代码重构、优化
- `docs` - 文档编写
- `test` - 测试相关
- `config` - 配置设置

## 验收标准编写

✅ **好的验收标准**：
- 实现用户认证 API（支持 POST /auth/login）
- 单元测试覆盖率 ≥ 80%
- 性能测试通过（响应时间 < 200ms）

❌ **不好的验收标准**：
- 完成认证功能（太宽泛）
- 写测试（不够具体）

## 组织结构示例

```markdown
# 待完成任务 (TODO)

## 用户管理模块

### [P1] TASK-001 实现用户注册
...

## 数据导出模块

### [P1] TASK-003 实现 CSV 导出
...
```

## 常见问题

**Q: 如何分配任务编号？**
A: 按照顺序分配（TASK-001, TASK-002...），保持唯一性。

**Q: 任务太大怎么办？**
A: 如果预计 > 3 天，拆分成多个小任务，使用子任务组织。
