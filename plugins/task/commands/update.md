# 更新任务进展

定期更新 `@.claude/task/in-progress.md` 中的任务进展，准确反映项目现状。

## 工作流程

### 第 1 步：打开任务文件

编辑 `@.claude/task/in-progress.md`

### 第 2 步：更新进度字段

```markdown
- **进展**: {百分比}% 已完成 - {具体描述}
- **完成的验收标准**:
  - [x] 已完成的标准 - 说明完成方式
  - [ ] 未完成的标准
- **下一步**: {接下来的工作}
```

### 进度百分比参考
- `10%` - 刚开始，框架搭建
- `30%` - 核心功能部分实现
- `60%` - 核心功能基本完成
- `90%` - 功能完成，进入优化阶段
- `100%` - 完全完成，所有标准满足

## 更新示例

### 示例 1：正常进行

```markdown
### [P1] TASK-001 实现用户认证 API

- **进展**: 35% 已完成 - 完成了 JWT 令牌生成，正在实现刷新逻辑
- **完成的验收标准**:
  - [x] 实现用户注册 API - 完成端点和数据验证
  - [ ] 实现用户登录 API - 进行中
  - [ ] 实现令牌刷新 API
  - [ ] 编写单元测试
- **下一步**:
  1. 完成登录 API 实现
  2. 添加错误处理
  3. 编写测试
```

### 示例 2：遇到阻塞

```markdown
- **进展**: 30% 已完成 - 数据库性能问题导致阻塞
- **阻塞因素**: 查询响应超过 5s，需要优化
- **解决方案**: 创建 TASK-050 单独处理数据库优化
- **关联任务**: TASK-050
- **下一步**: 等待 TASK-050 完成后继续
```

### 示例 3：即将完成

```markdown
- **进展**: 95% 已完成 - 所有功能实现，最后进行集成测试
- **完成的验收标准**:
  - [x] 实现用户注册 API - 87% 覆盖率测试
  - [x] 实现用户登录 API - 完成所有场景测试
  - [x] 实现令牌刷新 API
  - [ ] 编写单元测试 - 最后总结和清理
- **下一步**: 完成测试，准备 PR 提交
```

## 检查清单

更新前确保：

- [ ] 所有完成的工作都已记录
- [ ] 验收标准的勾选状态准确反映实际
- [ ] 进度百分比合理
- [ ] 下一步的工作清晰具体

## 最佳实践

✅ 应该做：
- 每天更新进度（至少一次）
- 记录具体的完成情况
- 及时识别和报告问题

❌ 不应该做：
- 忘记更新
- 进度百分比与实际不符
- 隐瞒问题和阻塞
