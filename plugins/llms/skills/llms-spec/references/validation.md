# 验证规则

## 结构检查

### 必需

- [ ] 文件以 `# 项目名称` H1 标题开头
- [ ] 整个文件只有一个 H1
- [ ] 文件命名为 `llms.txt`（小写）

### 推荐结构

- [ ] H1 后紧跟引用块 `>` 摘要
- [ ] 详细内容部分不含任何标题（H2-H6）
- [ ] H2 部分包含文件列表
- [ ] `## Optional` 仅用于次要信息

## 链接校验

### 格式检查

- [ ] 列表项以 `- ` 开头
- [ ] 包含 Markdown 超链接 `[title](url)`
- [ ] 描述（如有）以 `: ` 分隔
- [ ] 解析正则匹配：`-\s*\[(?P<title>[^\]]+)\]\((?P<url>[^)]+)\)(?::\s*(?P<desc>.*))?`

### 本地文件

- [ ] 路径相对于项目根目录
- [ ] 使用正斜杠 `/`
- [ ] 文件实际存在

### 远程 URL

- [ ] 包含协议（`http://` 或 `https://`）
- [ ] URL 格式正确
- [ ] 用户明确要求联网验证时再检查可访问性

## 内容质量

### 简洁性

- [ ] 摘要不超过 100 字
- [ ] 每个链接描述 ≤ 1 句话
- [ ] 无冗余内容

### 描述质量

- [ ] 每个链接有描述
- [ ] 描述说明文件用途
- [ ] 无未解释的术语

### 分组合理性

- [ ] Docs、Examples、Optional 分组清晰
- [ ] 重要文档在前
- [ ] Optional 仅含次要信息

## 常见错误

| 错误 | 正确做法 |
|---|---|
| 详细内容含 H2-H6 标题 | 详细内容只用段落/列表 |
| 链接缺少描述 | 添加 `: description` |
| Optional 含重要文档 | 移到 Docs 或 Examples |
| 描述过长 | 简化到 ≤ 1 句话 |
| 使用 H3/H4 子标题 | 只用 H1 和 H2 |
| 使用代码块替代链接 | 用链接指向代码文件 |
| 使用表格 | 用列表替代 |
