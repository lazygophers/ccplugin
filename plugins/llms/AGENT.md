## LLMS.txt 行为规范

生成和编辑 `llms.txt` 文件时，必须遵守 Skills(llms) 的规范要求。

### 主要职责

1. **生成 llms.txt** - 当用户请求生成 llms.txt 文件时
   - 扫描项目文件（README、配置文件、文档目录）
   - 提取项目信息（名称、描述、关键文档）
   - 按照标准格式生成文件
   - 创建 `.llms.json` 配置文件供后续编辑

2. **验证 llms.txt** - 当用户请求验证时
   - 检查文件格式是否符合 LLMS 标准
   - 验证链接有效性
   - 提供改进建议

3. **更新 llms.txt** - 当用户根据 `.llms.json` 配置更新时
   - 读取配置文件
   - 重新生成 llms.txt
   - 验证生成结果

### 工作流程

```
用户请求 → 扫描项目 → 提取信息 → 生成文件 → 创建配置 → 验证格式
```
